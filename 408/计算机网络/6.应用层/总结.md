# 应用层协议总结

> 应用层协议定义了运行在不同端系统上的应用程序进程之间如何相互传递报文。

---

## 核心协议总览

| 协议 | 全称 | 提供的服务 | 传输层协议 | 端口号 | C/S / P2P |
|:----:|------|-----------|:--------:|:-----:|:--------:|
| **DNS** | Domain Name System | 域名 → IP 地址解析 | UDP（主要）/ TCP | **53** | C/S |
| **FTP** | File Transfer Protocol | 文件上传/下载、目录管理 | TCP | **21**（控制）/ **20**（数据） | C/S |
| **TFTP** | Trivial File Transfer Protocol | 简单文件传输（无认证、无目录操作） | UDP | **69** | C/S |
| **NFS** | Network File System | 远程文件透明访问（挂载远程目录） | TCP / UDP | **2049**（NFSv4） | C/S |
| **SMTP** | Simple Mail Transfer Protocol | 邮件**发送** + 服务器间**中继转发** | TCP | **25** | C/S |
| **POP3** | Post Office Protocol v3 | 邮件**接收**（下载到本地） | TCP | **110** | C/S |
| **IMAP** | Internet Message Access Protocol | 邮件**接收**（在线管理，服务器存储） | TCP | **143** | C/S |
| **HTTP** | HyperText Transfer Protocol | 万维网——网页浏览、资源请求 | TCP | **80** | C/S |
| **HTTPS** | HTTP over SSL/TLS | 安全的万维网访问（加密） | TCP | **443** | C/S |
| **Telnet** | Telecommunication Network | 远程终端登录 | TCP | **23** | C/S |
| **SSH** | Secure Shell | 安全的远程登录 + 文件传输（SFTP） | TCP | **22** | C/S |
| **DHCP** | Dynamic Host Configuration Protocol | 动态分配 IP 地址、子网掩码、网关、DNS 等 | UDP | **67**（服务器）/ **68**（客户） | C/S |
| **SNMP** | Simple Network Management Protocol | 网络设备监控与管理 | UDP | **161**（代理）/ **162**（管理站 Trap） | C/S |
| **RTP** | Real-time Transport Protocol | 实时多媒体数据传输（音视频流） | UDP | 动态分配（RTCP 为 RTP+1） | — |
| **SIP** | Session Initiation Protocol | VoIP 呼叫的建立、修改、终止 | TCP / UDP | **5060** | — |

## 分类速记

### 按功能分类

| 功能类别 | 协议 |
|:--------:|------|
| **域名解析** | DNS |
| **文件传输** | FTP、TFTP、NFS |
| **电子邮件** | SMTP（发）、POP3（收-下载）、IMAP（收-在线管理） |
| **万维网** | HTTP、HTTPS |
| **远程登录** | Telnet、SSH |
| **地址分配** | DHCP |
| **网络管理** | SNMP |
| **多媒体通信** | RTP/RTCP、SIP |

### 按传输层协议分类

| 传输层 | 协议 |
|:-----:|------|
| **TCP** | FTP、SMTP、POP3、IMAP、HTTP、HTTPS、Telnet、SSH、NFS（v4） |
| **UDP** | DNS（主要）、TFTP、DHCP、SNMP、RTP、NFS（v2/v3） |
| **TCP + UDP 均可** | DNS（UDP 为主，超 512B / 区域传送用 TCP）、SIP |

## 关键对比速记

| 对比维度 | 协议 A | 协议 B |
|:--------:|--------|--------|
| **发送 vs 接收邮件** | **SMTP**（发，端口 25） | **POP3**（收，端口 110）/ **IMAP**（收，端口 143） |
| **文件传输：重量 vs 轻量** | **FTP**（TCP，21+20，功能丰富） | **TFTP**（UDP，69，仅上传下载） |
| **文件传输：复制 vs 透明** | **FTP**（复制整个文件） | **NFS**（挂载后透明访问，按需读写） |
| **邮件存储：本地 vs 服务器** | **POP3**（下载到本地，默认删除） | **IMAP**（保留服务器，多设备同步） |
| **网页：明文 vs 加密** | **HTTP**（80，明文） | **HTTPS**（443，TLS 加密） |
| **远程登录：明文 vs 加密** | **Telnet**（23，明文） | **SSH**（22，加密） |
| **有状态 vs 无状态** | **FTP**（有状态，控制连接保持） | **HTTP**（无状态，每个请求独立） |
| **带外 vs 带内** | **FTP**（带外，控制与数据分离） | **HTTP**（带内，控制与数据同一连接） |
| **推送 vs 拉取** | **SMTP**（Push，主动推送邮件） | **POP3 / IMAP**（Pull，主动拉取邮件） |

---

## ❗ 易错点总结

1. **DNS 主要用 UDP（53）**，但区域传送和超 512B 响应必须用 TCP
2. **FTP 使用两个端口**——21（控制）+ 20（数据），别记混
3. **TFTP 用 UDP（69）**——不要因为名字有 FTP 就以为也用 TCP
4. **SMTP 只负责"发"**（25）；**POP3（110）和 IMAP（143）负责"收"**——发送和接收用不同协议
5. **HTTP（80）vs HTTPS（443）**——端口是最直接的区分
6. **Telnet（23）是明文**，**SSH（22）是加密**——别记反
7. **DHCP 用两个端口**——服务器 67、客户 68
8. **FTP 是有状态协议**（控制连接一直保持），**HTTP 是无状态协议**（每个请求独立）
9. **FTP 是带外传送**，**HTTP 是带内传送**
10. **POP3 邮件默认下载到本地后删除服务器副本**；**IMAP 邮件始终保留在服务器**
