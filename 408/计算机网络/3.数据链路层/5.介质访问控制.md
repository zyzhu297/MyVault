# 介质访问控制的基本概念

> **介质访问控制（MAC, Medium Access Control）**：数据链路层的一个子层，负责协调多个结点**共享同一条广播信道**时的发送权分配，避免信号冲突。

> [!important] 为什么需要介质访问控制？
> 在广播网络中（如以太网、无线局域网），多个结点共享同一条通信信道。如果两个或多个结点同时发送数据，信号会在信道上发生**碰撞（冲突）**，导致所有发送失败。MAC 子层的核心任务就是**决定谁在什么时候可以发送**。

# 介质访问控制的分类

> 介质访问控制协议可分为三大类：**信道划分**、**随机访问**和**轮询访问**。

|   分类   |                  信道划分（静态分配）                  |                         随机访问（动态分配/争用）                         |           轮询访问（动态分配/受控）            |
| :----: | :------------------------------------------: | :-----------------------------------------------------------: | :--------------------------------: |
| **协议** | 频分复用 FDM<br>时分复用 TDM<br>波分复用 WDM<br>码分复用 CDM | ALOHA（纯/时隙）<br>CSMA（各种坚持策略）<br>CSMA/CD（以太网）<br>CSMA/CA（无线局域网） | 令牌传递（Token Passing）<br>轮询（Polling） |
| **冲突** |                     不冲突                      |                             可能冲突                              |                不冲突                 |
| **缺点** |                   低负载时资源浪费                   |                           高负载时冲突严重                            |               有轮询开销                |

# 信道划分介质访问控制

> **原理**：将信道资源（频率、时间、波长、码型等）划分为若干互不重叠的子信道，每个结点独享一个子信道。**静态分配，无冲突。**

## 频分复用（FDM, Frequency Division Multiplexing）

> 将信道的总带宽划分为多个**互不重叠的频带（子信道）**，每个结点独享一个频带；各子信道分配的带宽可以不同，但**总和不得超过信道的总带宽**

| 特点     | 说明                                                             |
| ------ | -------------------------------------------------------------- |
| **优点** | ① 实现简单<br>② 无冲突<br>③ 各结点可同时通信（不同频率）                            |
| **缺点** | ① 频带之间需要**隔离频带**，防止相邻信道干扰（浪费带宽）<br>② 结点数固定时不灵活<br>③ 低负载时空闲频带浪费 |
| **应用** | 广播电台、电视、ADSL                                                   |

## 时分复用（TDM, Time Division Multiplexing）

> 将时间划分为**等长的时分复用帧（TDM 帧）**，每个 TDM 帧再划分为固定数量的**时隙**，每个结点独占一个时隙。

| 特点 | 说明 |
|------|------|
| **优点** | ① 无冲突<br>② 每个结点固定分配时隙，公平 |
| **缺点** | ① 结点不发送数据时**时隙浪费**<br>② 结点数固定，不灵活 |
| **应用** | 传统电话网络（PSTN） |

**统计时分复用（STDM, Statistical TDM）：** 按需分配时隙，只给有数据要发送的结点分配，提高利用率。

| 类型 | 说明 |
|------|------|
| **同步 TDM** | 固定分配时隙，不发送也占着 |
| **统计 TDM（异步 TDM）** | 动态分配，按需占用，需额外携带地址信息 |

## 波分复用（WDM, Wavelength Division Multiplexing）

> **光的频分复用**。将不同波长的光信号复用到同一根光纤中传输。

| 特点 | 说明 |
|------|------|
| **原理** | 同 FDM，只是承载介质为光纤，划分依据为波长 |
| **DWDM（密集波分复用）** | 波长间隔极小，一根光纤可传输数百路信号 |

## 码分复用（CDM, Code Division Multiplexing / CDMA）⭐

采用不同编码区分各路原始信号的复用方式；CDM 既**共享信道的频率，有共享时间**；实际上更常用的名词是**码分多址（Code Division Multiple access，CDMA）**
> **原理**：各结点使用**相互正交的码片序列（Chip Sequence）** 对数据进行编码，所有结点可**同时同频**通信，各路数据在信道中**线性相加**，接收方通过码片序列的正交性分离各结点信号

### 码片序列的基本概念

| 概念           | 说明                                       |
| ------------ | ---------------------------------------- |
| **码片（Chip）** | CDMA 中数据的最小单位，每个比特用多个码片表示                |
| **码片序列**     | 每个结点分配一个唯一的 $m$ 位码片向量（通常 $m=8$ 或 $m=64$） |
| **正交性**      | 任意两个结点的码片序列的规格化内积为 0                     |

**码片序列的表示：**

- 码片序列中的 0 用 **-1** 表示，1 用 **+1** 表示
- 码片序列长度通常为 $m$ 位（$m$ 个码片）

> [!example] 码片序列示例
> 若结点 A 的码片序列为 `00011011`（8 位），则写成向量形式：
> $$A = (-1, -1, -1, +1, +1, -1, +1, +1)$$

### 编码规则

| 发送的比特 | 实际发送信号 |
|:--------:|------------|
| **1** | 发送码片序列本身（$S$） |
| **0** | 发送码片序列的反码（$-S$） |

### 正交性与内积

> 设结点 A 的码片向量为 $\vec{S}$，结点 B 的码片向量为 $\vec{T}$，则：
> $$S \cdot T \equiv \frac{1}{m}\sum_{i=1}^{m}S_i \times T_i = 0$$
> 即 $\vec{S}$ 和 $\vec{T}$ **正交**。

| 内积 | 含义 |
|:---:|------|
| $S \cdot S = 1$ | 规格化内积：自己和自己 = 1 |
| $S \cdot (-S) = -1$ | 和自己的反码 = -1 |
| $S \cdot T = 0$ | 和正交序列的内积 = 0（不同结点间） |

### 解码规则

> 接收方用某个结点的码片序列与收到的叠加信号做**规格化内积**：
> - 结果为 **1** → 该结点发送了 1
> - 结果为 **-1** → 该结点发送了 0
> - 结果为 **0** → 该结点没有发送

| 特点 | 说明 |
|------|------|
| **优点** | ① 所有用户同时同频通信<br>② 抗干扰能力强<br>③ 保密性好（码片序列相当于密钥） |
| **缺点** | ① 码片序列需严格正交<br>② 带宽扩展（扩频通信） |
| **应用** | 3G 移动通信（CDMA2000、WCDMA）、GPS |

# 随机访问介质访问控制

> **原理**：各结点**随机竞争**信道使用权，发送前不进行协调。**可能发生冲突**，协议核心在于**如何检测/避免冲突**和**冲突后如何处理**。

## ALOHA 协议

### 纯 ALOHA（Pure ALOHA）

> **想发就发**。任何结点有数据要发送时立即发送。若发生冲突（未收到确认），则**随机等待一段时间后重传**，直至发送成功

该方法冲突概率较高，吞吐量很低

### 时隙 ALOHA（Slotted ALOHA）

> **改进**：将时间划分为离散的时隙，结点只能在**时隙的开始时刻**发送数据，每一帧的发送时间必须小于或等于时隙长度；这种规定减少了发送的随意性，从而降低了冲突概率，提高了信道利用率

| 对比 | 纯 ALOHA | 时隙 ALOHA |
|------|:-------:|:--------:|
| **发送时机** | 随时 | 只能在时隙起点 |
| **冲突窗口** | $2T_0$ | $T_0$ |
| **最大吞吐量** | **18.4%** | **36.8%** |
| **实现复杂度** | 低（不需要同步） | 中（需要全网时钟同步） |

> [!tip] 两个 ALOHA 的吞吐量公式对比
> 时隙 ALOHA = 纯 ALOHA 的 2 倍（因为冲突窗口减半）：
> - 纯 ALOHA：$S = Ge^{-2G}$，$S_{max} = \frac{1}{2e} \approx 0.184$
> - 时隙 ALOHA：$S = Ge^{-G}$，$S_{max} = \frac{1}{e} \approx 0.368$

## CSMA 协议（Carrier Sense Multiple Access）

> **改进**：在发送前**先监听信道（载波监听）**，如果信道忙则推迟发送，减少冲突概率。

### 三种坚持策略

| 策略            | 行为                                                       | 冲突概率 | 延迟  |
| ------------- | -------------------------------------------------------- | :--: | :-: |
| **1-坚持 CSMA** | 信道忙 → 持续监听<br>信道闲 → **立即发送**（概率 1）                       |  高   |  低  |
| **非坚持 CSMA**  | 信道忙 → **随机等待**再监听<br>信道闲 → 立即发送                          |  中   |  中  |
| **p-坚持 CSMA** | 信道闲 → 以**概率 p** 发送，以概率 1-p 推迟<br>信道忙 → 持续监听（适用于**时隙信道**） |  低   | 较高  |
>[!note] p-坚持 CSMA
>在时隙开始时监听信道；若信道忙则等到下一个时隙再监听；推迟时也是等到下一个时隙再尝试；整个过程中**持续监听**

> [!important] 三种策略的本质区别
> - **1-坚持**：信道一空就发 → 多个等待结点同时发送 → 必然冲突 → "冲动型"
> - **非坚持**：信道忙就歇会儿 → 减少同时等待的结点 → 降低冲突 → "佛系型"
> - **p-坚持**：信道空闲时掷骰子决定发不发 → 进一步降低冲突 → "理性型"

> [!warning] CSMA 仍有冲突可能
> 载波监听只能减少冲突，**不能完全避免**！因为有**传播时延**——结点 A 开始发送后，信号到达结点 B 需要时间，在这段时间内 B 监听到的信道仍是空的，B 可能也开始发送 → 冲突！

## CSMA/CD 协议（CSMA with Collision Detection）⭐

> 以太网使用的 MAC 协议。在 CSMA 基础上增加了**碰撞检测**能力：发送过程中持续监听信道，一旦检测到碰撞立即**停止发送**，缩短无效占用时间。

- **载波监听**是指每个站点在发送前和发送过程中都必须持续监听信道
- **冲突检测（Collision Detection）** 即边发送边检测，适配器在发送数据的同时监听信道上的电压变化；当检测到电压变化幅度超过特定门限值时，即判定为发生冲突，立即停止发送，并等待一段**随机**时间后重试
- 工作流程概括为：**先听后发，边听边发，冲突停发，随即重发**

### 争用期（Contention Period）

> **争用期**（碰撞窗口）：从开始发送到能检测到碰撞的**最长时间**，等于**端到端往返传播时延** $2\tau$。

#### 冲突时间分析

设单程端到端传播时延为 $\tau$，A 在 $t=0$ 时刻开始发送一帧（帧传输时间为 $T_F$），B 在 $t=\tau-\sigma$ 时刻开始发送（$\sigma > 0$ 且极小，即 A 的信号即将到达 B 但尚未到达）

- **$t=0$**：A 开始发送帧
- **$t=\tau-\sigma$**：B 监听到信道仍为空（A 的信号还没到），B 开始发送
- **$t=\tau-\sigma/2$**：数据发生碰撞，但是两个站点都没有察觉
- **$t=\tau$**：A 的信号到达 B，但 B 已经开始发送 → B 立刻检测到碰撞
- **$t=2\tau-\sigma$**：B 的信号到达 A → **A 检测到碰撞**，至此两个站点的发送均失败，都需推迟一段时间后重传

A 从开始发送到检测到碰撞，共经过 **$2\tau-\sigma$**。

> 当 $\sigma \to 0$（B 恰好在 A 信号到达前瞬间发送），A 检测到碰撞的时间趋近 **$2\tau$**——这就是**最坏情况**，也是争用期的上界。

**结论**：经过 $2\tau$ 后仍未检测到碰撞，说明信道已被 A 成功抢占，后续不会再发生冲突。
**争用期内的冲突：** 只有**在争用期内发生冲突**才能被检测到。争用期过后，发送方已经占用信道，不会再发生冲突。

> [!important] 争用期的关键公式
> $$\text{争用期} = 2\tau = \frac{2 \times \text{端到端距离}}{\text{信号传播速率}}$$

### 最短帧长与争用期的关系

> CSMA/CD 要求：**发送一帧的时间 $\geq$ 争用期**。帧不能太短，否则发送完毕了还没检测到冲突。取临界值即得最短帧长：

$$L_{min} = 2\tau \times R$$

其中 $L_{min}$ 为最短帧长（bit），$R$ 为数据速率，$\tau$ 为单程端到端传播时延。

> [!note] 比特时间
> 发送 1 比特所需的时间，即 $1/R$（单位：秒/bit）。争用期用比特时间表示即为 $2\tau \times R$，含义是 **"争用期内可以发送的比特数"**。
>
> 争用期固定时，比特时间随速率反比变化——速率越高，比特时间越短，同样的争用期内可以发送更多比特，最短帧长也必须相应增大。换言之，**速率提升 → 最短帧长也必须提升**才能保证发送时间覆盖整个争用期。

> [!example] 以太网最短帧长计算
> **10 Mbps 以太网**：$\tau = 25.6\ \mu s$（最长 2.5 km），$R = 10$ Mbps
>
> $$L_{min} = 2 \times 25.6 \times 10^{-6} \times 10 \times 10^6 = 512\ \text{bit} = 64\ \text{byte}$$
>
> 这就是**以太网最小帧长 64 字节**的由来！
>
> 若数据不足 64 字节 → **填充（Padding）** 到 64 字节。

### 截断二进制指数退避算法

> CSMA/CD 碰撞后采用**截断二进制指数退避（Truncated Binary Exponential Backoff）** 来决定重传时机。

**算法描述：**

1. 碰撞发生后，退避时间从 $\{0, 1, 2, \ldots, 2^k-1\}$ 中**随机选取**一个数 $r$
2. 第 $n$ 次重传时：$k = \min(n, 10)$（最多到 10）
3. 退避时间 = $r \times 2\tau$（$2\tau$ 为争用期）
4. **重传 16 次仍失败** → 丢弃该帧，向上层报告错误

> [!tip] 退避算法的设计思想
> 碰撞次数越多 → 说明网络负载越重 → 退避范围指数增长 → 降低再次碰撞概率
> "截断"意味着退避范围的指数增长最多到 $2^{10}-1$，不再继续增大

**强化碰撞机制**：此外，以太网规定：一旦检测到重读，站点除立即停止发送数据外，还需继续发送 32 比特或 48 比特的**人为干扰信号**，以确保所有站点都能感知到冲突
### CSMA/CD 协议归纳
1. 准备发送：适配器从网络层获取数据报，封装成帧，放入缓存
2. 检测信道（载波监听）：
   - 信道空闲（**9.6μs 时间**内无信号）→ 开始发送
   - 信道忙 → 持续监听，直到空闲后立即发送
3. 发送过程中持续检测信道，可能出现两种情况：
   - **未检测到碰撞** → 发送完成，帧成功交付
   - **检测到碰撞** → 立即停止发送，发送 32/48 bit 人为干扰信号（强化碰撞），执行截断二进制指数退避算法
4. 退避结束后返回步骤 2 重试；**重传 16 次仍失败** → 丢弃该帧，向上层报告错误

## CSMA/CA 协议（CSMA with Collision Avoidance）⭐

> 无线局域网（**IEEE 802.11 / Wi-Fi**）使用的 MAC 协议。无线信道中**无法同时发送和接收**（半双工），因此无法实现碰撞检测，只能**避免碰撞**。

> [!important] 为什么无线不能用 CSMA/CD？
> 1. **信号衰减大**：无线信号的接收功率远小于发送功率，且信号强度动态范围极大，无法边发边听（碰撞检测需要对比收发信号）
> 2. **隐藏站问题**：A 和 C 都能与 B 通信但互相感知不到对方，A 发数据给 B 时 C 也发 → 在 B 处碰撞
> 3. **暴露站问题**：B 发数据给 A，C 想发给 D → C 监听到 B 在发就等待 → 实际上 C 和 D 之间通信不影响 A 和 B


- **冲突避免（Collision Avoidance）** 即通过随机退避、RTS/CTS 预约、ACK 确认等机制尽可能减少碰撞的发生概率，而非在碰撞发生后检测；**并不是可以完全避免冲突**，而是要**尽量降低冲突发生的概率**
- 工作流程概括为：**先打招呼，再发数据**

鉴于无线信道的通信质量远不如有线信道，802.11 协议在数据链路层引入**确认/重传（ARQ）** 机制：每个站点发送完**一帧**后，必须收到接收方的确认帧（ACK），才能继续发送下一帧；因此采用的是一种**停止-等待式的可靠传输协议**
**两种介质控制访问方式**：

| 方式      | 全称                                         |   类型   | 说明                                                              |
| ------- | ------------------------------------------ | :----: | --------------------------------------------------------------- |
| **DCF** | Distributed Coordination Function（分布式协调功能） | **必选** | 所有站点平等争用信道，采用 **CSMA/CA + 随机退避**，是实际部署中使用的模式                    |
| **PCF** | Point Coordination Function（点协调功能）         |   可选   | AP 集中控制，通过**信标帧（Beacon）** 建立无竞争期（CFP），依次轮询各站点是否需要发送；实际中**很少使用** |

802.11 协议规定，各站点在发送数据前**必须检测信道**，若信道忙，则禁止发送；为进一步减少冲突，还要求及时检测到信道空闲，也需再等待一段短暂时间（确保信道持续空闲），方可发送帧；该时间称为**帧间间隔（InterFrame Space，IFS）**，其长度取决于帧的类型

| 帧间间隔 | 全称 | 长度 | 用途 |
|:------:|------|:--:|------|
| **SIFS** | Short IFS（短帧间间隔） | 最短 | 高优先级控制帧使用：ACK、CTS、数据分片后的后续片段 |
| **PIFS** | PCF IFS（点协调帧间间隔） | 中间 | PCF 模式下 AP 轮询使用，长度 = SIFS + 1 个时隙，确保 AP 优先抢占信道 |
| **DIFS** | DCF IFS（分布式协调帧间间隔） | 最长 | DCF 模式下数据帧和 RTS 帧使用，长度 = SIFS + 2 个时隙，优先级最低 |

> [!tip] IFS 长度关系：SIFS < PIFS < DIFS
> 帧间间隔越短 → 优先级越高。ACK 等控制帧用 SIFS（最短）→ 确保及时确认；AP 轮询用 PIFS（中间）→ AP 比普通站点优先；普通数据帧用 DIFS（最长）→ 最低优先级。

### 争用期和随机退避

> 与 CSMA/CD 不同，CSMA/CA **没有碰撞检测**能力，因此也不存在类似"$2\tau$ 争用期"的概念。CSMA/CA 通过**竞争窗口（Contention Window, CW）+ 随机退避**机制来协调各站点的发送时机，避免碰撞。

**CSMA/CA 的争用机制：**

站点检测到信道**空闲且持续 DIFS** 后，进入争用期，从竞争窗口中**随机选取**一个退避时隙数，每个空闲时隙减 1，直至减为 0 后才发送当且仅当**检测到信道空闲且这个数据帧是要发送的第一个数据帧**时，才不使用退避算法（直接发送），**其余情况均需执行退避**；具体为：

1. 发送第一个数据帧之前检测到信道忙
2. 未收到确认，要重传数据帧时
3. 每次确认后要发送下一个数据帧

**退避算法**与 CSMA/CD 类似，第 $n$ 次退避在 $[0, \cdots, (2^{4+k} - 1)]$ 个时隙中随机选择一个，当时隙范围最大达到 1023 时（对应第 6 次重传），就不再增加

### CSMA/CA 协议归纳
1. 若站点**首次尝试**发送且检测到信道空闲并持续 **DIFS** → 可跳过退避**直接发送**整个数据帧；若信道忙则仍需等待 DIFS 后进入退避
2. 站点从竞争窗口 $\{0, 1, \ldots, CW-1\}$ 中**随机选择**一个退避计数值，信道每空闲一个时隙减 1，信道忙则**冻结**计数器（**只计算总空闲时间**），并继续等待，直至信道变为空闲（称为*推迟接入*）；若信道空闲，且在时间 DIFS 内持续空闲，则开始**争用信道**，进行倒计时；当计时器减至 0 时（尽可能发生在信道持续空闲期间），站点立即发送整个数据帧并等待确认
3. 发送站收到 ACK 确认后，若仍有后续帧待发送，则转到步骤 2；若在规定时间（由重传计时器控制）内**未收到 ACK** → $CW$ 加倍（$CW \leftarrow \min(2 \times CW,\ CW_{max})$），转到步骤 2 重试

### 虚拟载波监听

> 除了通过物理层检测信道能量的**物理载波监听**外，802.11 还引入了**虚拟载波监听**机制，通过帧头部中的**持续时间（Duration）字段**告知其他站点信道将被占用的时长。

**工作原理：**

1. 每个站点维护一个 **NAV（Network Allocation Vector，网络分配向量）** 定时器
2. 站点收到任何 802.11 帧（RTS/CTS/Data/ACK）后，读取其 Duration 字段
3. 若该帧不是发给自己的 → 将 Duration 值写入 NAV，NAV 倒计时
4. **NAV > 0 时，站点认为信道忙**，即使物理层检测到信道空闲也不发送

> 信道空闲的判定条件：**物理载波监听空闲** $\land$ **NAV = 0**，两者同时满足才认为信道可用。

| 类型 | 机制 | 特点 |
|:----:|------|------|
| 物理载波监听 | 物理层检测信号能量 | 只能感知通信范围内的信号 |
| 虚拟载波监听 | 通过 Duration 字段设置 NAV | 能"预知"信道占用，解决隐藏站问题 |

> [!tip] 虚拟载波监听的意义
> 即使站点 A 听不到站点 C 的信号（隐藏站），但 A 可能听到 B 回复给 C 的 CTS，CTS 中含有 Duration → A 设置 NAV → A 知道信道会被占用 → 避免碰撞。这就是 RTS/CTS 能够解决隐藏站问题的核心机制。

> 解决**隐藏站问题**的机制：发送方先发送短控制帧预约信道，接收方回应确认，附近结点听到后推迟发送。

**NAV（网络分配向量）：** RTS/CTS 帧中包含一个**持续时间字段**，告知附近结点信道将被占用的时长，听到的结点设置 NAV 定时器，在此期间不发送。
![[使用 RTS 和 CTS帧的冲突避免.png]]
**信道预约方法**：
1. A 站向 AP 发送数据帧 DATA 前，先检测信道；若信道空闲，则等待时间 DIFS 后，**广播发送**一个短控制帧 **RTS（Request to Send，请求发送）**，其中包含本次通信所需的持续时间（\[SIFS+CTS+SIFS+DATA+SIFS+ACK]，即从成功收到 RTS 帧起，到 AP 完成 ACK 发送为止的时间）写入 RTS 帧首部
2. 若 AP 正确收到 RTS 帧且信道空闲（物理载波监听 + NAV = 0），则等待 **SIFS** 后**广播回复 CTS（Clear to Send，允许发送）** 帧，CTS 帧首部同样包含持续时间字段（\[SIFS+DATA+SIFS+ACK]，即 A 站收到 CTS 到本次通信全部结束所需的时间），收到 CTS 的附近结点据此更新自己的 NAV
3. A 站正确收到 CTS 帧后，等待 **SIFS**，即可**发送 DATA 数据帧**；其首部写入时间\[DATA+SIFS+ACK]，即从开始接收数据帧起，到 AP 完成 ACK 发送为止的时间；AP 正确接收 DATA 后等待 **SIFS**，向 A 站**单播回复 ACK** 确认帧，至此一次 RTS/CTS 握手通信完成

> [!tip] RTS/CTS 中 RTS 也可能冲突
> RTS 帧本身也可能发生碰撞，但由于 RTS 很短，冲突代价小。碰撞后通过二进制指数退避重传 RTS。

## CSMA/CD 协议与 CSMA/CA 协议的区别

| 对比维度        |                                               CSMA/CD                                                |                                     CSMA/CA                                     |
| ----------- | :--------------------------------------------------------------------------------------------------: | :-----------------------------------------------------------------------------: |
| **应用场景**    |                                        有线局域网（以太网 IEEE 802.3）                                         |                            无线局域网（Wi-Fi IEEE 802.11）                             |
| **核心机制**    |                                      碰撞检测（Collision Detection）                                       |                            碰撞避免（Collision Avoidance）                            |
| **冲突处理**    |                                           边发送边检测，检测到碰撞立即停发                                           |                          发送前随机退避，尽量避免碰撞，通过 ACK 判断是否成功                           |
| **争用期**     |                                        固定值 $2\tau$（端到端往返传播时延）                                        |                               无固定争用期，通过竞争窗口（CW）管理                               |
| **退避触发**    |                                              检测到碰撞后触发退避                                              |                           发送前主动退避（除首次空闲可跳过），失败后 CW 加倍                           |
| **计数器行为**   |                                             每次碰撞后重新随机选取                                              |                               信道忙时**冻结**，空闲后从断点继续                               |
| **退避范围**    | 第 $n$ 次重传：$k=\min(n,10)$，在 $[0,2^k-1]$ 中随机选 $r$，退避 $r \times 2\tau$；最大 $2^{10}-1=1023$，重传 **16** 次放弃 | 第 $n$ 次重传：在 $[0,2^{4+k}-1]$ 个时隙中随机选（$k$ 起点多 4）；CW 最大 **1023**（第 **6** 次重传后不再增大） |
| **确认机制**    |                                          无需 ACK（通过检测碰撞判断成败）                                          |                          需 ACK 确认（SIFS 后回复），无 ACK 则重传                           |
| **最短帧长**    |                                          有（64 字节），由争用期和速率决定                                          |                             无此硬性约束（通过退避 + ACK 处理碰撞）                             |
| **隐藏站/暴露站** |                                           不涉及（有线信道不存在此问题）                                            |                          存在，通过 RTS/CTS + NAV 虚拟载波监听解决                           |
| **载波监听**    |                                               仅物理载波监听                                                |                              物理载波监听 + 虚拟载波监听（NAV）                               |
| **工作流程**    |                                         先听后发，边听边发，冲突停发，随即重发                                          |                              先听后发，空闲等待，随机退避，边发边确认                               |
| **全双工/半双工** |                                             可全双工（交换式以太网）                                             |                                 半双工（无线收发不能同时进行）                                 |
| **传输介质**    |                                               双绞线 / 光纤                                               |                                      无线电波                                       |
# 轮询访问介质访问控制

> **原理**：由一个**主结点**或通过**令牌**来协调各结点的发送权，本质上是一种**无冲突**的访问机制。

## 令牌传递（Token Passing）

> 一个称为**令牌（Token）** 的特殊帧在环状网络中循环传递，只有**持有令牌的结点**才能发送数据；令牌是有一个特殊控制帧，本身不包含数据，仅用于控制信道使用，确保同一时刻只有一个站点独占信道

站点发送完**一帧**后，应释放令牌，以便其他站点使用；由于令牌按顺序依次传递，**如果节点没有数据要发送，则立即传递令牌**，所有联网计算机对信道的访问权是公平的
## IEEE 802.5 令牌环网络中令牌和数据的传递过程

> 令牌环采用**单向环形拓扑**，令牌和数据帧沿固定方向（如顺时针）逐站传递。当网络空闲时，由一个特殊控制帧——**令牌（Token）** 在环上循环流动。

**令牌传递流程：**

1. 网络空闲时，**令牌帧**（Token Frame）沿环逐站传递，只有持有令牌的站才能发送数据。
2. 某站点**有数据要发送**时，等待令牌到达并**捕获令牌**（将令牌帧的令牌位从"空闲"改为"忙"），随后在令牌后附加待发送的数据和目的地址，形成**数据帧**发往下一站。
3. 数据帧沿环传递，**每个中间站**收到帧后：
   - 检查目的地址是否匹配本机 → 若匹配则**复制**数据到缓冲区，并在帧状态字节中标记"已复制"；
   - 无论是否匹配，均将帧**转发**给下一站。
4. 数据帧绕环一周回到**源发送站**后，源站检查状态位：
   - 若目的站已复制（接收成功）→ 源站**从环上移除**该帧；
   - 若未被复制 → 说明目的站不存在或未工作。
5. 源站移除数据帧后，立即**释放一个新令牌**到环上，供下游站点使用，保证公平性。

> **补充说明：** 站点每次只能发送**一帧**，发完后必须释放令牌，不能连续霸占信道。

| 特点     | 说明                                            |
| ------ | --------------------------------------------- |
| **优点** | ① **无冲突**<br>② **高负载时效率高**（信道利用率高）<br>③ 支持优先级 |
| **缺点** | ① 令牌维护开销大<br>② 令牌丢失会导致全网瘫痪                    |

## 轮询（Polling）

> **主结点**轮流询问各从结点是否有数据要发送，从结点收到轮询帧后才能发送。

| 特点 | 说明 |
|------|------|
| **优点** | 无冲突，主结点可集中控制 |
| **缺点** | ① 轮询帧本身有开销<br>② 单点故障（主结点坏了全网瘫痪）<br>③ 结点多时等待时间长 |

# ❗ 易错点总结

**关于 CDMA：**

1. **码片序列中的 0 用 -1 表示**，1 用 +1 表示。发送比特 0 → 发送码片的反码（即乘以 -1）；发送比特 1 → 发送码片本身

2. CDMA 解码是做**规格化内积**，不是普通内积！要除以码片序列长度 $m$

3. 规格化内积结果为 **1 → 发 1**，**-1 → 发 0**，**0 → 没发**

4. 正交码片的内积为 0，但必须用规格化内积计算；自己和自己的内积 = 1，自己和反码的内积 = -1

5. CDMA 中所有结点**同时同频**通信，通过正交码片序列分离信号

**关于信道划分（整体）：**

6. FDM/TDM/WDM/CDM 都是**静态划分、无冲突**，但低负载时资源浪费；高负载时效率高

7. STDM（统计时分复用）按需动态分配时隙，克服了同步 TDM 空闲时隙浪费的缺点；代价是需要额外携带地址信息

8. TDM 帧 = 等长的时分复用帧，每个 TDM 帧划分为固定数量的时隙；STDM 帧的时隙数 < 结点数

**关于 ALOHA：**

9. **纯 ALOHA 最大吞吐量为 $1/(2e) \approx 0.184$**，冲突窗口为 2 个帧时

10. **时隙 ALOHA 最大吞吐量为 $1/e \approx 0.368$**，恰好是纯 ALOHA 的 2 倍（因为冲突窗口减半 → $T_0$）

11. 时隙 ALOHA 需要**全网时钟同步**，这是它的主要代价

**关于 CSMA（三种坚持策略）：**

12. **CSMA 不能完全避免冲突**！因为存在传播时延（B 检测到信道空闲时 A 已经在发送但信号还没传到 B）

13. 三种策略的本质区别：**1-坚持**（"冲动型"→ 信道空立刻发）→ 冲突概率最高，延迟最低；**非坚持**（"佛系型"→ 忙则随机等）→ 中；**p-坚持**（"理性型"→ 掷骰子决定）→ 冲突概率最低，仅适用于时隙信道

14. 只有在**争用期 $2\tau$ 内**的冲突才能被检测到，超过争用期后发送方已经占用信道

**关于 CSMA/CD（以太网 IEEE 802.3）：**

15. 核心流程：**先听后发，边发边听，冲突停发，随即重发**

16. **最短帧长公式**：$L_{min} = 2\tau \cdot R$，本质是"发送一帧的时间 $\geq$ 争用期"——帧不能太短，否则发完了还没检测到碰撞

17. 以太网最小帧长 **64 字节**（512 bit），由 10 Mbps 以太网的 $\tau = 25.6\ \mu s$ 决定；不足 64 字节 → **填充（Padding）**

18. **争用期 = $2\tau$**（端到端往返时间），不是 $\tau$！

19. 截断二进制指数退避：第 $n$ 次重传时退避窗口 $k = \min(n, 10)$，从 $[0, 2^k-1]$ 中随机选 $r$，退避时间 = $r \times 2\tau$

20. 退避窗口最多到 $2^{10} - 1 = 1023$（截断）；**重传上限 16 次**，16 次后丢弃帧向上层报告

21. 检测到碰撞后除立即停发外，还需发送 32 或 48 bit **人为干扰信号（强化碰撞）**，确保所有站点都感知到冲突

22. 信道空闲的判定：**9.6 μs** 时间内无信号 → 认为空闲

**关于 CSMA/CA（Wi-Fi IEEE 802.11）：**

23. 无线局域网使用 CSMA/CA 而非 CSMA/CD：① 无线信号衰减大，收发功率差距悬殊 → **无法边发边检测碰撞**（无法实现 CD）；② 存在隐藏站/暴露站问题

24. **隐藏站**：A 和 C 互相感知不到，但都向 B 发送 → 在 B 处碰撞；**暴露站**：B→A 时 C 因监听到 B 在发而等待，但 C→D 不会干扰 A←B（不应该等）

25. **RTS/CTS 解决隐藏站问题**：发送方先发短控制帧（RTS）预约信道，接收方回复 CTS，附近结点听到后通过 **NAV（网络分配向量）** 推迟发送

26. 信道空闲的判定条件：**物理载波监听空闲 AND NAV = 0**（两者同时满足才认为可用）

27. **SIFS < PIFS < DIFS**：帧间间隔越短 → 优先级越高。ACK/CTS 用 SIFS（最短），AP 轮询用 PIFS（中间），普通数据帧用 DIFS（最长）

28. 三种帧间间隔的长度关系：PIFS = SIFS + 1 个时隙，DIFS = SIFS + 2 个时隙；单位是时隙而非绝对时间

29. CSMA/CA **首次发送特殊规则**：当且仅当检测到信道空闲且这是要发送的**第一个数据帧**时，才可不退避直接发送；其余情况（检测到忙、重传、发下一帧）均需退避

30. CSMA/CA 退避计数器行为：信道忙时**冻结**计数器，空闲后从**断点继续**（只计算总空闲时间）；CSMA/CD 每次碰撞后**重新随机选取**

31. CSMA/CA 退避范围：第 $n$ 次重传时 CW 从 $[0, 2^{4+k}-1]$ 中随机选（$k$ 比 CSMA/CD 多 4），CW 达到 1023（**第 6 次重传**）后不再增大

32. 发送后等待 **SIFS** 接收 ACK；未收到 ACK → CW 加倍（$CW \leftarrow \min(2CW, CW_{max})$）→ 重传

33. CSMA/CA **没有固定争用期 $2\tau$**，用竞争窗口（CW）管理；也**没有最短帧长**的硬性约束

34. RTS/CTS 中的 RTS 帧本身**也可能碰撞**（比如两个站点同时发 RTS），但由于 RTS 很短，冲突代价小

**CSMA/CD vs CSMA/CA 对比易错：**

35. CSMA/CD 可以有线全双工（交换式以太网），CSMA/CA 只能是半双工（无线收发不能同时）

36. CSMA/CD 无需 ACK（检测碰撞即可判断成败），CSMA/CA 需要 ACK 确认

37. CSMA/CD 仅物理载波监听，CSMA/CA = 物理载波监听 + 虚拟载波监听（NAV）

38. DCF（分布式协调）是 CSMA/CA 的必选模式，实际部署中使用；PCF（点协调）可选，很少使用

**关于轮询访问：**

39. 令牌环中**只有持有令牌的结点才能发送**数据；令牌丢失 → 全网瘫痪

40. **源站移除数据帧**（不是目的站！）：数据帧绕环一周回到源站后，由源站检查状态位并从环上移除——这是一个常见陷阱

41. 站点发送完**一帧**后必须立即释放令牌，不能连续霸占

42. 令牌环中如果结点没有数据要发送 → **立即传递令牌**给下一站

**关于三种 MAC 协议的负载效率：**

43. 低负载时效率：随机访问 > 信道划分 ≈ 轮询访问

44. 高负载时效率：信道划分 ≈ 轮询访问 > 随机访问
