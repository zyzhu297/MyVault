# 网桥基本概念

**网桥**工作在**数据链路层**，用于连接两个局域网网段，具有识别帧和转发帧的能力，根据帧首部的目的地址及自身的帧转发表，决定式转发还是丢弃所收到的帧，从而有效过滤通信量；
- 网桥**隔离冲突域**，但**不隔离广播域**
- 使用**生成树协议**（STP）消除环路
- 如今已被以太网交换机取代

# 以太网交换机

以太网交换机（Ethernet Switch）也称二层交换机，其中“二层”指其工作在数据链路层；本质上是一个**多端口网桥**，能够将网络划分为多个**较小的冲突域**，从而为每个用户提供更高的带宽，通过硬件（ASIC）实现高速帧转发;
在**使用集线器**构建的传统*共享式以太网*中，所有用户共享同一传输介质；例如，在 10Mb/s 网络中，若有 N 个用户，则每个用户的平均带宽仅约为 10/N Mb/s，且同一时刻只允许一台主机发送数据（半双工），否则会发生冲突。而**以太网交换机**将网络划分为多个较小的冲突域，每个端口提供独立的信道，独享带宽，不同端口对之间可同时并行通信，因此一台有 N 个端口的交换机的**总吞吐量可达 $N \times 10 Mb/s$** ，从而显著提升网络整体吞吐量。

## 交换机的原理和特点

### 交换模式

1. **存储转发**（Store-and-Forward）
   - 原理：将整个帧完整接收并缓存，进行 CRC 校验，确认无误后再从对应端口转发出去
   - 优点：提供差错检测，能过滤掉错误帧、冲突碎片，可靠性高；支持不同速率端口间的转换
   - 缺点：**转发延迟大**（需接收完整帧后才开始转发）
   - 适用：对可靠性要求高的网络环境，或需要连接不同速率端口的场景

2. **直通交换**（Cut-Through）
   - 原理：只读取帧的前 6 字节（目的 MAC 地址），查表后立即从对应端口转发，**不进行差错检测**
   - 优点：转发延迟极小（几乎一读到目的地址就转发）
   - 缺点：不检测 CRC 错误，会转发错误帧和冲突碎片（< 64 字节），浪费带宽
   - 适用：对延迟敏感、链路质量好的网络环境

3. **无碎片**（Fragment-Free）
   - 原理：接收前 64 字节后开始转发——冲突碎片一定小于 64 字节，因此能过滤掉碎片但不做完整 CRC 校验
   - 优点：在延迟和可靠性之间折中，能过滤冲突碎片，延迟适中
   - 缺点：无法检测大于 64 字节帧中的 CRC 错误
   - 适用：一般企业网络，兼顾延迟和基本可靠性

交换机通常配备多种速率的端口，如 10Mb/s、100Mb/s 端口，以及**多速率自适应**端口

### 主要特点

1. **多端口并行通信**：不同端口对之间可以同时通信，无冲突
2. **全双工**：支持全双工模式，发送和接收可同时进行（**无需 CSMA/CD**）
3. **带宽独占**：每个端口独享带宽，而非共享
4. **隔离冲突域**：每个端口是一个独立的冲突域
5. **即插即用**：自学习机制，无需人工配置
6. 当端口连接**集线器**时，由于集线器连接的所有设备均处于同一冲突域中，**必须使用 CSMA/CD 协议**，且只能工作在**半双工方式**
7. 交换机采用专用交换结构芯片，因此交换速率较高

### 工作原理

1. 交换机内部维护一张 **MAC 地址表**（交换表）
2. 收到数据帧后，提取源 MAC 和目的 MAC
3. 通过源 MAC → **自学习**（记录源 MAC 与端口的映射）
4. 通过目的 MAC → **查表转发**：
   - 命中 → 从对应端口**单播转发**
   - 未命中 → **泛洪**（除接收端口外的所有端口，交换表中已有内容的端口也要转发）
   - 广播/组播 → 不查询转发表，直接泛洪

## 交换机的自学习功能

### 自学习过程

交换机通过检查收到的每一帧的**源 MAC 地址**来学习：

1. **收到帧时**，记录 `源MAC地址 → 接收端口` 的映射关系
2. 将该映射写入 MAC 地址表，并**重置老化计时器**
3. 若源 MAC 已存在于表中但端口发生变化 → **更新端口号**
4. **老化机制**：每个表项有生存时间（通常 300s），到期未刷新则删除

为提升可靠性，网络中往往会在交换机之间增设**冗余链路**作为备份；但冗余链路会形成**环路**，导致帧在环路中循环转发，引发**广播风暴**和交换表震荡。为此需要使用**生成树协议**（STP），在逻辑上阻塞某些端口，将物理环路变为无环的逻辑树形拓扑，从而消除环路。

# 共享式以太网

## 共享式以太网的基本特点

共享式以太网是指以**集线器（Hub）** 为中心连接设备构建的局域网；所有主机通过双绞线接入集线器，**共享同一物理传输介质和总带宽**；从逻辑上看，整个网络构成**单一的局域网段**，所有通信均发生在同一冲突域和广播域；
### 核心特征

- **工作在物理层**：仅对电信号进行再生放大，不识别帧结构，无法识别 MAC 地址，也无任何智能转发或过滤功能
- **共享信道**：所有站点共享同一总带宽，效率低下（如 10Mbps 由所有站点均分）
- **半双工通信和 CSMA/CD 协议**：由于介质共享，同一时刻只能有一个站点发送数据，否则发生冲突；必须通过载波监听多点接入/碰撞检测来协调各站点发送
- **同一冲突域和广播域**：所有站点属于同一个冲突域，任意两台同时发送就会冲突
- **盲目广播所有帧**：任一站点发送的帧，其他所有站点都能收到，并自行判断是否接收；不仅浪费带宽，还存在通信隐私泄露的风险

正因上述缺陷，如今共享式以太网已被淘汰

## 共享式与交换式以太网的转发行为对比

### 核心差异一览

| 对比维度        | 共享式以太网（集线器）    | 交换式以太网（交换机）          |
| ----------- | -------------- | -------------------- |
| **工作层次**    | 物理层            | 数据链路层                |
| **转发依据**    | 电信号再生放大        | MAC 地址定向转发           |
| **通信方式**    | 半双工            | 支持全双工                |
| **带宽分配**    | 所有端口共享带宽       | 每个端口独占带宽             |
| **冲突域**     | 所有端口在同一冲突域     | 每个端口独立冲突域            |
| **广播域**     | 所有端口在同一广播域     | 所有端口在同一广播域（VLAN 可隔离） |
| **CSMA/CD** | 必须使用           | 全双工时不需要              |
| **帧过滤**     | 无（全部转发）        | 有（基于 MAC 地址选择性转发）    |
| **安全性**     | 低（所有站点都能看到所有帧） | 较高（单播帧只发到目的端口）       |
| **可扩展性**    | 差（冲突限制规模）      | 好（可级联/堆叠）            |

### 转发行为对比

**场景**：A 发送单播帧给 B

| 共享式（集线器） | 交换式（交换机） |
|------------------|------------------|
| 帧从集线器的一个端口进入 | 帧从交换机的一个端口进入 |
| 集线器将帧**复制到所有其他端口**（广播式转发） | 交换机查表找到 B 所在端口 |
| 所有站点（包括非目的站点）都收到该帧 | **仅 B 所在端口**收到该帧 |
| A 发送期间其他站点不能发送（半双工/CSMA/CD） | 其他端口对之间仍可并行通信 |

# 总结：数据链路层设备对照

| 设备 | 工作层 | 隔离冲突域 | 隔离广播域 | 转发依据 |
|------|--------|-----------|-----------|---------|
| 集线器（Hub） | 物理层 | ❌ 不隔离 | ❌ 不隔离 | 电信号（广播） |
| 网桥（Bridge） | 数据链路层 | ✅ 隔离 | ❌ 不隔离 | MAC 地址 |
| 交换机（Switch） | 数据链路层 | ✅ 隔离 | ❌ 不隔离 | MAC 地址 |
| 路由器（Router） | 网络层 | ✅ 隔离 | ✅ 隔离 | IP 地址 |

> **考试重点**：
> 1. 交换机/网桥隔离冲突域但不隔离广播域
> 2. 交换机的自学习过程（源 MAC 学习 + 目的 MAC 查表转发）
> 3. 共享式 vs 交换式的带宽分配和转发行为差异
> 4. 三种交换模式（存储转发/直通/无碎片）的特点
