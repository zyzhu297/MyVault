# 广域网的基本概念

> **广域网（WAN, Wide Area Network）**：覆盖范围从几十公里到几千公里，连接不同地区的局域网或城域网；主要任务是**长距离运送主机所发送的数据**

|  比较维度  |     广域网（WAN）     |          局域网（LAN）           |
| :----: | :--------------: | :-------------------------: |
|  覆盖范围  |    几十公里~几千公里     |           几米~几公里            |
|  数据速率  |  较低（Kbps~Gbps）   |     较高（10 Mbps~10 Gbps）     |
|  误码率   |        较高        | 较低（$10^{-11} \sim 10^{-8}$） |
| OSI 层次 | 三层：物理层、数据链路层、网络层 |        两层：物理层、数据链路层         |
|  拓扑结构  |      网状/不规则      |          总线、星形、环形           |
|  通信方式  |    点对点（存储转发）     |            广播/多播            |
|  使用协议  | PPP、HDLC、帧中继 |   以太网（IEEE 802.3）、令牌环（IEEE 802.5）、Wi-Fi（IEEE 802.11）    |
|  着重点   |      强调资源共享      |           强调数据传输            |
- **一个局域网可通过广域网与另一个局域网通信**
- 广域网和局域网都是互联网的重要构件，从互联网角度看，两者地位平等，不存在包含关系
- 当主机在广域网或局域网的内部通信时，仅需使用该网络的物理地址

---

> [!important] 广域网 vs 互联网
> - **概念层次不同**：广域网是**网络层以下**的通信子网概念，解决长距离数据传输；互联网是**网络的网络**，通过 IP 协议将各种异构网络互连
> - **协议层次**：WAN 工作在**物理层 + 数据链路层**；互联网的核心协议是**网络层的 IP 协议**
> - **所有权**：WAN 由电信运营商建设和管理；互联网无单一所有者，是全球范围的分布式网络
> - **地址体系**：WAN 使用数据链路层地址（如 HDLC 地址、PPP 无意义地址）；互联网使用 IP 地址
> - **关系**：广域网是互联网的**通信基础设施之一**，互联网通过路由器将**不同类型的网络**（LAN + WAN + MAN）互连起来

> [!note] 一句话辨析
> - **局域网（LAN）**：小范围、广播式、自建自管 → 以太网是典型
> - **广域网（WAN）**：跨区域、点对点、运营商建管 → 为互联网提供远程传输链路
> - **互联网（Internet）**：用 IP 协议把 LAN、WAN、MAN 等各种异构网络互连起来的"网络的网络"

## 广域网的组成

> 广域网由**通信子网**（传输网络）和**资源子网**（端系统）组成。通信子网的核心设备是**结点交换机（分组交换设备）**，负责**存储转发分组**

```
(端系统) ── 路由器 ── 广域网链路 ── 路由器 ── (端系统)
```

## 广域网的数据链路层协议

|    协议    | 全称                           | 特点                            | 408 考查 |
| :------: | ---------------------------- | ----------------------------- | :----: |
| **PPP**  | Point-to-Point Protocol      | 更简单，面向字节，支持多种网络层协议、认证、差错检测    |  ⭐⭐⭐   |
| **HDLC** | High-Level Data Link Control | 面向比特，同步传输，ISO 标准，现实中已被 PPP 取代 | 大纲已删除  |
| **帧中继**  | Frame Relay                  | 简化版 X.25，无差错和流量控制，检错即丢弃       |        |

> [!note] HDLC vs PPP
> - **HDLC** 是面向比特的同步协议，使用零比特填充法实现透明传输
> - **PPP** 是面向字节的协议，使用字节填充法实现透明传输
> - PPP 比 HDLC 更广泛用于因特网接入（如拨号上网、PPPoE）

# 点对点协议（PPP）⭐

> **PPP（Point-to-Point Protocol）**：一种**面向字节**的数据链路层协议，用于在两个结点之间建立直接的**点对点连接**；**是有连接不可靠服务**，是目前使用最广泛的广域网数据链路层协议。

在以太网中运行的 PPP 称为 PPPoE（PPP over Ethernet），它是对 PPP 的扩展；PPPoE 将 PPP 帧封装在以太网帧中，**用户通过 ADSL 宽带上网时使用的就是 PPPoE**

## PPP 的组成

> PPP 由三个部分组成：

|     组件      | 全称                             | 功能                                                         |
| :---------: | ------------------------------ | ---------------------------------------------------------- |
| **封装/成帧方式** | Encapsulation                  | 将网络层数据报封装成 PPP 帧，支持多种网络层协议（IP、IPX 等），其长度受**最大传送单元（MTU）**限制 |
| **链路控制协议**  | LCP (Link Control Protocol)    | 建立、配置、测试和终止数据链路连接，协商选项（如 MRU、认证方式）                         |
| **网络控制协议**  | NCP (Network Control Protocol) | 为不同的网络层协议建立和配置逻辑连接（如 IPCP 用于 IP、IPXCP 用于 IPX）              |

## PPP 帧格式

> PPP 帧格式类似于 HDLC 帧格式，但 PPP 是**面向字节**的，所有字段都是**整数个字节**。

![[PPP 帧的格式.png]]

|               字段               |  字节数  | 说明                                                     |
| :----------------------------: | :---: | ------------------------------------------------------ |
|         **标志 (Flag)**          |   1   | 定界符，固定值 **0x7E (01111110)**                            |
|        **地址 (Address)**        |   1   | 广播地址，固定值 **0xFF**（PPP 是点对点的，地址无意义）                     |
|        **控制 (Control)**        |   1   | 固定值 **0x03**（无序号帧 UI），表示 PPP 不使用编号                     |
|       **协议 (Protocol)**        |   2   | 标识信息字段中封装的数据类型（0x0021 → IP，0x8021 → IPCP，0xC021 → LCP） |
|      **信息 (Information)**      | ≤1500 | 网络层数据报，长度由 MRU 协商确定（默认 1500）                           |
| **FCS (Frame Check Sequence)** |   2   | CRC 帧检验序列，**只检错不纠错**                                   |
|         **标志 (Flag)**          |   1   | 结束定界符，0x7E                                             |

> [!tip] PPP 帧信息字段最小可为 0
> 由于 PPP 是点对点链路上（而非总线形网络），无需使用 CSMA/CD 协议，因此不存在最短帧长的限制，其信息字段最小可为 0B（相比之下，以太网帧的数据部分至少需 46B

### 透明传输

- 使用异步传输时，采用**字节填充法**，使用的转义字符为 **0x7D（01111101）**
- 使用同步传输时，采用**零比特填充法**，发送方每遇连续 **5 个 1** 后插入一个 **0**

## PPP 的工作过程

> 以用户拨号接入 ISP 的过程为例：用户拨号后，首先建立一条从用户到 ISP 的**物理层链接**；随后，用户向 ISP 发送一系列 LCP 分组（封装成 PPP 帧），协商链路参数并**建立 LCP 连接**；接着通过 NCP 配置网络层， ISP为用户分配一个临时 IP 地址；通信结束后，依次释放网络层、数据链路层和物理层连接；
![[PPP 的状态图|900]]
1. PPP 链路的起始与终止均为**链路静止**状态，此时用户与 ISP之间无物理层连接；
2. 用户主机通过**调制解调器**拨号，呼叫 ISP 的路由器
3. ISP 路由器的调制解调器**应答**，建立**物理连接**，双方调制解调器完成握手，物理层链路就绪，此时 PPP 进入**建立**状态
4. 用户主机向 ISP 路由器发送 **LCP 分组**（封装在 PPP 帧中），协商 PPP 参数（最大接收单元（MRU）、是否使用认证及认证方式（PAP/CHAP）、是否压缩等）协商成功后，LCP 链路建立，PPP 进入**鉴别**状态；若协商失败，则退回到**链路禁止**状态
5. 在**鉴别状态**下，若通信双方无需鉴别或身份鉴别成功，则进入**网络层协议状态**；若鉴别失败，则直接进入**链路终止**状态
6. 在**网络层协议**状态下，双方通过**NCP（IPCP）** 配置网络层参数（如分配 IP 地址），配置完成后，PPP 进入**链路打开状态**，双方即可开始数据通信
7. 数据通信完毕，若一方发送终止请求并在收到确认后，或链路发生故障，PPP 将进入**链路终止状态**；待调制解调器的载波信号停止后，最终返回**链路静止**状态

## PPP 的主要特点

| 特点            | 说明                               |
| ------------- | -------------------------------- |
| **不可靠服务**     | 不编号、不确认、不重传，只检错不纠错，交由上层处理        |
| **全双工点对点**    | 只支持点对点链路，**只支持全双工通信**            |
| **支持多种网络层协议** | 通过协议字段区分（IP、IPX 等），NCP 为每种协议单独配置 |
| **面向字节**      | 帧字段均为**整数个字节**，使用字节填充法实现透明传输     |

> [!important] PPP 不提供的功能
> - **不纠错**（只检错、丢弃，由上层处理）
> - **不提供流量控制**
> - **不提供可靠传输**（不使用序号和确认机制）
> - **不支持多点连接**（只支持点对点）
> - **不支持半双工或单工**（只支持全双工）

# HDLC 协议

> **HDLC（High-Level Data Link Control）**：ISO 制定的**面向比特**的数据链路层协议，支持同步传输和多种链路配置，是众多数据链路层协议的基础。

## HDLC 的三种站类型

| 站类型 | 英文 | 功能 |
|:---:|------|------|
| **主站** | Primary Station | 控制链路操作，发送命令帧 |
| **从站** | Secondary Station | 受主站控制，发送响应帧 |
| **复合站** | Combined Station | 兼具主站和从站功能，既可发送命令也可发送响应 |

## HDLC 的两种链路配置

| 配置类型 | 说明 | 站类型 |
|:---:|------|:---:|
| **非平衡配置** | 一个主站 + 多个从站 | 主站 + 从站 |
| **平衡配置** | 两个复合站点对点连接 | 复合站 + 复合站 |

## HDLC 的三种数据传送方式

| 方式 | 缩写 | 说明 |
|:---:|:---:|------|
| **正常响应方式** | NRM (Normal Response Mode) | 非平衡配置，从站只有收到主站轮询后才能发送 |
| **异步平衡方式** | ABM (Asynchronous Balanced Mode) | 平衡配置，双方均可随时发送 |
| **异步响应方式** | ARM (Asynchronous Response Mode) | 非平衡配置，从站可主动发送 |

## HDLC 帧格式

```
比特数:   8      8/16    8       16/变长      16       8
      ┌────────┬──────┬──────┬────────────┬───────┬─────────┐
      │  标志   │ 地址 │ 控制 │   信息      │ FCS   │ 标志    │
      │01111110│      │      │ (可变长)    │ (CRC) │01111110 │
      └────────┴──────┴──────┴────────────┴───────┴─────────┘
```

| 字段 | 说明 |
|:---:|------|
| **标志 F** | 定界符，固定 **01111110**（与 PPP 相同） |
| **地址 A** | 从站地址（命令帧）或发送站地址（响应帧），可扩展 |
| **控制 C** | 标识帧类型（信息帧 I / 监督帧 S / 无编号帧 U） |
| **信息 Info** | 网络层数据，仅 I 帧和部分 U 帧有 |
| **FCS** | CRC 校验，生成多项式 $G(x) = x^{16} + x^{12} + x^{5} + 1$ |

# PPP vs HDLC 对比总结

|   比较维度    |       PPP        |      HDLC       |
| :-------: | :--------------: | :-------------: |
| **面向方式**  |     **面向字节**     |    **面向比特**     |
|  **可靠性**  |    不可靠，无序号和确认    | 可靠（信息帧有序号和确认机制） |
|  **协议族**  |  LCP + NCP 分层设计  |      单一协议       |
|  **认证**   |   支持（PAP/CHAP）   |     不支持内置认证     |
| **多协议支持** | NCP 为每种网络层协议提供支持 |       有限        |
| **应用场景**  |    拨号接入、PPPoE    |   早期专线、帧中继底层    |
| **标准制定**  |       IETF       |       ISO       |
