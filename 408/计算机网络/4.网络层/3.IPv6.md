# IPv6 的特点

> **IPv6（Internet Protocol version 6）**：下一代互联网协议，从**根本上**解决 IPv4 地址耗尽问题。IPv6 地址长度从 32 位扩大到 **128 位**，地址空间是 IPv4 的 $2^{96}$ 倍，地球上每一粒沙子都可以分配一个 IP 地址。

> [!important] 为什么需要 IPv6？
> IPv4 的 32 位地址空间仅有约 43 亿个地址，2011 年 IANA 已将 IPv4 地址全部分配完毕。尽管 NAT、CIDR 等技术延缓了地址耗尽的进程，但这些只是"权宜之计"，IPv6 才是根本解决方案。

## IPv6 相比 IPv4 的主要改进

| 改进方面       |       IPv4        |               IPv6                | 说明          |
| ---------- | :---------------: | :-------------------------------: | ----------- |
| **地址空间**   |   32 位（约 43 亿）    | **128 位**（约 $3.4 \times 10^{38}$） | 彻底解决地址枯竭    |
| **首部格式**   | 可变长度（20~60 B），字段多 |         **固定 40 字节**，字段精简         | 提高路由器处理效率   |
| **分片机制**   |     路由器可执行分片      |        **仅源主机可分片**，路由器不分片         | 减轻路由器负担     |
| **首部校验和**  |     有（每跳重新计算）     |               **无**               | 减少每跳处理时间    |
| **选项机制**   |    选项置于首部可变部分     |        **扩展首部**（位于基本首部之后）         | 首部更简洁，扩展更灵活 |
| **QoS 支持** |   区分服务字段（8 bit）   |     **流标签**（20 bit），支持更好的 QoS     | 更适合实时多媒体    |
| **安全性**    |     可选（IPSec）     |       **内置 IPSec 支持**（扩展首部）       | 端到端安全性更好    |
| **自动配置**   |       DHCP        |   **无状态地址自动配置（SLAAC）** + DHCPv6   | 即插即用更便捷     |
| **NAT 需求** |     严重依赖 NAT      |      **原则上不需要 NAT**（恢复端到端通信）      | 简化网络结构      |

## IPv6 的其他重要特点

| 特点           | 说明                                                                                                                  |
| ------------ | ------------------------------------------------------------------------------------------------------------------- |
| **扩展的地址层次**  | IPv6 地址分为**网络前缀**和**接口标识**两级，支持多级地址层次结构（如全球路由前缀 → 子网 ID → 接口 ID），便于路由聚合和地址管理；相比 IPv4 的三级结构（网络号 + 子网号 + 主机号）更加灵活和可扩展 |
| **灵活的首部格式**  | 基本首部固定 40 字节，可选功能通过**扩展首部链**实现；扩展首部可灵活增删，且除逐跳选项外**中间路由器不处理**，既保证了功能可扩展性，又不拖慢转发速度                                    |
| **允许协议继续扩充** | 扩展首部机制使得 IPv6 协议本身**具有良好的可扩展性**——新增功能只需定义新的扩展首部类型，无需修改基本首部格式，保证向前兼容；本质原因是 IPv6 通过"下一个首部"字段形成链表，新协议可无缝接入             |
| **支持资源的预分配** | IPv6 的**流标签**字段（20 bit）允许源主机为属于同一"流"的分组打上标记，路由器可据此为特定流**预留带宽、缓存等资源**，为实时应用（如 VoIP、视频会议）提供 QoS 保障                    |

虽然 IPv4 与 IPv6 不兼容，但它与所有其他互联网协议（如 TCP、UCP、ICMP、IGMP 和 DNS 等）总体上兼容

> [!tip] IPv6 的设计哲学
> IPv6 的设计遵循"**简化路由器处理**"的原则：首部定长、校验和移除、路由器不分片，都是为了减少路由器每跳的处理开销，提高转发速度。

# IPv6 数据报的基本首部

> **IPv6 基本首部**：固定 **40 字节**，共 8 个字段（IPv4 首部有 13 个字段，其中部分功能被移除或移至扩展首部）。

![[IPv6 数据报的基本首部格式.png]]

## IPv6 基本首部的格式

| 字段 | 长度 | 说明 |
|------|:----:|------|
| **版本（Version）** | 4 bit | 指明 IP 协议版本，IPv6 = **6** |
| **通信量类（Traffic Class）** | 8 bit | 区分不同数据报的类别或优先级，与 IPv4 的区分服务字段功能类似，用于 QoS |
| **流标签（Flow Label）** | 20 bit | **IPv6 新增**，标记同一"流"的数据报序列，使路由器能对属于同一流的分组进行特殊处理（如实时音视频流） |
| **有效载荷长度（Payload Length）** | 16 bit | 基本首部**之后**的字节数（包括扩展首部 + 数据），最大 65535 字节 |
| **下一个首部（Next Header）** | 8 bit | 指明紧跟在基本首部之后的第一个扩展首部（或上层协议类型），功能类似 IPv4 的协议字段。若没有扩展首部，直接指明 TCP(6) 或 UDP(17) |
| **跳数限制（Hop Limit）** | 8 bit | 等同 IPv4 的 TTL 字段，每经过一个路由器减 1，减到 0 则丢弃 |
| **源地址（Source Address）** | 128 bit | 发送方 IPv6 地址 |
| **目的地址（Destination Address）** | 128 bit | 接收方 IPv6 地址（可以是最终目的地址或下一跳路由器的地址，取决于路由首部） |

> [!important] IPv6 首部的主要变化
> 与 IPv4 首部相比，IPv6 首部做了以下重要调整：
> - **取消了**：首部长度（定长 40B）、标识、标志、片偏移（移至分片扩展首部）、首部校验和（数据链路层和上层协议已有校验）、选项字段（移至扩展首部）
> - **新增了**：流标签（20 bit）
> - **重命名了**：TTL → 跳数限制，协议 → 下一个首部，总长度 → 有效载荷长度，服务类型 → 通信量类

## IPv6 的扩展首部

> **扩展首部（Extension Header）**：IPv6 将原来 IPv4 首部中的可选字段独立出来，放在基本首部和数据之间，形成扩展首部链。每个扩展首部通过"下一个首部"字段串联。

IPv6 扩展首部的处理方式：
- 扩展首部位于基本首部之后，上层协议数据之前
- 除**逐跳选项扩展首部**外，其他扩展首部仅由**源主机和目的主机**处理，**中间路由器不处理** — 这大幅减少了路由器的处理负担
- 多个扩展首部通过"下一个首部"字段形成**链表结构**
- 扩展首部的出现顺序有严格规定

| 扩展首部类型 | 说明 |
|-------------|------|
| **逐跳选项**（Hop-by-Hop Options） | 携带必须被路径上**每一个路由器**检查的信息（必须紧跟在基本首部之后） |
| **路由首部**（Routing Header） | 指定数据报必须经过的中间节点（源路由） |
| **分片首部**（Fragment Header） | 当源主机分片时使用，包含标识、片偏移和 MF 标志（**仅源主机可分片**） |
| **目的选项首部**（Destination Options） | 携带仅由目的主机检查的可选信息 |
| **认证首部**（Authentication Header, AH） | 提供数据完整性校验和源认证（IPSec 的一部分） |
| **封装安全有效载荷首部**（ESP Header） | 提供加密和可选的认证（IPSec 的一部分） |

> [!warning] 路由器不再分片
> IPv6 **不允许**中间路由器对数据报进行分片。如果数据报长度超过路径 MTU，路由器直接**丢弃**并返回 ICMPv6 "分组太大"差错报文。**分片仅由源主机**在发送前完成（通过 Path MTU Discovery 确定路径 MTU）。

# IPv6 地址

> **IPv6 地址**：128 位二进制数，通常用**冒号十六进制记法（Colon Hexadecimal）** 表示。IPv6 地址由**网络前缀**和**接口标识**两部分组成，不再区分 A/B/C 类。

## IPv6 地址的表示方法

### 基本表示（冒号十六进制记法）

将 128 位分为 8 组，每组 16 位，用 4 个十六进制数字表示，组间用冒号 `:` 分隔：

```
2001:0DB8:0000:0000:0000:FF00:0042:8329
```

### 缩写规则

| 规则        | 说明                                 | 示例                                        |
| --------- | ---------------------------------- | ----------------------------------------- |
| **省略前导零** | 每组内的前导零可以省略                        | `0042` → `42`，`0DB8` → `DB8`，`0000` → `0` |
| **零压缩**   | 连续的**全零组**可以用 `::` 代替（但**只能使用一次**） | `0000:0000:0000` → `::`                   |

使用缩写后：
```
2001:DB8::FF00:42:8329
```
（中间的三个全零组 `0000:0000:0000` 被 `::` 替代）

> [!warning] `::` 只能使用一次
> 如果有多个连续的全零组，`::` 只能压缩其中一个连续段，否则无法还原原始地址。例如 `2001:0:0:0:2:0:0:1` 只能缩写为 `2001::2:0:0:1` 或 `2001:0:0:0:2::1`，不能写成 `2001::2::1`。

### CIDR 前缀表示

IPv6 同样使用 CIDR 斜线记法表示网络前缀：

```
2001:DB8:1234::/48
```
表示前 48 位为网络前缀。

> [!tip] 嵌入 IPv4 的 IPv6 地址
> 过渡期间，IPv6 地址的最后 32 位可以嵌入一个 IPv4 地址，用点分十进制表示：
> ```
> ::FFFF:192.168.1.1
> ```
> 这种格式用于 IPv4/IPv6 混合环境。

## IPv6 地址的分类

IPv6 不再使用广播地址，而是用**组播**代替广播的功能。IPv6 地址分为三大类：

| 地址类型 | IPv4 对应 | 说明 |
|----------|:--------:|------|
| **单播（Unicast）** | 单播 | **一对一**通信，标识单个接口 |
| **组播（Multicast）** | 组播 | **一对多**通信，标识一组接口（取代了 IPv4 的广播） |
| **任播（Anycast）** | 无 | **IPv6 新增**，标识一组接口，数据报发往**最近的一个**（按路由协议度量值） |

> [!important] IPv6 取消了广播地址
> IPv6 中没有广播地址，广播的功能全部由**组播**代替。这样做的好处是减少了广播风暴的隐患，组播可以只发送给需要的节点。

## IPv6 单播地址的分类

| 类型                         | 地址格式                      | 说明                                      |
| -------------------------- | ------------------------- | --------------------------------------- |
| **全球单播地址**（Global Unicast） | `2000::/3`（当前分配）          | 等同于 IPv4 的公网地址，全球唯一，可在互联网上路由            |
| **链路本地地址**（Link-Local）     | `FE80::/10`               | 仅在同一链路上有效，**自动配置**生成，用于邻居发现等，**路由器不转发** |
| **唯一本地地址**（Unique Local）   | `FC00::/7`（`FD00::/8` 常用） | 等同于 IPv4 的私有地址，仅在本站点或组织内部使用             |
| **环回地址**（Loopback）         | `::1`                     | 等同于 IPv4 的 `127.0.0.1`，本机进程间通信          |
| **未指定地址**（Unspecified）     | `::`                      | 全 0 地址，等同于 IPv4 的 `0.0.0.0`，主机启动时使用     |

> [!tip] 链路本地地址
> IPv6 的链路本地地址 `FE80::/10` 是**每个启用了 IPv6 的接口自动配置**的地址，即使没有 DHCPv6 服务器或路由器，同一链路上的设备也能通过它进行通信。这为即插即用提供了基础。

## IPv6 组播地址

> **IPv6 组播地址**：格式为 `FF00::/8`，即以 `FF` 开头的地址均为组播地址，用于**一对多**通信。组播地址只能作为**目的地址**，不能作为源地址。

## IPv6 地址的配置方式

| 方式 | 说明 |
|------|------|
| **无状态地址自动配置（SLAAC）** | 主机根据路由器公告的前缀 + 自身接口标识（通常由 MAC 地址生成）自动生成 IPv6 地址，**无需 DHCP 服务器** |
| **DHCPv6** | 由 DHCPv6 服务器分配 IPv6 地址和其他配置信息（类似 IPv4 DHCP） |
| **静态配置** | 手动设置 IPv6 地址、前缀长度和默认网关 |

> [!info] SLAAC 是 IPv6 的一大亮点
> SLAAC（Stateless Address Autoconfiguration）使主机能够在没有 DHCP 服务器的情况下自动获取全球唯一的 IPv6 地址，实现真正的"即插即用"。主机通过 NDP（邻居发现协议）中的路由器请求（RS）和路由器公告（RA）消息获取网络前缀信息。

# 从 IPv4 向 IPv6 过渡

> IPv4 和 IPv6 将在相当长时间内**共存**。由于互联网规模庞大，不可能一夜之间全部切换到 IPv6。过渡策略的核心原则是：**逐步演进，保持互通**。

从 IPv4 向 IPv6 过渡的三种基本策略：

- **双协议栈（Dual Stack）**
	- 设备同时运行 IPv4 和 IPv6 两个协议栈，拥有 IPv4 和 IPv6 各一个地址
	- DNS 解析时优先选择 IPv6（AAAA 记录优先于 A 记录）
	- 应用最广泛，现代操作系统均默认支持

- **隧道技术（Tunneling）**
	- 将 **IPv6 数据报封装在 IPv4 数据报的数据部分**中传输（协议字段 = 41）
	- 用于连接跨越 IPv4 骨干网络的 IPv6 "孤岛"
	- 隧道两端路由器必须是双栈，中间 IPv4 网络对 IPv6 完全透明

- **协议转换 / NAT64**
	- 将 IPv6 数据报转换为 IPv4 数据报（反之亦然），需部署 NAT64 网关 + DNS64
	- 使纯 IPv6 主机能访问纯 IPv4 资源
	- 不仅转换地址，还要**改变 IP 协议版本**，开销较大

# ❗ 易错点总结

**IPv6 概述与特点**
1. IPv6 地址 **128 位**，地址空间是 IPv4 的 $2^{96}$ 倍；NAT 和 CIDR 只是 IPv4 的权宜之计
2. IPv6 首部**固定 40 字节**，共 8 个字段（精简是为了路由器快速处理）；IPv4 首部可变长 20~60 B
3. IPv6 **取消了首部校验和**（每跳重新计算太耗时，数据链路层和上层已有校验）、**取消了首部长度字段**（定长 40B）
4. IPv6 **取消了广播地址**，广播功能由**组播**代替；新增**任播地址**（发往最近的一个）

**IPv6 数据报**
5. **流标签**（20 bit）是 IPv6 新增字段，用于标记同一"流"的数据报，支持更好的 QoS
6. **下一个首部**字段连接扩展首部链；**只有逐跳选项扩展首部**被所有路由器处理，其他扩展首部只在两端处理
7. IPv6 **仅源主机可分片**，路由器不分片；超过路径 MTU 则丢弃并返回 ICMPv6 "分组太大"报文
8. IPv6 通过路径 MTU 发现机制确定分片大小；分片信息放在**分片扩展首部**中

**IPv6 地址**
9. 缩写规则：前导零可省略，连续全零组用 `::` 压缩但**只能用一次**
10. 链路本地地址 `FE80::/10`（自动配置，不跨路由器）；环回地址 `::1`（= `127.0.0.1`）
11. **SLAAC（无状态自动配置）** = 路由器公告前缀 + 接口标识自动生成地址，**无需 DHCP**
12. 全球单播地址 `2000::/3`；唯一本地地址 `FC00::/7`（= 私有地址）

**IPv4 → IPv6 过渡**
13. 三种过渡策略：**双协议栈**（同时支持两套协议）、**隧道技术**（IPv6 封装在 IPv4 中）、**NAT64**（协议转换）
14. **双协议栈**应用最广；**隧道技术**解决"孤岛互联"；**NAT64** 解决"最后一公里"互通
15. 隧道本质是**封装**：将 IPv6 数据报塞入 IPv4 数据报的数据部分；隧道两端路由器必须是双栈
16. DNS 解析时，双栈主机**优先使用 IPv6**（AAAA 记录优先于 A 记录）

**ICMPv6**
17. ICMPv6 整合了 ICMPv4、ARP、IGMP 的功能 → **NDP 替代 ARP**，**MLD 替代 IGMP**
18. NDP 使用五种报文：RS、RA、NS、NA、Redirect；其中 NS/NA 替代了 ARP 请求/响应
19. NDP 运行在 ICMPv6 之上，可利用 IPSec 防欺骗，比 ARP 更安全
20. ICMPv6 在 IPv6 中**不可或缺**，远比 ICMPv4 在 IPv4 中的重要性强
