# 移动 IP 的概念

> **移动 IP（Mobile IP）**：一种允许移动节点在改变网络接入点时，仍能使用其**永久 IP 地址（归属地址）** 与通信对端保持通信的技术。移动 IP 工作在**网络层**，对上层协议（TCP、UDP）透明——上层协议无需感知节点的移动。

> [!important] 为什么需要移动 IP？
> 在传统 IP 网络中，IP 地址同时具有**标识符**和**定位符**的双重作用——既标识主机身份，又指示主机所处的网络位置。当主机移动到另一个网络时，原 IP 地址的"定位符"部分失效，导致无法继续通信。移动 IP 通过引入**转交地址**，将标识与定位分离，解决了这一问题。

## 移动 IP 的关键术语

| 术语             | 英文                 | 说明                                                    |
| -------------- | ------------------ | ----------------------------------------------------- |
| **移动节点 (MN)**  | Mobile Node        | 可以从一个网络移动到另一个网络，且保持使用**归属地址**通信的主机                    |
| **归属网络**       | Home Network       | 移动节点**原始归属**的网络，其归属地址的网络前缀属于该网络                       |
| **归属地址**       | Home Address       | 移动节点在**归属网络**的永久 IP 地址，通信对端始终使用该地址与之通信                |
| **归属代理 (HA)**  | Home Agent         | 位于**归属网络**的路由器，负责截获发往移动节点的数据报，并通过隧道转发到移动节点当前所在的外地网络   |
| **外地网络**       | Foreign Network    | 移动节点**当前访问**的网络（非归属网络）                                |
| **外地代理 (FA)**  | Foreign Agent      | 位于**外地网络**的路由器，为移动节点提供路由服务，帮助解封来自归属代理的隧道数据，并转发给移动节点   |
| **转交地址 (CoA)** | Care-of Address    | ⭐移动节点在**外地网络**获得的临时 IP 地址，用于标识其当前位置。归属代理将数据报通过隧道发往该地址 |
| **通信对端 (CN)**  | Correspondent Node | 与移动节点通信的**对等实体**（可以是固定主机或另一个移动节点）                     |

> **转交地址**有两种方式获得：
> 1. **外地代理转交地址 (FA-CoA)**：使用外地代理的 IP 地址作为转交地址（多个移动节点可共享）
> 2. **共建转交地址 (Co-located CoA)**：移动节点通过 DHCP 等方式在外地网络自行获取一个临时 IP 地址，该地址仅供该节点独享

> [!warning] 关机搬家 ≠ 移动 IP
> 把电脑关机后从家里搬到办公室，**不属于移动 IP**。因为关机导致所有 TCP 连接已终止，到办公室重新开机后获取的是新 IP，网络会话从头开始——这只是普通的 **IP 地址重新配置**，移动 IP 的任何机制（代理发现、注册、隧道转发）都没有动用。
>
> 移动 IP 解决的是 **移动过程中保持通信不中断**——节点在物理位置改变的同时，TCP 连接不断、归属地址不变，上层协议完全无感知。关键区别在于：**连接是否需要"无缝"保持**。

# 移动 IP 通信过程

## 外地代理的两个重要功能

外地代理 FA 在移动 IP 中承担两个核心职责：

| 功能              | 说明                                                                          |
| --------------- | --------------------------------------------------------------------------- |
| **为移动节点提供转交地址** | 当使用 **FA-CoA** 方式时，FA 在自己的代理通告中宣告可供使用的转交地址（即 FA 自己的 IP 地址），移动节点将其作为注册时的转交地址 |
| **解封隧道并转发数据报**  | FA 收到归属代理通过隧道发来的封装数据报后，**解封**取出原始 IP 数据报，再根据目的地址（MN 的归属地址）将其**交付**给移动节点     |

>[!note]
>转交地址仅用于移动站、归属代理和外地代理之间的通信，**应用程序不会使用该地址**；
>外地代理向被访网络中的移动站发送 IP 分组时，直接通过其 MAC 地址进行链路层交付

## 移动 IP 基本通信流程

### 1. 本地通信

移动节点 MN 在**归属网络**时，完全按照**标准的 TCP/IP 协议栈**进行通信，不启用任何移动 IP 机制：

- MN 使用归属地址发送和接收数据报
- 归属代理 HA 不做任何干预
- 外地代理 FA 不参与

### 2. 漫游与注册

当 MN **漫游**到外地网络时，必须完成以下两步：

**（1）代理发现** —— MN 判断自己离开了归属网络：

| 方式 | 说明 |
|------|------|
| **被动发现** | 接收外地代理 FA 定期广播的 **ICMP 代理通告报文**，发现网络前缀与归属地址的网络前缀**不同**，判定自己在"外地" |
| **主动发现** | MN 发送 **ICMP 代理请求报文**，要求周围代理立即广播通告（避免被动等待） |

**（2）向归属代理注册转交地址**：

- MN 向外地代理 FA 发送**注册请求**（含归属地址、归属代理地址、转交地址）
- FA 将请求**转交地址**给归属代理 HA
- HA 验证后，在**绑定缓存**中记录"归属地址 ↔ 转交地址"的映射，返回**注册应答**
- FA 将应答转发给 MN，注册完成

> [!info] 绑定缓存 (Binding Cache)
> HA 维护的核心数据结构，每个条目包含：归属地址、转交地址、**注册有效期限**。若过期未续注册，条目自动删除，HA 不再为该 MN 做隧道转发。

### 3. 移动站接收数据（入方向：CN → MN）

这是**三角路由**的核心——通信对端 CN **不知道** MN 已经移动，数据报经过"绕道"抵达 

**关键步骤说明**：

| 步骤      | 操作                     | 要点                                                            |
| ------- | ---------------------- | ------------------------------------------------------------- |
| **(a)** | CN 用 MN 的**归属地址**发送数据报 | CN 完全不知 MN 已移动                                                |
| **(b)** | HA 截获后**隧道封装**         | HA 通过 **ARP 代理**截获帧；外层源地址=HA，外层目的地址=CoA（转交地址），协议号=4（IP-in-IP） |
| **(c)** | FA 解封并交付               | FA 解封取出原始数据报，查找访问者列表，交付给 MN                                   |

> [!important] ARP 代理是 HA 截获数据报的关键
> 当 CN 发送的数据报到达归属网络时，路由器需要知道 MN 的 MAC 地址才能将帧送达。HA 代替 MN **响应 ARP 请求**，用自己的 MAC 地址"冒充"MN，从而使发往 MN 的帧全部被 HA 接收。

### 4. 移动站发送数据（出方向：MN → CN）

MN 发送数据给 CN 时，**不经过 HA**，直接使用普通 IP 路由：

- **源地址**：MN 的**归属地址**（保持标识不变）
- **目的地址**：CN 的 IP 地址
- 数据报由外地网络的路由器按正常路由表直接转发，无需隧道

> [!important] 双向路径不对称
> 入方向（CN → MN）：CN → HA → FA → MN（三角路由）；出方向（MN → CN）：MN → CN（直接路由）。两者路径不对称，这正是**三角路由**得名的原因。

### 5. 移动与返回

MN 可能在不同网络之间持续移动，或最终回到归属网络：

**（1）移动到新的外地网络**：
- MN 在新网络重新进行代理发现 → 获取新转交地址 → 向 HA 重新注册
- HA 更新绑定缓存，后续隧道发往新 CoA

**（2）返回归属网络**：
- MN 回到归属网络，收到 HA 的代理通告，发现网络前缀与自己相同
- MN 向 HA 发送**注销请求**，HA 删除绑定缓存条目
- MN 恢复正常 IP 通信模式，不再使用移动 IP

## 网络层增加的四个新功能

移动 IP 的引入，使得网络层在标准 IP 协议之上**新增了四个功能**：

| 新功能               | 英文                                        | 说明                                                                                                          |
| ----------------- | ----------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| **移动站到外地代理的登记协议** | MN-FA Registration Protocol               | 移动节点到达外地网络后，与外地代理 FA 之间完成**本地交互**——MN 向 FA 发送登记请求，FA 为 MN 分配或确认转交地址，并为后续向归属代理的正式注册做好准备。这是 MN 进入外地网络后最先执行的步骤 |
| **代理发现**          | Agent Discovery                           | 移动节点通过接收代理通告或主动发送代理请求，判断自己是在归属网络还是外地网络，并发现可用的外地代理及转交地址。代理发现是登记协议执行的**前提**                                   |
| **注册**            | Registration                              | 移动节点通过 FA 将当前的转交地址告知归属代理 HA，使 HA 建立绑定缓存，能够将发往 MN 的数据报正确转发到 MN 当前所在位置；注册信息有**有效期**，过期需续注册                    |
| **隧道封装与解封**       | Tunneling / Encapsulation & Decapsulation | HA 将发往 MN 归属地址的原始 IP 数据报**整个封装**进新的 IP 数据报（外层目的地址=CoA）；外地代理 FA（或 MN 自己）收到后**剥离外层首部**，还原出原始数据报交付给 MN         |

> [!tip] 四个功能的关系
> **代理发现**是前提（知道自己已离开归属网络）→ **移动站到 FA 的登记协议**是入口（与外地代理建立本地联系、获取转交地址）→ **注册**是准备（通过 FA 告知 HA 当前转交地址）→ **隧道封装与解封**是执行（把数据穿越网络送到处于外地的 MN）。四个功能协同工作，使得移动 IP 对上层协议**完全透明**。

# 总结

| 要点 | 说明 |
|------|------|
| **核心问题** | 移动后原 IP 地址"定位符"意义失效 |
| **解决方案** | 引入**转交地址 CoA**实现标识与定位分离 |
| **FA 的两个功能** | 提供转交地址；解封隧道并转发数据报 |
| **五个通信流程** | 本地通信 → 漫游与注册 → 接收数据（入）→ 发送数据（出）→ 移动与返回 |
| **四个新增功能** | 移动站到 FA 登记协议、代理发现、注册、隧道封装与解封 |
| **三角路由** | CN → HA → FA → MN，双向路径不对称 |
| **MIPv4 vs MIPv6** | MIPv4 依赖 FA；MIPv6 支持共建 CoA、内置路由优化 |
