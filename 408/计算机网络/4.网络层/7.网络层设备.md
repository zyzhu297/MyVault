# 冲突域和广播域

> **冲突域**和**广播域**是理解网络设备工作原理的两个核心概念，它们决定了不同网络设备（集线器、交换机、路由器）对网络流量的隔离能力

## 冲突域（Collision Domain）

> **冲突域**：同一时间内只能有一台设备发送数据的网络区域。如果两台设备同时发送数据，就会发生**冲突（Collision）**，数据被破坏，需要重传。

| 要点       | 说明                            |
| -------- | ----------------------------- |
| **产生原因** | 共享介质（如同轴电缆、集线器）上，多个节点竞争同一物理信道 |
| **工作机制** | 采用 CSMA/CD 协议检测冲突并退避重传        |
| **影响**   | 冲突域越大，冲突概率越高，网络效率越低           |

## 广播域（Broadcast Domain）

> **广播域**：网络中能接收同一广播帧的所有节点的集合。一台设备发送广播帧（目的 MAC 地址为 FF-FF-FF-FF-FF-FF），该帧能被广播域内的所有其他设备收到。

| 要点       | 说明                                     |
| -------- | -------------------------------------- |
| **产生来源** | ARP 请求、DHCP 发现等协议会发送广播帧                |
| **影响**   | 广播域过大时，广播帧泛洪会消耗大量带宽和 CPU 资源，形成**广播风暴** |
| **隔离方式** | 只有**网络层设备（路由器）**能隔离广播域                 |

## 三种设备对冲突域和广播域的影响

> 这是**高频考点**，也是理解三个层次设备差异的关键。

| 设备 | 工作层次 | 隔离冲突域 | 隔离广播域 |
|:----:|:--------:|:----------:|:----------:|
| **集线器（Hub）** | 物理层 | ❌ 不隔离 | ❌ 不隔离 |
| **交换机（Switch）** | 数据链路层 | ✅ **隔离**（每个端口一个冲突域） | ❌ 不隔离（默认所有端口在同一广播域） |
| **路由器（Router）** | 网络层 | ✅ **隔离** | ✅ **隔离**（每个端口一个广播域） |

> [!important] 关键结论
> - **集线器**：所有端口处于**同一冲突域**和**同一广播域**
> - **交换机**：每个端口是一个**独立的冲突域**，但默认所有端口处于**同一广播域**（VLAN 可分割广播域，但这是二层技术，考试中默认交换机不隔离广播域）
> - **路由器**：每个端口既是**独立的冲突域**，又是**独立的广播域**——路由器**不会转发广播包**，从根本上隔断了广播域

> [!tip] 记忆技巧
> 网络层次越高，隔离能力越强：
> - 物理层设备：什么都隔离不了
> - 数据链路层设备：隔离冲突域，不隔离广播域
> - 网络层设备：冲突域和广播域都能隔离

# 路由器的组成和功能

> **路由器（Router）**：工作在**网络层**的核心设备，用于连接不同的网络并实现分组转发。路由器本质是一台**专用的计算机**，由硬件和软件组成，核心功能是**路由选择**和**分组转发**；从模型角度来看，路由器是工作在网络层的设备，其转发决策基于 IP 地址；为完成此功能，其端口还要具备物理层和数据链路层的处理能力

## 路由器的基本结构

路由器主要由两大部分组成：**路由选择部分**（控制平面）和**分组转发部分**（数据平面）。

### 1. 输入端口（Input Port）

> 完成**物理层**比特流接收、**数据链路层**解封装提取 IP 分组，并根据目的 IP 查转发表确定输出端口。端口处设有**缓冲区**，用于暂存等待处理的到达分组，同时可在此进行**差错检测**。

### 2. 输出端口（Output Port）

> 将 IP 分组重新封装成帧，转换为比特流从输出链路发送。端口处设有**缓冲区**，若到达分组速率超过链路带宽则需排队等待，缓冲区满时到达的分组将被**丢弃**。

### 3. 交换结构（Switching Fabric）

> **交换结构**是连接输入端口和输出端口的"内部总线"，负责将分组从输入端口高速转移到输出端口。

| 交换方式 | 说明 | 特点 |
|----------|------|------|
| **通过内存交换** | 分组存入共享内存，CPU 查表后将分组复制到输出队列 | 最原始，速度受内存带宽限制 |
| **通过总线交换** | 输入端口通过共享总线将分组直接传输到输出端口 | 同一时间只能有一个分组通过总线，存在总线争用 |
| **通过交叉开关交换** | 每条输入到输出之间有一条独立通路（N×N 矩阵开关） | 支持多个分组同时传输，速度快但成本高 |

### 4. 路由选择处理机（Routing Processor）

> 路由选择处理机执行**路由协议**，与邻近路由器交换路由信息，计算并维护路由表。属于**控制平面**功能，通常由**软件（CPU）** 执行，不像转发那样需要纳秒级速度。

| 功能 | 说明 |
|------|------|
| **运行路由协议** | 与邻居路由器交换路由信息（RIP、OSPF、BGP 等） |
| **计算路由表** | 根据路由算法计算到达各目的网络的最佳路径 |
| **维护路由表** | 感知拓扑变化，动态更新路由表 |
| **导出转发表** | 将路由表中的最优路由导出为供硬件高速查表的转发表 |

## 路由器的核心功能

| 功能 | 说明 | 所属平面 |
|------|------|:--------:|
| **路由选择（Routing）** | 根据路由协议和算法为分组选择端到端传输路径 | 控制平面 |
| **分组转发（Forwarding）** | 根据转发表将分组从输入端口转移到输出端口 | 数据平面 |
| **异构网络互联** | 连接使用不同协议、技术的网络 | —— |
| **广播隔离** | 路由器不转发广播帧，每个端口是一个独立广播域 | —— |
| **安全与过滤** | 通过 ACL（访问控制列表）对分组进行过滤 | —— |
| **NAT 转换** | 实现私有地址和公有地址之间的转换 | —— |

> [!warning] "路由器不会从分组入口转发"
> 这是一个重要的基本规则：路由器收到一个分组后，**不会从接收该分组的入口再转发回去**。即如果转发表查找结果显示下一跳在同一条链路上，路由器**不会将分组原路返回**。这是为了避免路由环路和无意义的重复传输。

## 路由器与交换机的对比

| 比较维度 | 路由器 | 二层交换机 |
|:--------:|:------:|:----------:|
| **工作层次** | 网络层（第三层） | 数据链路层（第二层） |
| **转发依据** | **目的 IP 地址** | **目的 MAC 地址** |
| **转发单位** | IP 分组（数据报） | 帧（Frame） |
| **寻址方式** | IP 地址（逻辑地址） | MAC 地址（物理地址） |
| **广播隔离** | ✅ 隔离广播域 | ❌ 不隔离广播域 |
| **异构网络互联** | ✅ 可以 | ❌ 不可以 |
| **路由表/转发表** | 有（路由协议动态生成） | 有（MAC 地址表，自学习） |
| **分组修改** | 修改 TTL、校验和、MAC 地址（NAT 时还修改 IP） | 不修改帧内容，仅转发 |
| **主要用途** | 连接不同网络（广域网/互联网） | 连接同一网络内的主机（局域网） |

> [!info] 三层交换机
> **三层交换机** = 二层交换 + 三层路由，可以理解为"带路由功能的交换机"。它既有二层交换的高速转发能力，也能基于 IP 地址进行三层转发（通常使用硬件 ASIC），但功能没有路由器全面。**三层交换机能隔离广播域吗？** 能——因为它具备 IP 路由功能，不会转发广播帧到其他 VLAN/子网。

# 路由表与分组转发

> 路由表是路由器的"地图"，分组转发是路由器按"地图"执行的动作。**转发表由路由表得出**，其表项与路由表项有直接对应关系；但两者设计目标不同：**路由表应使路由计算优化**，便于动态更新和拓扑变化处理；**转发表**应使高速查询优化，用于实际的数据转发过程，

## 转发与路由选择的区别

> **转发**（数据平面）和**路由选择**（控制平面）是路由器中两个相互独立但又紧密配合的核心功能。

|    比较维度    |      转发（Forwarding）       |     路由选择（Routing）      |
| :--------: | :-----------------------: | :--------------------: |
|  **所属平面**  |         **数据平面**          |        **控制平面**        |
|   **作用**   |      将分组从输入端口转移到输出端口      |     为分组确定端到端的传输路径      |
|  **核心操作**  |        查转发表，匹配输出端口        |     运行路由协议，计算最佳路径      |
|  **时间尺度**  |    **纳秒级**（每个分组到达即执行）     |  **秒~分钟级**（周期性或触发式更新）  |
|  **实现方式**  |        **硬件**也可软件         |   **软件**（CPU 运行路由协议）   |
| **核心数据结构** | **转发表**（Forwarding Table） | **路由表**（Routing Table） |
|  **关注范围**  |      单跳（当前路由器 → 下一跳）      |   全局（源端 → 目的端的整条路径）    |

> [!tip] 一句话区分
> - **路由选择**：决定"走哪条路"（规划地图）
> - **转发**：执行"把分组送出去"（按地图走）
> - 路由选择为转发生成转发表，转发依赖转发表高速执行

## 路由表的结构

路由表的每条记录通常包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **目的网络地址** | 分组要到达的目标网络（可以是网络地址、子网地址或主机地址） | `192.168.1.0` |
| **子网掩码** | 与目的网络地址配合，确定网络前缀长度 | `255.255.255.0`（或 `/24`） |
| **下一跳地址** | 到达目的网络需要经过的下一个路由器的 IP 地址 | `10.0.0.1` |
| **输出接口** | 从路由器的哪个端口将分组发出 | `eth0`、`GigabitEthernet0/1` |
| **度量值（Metric）** | 到目的网络的代价，用于比较不同路由的优劣 | 跳数、带宽、延迟等 |
| **管理距离（AD）** | 标识路由来源的可信度，值越小越可靠 | 直连 0、静态 1、OSPF 110、RIP 120 |

### 路由的类型

| 类型 | 说明 | 管理距离（典型值） |
|:----:|------|:-----:|
| **直连路由** | 路由器端口直接连接的网络，自动生成 | 0 |
| **静态路由** | 由管理员手动配置的路由项 | 1 |
| **动态路由** | 由路由协议（RIP、OSPF、BGP 等）自动学习到的路由 | 90~200 |
| **默认路由** | `0.0.0.0/0`，匹配所有目的地址，作为"兜底"路由 | 可配置 |

### 路由优先级规则

当到达同一目的网络有多条路由时，选择顺序为：

1. **最长前缀匹配（Longest Prefix Match）**：掩码越长（网络前缀越长）、越精确的表项优先
2. **管理距离（Administrative Distance）**：前缀长度相同时，AD 越小越优先（不同协议来源的比较）
3. **度量值（Metric）**：同一协议的多条路由中，度量值越小越优先

## 分组转发的完整流程

|     步骤      | 操作                           | 说明                                                  |
| :---------: | ---------------------------- | --------------------------------------------------- |
|  **① 接收帧**  | 输入端口物理层接收比特流，组装成帧            | 进行 CRC 校验，出错则丢弃                                     |
|  **② 解封装**  | 去除帧头和帧尾，提取 IP 分组             | 数据链路层 → 网络层                                         |
| **③ 校验与处理** | 检查 IP 首部校验和，递减 TTL           | **TTL 减 1，若 TTL=0 则丢弃并发送 ICMP 超时报文**                |
| **④ 查转发表**  | 提取目的 IP 地址，在转发表中执行**最长前缀匹配** | 这是分组转发最核心的一步                                        |
| **⑤ 确定出口**  | 根据匹配到的表项获取"下一跳 IP 地址"和"输出接口" | 若未匹配到（含默认路由），则丢弃分组                                  |
| **⑥ 封装成帧**  | 将 IP 分组重新封装为帧，填入源/目的 MAC 地址  | 源 MAC = 出接口 MAC，目的 MAC = 下一跳路由器接口 MAC（或最终目的主机的 MAC） |
|  **⑦ 发送帧**  | 输出端口物理层将帧转换为比特流发送到链路         | 分组继续在互联网中逐跳传输                                       |

### 路由器对分组的修改

> [!important] 路由器**必须修改**分组首部的以下字段：
>
> - **TTL（生存时间）**：每经过一个路由器减 1
> - **首部校验和（Header Checksum）**：因 TTL 改变，需**重新计算**校验和（IPv6 无此字段，因已取消首部校验和）
>
> 在 NAT 场景下，路由器还会修改**源 IP 地址**或**目的 IP 地址**。

> [!info] 路由器**不会修改**的内容：
> - IP 分组的**数据部分**（payload）
> - 传输层的端口号（除非启用了 NAPT/PAT）

> [!note] 关键点回顾
> - **MAC 地址每跳都变**（源 MAC = 当前发出接口 MAC，目的 MAC = 下一跳接收接口 MAC）
> - **IP 地址始终不变**（源 IP 和目的 IP 在传输过程中不改变，除非经过 NAT）
> - **TTL 每跳减 1**，为 0 时丢弃

# ❗ 易错点总结

1. **集线器不隔离冲突域和广播域**，交换机隔离冲突域但不隔离广播域（默认），**路由器两者均隔离**
2. 路由器的**路由选择**（控制平面）和**分组转发**（数据平面）是两个独立的功能，前者靠软件/CPU，后者靠硬件/TCAM
3. **路由表** ≠ **转发表**：路由表由路由协议生成，转发表由路由表导出并针对高速查找优化
4. **最长前缀匹配**：CIDR 下转发时选择掩码最长的匹配路由，而非默认路由或聚合路由
5. 路由器**不会从分组进入的端口将分组转发回去**
6. 分组经过路由器时，**源/目的 IP 不变**（除非 NAT），**MAC 地址每跳改变**，**TTL 每跳减 1**
7. **三层交换机**同时具备二层的交换和三层的路由能力，可以隔离广播域（因为能路由）
8. **路由器不转发广播帧**，这是它隔离广播域的底层原因——如果查表发现目的地址是广播地址，路由器直接丢弃
9. 转发分组时，路由器可能对 IP 分组执行**分片**操作（若输出链路 MTU 小于分组长度且 DF=0）
