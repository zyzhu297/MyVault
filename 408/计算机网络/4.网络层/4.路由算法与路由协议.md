# 路由算法

路由算法是网络层确定数据包从源到目的地传输路径的核心机制，是路由选择协议的核心。路由算法负责**为路由器生成并维护路由表**，找到源路由器到目的路由器的**最佳**路径，通常最佳路径指**费用最低的路径**，该费用可根据跳数、带宽、延迟等因素定义

### 路由算法的分类

| 分类方式 | 类型 | 说明 |
|---------|------|------|
| **按决策方式** | 静态路由 | 人工手动配置，适用于小型、稳定的网络，开销小但无法自适应拓扑变化 |
| | 动态路由 | 路由器自动发现路由并更新，适用于大型、变化的网络 |
| **按信息范围** | 全局信息（链路状态算法） | 每个节点拥有完整网络拓扑，如 OSPF |
| | 分散信息（距离向量算法） | 每个节点仅知邻居信息，如 RIP |
| **按适应能力** | 静态自适应 | 路由随时间缓慢变化 |

## 距离向量（Distance-Vector, DV）算法

> **核心思想**：每个路由器维护到每个目的地的距离向量，定期与**直接邻居**交换整个路由表。

### Bellman-Ford 方程

$$d_x(y) = \min_v \{ c(x, v) + d_v(y) \}$$

- $d_x(y)$：节点 $x$ 到节点 $y$ 的最小代价
- $c(x, v)$：$x$ 到邻居 $v$ 的直接链路代价
- 遍历所有邻居 $v$，选择代价最小的路径

### DV 算法的关键性质

#### 1. 只需要局部信息

DV 算法是**分布式、异步**的。每个节点**仅需与直接邻居通信**，不需要掌握全局网络拓扑：

- 每个节点**只维护**到所有目的地距离的估计值（距离向量）
- 节点**只与直接邻居交换**路由信息，无需泛洪
- 当本节点路由表变化时，通知所有邻居；否则不发送更新（触发更新）
- 这种局部性使得 DV 算法实现简单、开销小，但也是收敛慢的根源

#### 2. 最优子结构（Bellman 方程的理论基础）

> **两点之间的最短路径，一定包含该路径上任意子路径的最短路径。**

设节点 $x$ 到 $y$ 的最短路径为 $x \to v_1 \to v_2 \to \cdots \to y$，则对于路径上的任意中间节点 $v_k$：
- $x$ 到 $v_k$ 的子路径一定是 $x$ 到 $v_k$ 的最短路径
- $v_k$ 到 $y$ 的子路径一定是 $v_k$ 到 $y$ 的最短路径

**为什么成立？** 反证法：若存在更短的子路径，替换后能得到一条比原来更短的 $x \to y$ 路径，与"原路径最短"矛盾。

**与 Bellman-Ford 方程的关系**：
$$d_x(y) = \min_v \{ c(x, v) + d_v(y) \}$$

该方程正是最优子结构的直接体现：$x$ 到 $y$ 的最短距离 = 从所有邻居 $v$ 中找一个，使得"$x$ 到 $v$ 的直接代价 + $v$ 到 $y$ 的最短代价"最小。其中 $d_v(y)$ 就是子问题的最优解。

#### 3. 每个节点的路由信息

在距离向量算法中，每个节点 $x$ 维护以下数据结构：

| 数据结构                      | 说明                                                                        |
| ------------------------- | ------------------------------------------------------------------------- |
| **距离向量（Distance Vector）** | $D_x = [D_x(y_1), D_x(y_2), \dots, D_x(y_n)]$，从 $x$ 到每个目的节点 $y$ 的当前最小代价估计 |
| **路由表（Routing Table）**    | 每条表项包含：`<目的网络, 距离（跳数/代价）, 下一跳路由器>`                                        |
| **邻居代价**                  | $c(x, v)$：从 $x$ 到每个直接邻居 $v$ 的链路代价                                         |
| **邻居的距离向量**               | 存储从每个邻居收到的距离向量**副本**，用于本地计算                                               |

**路由表的更新逻辑**：节点 $x$ 收到邻居 $v$ 发来的距离向量后，对每个目的地 $y$ 计算：
- $D_x(y) = \min(c(x, v) + D_v(y),\ \text{当前} D_x(y))$
- 若更新了 $D_x(y)$，则下一跳改为 $v$

### 算法特点

- **好消息传得快**：链路代价减小时，迅速收敛
- **坏消息传得慢（无穷计数问题）**：链路断开后，路由环路导致距离缓慢增长到"无穷大"
- **更新报文的大小与网络节点数成正比**，大型网络中可能产生较大的通信开销

> [!note] 收敛（Convergence）
> **收敛**是指网络中所有路由器的路由表达成一致（即路由信息稳定不变）的状态或过程。路由算法从网络拓扑变化到所有路由器重新计算出正确路由的时间称为**收敛时间**。
> - **收敛快** = 网络不稳定时间短，路由环路的持续时间短
> - **收敛慢** = 长时间处于不一致状态，可能导致路由环路、丢包
> - DV 算法（如 RIP）收敛慢是其主要缺点，也是无穷计数问题的表现

### 无穷计数问题的解决

| 方法 | 原理 |
|------|------|
| **毒性逆转（Poisoned Reverse）** | 如果 $z$ 通过 $y$ 到达 $x$，则 $z$ 告诉 $y$：$D_z(x) = \infty$ |
| **定义最大跳数** | RIP 中最大跳数为 15，16 表示不可达 |
| **水平分割（Split Horizon）** | 不将从某接口学到的路由再从该接口发送回去 |
| **触发更新** | 路由变化时立即发送更新，不等周期性更新 |
| **抑制计时器** | 路由失效后等待一段时间再接受新路由，避免使用错误信息 |

> **注意**：毒性逆转仅对涉及两个节点的环路有效，三个及以上节点的环路仍可能发生。

## 链路状态（Link-State, LS）算法

> **核心思想**：每个节点通过泛洪获得完整网络拓扑，使用 [[3.图的应用#Dijkstra 求单源最短路径|Dijkstra 算法]]独立计算最短路径。

### 每个节点执行的两项任务

1. **主动探测**：每个节点主动探测与所有直接邻居的连通性和链路代价，封装为**链路状态通告（LSA）**
2. **定期泛洪**：将 LSA 洪泛（Flooding）到全网，同时按固定周期（如 OSPF 每 **30 分钟**）刷新。洪泛时通过**序号**和**生存期（Age）** 防止重复转发和过期信息残留

> 洪泛完成后，每个节点拥有相同的完整拓扑（LSDB），各自独立运行 Dijkstra 算法计算出**天然无环路**的路由表。

### LS 算法的主要优点

1. **各节点独立路由决策，不依赖邻居**：每个节点基于全局 LSDB 独立运行 Dijkstra 计算路由表，路由选择不依赖于邻居传递的距离向量，因此**不会出现无穷计数问题**
2. **可获得完整网络拓扑**：洪泛使每个节点最终拥有全网一致的拓扑视图（LSDB），路由计算基于全局信息而非局部估计，**且无环**
3. **链路状态报文仅包含本节点的直接链路信息**：每次交换的 LSA 只携带发送节点的本地邻接关系，报文大小与邻居数量相关，而不像 DV 交换整个路由表（大小与全网节点数成正比）
4. **收敛迅速**：链路状态变化立即洪泛触发全网重算，不需要 DV 的逐跳等待
5. **在大网络中有更好的可扩展性**：通过划分区域（Area），LSDB 规模和 SPF 计算量限制在区域内，辅以区域间路由汇总，使 LS 协议（如 OSPF）能支持大规模网络

### LS 算法的问题

- **震荡（Oscillation）**：当链路代价与流量相关时，路由可能反复切换，导致震荡
  - **解决**：让路由器在不同时间运行 LS 算法，或避免基于流量的链路代价

### DV 与 LS 算法对比

| 特性 | 距离向量（DV） | 链路状态（LS） |
|------|---------------|---------------|
| **信息范围** | 仅邻居 | 全网泛洪 |
| **交换内容** | 整个路由表 | 链路状态信息 |
| **计算方式** | 分布式 Bellman-Ford | 集中式 Dijkstra |
| **收敛速度** | 慢（无穷计数） | 快 |
| **健壮性** | 错误会传播全网 | 错误影响有限 |
| **复杂度** | 简单、开销小 | 复杂，$O(n^2)$ |
| **典型协议** | RIP | OSPF |

# 分层次的路由选择协议

## 自治系统（Autonomous System, AS）

> **自治系统（AS）**：由**单一机构管理**、使用**统一路由策略**的一组路由器集合。每个 AS 拥有全球唯一的 **AS 号（ASN）**。

### 为什么要分层？

- **规模问题**：互联网路由器数量巨大，所有路由器维护全局路由表不现实
- **管理自治**：不同 ISP/组织希望独立管理内部网络，隐藏内部拓扑
- **策略需求**：不同网络间需要基于商业关系的路由策略

### 路由协议的层次划分

| 层次        | 协议类型                                   | 说明                | 典型协议             |
| --------- | -------------------------------------- | ----------------- | ---------------- |
| **AS 内部** | 内部网关协议（IGP, Interior Gateway Protocol） | 在同一个 AS 内部运行，关注效率 | RIP, OSPF, IS-IS |
| **AS 之间** | 外部网关协议（EGP, Exterior Gateway Protocol） | 在不同 AS 之间运行，关注策略  | BGP-4            |

自治系统之间的路由选择也称**域间路由选择**，自治系统内部的路由选择也称**域内路由选择**
### 层次路由的转发过程

1. 源主机 → 源 AS 内转发 → 到达**网关路由器（Gateway Router）**
2. 网关路由器使用 EGP 将数据包转发到目标 AS
3. 目标 AS 内使用 IGP 转发到目的主机

# 路由信息协议

> **RIP（路由信息协议，Routing Information Protocol）**：基于**距离向量**算法的内部网关协议（IGP），使用**跳数（Hop Count）** 作为度量，是一种**分布式、基于距离向量的**路由协议

## RIP 的规定

- **使用跳数作为度量**：RIP 以**跳数（Hop Count）** 衡量到达目的网络的距离，每条链路代价为 1。**RIP 认为"好"的路由是经过路由器数量最少（即跳数最少）的路由**，不考虑带宽、延迟等其他因素
- **一条路径最多 15 个路由器**：RIP 规定最大跳数为 **15**，跳数 **16 表示网络不可达**。这决定了 RIP 仅适用于小型网络，网络直径不超过 15 跳
- **仅和相邻路由器交换信息**：RIP 路由器只与**直接邻居**交换路由信息，不向全网洪泛
- **交换完整路由表**：交换的内容是路由器所知的**全部路由信息**（自己的完整路由表）
- 网络中的每个路由器**都需维护一个距离向量**，即其到所有目的网络的距离记录
- **周期性交换**：按固定时间间隔交换路由信息，默认每 **30 秒**一次（存在 25~35s 随机抖动以避免同步）
- **属于应用层协议**：RIP 报文使用 **UDP** 端口 **520** 封装，由应用层进程处理

### 每个路由器维护的信息

每个 RIP 路由器维护一张**路由表**，每条表项包含三个关键字段：

| 关键字段 | 说明 |
|---------|------|
| **目的网络** | 数据包要到达的目标网络地址 |
| **距离（跳数）** | 到达目的网络需要经过的路由器数量（即跳数），最优情况下直接交付跳数为 0 或 1 |
| **下一跳路由器** | 到达目的网络应转发的下一个路由器地址 |

> "距离"实际为该路由器到目的网络所经过的路由器数目，**最少跳数才是"最短距离"**。

## RIP 基本工作原理

### 距离向量交换

RIP 基于距离向量算法：每个路由器维护到每个目的网络的距离（跳数），并与**直接邻居**周期性交换路由表。

### 路由更新规则

1. 收到邻居发来的 RIP 报文（路由表）
2. 将报文中每条路由的**跳数 +1**（经过一跳到达邻居）
3. 与自身路由表比较：
   - 若该网络**不在**路由表中 → **添加**该路由（表示发现新网络）
   - 若**在**路由表中，且下一跳相同 → **更新**（无论新旧值大小，要以**最新**的消息为准）
   - 若**在**路由表中，且下一跳不同：
     - 新跳数 **<** 旧跳数 → **更新**（找到更优路径）
     - 新跳数 **>=** 旧跳数 → 保持
1. 若连续 180 秒（RIP 默认超时时间）未收到某相邻路由器的更新报文，则将其标记为**不可到达**，即将对对应路由项的距离设置为 16（表示不可达）

### RIP 的定时器

| 定时器                        | 时长   | 作用                             |
| -------------------------- | ---- | ------------------------------ |
| **更新定时器（Update Timer）**    | 30s  | 周期性发送路由更新（平均 25~35s 随机抖动，避免同步） |
| **无效定时器（Invalid Timer）**   | 180s | 超时未收到某路由更新 → 标记为不可达（跳数 = 16）   |
| **刷新定时器（Flush Timer）**     | 240s | 不可达路由彻底删除                      |
| **抑制定时器（Hold-down Timer）** | 180s | 抑制新路由的接收，防止路由震荡                |

# RIP 的优缺点

**优点**：
- 实现简单，开销小，收敛过程较快
- 适用于小型网络
- **好消息传得快**：网络中出现更短路径时（如新增链路或链路代价降低），路由更新信息能够迅速传播，网络快速收敛

**缺点**：
- **坏消息传得慢（无穷计数问题）**：链路断开或路由器故障时，路由信息更新缓慢，导致路由环路长时间存在，收敛极慢

> [!example] 坏消息传得慢示例
> 三个路由器串联：**A — B — C**。初始时 B 到 A 距离=1，C 到 A 距离=2（经 B）。
>
> **当 A-B 链路断开后**：
> 1. B 检测到 A 不可达，但恰好此时收到 C 的路由表（C 说"我到 A 距离=2"）
> 2. B 心想："C 能到 A，那我经 C 到 A 距离=2+1=**3**"（形成环路 B→C→B→A?）
> 3. B 更新后通知 C，C 又更新为 3+1=**4**，再通知 B...
> 4. 如此反复，距离值从 1 → 3 → 4 → 5 → ... 缓慢增长，直至 **16（不可达）** 才停止
>
> 这就是"无穷计数"——本应立即标记为不可达，却因环路慢慢数到 16。

- 最大跳数 15，不支持大型网络
- 周期性交换整个路由表，浪费带宽
- 仅以跳数为度量，不考虑带宽/延迟等

# 开放最短路径优先协议（Open Shortest Path First，OSPF）

## OSPF 的特点

1. **使用泛洪法**：OSPF 采用**洪泛法（Flooding）** 向全网发送路由信息——路由器收到 LSA 后向**除接收接口外的所有接口**转发，确保信息到达每一台路由器
2. **交换的信息是链路状态通告（LSA）**：OSPF 交换的不是完整路由表，而是**链路状态通告（LSA）**，LSA 仅包含发送节点与**直接邻居**的链路状态和代价，报文大小与邻居数量相关
3. **和所有路由器交换**：与 RIP 仅和直接邻居交换不同，OSPF 通过洪泛法向**全网所有路由器**发送 LSA，最终每台路由器拥有相同的 LSDB
4. **触发更新 + 定期刷新**：OSPF 在**链路状态发生变化时立即洪泛**，同时每 **30 分钟**定期刷新一次所有 LSA，防止过期信息残留
5. **属于网络层协议**：OSPF 报文直接封装在 **IP 数据报**中（协议号 = **89**），不使用 UDP 或 TCP

| 特性 | 说明 |
|------|------|
| **算法类型** | 链路状态（LS） |
| **度量标准** | 链路代价（Cost），通常与带宽相关：$Cost = 10^8 / \text{带宽(bps)}$ |
| **封装方式** | 直接使用 **IP 数据报**（协议号 = **89**） |
| **交换内容** | **链路状态通告（LSA, Link State Advertisement）** |
| **交换方式** | **洪泛法（Flooding）** 向所有路由器发送 |
| **交换时机** | 链路状态变化时触发更新，同时每 30 分钟刷新一次 |

> [!warning]
> OSPF 是**网络层协议**（协议号 89），而 RIP 是**应用层协议**（UDP 520），BGP 是**应用层协议**（TCP 179）。

## OSPF 基本工作原理

OSPF 的工作过程可概括为四个阶段：

### 1. 邻居发现（Neighbor Discovery）与可达性维护

- 路由器启动后，在所有 OSPF 接口上**每 10 秒**周期性发送 **Hello 报文**（组播地址 224.0.0.5），该报文称为**问候分组**
- 收到邻居的 Hello 后，若双方参数匹配（Area ID、Hello 间隔、认证等），则建立**邻居关系**
- 邻居关系为**双向连通**后，进入下一阶段
- **可达性维护**：若连续 **40 秒**（4 倍 Hello 间隔）未收到某邻居的 Hello 报文，则判定该邻居**不可达**，触发拓扑变更通告和路由重算

### 2. LSDB 初始同步（Database Synchronization）

> 该阶段为 **OSPF 启动或新邻居加入时**的一次性初始同步过程。

邻居关系建立后，双方的 LSDB 可能存在差异，需要同步至一致：

1. 双方通过 **DD 报文**交换各自 LSDB 的摘要（LSA 头部列表），以主从模式确定序列号
2. 比较摘要后，用 **LSR 报文**向对方请求本地缺少或版本较旧的 LSA
3. 对方用 **LSU 报文**将所请求的 LSA 完整发送过来
4. 接收方回复 **LSAck 报文**确认收到

> 同步完成后，两台路由器的 LSDB 达到一致，进入 **Full（完全邻接）** 状态。此后仅链路状态变化时才进行增量洪泛，不再需要全量初始同步。

### 3. 链路状态变更的实时传播

LSDB 同步完成后，每台路由器独立运行 **Dijkstra 算法**计算最短路径树（SPF Tree），生成路由表。此后 OSPF 进入稳定运行阶段——**一旦链路状态发生变化，立即触发实时传播**：

- 检测到变化的 router 生成新的 LSA，**立即洪泛**至全网
- 所有收到新 LSA 的路由器更新本地 LSDB，**重新运行 SPF** 重算路由
- 这种"触发式"机制使得路由变更**秒级收敛**，不需要像 RIP 一样等待周期性更新

> **关键**：OSPF 的"实时"体现在变化驱动的洪泛，而非定时——这正是链路状态算法收敛远快于距离向量算法的根本原因。

### 4. 持续维护（Ongoing Maintenance）

路由收敛后的维持阶段：

- **Hello 保活**：周期性发送 Hello 监测邻居状态（广播网络 10s，NBMA 30s），超时未收到则宣告邻居失效
- **触发更新**：链路状态变化时，立即生成新的 LSA 并洪泛到全网，路由器收到后重新运行 SPF 重算路由
- **定期刷新**：即使无变化，每 **30 分钟**刷新一次 LSA（Age 归零重新洪泛），防止过期信息残留。每条 LSA 的最大生存时间为 **60 分钟**（MaxAge），若 Age 达到 60 分钟仍未刷新，该 LSA 将被删除

> [!summary] OSPF 工作流程总结
> OSPF 的完整生命周期围绕四个阶段展开：
> 1. **邻居发现与可达性维护**：通过周期性 Hello 报文建立并维持邻居关系
> 2. **LSDB 初始同步**：DD 摘要交换 → LSR 请求 → LSU 响应 → LSAck 确认，达成 LSDB 一致
> 3. **链路状态变更的实时传播**：链路变化立即洪泛新 LSA，全网秒级重算路由，实现快速收敛
> 4. **持续维护**：Hello 保活 + 触发洪泛 + 30min 定期刷新，保持网络稳定运行


## OSPF 核心概念

### 链路状态数据库（LSDB）

- 每个路由器维护一个**链路状态数据库（LSDB）**，存储全网所有 LSA
- 同一区域内的所有路由器具有**相同的 LSDB**
- 基于 LSDB 使用 Dijkstra 算法计算以自己为根的**最短路径树（SPF Tree）**

### Hello 协议

OSPF 通过**Hello 报文**建立和维护邻居关系：

| 功能 | 说明 |
|------|------|
| **发现邻居** | 组播 Hello 报文到 224.0.0.5（AllSPFRouters） |
| **选举 DR/BDR** | 在多路访问网络中选举指定路由器 |
| **维持邻居** | 周期性发送 Hello（默认 10s/30s），4 倍间隔未收到则邻居失效 |

### 几种 OSPF 路由器类型

| 类型 | 说明 |
|------|------|
| **内部路由器（IR）** | 所有接口属于同一个区域 |
| **区域边界路由器（ABR）** | 连接主干区域和其他区域 |
| **骨干路由器（BR）** | 至少有一个接口在主干区域 Area 0 |
| **自治系统边界路由器（ASBR）** | 连接 OSPF 域和其他 AS（引入外部路由） |

## OSPF 区域的划分

> **区域（Area）**：将一个 AS 划分为若干区域，每个区域独立运行 LS 算法，**减少 LSA 洪泛范围和 LSDB 大小**

### 区域类型

| 区域类型                            | 说明                     |
| ------------------------------- | ---------------------- |
| **主干区域（Backbone Area）**         | Area 0，所有其他区域必须连接到主干区域 |
| **标准区域（Standard Area）**         | 普通区域                   |
| **末梢区域（Stub Area）**             | 不接受外部路由（Type 5 LSA）    |
| **完全末梢区域（Totally Stubby Area）** | 不接受外部路由和区域间汇总路由        |
| **非完全末梢区域（NSSA）**               | 允许引入有限外部路由（Type 7 LSA） |

### 划分区域的好处

- 区域内 LSA 洪泛只影响本区域，**减少了路由开销**
- 区域间通过**路由汇总（Summarization）**，缩小路由表
- **隔离网络不稳定**，某区域震荡不影响其他区域

## OSPF 报文类型（5 种）

| 名称 | 英文全称 | 作用 |
|------|---------|------|
| **Hello** | Hello | 发现邻居、选举 DR/BDR、维持邻居关系，确认链路双向连通性 |
| **数据库描述（DD）** | Database Description | 发送 LSDB 摘要信息，用于主从协商和 LSDB 同步 |
| **链路状态请求（LSR）** | Link State Request | 向邻居请求特定的 LSA |
| **链路状态更新（LSU）** | Link State Update | 携带具体 LSA 信息进行洪泛，是最核心部分 |
| **链路状态确认（LSAck）** | Link State Acknowledgment | 对收到的 LSU 进行可靠确认 |

## OSPF 的优缺点

**优点**：
- **支持为每条链路设置不同代价**：管理员可根据带宽、延迟等因素灵活指定每条链路的 Cost，实现流量工程和路径优化
- **支持等价多路径负载均衡**：当存在多条到同一目的网络且代价相同的路径时，OSPF 可将流量均匀分摊到这些路径上（ECMP, Equal-Cost Multi-Path）
- **支持鉴别功能**：OSPF 报文可携带认证信息（明文或 MD5），防止非法路由器注入虚假路由信息
- **支持可变长子网掩码（VLSM）和无分类编址（CIDR）**：OSPF 携带子网掩码信息，支持 VLSM 和 CIDR，适应现代 IP 编址需求
- **LSA 携带 32 位序号**：每个 LSA 带有 32 位递增序号，路由器只接受序号更大的新版 LSA，有效防止过期或重复的链路状态信息
- **收敛速度快**：链路状态变化立即洪泛触发全网重算
- **无环路**：基于全局 LSDB 运行 Dijkstra，天然避免路由环路
- **支持区域划分，可扩展性好**
- **开销与全网规模无直接关联**：LSA 仅包含本节点直连链路信息，报文大小取决于邻居数量而非全网节点数；区域划分使单台路由器的 LSDB 和 SPF 计算量仅与所在区域规模相关，不受全网规模增长影响（对比 RIP 每次需交换完整路由表）

**缺点**：
- 算法复杂，对路由器 CPU 和内存要求高
- 维护 LSDB 需要较大内存
- 初始 LSDB 同步开销大

## OSPF vs RIP 总结

| 特性 | RIP | OSPF |
|------|-----|------|
| **算法** | 距离向量（DV） | 链路状态（LS） |
| **协议层次** | 应用层（UDP 520） | 网络层（IP 协议号 89） |
| **度量** | 跳数 | 链路代价（带宽相关） |
| **交换内容** | 完整路由表 | LSA（链路状态信息） |
| **交换对象** | 仅邻居 | 全网洪泛 |
| **更新时机** | 30s 周期 | 触发 + 30min 刷新 |
| **收敛速度** | 慢 | 快 |
| **规模** | 小（≤15 跳） | 大（支持分层区域） |
| **路由选择** | 仅比较跳数 | 基于代价，可通过修改 cost 实现流量工程 |

# 边界网关协议

## BGP 概述

> **BGP（Border Gateway Protocol）**：AS 之间运行的**外部网关协议（EGP）**，是**不同自治系统**的路由器之间交换路由信息的协议，当前版本为 **BGP-4**，是互联网的核心路由协议。

### 为什么需要 BGP？

- IGP（RIP/OSPF）优化基于**性能指标**（跳数、代价）
- **AS 间路由需要基于策略**：商业关系（peer/transit）、流量工程、安全性
- IGP 无法扩展到全球互联网规模
- 不同 AS 之间需要独立路由决策，不能暴露内部拓扑
- AS 之间的路由选择必须考虑政治、安全或经济等有关因素

因此 BGP 只能力求寻找一条能够达到目的网络且比较好的路由（不能兜圈子），并非要寻找一条最佳路由；BGP 采用乐**路径向量**路由选择协议，它与**距离向量**协议（如 RIP）和链路状态协议（如 OSPF）都有很大的区别；**BGP 是应用层协议，基于 TCP 实现**

### 基本特征

| 特性 | 说明 |
|------|------|
| **算法类型** | **路径向量（Path Vector）**算法（DV 的变体） |
| **传输协议** | **TCP** 端口 **179** |
| **协议层次** | 应用层协议 |
| **交换内容** | **路由信息 + AS-PATH 属性** |
| **交换对象** | **BGP 对等体（Peer）** |
| **更新方式** | 仅增量更新（只发送变化的路由，不周期性更新整个表） |

>[!warning]
>BGP 是**应用层协议**（基于 TCP 179），但服务于网络层路由

## 路径向量（Path Vector）算法

### 与距离向量的关键区别

|           | 距离向量（RIP）    | 路径向量（BGP）                   |
| --------- | ------------ | --------------------------- |
| **通告内容**  | 到目标网络的距离（跳数） | 到目标网络的**完整 AS 路径**（AS-PATH） |
| **选路依据**  | 仅跳数          | **策略优先**（基于 AS-PATH 等多种属性）  |
| **环路避免**  | 毒性逆转 / 最大跳数  | 检查 AS-PATH 中是否已有自己的 AS      |
| **接收路由后** | 与自身表比较       | 经过**输入策略引擎**决策              |

### BGP 环路避免

> 路由器在收到路由时检查 AS-PATH 是否包含自己的 AS 号。若包含则**丢弃**该路由，防止环路。

## BGP 路由

BGP 使用丰富的路径属性来支持灵活的路由策略。

### BGP 路由的一般格式

BGP 路由器通过 **UPDATE 报文**通告路由信息，每条路由的格式为：

$$
\text{BGP 路由} = (\text{前缀},\ \text{BGP 属性}) = (\text{前缀},\ \text{AS-PATH},\ \text{NEXT-HOP})
$$

- **前缀**：目的网络的 IP 前缀及子网掩码长度，如 `192.168.0.0/16`
- **AS-PATH**（自治系统路径）：到达目的网络所经过的 AS 序列，如 `AS100 → AS200 → AS300`，每个自治系统由一个全局唯一的**自治系统号（ASN）标识**，BGP 路由没经过一个 AS，就向其 ASN 中加入 AS-PATH；可见，BGP 路由明确指出**所经过的 AS 路径**，但不指明具体经过哪些路由器
- **NEXT-HOP**（下一条）：到达目的网络的下一个 AS 边界路由器 IP 地址，如 `10.0.0.1`

> BGP 路由的核心特征在于携带了**完整的 AS-PATH**——这与 RIP（仅携带跳数）和 OSPF（仅携带本地链路状态）完全不同。AS-PATH 既用于选路策略，也用于环路检测。

### AS 中两类功能不同的路由器

在 BGP 体系下，一个 AS 内的路由器按功能分为两类：

- **边界路由器（Border Router / ASBR）**：位于 AS 边缘，同时运行 **IGP（如 OSPF）和 EGP（BGP）**，负责与其他 AS 交换路由信息。边界路由器将学到的外部路由通过 IGP 通告给内部路由器
- **内部路由器（Internal Router）**：位于 AS 内部，**仅运行 IGP**，不运行 BGP。内部路由器通过 IGP 获知到达外部网络需经由哪台边界路由器，无需维护全局 AS 路径信息

> 这种分工使内部路由器免受全球路由表膨胀的影响——内部只需知道"出 AS 走哪个边界"即可。

### BGP 的半永久 TCP 连接

BGP 路由器之间通过 **TCP 端口 179** 建立一条**半永久（semi-permanent）的 TCP 连接**：

- "半永久"意味着连接建立后**长期保持**，不会在路由交换完成后拆除，而是持续用于后续的路由更新和维护
- 这与 RIP/UDP（无连接）和 OSPF/IP（无连接洪泛）的通信模式**根本不同**——BGP 是有连接的、可靠的
- TCP 的可靠性保证了 BGP 无需像 OSPF 那样自己实现 ACK 和重传机制，同时也使得 BGP 仅需发送**增量更新**

### eBGP 和 iBGP

根据对等体是否属于同一 AS，BGP 会话分为两种：

|           | **eBGP（External BGP）**       | **iBGP（Internal BGP）**                |
| --------- | ---------------------------- | ------------------------------------- |
| **对等体关系** | 位于**不同 AS** 之间               | 位于**同一 AS** 内部                        |
| **关键区别**  | 通告路由时**添加自己的 AS 号**到 AS-PATH | 通告路由时**不修改** AS-PATH                  |
| **连接方式**  | 通常**物理直连**（TTL=1）            | **不要求物理直连**，可跨多跳，通过 IGP 可达即可（TTL=255） |

> **iBGP 水平分割**：从 iBGP 学到的路由不得再传给其他 iBGP 对等体。这是为了避免路由信息丢失——若不设此限制，某 iBGP 对等体学到的路由被转发给另一个对等体时，该路由可能被误认为是来自转发者而非原始来源，导致原始路由信息丢失。由于水平分割的存在，AS 内必须 **iBGP 全互联**（任意两台之间都建会话）

## BGP 路由选择

当 BGP 路由器存在多条到达同一目的网络的路由时，按以下优先级顺序选择最佳路径：

### 1. 选择本地偏好值（LOCAL-PREF）最高的路由

**LOCAL-PREF** 是 AS 内部设置的策略属性，值越大越优先。它反映了本地 AS 对出口点的偏好——管理员可通过调节 LOCAL-PREF 控制出站流量走向，即使高速链路过载，BGP 也无法自动将部分流量切换到空闲的低速链路

### 2. 选择 AS 跳数最少（AS-PATH 最短）的路由

若 LOCAL-PREF 相同，则比较 **AS-PATH 长度**，选择经过 AS 数量最少的路由。这类似于 RIP 的跳数最小原则，但度量的是 AS 数而非路由器数，AS 跳数最少的路由未必是最好的

### 3. 使用热土豆路由选择算法

若前两项相同，则比较到达 **NEXT-HOP** 的 IGP 代价，选择距离最近的出口边界路由器。所谓"热土豆"——将数据包**像烫手土豆一样尽快扔出本 AS**，选择离自己最近的出口，这时需要使用**内部网关协议**（如 RIP 或 OSPF），即使该出口离最终目的地更远。

> 热土豆路由 = 最小化本 AS 内的转发代价，而非端到端路径最优。

### 4. 选择 BGP 标识符（Router ID）数值最小的路由

若以上均相同，在 BGP 报文的首部有一个称为 **BGP 标识符的字段**，该字段是运行 BGP 路由器的唯一标识符，则选择 **BGP Router ID 数值最小**的路由作为最终决策，确保选路结果的确定性（打破平局）。

>[!summary] BGP 选路优先级
>LOCAL-PREF → AS-PATH 长度 → 热土豆（最近 NEXT-HOP）→ Router ID。核心思想是**策略优先，路径次之，性能最后**，与 IGP 的纯性能导向形成鲜明对比。

## BGP 的四种报文

当 BGP 刚启动时，BGP 对话的两端需要交换完整的 BGP 路由表；此后，仅在路由发生变化时发送更新部分；这种方式有助于节省网络带宽并降低路由器的处理开销
BGP 定义了四种报文类型，均基于 TCP 179 可靠传输：

| 报文                     | 作用                                                                                                                 |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **OPEN（打开报文）**         | 与相邻 BGP 路由器建立对等体关系，协商 BGP 版本、AS 号、Hold Time（保持时间）等参数。会话建立后进入 Established 状态                                        |
| **UPDATE（更新报文）**       | BGP 最核心的报文，用于**通告新的可达路由**或**撤销不可达路由**。新增路由时**一次只能加一条**（携带前缀和路径属性），撤销路由时**一次可撤销多条**                                 |
| **KEEPALIVE（保活报文）**    | 周期性发送（默认 **60 秒**），只包含 BGP 报文头（19 字节），用于确认对等体仍在线，防止 Hold Timer 超时断开；若在约定的**保持时间**（通常位 180 秒）内位收到任何 BGP 报文，则认为对方已失效 |
| **NOTIFICATION（通知报文）** | 检测到错误（如 Hold Timer 超时、报文格式错误、不支持的参数等）时发送，发送后**立即关闭 BGP 会话**                                                        |

> **408 重点**：OPEN 建立连接，UPDATE 传递路由信息，KEEPALIVE 维持会话（60s），NOTIFICATION 报错并关闭连接。BGP 只需增量更新，无需周期性交换完整路由表。

## 三种路由协议总结

| 特性       | RIP      | OSPF      | BGP                |
| -------- | -------- | --------- | ------------------ |
| **类型**   | IGP      | IGP       | EGP                |
| **算法**   | 距离向量（DV） | 链路状态（LS）  | 路径向量（PV）           |
| **协议层次** | 应用层      | 网络层       | 应用层                |
| **传输层**  | UDP 520  | IP 协议号 89 | TCP 179            |
| **交换内容** | 完整路由表     | 链路状态通告（LSA） | 路由信息 + 路径属性（AS-PATH 等） |
| **度量**   | 跳数       | 链路代价      | 路径属性（策略）           |
| **更新**   | 周期性（30s） | 触发更新      | 增量更新               |
| **收敛**   | 慢        | 快         | 较慢（策略计算）           |
| **适用**   | 小型网络     | 大中型企业网络   | 互联网 AS 间           |
| **关注点**  | 距离/效率    | 效率/拓扑     | 策略/商业关系            |
| **最大跳数** | 15       | 无限制       | 无限制（AS-PATH 长度有限制） |

# ❗ 易错点总结

**路由算法基础**
1. 路由算法负责生成和维护路由表，最佳路径 = **费用最低的路径**；费用可基于跳数、带宽、延迟等
2. 静态路由 = 人工配置（小网），动态路由 = 自动发现（大网）；全局信息 → LS（OSPF），分散信息 → DV（RIP）

**DV 算法**
3. 核心方程 $d_x(y) = \min_v\{c(x,v)+d_v(y)\}$，理论基础是**最优子结构**——最短路径的任意子路径也是最短的
4. DV 仅需**局部信息**（与直接邻居交换），分布式异步；每个节点维护距离向量 + 路由表 + 邻居代价
5. **好消息传得快，坏消息传得慢（无穷计数）**：链路断开后环路导致距离缓慢增长到 16
6. 五种缓解方法：毒性逆转、最大跳数、水平分割、触发更新、抑制计时器。毒性逆转仅对两节点环路有效
7. 收敛 = 全网路由表达成一致的过程；收敛时间越短越好；DV 收敛慢是其主要缺点

**LS 算法**
8. 两个任务：**主动探测**（封装 LSA）+ **定期泛洪**（全网洪泛，序号 + Age 防重复）；泛洪后全网 LSDB 一致
9. LS 优点：独立决策不依赖邻居（无无穷计数）、完整拓扑（无环）、LSA 仅含本地信息、收敛快、可分层扩展
10. 每个节点基于**相同的 LSDB** 各自运行 Dijkstra → **天然无环路**路由表
11. LS 问题：流量相关代价可能导致**震荡**——错开运行时间或避免基于流量的代价

**RIP**
12. **RIP 是应用层协议**（UDP 520），OSPF 是网络层（IP 89），BGP 是应用层（TCP 179）——常考点！
13. RIP 以**跳数**为度量，"好路由" = 跳数最少；最大 15 跳，16 = 不可达
14. 路由表三项关键字段：**目的网络、距离（跳数）、下一跳路由器**
15. 更新规则：收到路由 → 跳数 +1 → 不在表中则添加、同下一跳则更新（以最新为准）、不同下一跳则选跳数更小的
16. 四大定时器：更新 30s、无效 180s、刷新 240s、抑制 180s
17. RIPv1 有类广播，RIPv2 无类组播 + CIDR + 认证

**OSPF**
18. OSPF 五特点：泛洪法、交换 LSA、和所有路由器交换、触发 + 30min 刷新、网络层协议
19. 基本工作四阶段：Hello 邻居发现（10s 问候 / 40s 不可达）→ DD/LSR/LSU/LSAck 同步 LSDB → 链路变更实时洪泛 → Hello 保活 + 触发 + 30min 刷新
20. 五种报文：Hello、DD、LSR、LSU、LSAck；LSA MaxAge = **60 分钟**
21. OSPF 优点：每条链路设置代价、ECMP 负载均衡、鉴别认证、VLSM/CIDR、32 位序号、收敛快、无环、区域划分
22. 开销与**全网规模无直接关联**：LSA 大小取决于邻居数，区域限制了 LSDB/SPF 范围

**BGP**
23. BGP 路由格式：**(前缀, AS-PATH, NEXT-HOP)**；携带完整 AS 路径，区别于 RIP（仅跳数）和 OSPF（仅本地链路）
24. BGP 是**路径向量**协议：通告完整 AS-PATH → 检查 AS 号防环路，策略优先而非性能优先
25. BGP 选路四步：**LOCAL-PREF 最高 → AS-PATH 最短 → 热土豆（最近出口）→ Router ID 最小**
26. 热土豆路由 = 尽快扔出本 AS，选择离自己最近的出口，端到端路径不一定最优
27. BGP 半永久 TCP 连接（端口 179），可靠性由 TCP 保证，仅需**增量更新**
28. EBGP（不同 AS，直连 TTL=1，添加 AS 号）vs IBGP（同一 AS，不要求直连 TTL=255，不修改 AS-PATH）
29. iBGP 水平分割：iBGP 学到的路由不得再传其他 iBGP → **必须全互联**（每两台建会话），大型 AS 用 RR 或联盟优化
30. 四类报文：OPEN（建会话）、UPDATE（增一条 / 撤多条）、KEEPALIVE（60s）、NOTIFICATION（报错即关闭）

**三种协议对比**
31. RIP 交换**完整路由表**，OSPF 交换 **LSA**，BGP 交换**路由信息 + 路径属性**
32. 协议层次：RIP（应用层 UDP 520）、OSPF（网络层 IP 89）、BGP（应用层 TCP 179）
33. RIP 适合小网（跳数），OSPF 适合大中型（链路代价），BGP 适合 AS 间（策略）
