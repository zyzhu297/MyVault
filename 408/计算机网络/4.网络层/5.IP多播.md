# 多播的概念

**多播 (Multicast)**：一对多通信，源主机只发送一份数据，由路由器在分岔口复制转发，只有属于多播组的成员才能收到。

与单播和广播的对比：

| 通信方式 | 特点 | 缺点 |
|----------|------|------|
| **单播 (Unicast)** | 一对一，每个接收者独立发送 | 重复发送浪费带宽 |
| **广播 (Broadcast)** | 一对所有，子网内全部主机接收 | 不相关主机也被迫处理 |
| **多播 (Multicast)** | 一对多，只发送给加入组的成员 | 需要路由器支持 |

**多播的关键特征**：
- **多播组**：用 D 类 IP 地址标识，主机可随时加入或退出
- **尽力交付**：多播数据报使用 IP 协议，不保证可靠到达（与单播 IP 一样）
- **不产生 ICMP 差错**：多播数据报出错时不发送 ICMP 差错报文
- **不能用于产生 ICMP 报文的数据报**：ping 多播地址无意义

**多播节省带宽的原理**：若用单播向 $n$ 个接收者发送，需 $n$ 条相同数据流；多播只在网络分岔口复制，核心链路只需一条流。

# IP 多播地址

**D 类地址范围**：`224.0.0.0` ~ `239.255.255.255`（不分网络号和主机号），可支持 $2^{28}$ 个多播组

**分类**：

| 地址范围 | 用途 |
|----------|------|
| `224.0.0.0 ~ 224.0.0.255` | 预留的局域网多播地址（TTL=1） |
| `224.0.1.0 ~ 238.255.255.255` | 全球范围可用的多播地址 |
| `239.0.0.0 ~ 239.255.255.255` | 本地管理（私有）多播地址 |
| `224.0.0.0 ~ 224.0.0.255` | 链路本地多播（路由器不转发，TTL=1） |

> **注意**：224.0.0.0 ~ 224.0.0.255 的数据报 TTL 通常为 1，路由器不会转发到局域网外。

- 多播数据报也是“尽最大努力交付”，**不提供可靠交付**
- 多播地址**只能用于目的地址**，而不能用于源地址
- 对多播数据报**不产生 ICMP 差错报文**

>[!note] 多播机制仅适用于 UDP
>因为它能将同一报文同时发送给多个接收者；而 TCP 是面向连接的协议，建立的是一对一的端到端连接，无法支持一对多的多播通信

# 在局域网上进行硬件多播

由于**局域网支持硬件多播**，只需将 IP 多播地址映射为对应的多播 MAC 地址，并将 IP 多播数据报封装在以太网帧中；
**IANA 为多播分配的以太网 MAC 地址范围**：`01-00-5E-00-00-00` ~ `01-00-5E-7F-FF-FF`

**IP 多播地址 → MAC 地址的映射规则**：

1. MAC 地址前 25 位固定：`00000001 00000000 01011110 0`（即 `01-00-5E` 加上第 4 字节最高位 0）
2. IP 多播地址的低 23 位直接映射到 MAC 地址的低 23 位
3. IP 多播地址的高 5 位 **不参与映射**，被忽略

**映射不唯一的问题**：由于 IP 多播地址的高 5 位被忽略，$2^5 = 32$ 个不同的 IP 多播地址会映射到同一个 MAC 地址。例如：
- `224.1.1.1` 和 `225.1.1.1` 映射到相同的 MAC 地址

**解决**：主机网卡收到帧后仍需在网络层按 IP 地址过滤，丢弃不属于本组的报文。

**网卡过滤机制**：现代网卡支持精确多播过滤——只接收本机已加入的多播组对应的帧，减少 CPU 中断开销。

# IGMP 与多播路由协议

路由器要获得多播组的成员信息，需要利用**网际组管理协议（IGMP，Internet Group Management Protocol）**；连接到局域网上的多播路由器还必须和互联网上的其他多播路由器协同工作，以便用最小代价将多播数据报分发至所有组成员，这就需要使用**多播路由选择协议**

**IGMP 的功能**：让本地路由器知道本局域网上有哪些主机加入了哪些多播组

> IGMP 仅工作在**主机与本地路由器之间**，并非全局路由协议。

IGMP 报文封装在 IP 数据报中（协议号 = 2），TTL = 1（不离开本局域网），IGMP 作为 IP 的配套协议，用于协助 IP 实现多播功能，因此不将其视为一个单独的协议，而是整个 IP 协议的一个组成部分

IGMP 的工作可分为两个阶段：

**第一阶段**：当某台主机加入新的多播组时，它向多播组的组地址发送一个 IGMP 报文，声明自己要成为该组的成员。本地多播路由器收到 IGMP 报文后，利用多播路由选择协议将该组成员关系转发给互联网上的其他多播路由器。

**第二阶段**：本地多播路由器周期性探询本地局域网上的主机，以便知道这些主机是否还继续是该组的成员。只要有一台主机对某个组响应，多播路由器就认为这个组是活跃的。如果经过几次探询后没有主机响应，多播路由器就不再将该组的成员关系转发给其他多播路由器。

> 一个多播组中的成员关系是动态的——主机可以随时加入或退出。因此 IGMP 需要同时处理"加入"和"维持/离开"两种场景。

**多播路由协议**实际上就是要构建一棵**以源主机为根节点**的**多播转发树**，每个分组在每条链路上**仅被传送一次**，且参与转发的路由器不会收到重复的多播数据报；不同的多播组对应不同的转发树；同一个多播组，若源主机不同，则对应的转发数也不同
