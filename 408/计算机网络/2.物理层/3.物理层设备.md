# 基本概念

> [!info] 物理层设备
> 物理层设备主要对信号进行**放大、整形、再生**，以扩展网络的物理覆盖范围。物理层设备**不识别**帧结构、MAC 地址、IP 地址等上层信息，仅处理**比特信号**。

物理层主要设备：

| 设备 | 功能 | 端口数 |
|------|------|:------:|
| **中继器（Repeater）** | 将衰减的信号**放大整形**后转发，延长传输距离 | 2 个 |
| **集线器（Hub）** | **多端口的中继器**，将信号广播到所有端口 | 多个 |

> [!warning] 注意
> 物理层设备连接的网络在逻辑上属于**同一个冲突域**和**同一个广播域**。

# 中继器

## 工作原理

> [!info] 中继器（Repeater）
> 信号在传输介质上传输时，由于**衰减**和**噪声干扰**，信号会逐渐减弱和失真。中继器将接收到的**弱信号**进行**再生放大**，恢复为原始信号后继续传输。

## 中继器的功能

| 功能 | 说明 |
|------|------|
| **信号放大** | 将衰减的信号幅度恢复到原有水平 |
| **信号整形** | 去除信号中的噪声和失真，恢复信号原始波形 |
| **信号再生** | 将失真的信号重新生成标准信号（不是简单放大，而是"理解"后重新发送） |
| **延长传输距离** | 扩大网络的物理覆盖范围 |
| **连接不同传输介质** | 可以连接两种不同类型的介质（如双绞线转光纤） |

## 中继器的特点

中继器连接的两端属于**同一局域网的不同网段（而非不同子网）**，因此所连网段仍构成一个局域网，如果中继器发生故障，将影响相邻两个网段的正常工作

| 特点          | 说明                                    |
| ----------- | ------------------------------------- |
| **工作在物理层**  | 仅处理比特信号，不理解帧格式和 MAC 地址                |
| **无过滤功能**   | 不区分有效信号和噪声，**一并放大**（包括噪声和冲突信号）        |
| **不隔离冲突域**  | 中继器两端属于**同一个冲突域**                     |
| **不连接不同协议** | 不能连接两个使用不同 MAC 协议（如 CSMA/CD 和令牌传递）的网络 |
| **有数量限制**   | 受网络标准限制（如 10 Mbps 以太网的 **5-4-3 规则**）  |

> [!warning] 5-4-3 规则（10 Mbps 以太网）
> - 一个以太网中允许有 **5 个网段**
> - 最多使用 **4 个中继器**连接
> - 其中只有 **3 个网段**可以连接计算机（其余两个仅用于延长距离）
>
> 超过此限制会导致信号时序问题，冲突检测失效。

## 放大器 vs 中继器

| 比较维度 | 放大器（Amplifier） | 中继器（Repeater） |
|----------|:-----------------:|:-----------------:|
| **处理方式** | 简单放大信号幅度 | 再生整形后重发 |
| **噪声** | 信号和噪声**一并放大** | 去除噪声，重新生成信号 |
| **信号质量** | 信噪比可能变差 | 信号质量恢复 |
| **累积噪声** | 多次放大后噪声累积 | 每次重新生成，噪声不累积 |
| **适用范围** | 模拟信号 | 数字信号 |

# 集线器

## 工作原理

> [!info] 集线器（Hub）
> 集线器可以看作是一个**多端口的中继器**，它的每一个端口都是一个中继器。集线器从任意一个端口接收到信号后，将其**整形放大**，然后**广播**到其他所有端口，是标准的**共享式设备**

## 物理结构与逻辑结构

> [!info] 物理星型，逻辑总线
> 集线器在**物理拓扑**上呈**星型**（所有设备通过独立线缆连接到集线器），但在**逻辑上**等价于一条**共享总线**——所有端口共享同一传输介质，同一时刻只能有一个设备发送数据。

| 维度       |       物理结构       |       逻辑结构       |
| -------- | :--------------: | :--------------: |
| **拓扑形态** |        星型        |       总线型        |
| **连接方式** | 每台设备有独立线缆连到 Hub  |   所有端口共享同一传输介质   |
| **发送方式** | 一台发送，Hub 广播到所有端口 | 相当于总线上所有节点都能收到信号 |

> [!tip] 理解要点
> 集线器内部本质上就是将所有端口的接收和发送线路**直接电气连接**在一起（或通过背板总线互连），并没有任何交换或缓存逻辑。因此：
> - 一个端口收到的信号，会直接出现在其他所有端口的发送线路上（广播）
> - 如果两个端口同时有信号输入，信号会在内部叠加，产生**冲突**

## 集线器的特点

| 特点         | 说明                                             |
| ---------- | ---------------------------------------------- |
| **工作在物理层** | 与中继器相同，仅处理比特信号                                 |
| **广播式转发**  | 从一个端口接收，向**其他所有端口**转发                          |
| **共享带宽**   | 所有端口**共享**同一带宽（如 100 Mbps 集线器，所有端口共享 100 Mbps） |
| **半双工**    | 同时只能有一个设备发送数据，否则产生冲突                           |
| **一个冲突域**  | 集线器所有端口属于**同一个冲突域**                            |
| **不隔离广播域** | 集线器所有端口属于**同一个广播域**                            |


> [!warning] 集线器的局限
> 1. 所有端口共享带宽，设备越多，每台设备实际可用带宽越低
> 2. 不能同时收发（半双工），效率低
> 3. 存在冲突问题，随着设备增多冲突概率急剧上升
> 4. 安全性差——所有端口都能收到其他端口的数据（广播方式）
> 5. 现代网络中基本已被**交换机（数据链路层设备）**取代

## 集线器的分类

| 分类方式 | 类型 | 说明 |
|----------|------|------|
| **按端口数** | 4口、8口、16口、24口等 | 常见端口数 |
| **按带宽** | 10 Mbps、100 Mbps | 所有端口共享该带宽 |
| **按结构** | 独立式、堆叠式、模块式 | 不同部署方式 |

## 集线器的带宽共享问题

> [!info] 带宽计算
> 集线器所有端口共享同一带宽。一个有 $N$ 个端口的 100 Mbps 集线器，每台计算机实际可用的**平均带宽**为：
> $$\text{平均带宽} = \frac{\text{总带宽}}{N}$$
>
> 例如：8 口 100 Mbps 集线器，每台设备平均可用约 12.5 Mbps（实际更低，因为有冲突开销）。

# 中继器与集线器的比较

| 比较维度 | 中继器 | 集线器 |
|----------|:-----:|:-----:|
| **端口数** | 2 个 | 多个（4/8/16/24 等） |
| **本质** | 信号再生放大器 | 多端口中继器 |
| **转发方式** | 一端接收，另一端发送 | 一端接收，**广播**到其余所有端口 |
| **工作层次** | 物理层 | 物理层 |
| **冲突域** | 不隔离（同一冲突域） | 不隔离（同一冲突域） |
| **广播域** | 不隔离 | 不隔离 |
| **带宽** | 与所连介质带宽一致 | **共享带宽** |
| **典型应用** | 长距离链路的信号中继 | 早期局域网的中心节点 |
>[!warning]
>中继器和集线器**不具备缓存能力**，因此：
>- **无法连接不同协议的网段**：两端必须使用相同的 MAC 协议（如 CSMA/CD），否则无法正常工作
>- **无法连接不同速率的网段**：若两个网段速率不同（如 10 Mbps 与 100 Mbps），由于没有缓存，数据到达速率不匹配会导致**大量丢包**；即使勉强连通，整个网络也只能以**较低速率**运行

# 物理层设备与其他层设备的对比

> [!info] 各层典型设备
> 不同层次的网络设备工作在不同协议层，处理不同粒度的数据单元。

|    层次     | 设备      |     处理单元     | 隔离冲突域 | 隔离广播域 |
| :-------: | ------- | :----------: | :---: | :---: |
|  **物理层**  | 中继器、集线器 |   比特（bit）    |   ❌   |   ❌   |
| **数据链路层** | 网桥、交换机  |   帧（Frame）   |   ✅   |   ❌   |
|  **网络层**  | 路由器     | 分组/包（Packet） |   ✅   |   ✅   |

> [!tip] 记忆要点
> 判断一个设备工作在哪一层，看它**识别什么地址**：
> - 物理层设备：不识别任何地址，仅处理比特
> - 数据链路层设备：识别 **MAC 地址**（物理地址）
> - 网络层设备：识别 **IP 地址**（逻辑地址）


# 易错点总结

1. **中继器不能连接不同协议的网络**：中继器两端必须使用相同的 MAC 协议
2. **中继器不是放大器**：中继器是"再生整形"，不是简单放大；放大器连同噪声一起放大
3. **集线器 = 多端口中继器**：它本质上就是中继器，只不过端口更多
4. **集线器所有端口共享带宽**：这与交换机（每个端口独立带宽）有本质区别
5. **物理层设备不隔离冲突域**：中继器和集线器的所有端口都在**同一个冲突域**
6. **物理层设备不隔离广播域**：广播帧会广播到所有端口
7. **集线器是半双工的**：不能同时收发的根本原因是共享介质 + CSMA/CD
8. **5-4-3 规则**仅适用于 10 Mbps 以太网，快速以太网有不同规则
9. **交换机不是物理层设备**：交换机工作在数据链路层，能识别 MAC 地址，隔离冲突域
10. **物理层设备已基本被淘汰**：在现代以太网中，中继器和集线器已被交换机全面取代
