# 计算机网络的概念

计算机网络由若干节点和间接这些节点的链路组成；网络中的节点可以是计算机、集线器、交换机和路由器等；多个网络还可以通过路由器互连，形成互连网；
网络将多台计算机连接在一起，而互联网则将多个网络通过路由器连接在一起
**internet（互连网）**：是通用名词，泛指由多个计算机网络通过路由器互连而成的网络，可以使用任意协议
**Internet（互联网或因特网）**：是专用名词，特指当前全球最大的、开放的、使用 TCP/IP 协议族的特定计算机网络，必须使用 **TCP/IP 协议族**

# 计算机网络的组成

## 从组成部分看

| 组成部分 | 说明 |
|----------|------|
| **硬件** | 主机（端系统）、通信链路、交换设备（路由器、交换机等）、通信处理机（网卡） |
| **软件** | 网络操作系统、网络协议软件、网络管理软件、网络应用软件 |
| **协议** | 网络通信的规则和标准，是计算机网络的核心 |

## 从工作方式看

| 部分       | 组成                 | 作用               |
| -------- | ------------------ | ---------------- |
| **边缘部分** | 所有连接到网络上的主机（端系统）   | 用户直接使用，进行通信和资源共享 |
| **核心部分** | 由大量网络和连接这些网络的路由器组成 | 为边缘部分提供连通性和交换服务  |

> **端系统之间的通信方式**：
> - **C/S 模式**（客户/服务器）：客户是服务请求方，服务器是服务提供方
> - **P2P 模式**（对等连接）：每个主机既是客户也是服务器

## 从功能组成看

| 子网 | 组成 | 功能 |
|------|------|------|
| **通信子网** | 传输介质、通信设备、网络协议 | 实现数据传输、交换、控制和变换 |
| **资源子网** | 主机、终端、软件、数据资源 | 实现资源共享、数据处理 |

# 计算机网络的功能

| 功能 | 说明 |
|------|------|
| **数据通信** | 实现计算机之间各种信息的传输（最基本、最重要的功能） |
| **资源共享** | 硬件共享、软件共享、数据共享 |
| **分布式处理** | 将大型任务分配给多台计算机协同完成 |
| **提高可靠性** | 通过网络中的冗余和备份，各计算机互为替代 |
| **负载均衡** | 将工作均匀分配给网络中的各计算机 |

# 计算机网络的分类

## 按分布范围分类

| 类型              | 范围      | 典型应用                    |
| --------------- | ------- | ----------------------- |
| **广域网 (WAN)**   | 几十~几千公里 | 跨越国家、洲际的远程网络，采用**交换技术** |
| **城域网 (MAN)**   | 5~50 公里 | 城市范围，多采用以太网技术           |
| **局域网 (LAN)**   | 几十米~几公里 | 校园、企业、家庭，采用**广播技术**     |
| **个人区域网 (PAN)** | 10 米左右  | 蓝牙、ZigBee 等无线技术连接的设备    |

> ⚠️ **注意**：若中央处理机之间的距离非常近（如 1 米甚至更小），则一般称为**多处理机系统**，而非计算机网络。

## 按传输技术分类

| 类型 | 特点 |
|------|------|
| **广播式网络** | 所有计算机共享一个公共通信信道，一台计算机发送的数据可被所有计算机接收 |
| **点对点网络** | 每条物理线路连接一对计算机，数据通过存储转发方式从源到目的 |

## 按拓扑结构分类

| 拓扑结构    | 特点                             |
| ------- | ------------------------------ |
| **星型**  | 中央节点控制，便于管理，但中央节点故障则全网瘫痪       |
| **总线型** | 所有节点共享一条总线，结构简单，但故障隔离困难        |
| **环型**  | 数据单向流动，令牌环是其典型，环形拓扑故障定位容易      |
| **树型**  | 星型拓扑的扩展，分级结构，适用于分级管理           |
| **网状**  | 每个节点至少与其他两个节点相连，可靠性高，但成本和复杂度也高 |

## 按使用者分类

| 类型 | 说明 |
|------|------|
| **公用网** | 由电信公司出资建造，面向公众提供服务的网络 |
| **专用网** | 由某个部门/单位组建，不向公众开放的网络 |

# 电路交换、报文交换和分组交换

## 电路交换

电路交换的核心思想是：通信前先建立一条**专用的物理通路**，通信期间**独占**这条线路，通信结束后释放。

### 三个阶段

| 阶段       | 说明                                         |
| -------- | ------------------------------------------ |
| **建立连接** | 源节点向目的节点发送呼叫请求，沿途各节点分配资源，建立一条专用的物理通路       |
| **数据传输** | 建立连接后，双方通过这条专用通路进行数据传输，**比特流直接传送**，不经过存储转发 |
| **释放连接** | 通信结束后，释放占用的所有资源，通路被拆除                      |

### 特点

| 优点                 | 缺点                      |
| ------------------ | ----------------------- |
| 通信时延小（一旦建立连接，数据直达） | 建立连接时间长（拨号等待）           |
| 数据传输**有序**，无失序问题   | 线路利用率低（独占线路，通信间歇期线路空闲）  |
| 没有冲突（专用通路，不存在争用）   | 灵活性差（中间节点故障即通信中断）       |
| 实时性好（适用于语音、视频等连续流） | 不支持差错控制（中间节点不存储，无法纠错）   |
| 二进制比特流**透明传输**     | **不适用于突发性数据（计算机通信的特点）** |

> 🔑 **关键理解**：电路交换中，中间节点（交换机）只是将比特流从一个端口**直接转发**到另一个端口，**不进行存储转发**，也不检错。

## 报文交换

报文交换的核心思想是：**存储转发**（Store and Forward），以**整个报文**为单位进行传输。

### 工作过程

1. 发送方将整个报文（数据 + 目的地址 + 源地址等信息）发送给第一个交换节点
2. 交换节点**完整接收**报文后，暂存并检查无误，再根据目的地址选择下一跳转发
3. 逐跳转发，直到到达目的节点

### 特点

| 优点                    | 缺点                         |
| --------------------- | -------------------------- |
| 无需预先建立连接（无连接）         | 转发延迟大（必须**完整接收整个报文**后才能转发） |
| 线路利用率高（不独占线路，多个报文可共享） | 对交换节点的缓存要求大（需存储完整报文）       |
| 支持差错校验（逐跳存储后可检错）      | 错误处理低效，需要重传整个报文            |
| 可靠性较高（某跳失败可重传）        | 报文大小不固定，管理和控制复杂            |

## 分组交换

分组交换是报文交换的**改进版**，同样采用**存储转发**，但将报文**拆分成较小的分组（包）**，以分组为单位独立传输。

### 工作过程

1. 发送方将报文拆分成多个**等长分组（Packet）**
2. 每个分组携带首部（含目的地址、序号等控制信息），独立路由
3. 中间节点以分组为单位存储转发（不需要等整个报文到齐）
4. 目的节点根据序号将分组**重新组装**成原始报文

### 特点

| 优点                           | 缺点                   |
| ---------------------------- | -------------------- |
| 线路利用率高（分组小，多用户可交错发送）         | 需要拆分和重组，增加额外开销（首部信息） |
| 延迟较小（流水线效应：后续分组发送与前序分组转发可并行） | 分组可能失序到达（需重新排序）      |
| 灵活性强（分组可独立选择路由，绕过故障节点）       | 可能产生额外时延（存储转发 + 排队）  |
| 缓存需求小（只需存储分组，而非整个报文）         | 首部开销比例较大（短报文效率低）     |
| 降低出错概率和重传代价                  |                      |

### 分组交换的两种方式

| 方式 | 说明 | 代表技术 |
|------|------|----------|
| **数据报方式**（Datagram） | 每个分组独立路由，路径可能不同，无连接服务 | IP 协议 |
| **虚电路方式**（Virtual Circuit） | 先建立逻辑连接，分组沿同一路径传送，面向连接服务 | X.25、帧中继、ATM |

## 三种交换方式对比

| 对比维度            | 电路交换    | 报文交换         | 分组交换                |
| --------------- | ------- | ------------ | ------------------- |
| **是否建立连接**      | 是（物理连接） | 否            | 数据报：否 / 虚电路：是（逻辑连接） |
| **传输单位**        | 比特流     | 报文           | 分组（包）               |
| **存储转发**        | ❌ 不存储转发 | ✅ 存储转发       | ✅ 存储转发              |
| **线路利用率**       | 低（独占）   | 高            | 高                   |
| **是否需要额外的控制信息** | ❌       | ✅            | ✅ 且占比较大             |
| **传输时延**        | 最小（直达）  | 最大（整报存储）     | 中等（分组流水线）           |
| **可靠性**         | 低（无检错）  | 较高（逐跳检错）     | 较高（逐跳检错 + 可重传）      |
| **灵活性**         | 差       | 较好           | 最好                  |
| **适用场景**        | 实时语音、视频 | 电报、电子邮件（已淘汰） | **现代互联网**（数据业务）     |

# 计算机网络的主要性能指标

## 速率（数据率 / 比特率）

> 连接在计算机网络上的主机在数字信道上**传输数据的速率**。

- **单位**：bit/s（bps）、kbps（$10^3$ bps）、Mbps（$10^6$ bps）、Gbps（$10^9$ bps）、Tbps（$10^{12}$ bps）
- **注意**：速率指的是**额定速率**或**标称速率**，而非实际运行速率

## 带宽

> 带宽有两种含义：

| 含义           | 说明                                                                 |
| ------------ | ------------------------------------------------------------------ |
| **原始意义（频域）** | 信号具有的**频带宽度**，单位：赫兹(Hz)                                            |
| **计算机网络中**   | 网络中某信道传送数据的**最大能力**，即**最高数据率**，单位：bit/s；节点间实际可用带宽由中间所有设备的**最小值**决定 |

## 吞吐量

> 在**单位时间内**通过某个网络（或信道、接口）的**实际数据量**。

- 受到网络带宽或额定速率的限制
- 吞吐量受网络的额定速率限制

> 📌 **速率 vs 带宽 vs 吞吐量**：
> - **速率**：实际传输的速率
> - **带宽**：信道的最大传输能力（理论上限）
> - **吞吐量**：单位时间内实际成功传输的数据量

## 时延（Delay / Latency）

> 数据从网络（或链路）的**一端**传送到**另一端**所需的时间。总时延 = **发送时延 + 传播时延 + 处理时延 + 排队时延**

| 时延类型 | 定义 | 计算公式 | 影响因素 |
|----------|------|----------|----------|
| **发送时延**（传输时延） | 主机或路由器将分组的所有比特推向链路所需的时间 | $\frac{\text{分组长度}(bit)}{\text{发送速率}(bit/s)}$ | 分组长度、发送速率 |
| **传播时延** | 电磁波在信道中传播一定距离所需的时间 | $\frac{\text{信道长度}(m)}{\text{电磁波传播速率}(m/s)}$ | 信道长度、传播速率 |
| **处理时延** | 路由器为存储转发而进行必要处理所花费的时间 | — | 路由器的处理能力 |
| **排队时延** | 分组在路由器输入/输出队列中等待处理的时间 | — | 网络拥塞程度 |

> ⚠️ **容易混淆的点**：
> - **发送时延**发生在**机器内部**（网卡/路由器），取决于发送速率
> - **传播时延**发生在**传输介质**上，取决于距离和介质中信号的传播速率
> - 高速链路提高的是**发送速率**，减小的是**发送时延**，对**传播时延无影响**

## 时延带宽积

> **时延带宽积 = 传播时延 × 带宽**

- 表示管道中容纳的**比特数量**，即"以比特为单位的链路长度"
- 含义：当发送端连续发送数据时，第一个比特到达终点之前，发送端已发出的比特数

## 往返时延（RTT）

> 从发送端发送数据开始，到发送端收到来自接收端的**确认**所经历的**总时间**。

- 包括：往返的传播时延 + 中间节点的处理时延 + 排队时延 + 发送确认报文的发送时延
- **有效数据率 = 数据长度 / (发送时间 + RTT)**（使用停止等待协议时）
>[!warning] 往返时延不包括数据分组本身的发送延迟

## 利用率

| 类型        | 定义                                |
| --------- | --------------------------------- |
| **信道利用率** | 某信道在**时间**上的利用效率 = 有数据通过的时间 / 总时间 |
| **网络利用率** | 全网所有信道的信道利用率的**加权平均值**            |


> ⚠️ **利用率和时延的关系**：
> $$D = \frac{D_0}{1 - U}$$
> - $D_0$：网络空闲时的时延
> - $U$：网络利用率
> - $D$：网络当前的时延
> - **信道利用率并非越高越好，当利用率接近 1 时，时延急剧增大（类似交通拥塞）

# 计算机网络的标准化工作及相关组织

## 标准化工作

> **RFC（Request For Comments）**：互联网技术标准的形式，由**互联网工程任务组 (IETF)** 发布。

- RFC 文档的四个阶段：
  1. **互联网草案**（Internet Draft）：非正式文档
  2. **建议标准**（Proposed Standard）：较稳定
  3. **草案标准**（Draft Standard）：已被广泛使用
  4. **互联网标准**（Internet Standard）：最终标准

## 相关国际组织

| 组织 | 全称 | 主要工作 |
|------|------|----------|
| **ISO** | 国际标准化组织 | 制定 OSI/RM 参考模型 |
| **ITU** | 国际电信联盟 | 制定通信领域的标准（如 X 系列、V 系列） |
| **IEEE** | 电气与电子工程师协会 | 制定局域网标准（如 IEEE 802 系列） |
| **IETF** | 互联网工程任务组 | 制定互联网相关标准，发布 RFC 文档 |

# ❗ 易错点总结

1. **速率 ≠ 带宽**：速率是实际值，带宽是理论上限
2. **发送时延 ≠ 传播时延**：发送在"机器内部"，传播在"链路上"
3. **高速链路只减小发送时延**，不减小传播时延
4. **网络利用率越高 ≠ 性能越好**：利用率接近 100% 时，时延急剧增大
5. **广域网采用交换技术**，**局域网采用广播技术**
6. **RFC 不是组织**，而是 IETF 发布的文档名称
