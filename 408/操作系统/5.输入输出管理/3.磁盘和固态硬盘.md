# 磁盘（Disk）

磁盘（硬盘）由多个**盘片** (Platter) 组成，每个盘片有上下两个**盘面** (Surface)，盘面上涂有磁性材料用于存储数据，**相邻磁道及扇区间留有间隙**，以免读/写干扰
## 基本概念
- **磁道 (Track)**：每个盘面上被划分为多个同心圆环，每个圆环称为一个磁道；盘面边缘为 0 号磁道，向内编号递增，磁道宽度和磁头相当
- **扇区 (Sector)**：每个磁道被划分为若干弧段，每个弧段称为一个扇区，是磁盘**读/写的最小物理单位**，通常每个扇区大小为 512 字节
- **柱面 (Cylinder)**：由所有盘面上同一编号的磁道组成的圆柱体，称为柱面；同一柱面上的数据可以**不移动磁臂**而直接访问（仅需切换磁头）
- **磁头 (Head)**：每个盘面对应一个磁头，负责读/写该盘面上的数据，所有磁头固定在**磁臂**上统一移动
>[!note] 磁道存储密度
>传统设计中，每个磁道扇区数相同，由于外道周长更大，其**线性存储密度**低于内道，磁盘的存储能力受限于最内道的最大存储密度
>现代磁盘采用**区域记录（MBR）** 技术：将盘面划分为若干环带，同一环带内的所有磁道具有相同的扇区数，外层环带的磁道扇区数多于内层环带
## 磁盘驱动器
磁盘安装在磁盘驱动器中，驱动器由**机械部件**和**电子部件**共同组成，负责控制磁盘的旋转、磁头定位以及数据的读/写传输

### 机械部件
- **盘片 (Platter)**：涂有磁性材料的圆形盘片，数据存储的实际载体
- **主轴 (Spindle)**：带动所有盘片以恒定角速度高速旋转（常见转速 5400rpm / 7200rpm / 10000rpm / 15000rpm）
- **磁头 (Head)**：每个盘面配有一个磁头，负责读/写数据；所有磁头固定在磁臂上联动
- **磁臂 (Actuator Arm)**：承载所有磁头，由音圈电机驱动在盘面上沿半径方向移动，将磁头定位到目标磁道
- **音圈电机 (Voice Coil Motor)**：驱动磁臂运动的执行机构，响应速度快、定位精度高

### 电子部件
- **控制器 (Controller)**：驱动器的核心控制单元，负责解析主机命令、调度磁头移动、管理数据缓存
- **缓存 (Cache/Buffer)**：内置高速缓存（通常 8MB~256MB），用于缓冲读/写数据，减少实际磁盘访问次数
- **接口 (Interface)**：连接磁盘驱动器与主机（或磁盘控制器）的通信接口，常见类型为 SATA、SAS、IDE（已淘汰）

### 工作流程
1. 主机通过接口发送读/写请求（含逻辑块地址）
2. 控制器解析请求，将逻辑地址转换为物理地址（**柱面号→磁头号→扇区号**）
3. 音圈电机驱动磁臂将磁头定位到目标磁道（**寻道**）
4. 盘片旋转，目标扇区转至磁头下方（**旋转等待**）
5. 磁头完成数据的读取或写入（**数据传输**）

## 磁盘的分类

### 按磁头分类
- **固定头磁盘**：每个磁道配置一个磁头，无需移动磁臂即可访问任意磁道；寻道极快但磁头数量庞大、成本极高，仅用于少数专用高速存储场景
- **活动头磁盘（移动头磁盘）**：每个盘面仅配有一个磁头（装在磁臂上），需移动磁臂将磁头定位到目标磁道；成本低但存在寻道延迟，是当前主流方案

### 按盘片可换性分类
- **固定盘磁盘**：盘片**永久密封**在驱动器内部不可更换，防尘防污染、可靠性高，是最常见的磁盘形式
- **可换盘磁盘**：盘片制成**可插拔的盘组**，可以更换不同盘组来扩展容量或备份数据；灵活性高但密封性差，对环境要求较高

>[!info] 温彻斯特盘
>固定头/活动头×固定盘/可换盘交叉组合可产生多种磁盘类型。现代硬盘的主流形态是**温彻斯特盘**——盘片、磁头、磁臂一体化密封，属于**活动头 + 固定盘**的组合；防尘防污染，可靠性高，成本适中

## 磁盘地址表示
磁盘的物理地址由三维坐标定位：**柱面号（磁道号） → 磁头号（盘面号） → 扇区号**，编号从 0 开始

# 磁盘管理
磁盘管理涵盖磁盘从出厂到日常使用的全生命周期，包括：**初始化（低级格式化→分区→高级格式化）→ 引导启动 → 坏块处理 → 碎片整理 → 交换空间管理**

## 磁盘初始化
>[!note] 初始化的三步流程
>一块新磁盘投入使用前，必须依次经历 **低级格式化 → 分区 → 高级格式化** 三个步骤，缺一不可

### 低级格式化（物理格式化）
- 将磁盘**划分为扇区**，每个扇区包含**数据区**和**头部/尾部**，头部尾部包含了一些磁盘控制器的使用信息，其中利用磁道号、磁头号和扇区号来标志一个扇区，利用 CRC 字段对扇区校验
- 低级格式化在出厂前完成，一般用户不需要执行
### 分区
- 将磁盘划分为一个或多个独立的**分区**（每个分区可安装不同文件系统）
- 每个分区信息存放在磁盘的**分区表**中（如 MBR 分区表位于磁盘 0 扇区）
### 高级格式化（逻辑格式化）
- 在每个分区上创建**文件系统**的数据结构，包括空闲空间管理信息、根目录等
- 操作系统通过逻辑格式化将裸分区变为可用的存储空间，由于扇区单位过小，为提高 I/O 效率，操作系统将多个连续扇区组合成一簇（在文件系统中也称块）；为简化管理，通常将**一簇分配给单个文件**，因此文件占用空间为簇的整数倍
## 引导块
- 计算机加电后，CPU 从 ROM/Flash 中执行**BIOS/UEFI** 引导程序
- BIOS/UEFI 从磁盘的**引导块**（第零扇区，即 MBR 或 GPT 头）加载引导程序
- **主引导记录 (MBR)** 位于 0 柱面 0 磁头 0 扇区，包含引导代码和分区表以及一个标志位，用于指明应从哪个分区引导
- 引导程序找到操作系统内核所在分区，加载内核到内存并启动
## 坏块处理
磁盘在制造或使用过程中可能产生**坏块**（无法正常读/写的扇区），处理方式有：

- 对于简单磁盘，通常在**逻辑格式化时**使用检查工具标记坏块，并在 FAT 表中标记为不可用，从而避免程序使用
- 对于现在复杂磁盘，控制器内部维护一份**坏块列表**，该列表在出厂**低级格式化**时初始化，并在使用过程中动态更新；低级格式化会预留一部分备用扇区，这些扇区对操作系统*透明*；当发现坏块时，控制器自动将其映射到备用扇区，实现逻辑替换，此机制称为**扇区备用**
- 通过软件标记或硬件重映射，使文件系统**不再分配使用**坏块
- 对于包含有用数据的坏块，尽可能读取数据并转移到正常扇区

## 交换空间管理

[[2.虚拟内存管理#从何处调入页面|交换区]]
### 基本概念
- 当物理内存不足时，操作系统将内存中暂时不用的页面**换出到磁盘**的交换空间，腾出内存供当前进程使用
- 交换空间是磁盘上的一块**专用区域**，作为内存的扩展
- 交换空间可以是**独立的磁盘分区**（交换分区），也可以是文件系统内的**交换文件**

### 交换位置的优劣对比
| 位置 | 优点 | 缺点 |
|------|------|------|
| 独立交换分区 | 访问速度快（无文件系统开销）、不受文件系统碎片影响 | 大小固定，调整不便 |
| 交换文件 | 大小灵活可动态调整、创建方便 | 需通过文件系统访问，额外开销大 |

### 交换策略
- **换出策略**：选择哪些页面换出（与页面置换算法配合，如 LRU、Clock 等）
- **换入策略**：何时将页面从交换空间换入
  - **请求调页**：访问缺失页面时才换入
  - **预调页**：提前预判即将使用的页面并换入，减少缺页

>[!warning] 交换空间与虚拟内存
>交换空间是实现**虚拟内存**的底层基础。当系统频繁在内存与交换空间之间换入换出页面时，会产生**抖动 (Thrashing)** 现象，此时系统性能急剧下降

# 磁盘的性能指标
### 磁盘格式化容量
- **非格式化容量**：磁盘上所有物理存储位的总和
- **格式化容量**：实际可用于存储数据的容量，计算公式为：
$$\text{格式化容量} = \text{盘面数} \times \text{每面磁道数} \times \text{每道扇区数} \times \text{每扇区字节数}$$

### 磁盘访问时间 $T_a$
磁盘完成一次读/写操作的总时间由三部分组成：
$$T_a = T_s + T_r + T_t$$

- **寻道时间 $T_s$**：将磁头移动到目标磁道所需的时间
  $$T_s = m \times n + s$$
  其中 $m$ 为跨越磁道数，$n$ 为跨越每条磁道的时间，$s$ 为磁臂启动时间

- **旋转延迟 $T_r$**：目标扇区旋转到磁头下所需的时间，取决于磁盘转速 $r$（转/分）
  $$T_r = \frac{1}{2r}$$
  **通常取平均值**，即旋转半圈所需的时间

- **传输时间 $T_t$**：读出或写入数据的时间，取决于数据传输速率
  $$T_t = \frac{b}{r \times N}$$
  其中 $b$ 为读/写的字节数，$N$ 为每条磁道上的字节数

>[!important] 访问时间的主要瓶颈
>在三部分时间中，**寻道时间 $T_s$ 通常占比最大**，是影响磁盘性能的关键因素，因此磁盘调度算法的核心目标是**减少平均寻道时间**

# 磁盘调度算法
调度目标：**在多个 I/O 请求同时到达时，确定磁头移动的服务顺序，使平均寻道时间最小**

以下假定磁头当前位于 100 号柱面，请求序列为：55, 58, 39, 18, 90, 160, 150, 38, 184，并以该例说明各算法

## 先来先服务 (FCFS)
### 基本思想
按照请求到达的**先后顺序**依次服务，不进行任何优化排序

### 特点
- **优点**：公平、实现简单，不会饥饿
- **缺点**：平均寻道距离大，性能最差；磁头频繁"来回折返"，寻道不连续

## 最短寻道时间优先 (Shortest Seek Time First, SSTF)
### 基本思想
优先服务**距离当前磁头位置最近**的请求，使每次寻道距离最短

### 特点
- **优点**：平均寻道时间比 FCFS 大幅降低（如上例 498 → 248），性能良好
- **缺点**：可能导致**饥饿**——若不断有新请求出现在磁头当前位置附近，远距离请求可能长期得不到服务
>[!warning] SSTF 的饥饿问题
>SSTF 本质上是**贪心算法**，它不是最优的寻道方案（无法保证全局最短），但与最优算法的差距通常不大

## 扫描算法 (SCAN / 电梯算法)
### 基本思想
磁头从当前位置开始，**沿一个方向移动并服务途中所有请求**，到达该方向尽头后**反向移动**，继续服务反方向上的请求，如此往复

### 特点
- **优点**：消除了 SSTF 中的饥饿问题，性能良好
- **缺点**：磁道两端的请求（最内圈和最外圈附近）平均等待时间较长；且 SCAN 会扫描到请求最远处的尽头（即使尽头无请求），访问局部性方面不如 FCFS 和 SSTF

## 循环扫描算法 (C-SCAN)
### 基本思想
与 SCAN 类似，但磁头**仅在单一方向移动时服务请求**，反向移动时快速返回到起始端**不服务任何请求**，到达起始端后再沿同一方向扫描服务

### 特点
- **优点**：各磁道的等待时间更加**均匀**
- **缺点**：返回途中的空跑增加了总寻道路程


>[!note] LOOK 和 C-LOOK
>C-SCAN 的返回途中不服务；现代磁盘实际更多使用 LOOK 和 C-LOOK（见下文），这两种算法仅扫描到**最远请求处**即反向，不会扫到磁盘物理尽头

## LOOK 算法与 C-LOOK 算法
SCAN 和 C-SCAN 中磁头必须移动到磁盘的**物理尽头**才反向，而**LOOK 和 C-LOOK 仅需移动到请求序列的最远端**即可反向

- **LOOK**：SCAN 的改进版，磁头在当前方向无请求时**提前反向**，无需到达物理尽头
- **C-LOOK**：C-SCAN 的改进版，磁头在当前方向无请求时提前反向并**快速返回到最小请求处**（而非物理最小端）

## NstepSCAN 算法和 FSCAN 算法

SCAN 和 C-SCAN 虽然解决了饥饿问题，但当请求密集到达时，可能出现**磁臂粘着**——磁头长时间滞留在某个区域服务新请求，远端请求仍然迟迟得不到响应。NstepSCAN 和 FSCAN 正是为了解决这一问题

### NstepSCAN 算法
- **基本思想**：将请求队列划分为**长度**为 $N$ 的子队列，每次只处理其中一个子队列（用 SCAN），新到达的请求放入下一个子队列
- 在一个子队列被服务期间，新到达的请求不会打断当前的服务顺序
- 当 $N$ 足够大时，NstepSCAN 退化为普通 SCAN；当 $N = 1$ 时，退化为 FCFS

### FSCAN 算法
- **基本思想**：将请求队列分为**两个子队列**——一个"服务队列"和一个"等待队列"（FSCAN 可看作 $N=2$ 的 NstepSCAN 特例）
- 当前服务队列按 SCAN 算法处理，期间所有新到达的请求进入等待队列
- 当前服务队列全部处理完毕后，两队列**角色互换**，开始处理等待队列中的请求，新请求放入原服务队列
- **优点**：实现简单，完全避免了粘着问题，且对新老请求相对公平

| 算法        | 核心思路        | 与 SCAN 的关系               |
| --------- | ----------- | ------------------------ |
| NstepSCAN | 分多个子队列，分批服务 | $N$ 大 → SCAN，$N$ 小 → 更公平 |
| FSCAN     | 双队列轮换       | NstepSCAN 在 $N=2$ 的特例    |

## 磁盘调度算法的综合对比
| 算法     | 优点           | 缺点         | 饥饿  |
| ------ | ------------ | ---------- | :-: |
| FCFS   | 公平、简单        | 性能最差、寻道路程长 |  无  |
| SSTF   | 平均寻道短、性能好    | 可能饥饿、非全局最优 |  有  |
| SCAN   | 无饥饿、性能好      | 两端请求等待不均   |  无  |
| C-SCAN | 响应时间更均匀      | 总路程可能更大    |  无  |
| LOOK   | SCAN 的实用优化   | 需追踪请求分布    |  无  |
| C-LOOK | C-SCAN 的实用优化 | 需追踪请求分布    |  无  |

# 减少延迟的方法

磁盘访问延迟主要由 $T_s$（寻道时间）和 $T_r$（旋转延迟）构成，减少旋转延迟时间也是提高磁盘 I/O 效率的重要途经，除调度算法外，还可通过物理布局优化来减少延迟

## 扇区交替编号
- 盘片持续高速旋转，若相邻逻辑扇区在物理上连续排列，读完一个扇区后下一个扇区可能已转过磁头，需**等待一整圈**才能继续
- **解决**：对扇区进行**交替编号**（如 0, 4, 8, 1, 5, 9, 2, 6, 10...），使连续的逻辑扇区在物理上间隔排列，读完一个扇区时下一个刚好转到磁头下方

## 柱面倾斜 (Cylinder Skew)
- 当磁头从一条磁道切换到相邻磁道时，由于磁头切换需要时间，若相邻磁道的 0 号扇区物理对齐，磁头到达新磁道时 0 号扇区已转过，同样需等一整圈
- **解决**：使相邻磁道的扇区起始位置错开一定角度（倾斜），保证磁头切换完成时目标扇区刚好到达

## 盘面交错
- 多盘面结构中，切换磁头也需要时间（电子切换，微秒级）
- 同理可通过相邻盘面间扇区编号的错位来消除等待

>[!important] 核心思想
>延迟优化的本质是**让机械动作与盘片旋转"对齐"**——读完一个扇区后，磁头和盘片恰好准备好读取下一个目标扇区，避免空转等待

# 提高磁盘 I/O 速度的方法

文件访问速度时衡量文件系统性能最重要的因素，可从以下三个方面来优化：
- 改进文件的目录结构和检索目录的方法，以减少对目录的查找时间，见[[2.目录]]
- 选取好的文件存储结构，以提高对文件的访问速度，见[[3.文件]]
- 调高磁盘 I/O 速度，以实现文件数据在磁盘和内存间的快速传送
## 磁盘高速缓存 (Disk Cache)
- 见[[2.设备独立性软件#磁盘高速缓存（Disk Cache）]]
- 在**内存**中开辟一块区域作为磁盘的**高速缓存**，缓存最近读/写的磁盘块
- **读命中**时直接从内存返回数据，无需访问磁盘
- **写缓存**时可采用不同策略：
  - **写穿 (Write-Through)**：数据同时写入缓存和磁盘，安全但速度慢
  - **写回 (Write-Back)**：数据先写入缓存，延迟写回磁盘，速度快但有数据丢失风险

- 即上文介绍的磁盘调度算法

## 提前读 (Read-Ahead)
- 根据程序的顺序访问特征，在读当前块的同时**预先将后续几个连续的块读入缓存**
- 适用于顺序文件读取（如视频播放），可显著减少磁盘访问次数

## 延迟写 (Delayed Write)
- 修改一个缓冲区的数据时，不立刻写回磁盘，而仅设置延迟写标志，仅当该缓冲区被替换时，才真正将数据写入磁盘
- 更多修改的数据积累后**批量写回**，减少磁盘寻道次数，并有机会将多次写入合并为一次

## 优化物理块分布
- 上文的扇区编号优化
- 对于采用链接或索引方式组织的文件，应尽量将其所属的块安排在同一磁道或相邻磁道上，以减少寻道时间
- 将若干连续扇区组成簇，以簇为单位对文件进行分配，能有效降低磁头的平均移动距离

## 虚拟盘 (RAM Disk)
- 用内存模拟磁盘，数据存储在 RAM 中，访问速度极快（无机械延迟）
- 适用于临时文件、编译中间产物、对 I/O 延迟敏感的数据等不需要持久化的数据
- 缺点是掉电数据丢失，且占用宝贵内存资源
>[!important] 虚拟盘和磁盘高速缓存的区别
>虚拟盘的内容完全由用户控制，而磁盘高速缓存的内容由操作系统透明管理

## RAID 技术概述
- 见[[4.外部存储器]]
- RAID (Redundant Array of Independent Disks，独立磁盘冗余阵列) 将多块物理磁盘组合成一个逻辑磁盘
- **核心目标**：提高 I/O 速度（并行读写） + 提高可靠性（数据冗余）
- 常见级别：

| RAID 级别 | 原理 | 优点 | 缺点 |
|----------|------|------|------|
| RAID 0 | 条带化，数据分散到多盘 | 读写速度翻倍 | 无冗余，一盘坏全毁 |
| RAID 1 | 镜像，数据完整复制到两盘 | 读速提升、容错强 | 磁盘利用率仅 50% |
| RAID 5 | 条带化 + 分布式奇偶校验 | 兼顾速度与容错 | 写开销大 |
| RAID 10 | RAID 1 + RAID 0 | 速度与容错兼得 | 磁盘利用率 50% |

# 固态硬盘 (SSD)
## SSD 的结构与原理
固态硬盘基于**闪存** (Flash Memory) 技术，核心存储单元是**NAND 闪存芯片**替代了机械驱动器，*闪存翻译层*负责将 CPU 发出的逻辑块读写请求转换为对底层物理闪存的读/写控制信号，替代了磁盘控制器的角色
### 存储层次
- **芯片→块 (Block)→页 (Page)**，每个块包含若干页，每页通常为 4KB 或 8KB
- 读/写操作以**页**为单位，擦除操作以**块**为单位
### 读/写/擦除特性
- **读**：速度快（微秒级），以页为单位
- **写**：速度较快，以页为单位，但只能向已擦除的页写入
- **擦除**：以块为单位（一个块包含多个页），擦除操作耗时且会消耗闪存寿命

>[!warning] 随机写入速度较慢
>闪存的重要特性：**不能原地覆盖写 (no in-place update)**。若要对已写入的页再次写数据，必须**先擦除整个块**，再重新写入；因此 SSD 的写入性能受限于擦除操作的频率和开销

## 磨损均衡 (Wear Leveling)
### 问题背景
每个闪存块有**擦写寿命有限**（通常几千至数万次），频繁擦除同一块会使其过早失效
### 磨损均衡策略
- **动态磨损均衡**：在写入数据时，优先选择擦写次数较少的空闲块，避免反复写入同一区域，从而将写入负载分配到更多物理块上
- **静态磨损均衡**：是更高级的策略，即使没有新数据写入，控制器也会定期扫描并自动迁移数据，将高磨损块中的有效数据迁移到低磨损块中，使高磨损块转为以读为主，低磨损块承担更多写入任务，进一步均衡整体寿命；**效果更好但开销更大**
### 实现机制
SSD 内部维护**FTL (Flash Translation Layer，闪存转换层)**：
- 将逻辑地址映射到物理地址
- 记录每个块的擦除次数
- 管理垃圾回收（GC）将有效数据合并整理，释放更多空闲块

## SSD 与 HDD 对比
| 对比维度    | HDD（机械硬盘）      | SSD（固态硬盘）         |
| ------- | -------------- | ----------------- |
| 存储介质    | 磁性盘片           | NAND 闪存芯片         |
| 机械部件    | 盘片、磁头、磁臂       | 无                 |
| 寻道/旋转延迟 | 有（毫秒级）         | 无                 |
| 随机读/写   | 慢（磁头需定位）       | 快（无机械延迟）          |
| 连续读/写   | 快（100~200MB/s） | 更快（500MB/s~数GB/s） |
| 寿命限制    | 机械磨损（寿命长）      | 擦除次数有限            |
| 价格/容量   | 低/大（TB 级）      | 高/中等              |
| 抗震性     | 弱（怕震动）         | 强                 |
| 功耗      | 高              | 低                 |
| 噪音      | 有              | 无                 |

>[!summary] 本章总结
> 
> **一、磁盘结构**
> 
> 1. **四要素**：**盘片** (Platter) → **磁道** (Track) → **扇区** (Sector，最小读写单位，512B) → **柱面** (Cylinder，同编号磁道组成的圆柱体，无需移臂即可访问)
> 2. **磁盘驱动器** = 机械部件（盘片/主轴/磁头/磁臂/音圈电机）+ 电子部件（控制器/缓存/接口）；工作流程：寻道 → 旋转等待 → 数据传输
> 3. **磁盘分类**：按磁头（固定头 vs 活动头）、按盘片可换性（固定盘 vs 可换盘）；温彻斯特盘 = **活动头 + 固定盘**，现代主流
> 4. **物理地址**：三维定位——**柱面号（磁道号）→ 磁头号（盘面号）→ 扇区号**
> 
> **二、磁盘管理**
> 
> 1. **初始化三步流程**：低级格式化（划扇区，出厂完成）→ 分区（划磁盘区域，存分区表）→ 高级格式化（创建文件系统、空闲空间管理、根目录）；操作系统以**簇（块）** 替代扇区作为分配单位
> 2. **引导块**：加电 → ROM/Flash 执行 BIOS/UEFI → 读磁盘第 0 扇区 MBR/GPT → 加载引导代码 + 分区表 → 定位 OS 内核分区 → 启动
> 3. **坏块处理**：硬件层面（ECC 纠错 + 备用扇区透明重映射）和 OS 层面（逻辑格式化时标记坏块、FAT 标记不可用）；现代磁盘采用**扇区备用**机制，控制器维护坏块列表并自动逻辑替换
> 4. **交换空间**：内存不足时换出页面到磁盘；交换分区（快但大小固定）vs 交换文件（灵活但有文件系统开销）；换入策略：请求调页 vs 预调页
> 
> **三、磁盘性能指标**
> 
> 1. **格式化容量** = 盘面数 × 每面磁道数 × 每道扇区数 × 每扇区字节数
> 2. **访问时间** $T_a = T_s + T_r + T_t$：寻道时间 $T_s$（$m \times n + s$）+ 旋转延迟 $T_r$（$\frac{1}{2r}$，均值取半圈）+ 传输时间 $T_t$（$\frac{b}{r \times N}$）
> 3. ⭐ **$T_s$（寻道时间）占比最大** → 调度算法的核心目标是减少平均寻道时间
> 
> **四、磁盘调度算法（8 种，从简单到工程化演进）**
> 
> 1. **FCFS**：按到达顺序服务；公平简单、无饥饿，但性能最差（寻道路程最长、磁头频繁折返）
> 2. **SSTF**：贪心地选最近请求；性能大幅提升（如上例 498→248），但远距离请求可能饥饿，非全局最优
> 3. **SCAN（电梯算法）**：单向扫到头再反向；消除饥饿、性能好，但两端请求等待不均、扫到物理尽头才回头
> 4. **C-SCAN**：仅单向服务、反向空跑；各磁道等待时间**更均匀**，但总路程更大
> 5. **LOOK / C-LOOK**：SCAN/C-SCAN 的实用优化——**仅扫到最远请求处即反向**，避免物理尽头空跑
> 6. **NstepSCAN**：分 N 个子队列分批 SCAN；解决密集请求下的「磁臂粘着」；N 大→SCAN，N=1→FCFS
> 7. **FSCAN**：NstepSCAN 在 N=2 的特例，双队列轮换；实现简单，彻底避免粘着
> 
> **五、减少延迟的方法（物理布局对齐）**
> 
> 1. **扇区交替编号**：逻辑连续扇区物理间隔排列，读完一个时下一个刚好转到
> 2. **柱面倾斜 (Cylinder Skew)**：相邻磁道扇区起始位错开，抵消磁头切换时间
> 3. **盘面交错**：相邻盘面扇区编号错位，抵消磁头电子切换延迟
> 4. ⭐ 核心思想：让**机械动作与盘片旋转对齐**，避免空转等待
> 
> **六、提高磁盘 I/O 速度的方法（系统层面）**
> 
> 1. **磁盘高速缓存**：内存缓存磁盘块；写策略——写穿（安全但慢）vs 写回（快但有丢失风险）
> 2. **提前读 (Read-Ahead)** + **延迟写 (Delayed Write)**：利用顺序访问特征预读后续块；批量延迟写回减少寻道
> 3. **优化物理块分布**：将相关块安排在同磁道/相邻磁道；以簇为单位分配降低平均移臂距离
> 4. **虚拟盘 (RAM Disk)**：内存模拟磁盘，极快但掉电丢失；内容由用户控制（vs 高速缓存由 OS 透明管理）
> 5. **RAID 技术**：多盘组合为逻辑盘；RAID 0（条带化，快但无冗余）→ RAID 1（镜像，利用率 50%）→ RAID 5（条带化+分布式奇偶校验，兼顾）→ RAID 10（RAID 1+0，速度容错兼得）
> 
> **七、固态硬盘 (SSD)**
> 
> 1. **存储层次**：芯片→块 (Block)→页 (Page)；读/写以**页**为单位，擦除以**块**为单位
> 2. **关键约束**：不能原地覆盖写 (no in-place update)，写入前必须先擦除整个块 → 随机写入性能受限
> 3. **磨损均衡**：动态（优先选低磨损空闲块）+ 静态（定期将冷数据搬移到高磨损块）；**FTL（闪存转换层）** 负责地址映射与 GC
> 4. **SSD vs HDD**：SSD 随机读写极快、无机械延迟、抗震低功耗，但单位容量贵、擦除次数有限；主流方案为 **SSD 系统盘 + HDD 数据盘**
