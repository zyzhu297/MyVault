# 手工操作阶段（无操作系统）

最早的计算机没有操作系统，用户独占全机
## 人工操作方式

- 用户先将程序纸带（或卡片）装入输入机，然后启动输入机将程序和数据送入计算机
- 通过控制台开关启动程序运行
- 计算完毕，用户取走打印结果并卸下纸带

>[!failure] 缺点
>1. **用户独占全机**：一台计算机为一个人所独占，资源利用率极低
>2. **CPU等待人工操作**：在人工装/卸纸带及操作开关期间，CPU处于空闲状态，**人机速度矛盾**严重
>3. **串行执行**：程序只能一个接一个地串行执行

## 脱机输入/输出（Off-Line I/O）方式

为解决CPU与I/O设备之间巨大的速度差距，在手工操作阶段后期引入了**脱机I/O技术**
**核心思想**：用一台相对廉价的**外围机（卫星机）** 专门处理I/O操作，将低速的纸带/卡片设备与高速的主机（Host）分离，主机只与相对高速的磁带进行数据交换

**工作流程**：

1. **脱机输入**：外围机将纸带（或卡片）上的程序和数据读入，预先转存到磁带上
2. **联机处理**：将存有程序的磁带装入主机，主机从高速磁带上读取程序和数据并执行
3. **脱机输出**：主机将计算结果写入磁带，再由外围机从磁带读出结果并打印输出

>[!note] 脱机I/O的意义
>1. **减少CPU等待**：主机只需与高速的磁带打交道，不再直接受低速I/O设备拖累，有效**缓和了人机速度矛盾**
>2. **提高并行度**：主机处理作业时，外围机可同时进行下一批作业的输入/输出，实现了I/O与计算的**初步并行**
>3. **过渡性方案**：脱机I/O并未从根本上解决人工干预问题（仍需人工装卸磁带），但为后续**批处理技术**的引入奠定了硬件基础

# 批处理阶段（操作系统开始出现）

为解决人机速度矛盾，引入了**批处理技术**
## 单道批处理系统

引入**监督程序（Monitor）** 常驻内存，先将一批作业以脱机方式输入磁带，监督程序将磁带上的第一个作业装入内存，该作业运行结束后，控制权交还给监督程序，再由监督程序装入下一个作业，如此反复，直至全部作业处理完毕

>[!note] 特征
>- **自动性**：作业自动一个接一个执行，无需人工干预
>- **顺序性**：各道作业按顺序进入内存，按顺序完成
>- **单道性**：内存中始终仅有一道程序在运行

>[!failure] 缺点
>CPU与I/O设备速度不匹配，程序发出I/O请求后，CPU必须等待I/O完成才能继续，**CPU利用率仍然较低**，用户无法了解程序运行状态或进行干预

## 多道批处理系统

**多道程序设计技术**允许多个程序同时进入内存并交替运行，是现代操作系统的核心思想
用户提交的作业首先存放在**外存**的后备队列中，作业调度程序按一定算法从队列中选取若干作业调入内存，使其在管理程序下并发执行，并共享系统资源，当一道程序因I/O请求而暂停运行时，CPU 立即转去执行另一道程序，该机制依赖**中断技术**，使系统各部件尽可能保持忙碌

>[!note] 核心特点
>- **多道**：内存中同时驻留多道程序
>- **宏观上并行**：多道程序同时在运行，先后开始但都未结束
>- **微观上串行**：任意时刻CPU上仅有一道程序在执行，轮流占用 CPU，交替进行
>- **无序性**：各道程序的开始、完成顺序与其进入内存的顺序无对应关系

>[!success] 优点
>1. **CPU利用率高**：利用程序之间的I/O空档切换执行，减少CPU空闲等待
>2. **系统吞吐量大**：单位时间内完成的总工作量显著提升
>3. **资源利用率高**：CPU、内存、I/O设备均可被多道程序共享

>[!failure] 缺点
>1. **作业平均周转时间长**：作业需要排队等待进入内存和处理
>2. **无交互能力**：用户提交作业后直到完成才能获得结果，调试困难

### 需要解决的关键问题

多道批处理系统催生了操作系统需要解决的核心问题：

- **处理机管理**：如何分配CPU给各道程序
- **存储器管理**：如何为多道程序分配内存并保证互不干扰
- **设备管理**：如何分配和调度I/O设备
- **文件管理**：如何组织和管理大量程序与数据
- **作业管理**：如何组织和调度作业

# 分时操作系统

⭐**分时技术**将处理器的运行时间划分为很短的**时间片**，按时间片轮流将处理器分配给各联机作业使用，以**人机交互**为核心目标，主要用于交互式作业而非批处理作业

>[!note] 基本特征
>1. **多路性（同时性）**：允许多个用户通过各自终端同时使用一台计算机
>2. **独立性**：各用户彼此独立操作，互不干扰，感觉不到他人存在
>3. **及时性**：用户请求能在较短时间间隔内得到响应
>4. **交互性**：用户可通过终端直接与系统进行人机对话

# 实时操作系统

为保证某些任务具有**较高时间约束和可靠性**而发展起来的系统，要求系统能在**严格时间范围内**对外部事件做出响应，根据**时限要求的严格程度**，实时操作系统可分为：

- **硬实时系统**：必须在**绝对严格**的截止时间内完成任务，否则会造成严重后果（如导弹制导、自动驾驶）
- **软实时系统**：允许偶尔错过截止时间而不至于造成灾难性后果（如视频播放、网络通信）

>[!note] 基本特征
>- **强时效性**：在严格的时间约束内完成处理
>- **高可靠性**：系统必须高度可靠，不能出现故障（通常采用多级容错措施）
>- **确定性**：任务执行时间必须是可预测和可确定的

# 网络操作系统

将计算机网络中的各台计算机**有机地结合起来**，提供统一的**全网范围**的资源管理和共享能力；该系统中，各计算机保持**独立运行**，用户需要**显式**指定远程资源的位置才能访问，系统不提供统一的全局视图

>[!note] 主要功能
>- 网络通信管理：实现计算机之间的数据通信
>- 网络资源管理：实现网络中**共享资源**（如文件、打印机）的统一管理
>- 网络安全管理：提供网络访问控制、身份认证等安全服务
>- 网络服务：提供电子邮件、文件传输、远程登录等网络服务

# 分布式操作系统

系统中若干台计算机**相互协作**完成同一个共同任务，各计算机地位平等，没有主从关系，系统资源可被所有用户**透明地共享**

>[!note] 特点
>- **分布性**：物理资源和处理能力在地理上分散
>- **透明性**：对用户隐藏系统的分布性，表现为**单一系统映像**
>- **并行性**：多台计算机可以**同时**执行，可将一个任务自动分解为多个子任务在多个节点上**并行执行**，从而加速计算过程
>- **统一性**：用一个全局的操作系统管理所有资源
>- **可拓展性与容错能力**：支持动态重组
## 与网络操作系统的区别

| 对比维度     | 网络操作系统       | 分布式操作系统           |
| -------- | ------------ | ----------------- |
| **耦合程度** | 松耦合          | 紧耦合               |
| **通信方式** | 通常基于消息传递     | 支持共享存储器或消息传递      |
| **资源视图** | 用户需指明资源所在节点  | 用户看到的是一台逻辑上的统一计算机 |
| **任务执行** | 各节点各自运行，相对独立 | 任务分布到多台计算机上协作完成   |
| **管理方式** | 各节点独立管理自身资源  | 统一管理和调度全网资源       |

# 微机操作系统

面向个人计算机的操作系统，根据**同时支持的用户数**和**同时运行的任务数**，可分为三类：

## 单用户单任务操作系统

同一时间只允许**一个用户**使用计算机，且该用户一次只能运行**一个程序**，用户独占全机资源，结构简单

- **特点**：无并发，无多用户隔离，资源利用率低
- **典型代表**：CP/M、MS-DOS

## 单用户多任务操作系统

同一时间只允许**一个用户**使用计算机，但该用户可以同时启动并运行**多个程序**（多任务并发）；是目前桌面操作系统的主流形态

- **特点**：支持多任务并发，CPU与外设并行，用户界面友好，追求**易用性**和**丰富的应用生态**，兼具良好的人机交互性、高效率和稳定性
- **典型代表**：Windows、macOS、Linux（桌面发行版）

## 多用户多任务操作系统

同一时间允许多个**用户**通过各自的终端同时使用同一台计算机，每个用户又可以同时运行**多个程序**

- **特点**：提供完善的**用户隔离**和**资源分配**机制（每个用户有独立账户、文件权限、进程空间），强调**安全性**和**公平调度**
- **典型代表**：UNIX、Linux服务器版和 Windows Server 系列

>[!summary] 操作系统发展总结
>操作系统的发展主线是从**人工操作**（完全无操作系统）→ **脱机I/O**（缓解人机速度矛盾）→ **批量处理**（追求资源利用率与自动化）→ **分时系统**（追求交互性与及时响应）→ **实时系统**（追求严格时效与高可靠性）→ **网络/分布式系统**（追求资源共享与协同）→ **微机操作系统**（追求易用性与生态），每一阶段都回应了当时最核心的技术与应用需求
