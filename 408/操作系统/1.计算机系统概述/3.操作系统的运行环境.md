# 处理器运行模式

计算机系统中，CPU 执行程序时，根据对资源和机器指令的使用权限，将 CPU 的运行模式划分为不同级别

## 用户态（目态）与内核态（管态/核心态）

| 运行模式    | 别名     | 权限  | 说明                                   |
| ------- | ------ | --- | ------------------------------------ |
| **用户态** | 目态     | 低   | CPU 只能执行**非特权指令**，只能访问受限的地址空间        |
| **内核态** | 管态/核心态 | 高   | CPU 可以执行**一切指令**（包括特权指令），可访问所有内存和寄存器 |

>[!important] 区分依据
>CPU 处于**用户态**还是**内核态**，由**程序状态字寄存器（PSW）** 中的某一位（状态位/模式位）来标识，0 为内核态，1 为用户态

## 特权指令与非特权指令

- **特权指令**：只能在内核态下执行的指令，涉及对系统资源的访问和控制，如 I/O 指令、开关中断指令、设置时钟指令、内存清零指令、停机指令、**返回用户态**等
- **非特权指令**：用户态和内核态均可执行的指令，如普通的算术运算、逻辑运算、数据传送等

>[!warning] 用户态执行特权指令将产生**异常**（陷入），CPU 会切换到内核态并由操作系统接管处理

## 内核态与用户态的切换

CPU 的状态切换通过**中断/异常机制**实现：

| 切换方向      | 触发方式                 | 说明                    |
| --------- | -------------------- | --------------------- |
| 用户态 → 内核态 | 中断、异常、系统调用（陷入）       | **唯一途径**，由硬件自动完成状态切换  |
| 内核态 → 用户态 | 操作系统通过设置 PSW 的特权指令实现 | 在返回用户程序即将控制权交还**前**完成 |

>[!note] 切换时的操作
>1. 保存当前程序的上下文（PC、PSW、通用寄存器等）到内核栈
>2. 将 CPU 模式切换为内核态，加载内核的地址空间
>3. 根据中断/异常向量表跳转到对应的处理程序

# 中断和异常

## 中断的概念

**中断（Interrupt）** 是指 CPU 在执行程序过程中，由于系统内部或外部发生了某个**急需处理的事件**，CPU **暂停**当前程序的执行，转而执行相应的中断处理程序，处理完毕后再**返回**被中断的程序继续执行的过程，这一机制由**硬件**强制保障，确保状态切换的**安全性**和**原子性**

>[!important] 中断的作用
>- **实现并发**：使 CPU 可以在多道程序之间切换，是实现多道程序并发执行的基础
>- **处理异常**：当程序出现除零、越界等错误时，通过异常机制将控制权交给操作系统
>- **响应外设**：CPU 可及时响应外部 I/O 设备的请求，无需轮询等待
>- **提供系统调用**：用户程序通过陷入（Trap）指令请求操作系统服务

## 中断的分类

故障和自陷异常属于*软件中断*（程序性异常）；终止异常和外中断属于*硬件中断*

### 内中断（异常）

由 CPU 内部产生，与当前执行的指令**同步**发生，异常一旦发生，就必须立即处理，**不可被屏蔽**

| 类型            | 产生原因               | 示例                                | 处理方式            |
| ------------- | ------------------ | --------------------------------- | --------------- |
| **故障（Fault）** | 指令执行中可被纠正的错误       | 缺页异常、非法操作码、除零、**运算溢出**            | 处理后**重新执行**当前指令 |
| **自陷（Trap）**  | 程序主动请求 OS 服务（系统调用） | `int 0x80`、`syscall`、<br>**访管指令** | 处理后执行**下一条**指令  |
| **终止（Abort）** | 无法恢复的**硬件错误**      | 内存校验错、总线错误                        | **直接终止**当前进程    |

### 外中断（中断）

由 CPU 外部的事件触发，与当前指令**异步**发生

| 类型                 | 产生原因               | 示例              |
| ------------------ | ------------------ | --------------- |
| **可屏蔽中断** <br>INTR | 外设发出的可通过设置屏蔽位忽略的中断 | 打印机完成、磁盘 I/O 完成 |
| **不可屏蔽中断** <br>NMI | 必须立即响应的紧急硬件故障      | 电源掉电、内存奇偶校验错    |

>[!summary] 内中断（异常）与外中断的区别
>| 对比维度 | 内中断（异常） | 外中断 |
>|---------|--------------|--------|
>| **触发源** | CPU 内部，与当前指令相关 | CPU 外部，与当前指令无关 |
>| **同步性** | 同步于指令执行 | 异步于指令执行 |
>| **可预测性** | 除陷阱外通常不可预测 | 完全不可预测 |
>| **响应时机** | 指令执行期间或执行后 | 当前指令执行完毕后 |

## 中断的处理过程

通常情况下，中断和异常的具体处理由操作系统及设备驱动器完成
```
1. 关中断（防止新的中断嵌套干扰现场保存）
2. 保存断点（PC、PSW 等存入内核栈）
3. 识别中断源（硬件向量法或软件查询法）
4. 保存现场（通用寄存器等）
       ↓
5. 执行中断服务程序（核心处理逻辑）
       ↓
6. 恢复现场（恢复通用寄存器）
7. 开中断
8. 返回断点（恢复 PC、PSW，继续执行原程序）
```

## 中断处理与子程序调用的区别

核心差异在于前者是**硬件被动响应**，后者是**软件主动调用**：

| 对比维度     | 子程序调用              | 中断处理                                                            |
| -------- | ------------------ | --------------------------------------------------------------- |
| **触发方式** | 软件主动（`CALL`/`JSR`） | 硬件被动（内外事件**随机触发**）                                              |
| **特权切换** | 不涉及，同一模式运行，有主从关系   | 通常伴随**用户态→内核态**切换，两程序相互独立，无固定关联                                 |
| **现场保存** | 仅 PC + 少量寄存器       | **完整上下文**（PC+**PSW**+通用寄存器），其中PC 和 PSW 由**硬件自动保存**，通用寄存器由操作系统保存 |
| **入口寻址** | 编译时固定的函数地址         | 通过**中断向量表**动态查找                                                 |
| **返回指令** | `RET`（仅恢复 PC）      | `IRET`/`RETI`（恢复 PC+PSW）                                        |
| **开关中断** | 无此需要               | 执行期间需**关中断**保护现场，对多个同时到达的中断请求进行优先级裁决                            |

# 系统调用

**系统调用（System Call）** 是操作系统提供给应用程序（程序员）的**程序接口**，是应用程序请求操作系统内核服务的**唯一合法途径**，**不同系统提供不同的系统调用**

>[!important] 系统调用与库函数的区别
>- **系统调用**：运行在内核态，是 OS 内核提供的服务入口，本质上是特殊的**陷阱指令（访管指令）**触发的内中断
>- **库函数**：运行在用户态，通常是对系统调用的封装，目的是简化编程（如 `printf()` 封装了 `write()` 系统调用）
>- 纯计算类库函数（如 `sqrt()`、`strlen()`）完全在用户态执行，不涉及任何系统调用
>
>用户程序可以通过**库函数**间接使用系统调用，也可以直接使用系统调用（如 `syscall` 指令）

## 系统调用的处理过程

1. **用户程序**将系统调用号和参数传入约定的寄存器或栈中
2. 执行**陷入指令（访管指令）**，触发从用户态到内核态的切换
3. **CPU 硬件**自动保存断点（PC、PSW），根据陷阱向量表跳转到系统调用入口
4. **操作系统内核**根据系统调用号查表，调用对应的内核函数
5. 内核函数执行完毕后，将返回值存入寄存器
6. 执行**返回指令**，CPU 切换回用户态，从下一条指令继续执行

## 系统调用与普通过程调用的区别

核心差异在于系统调用涉及**用户态→内核态的模式切换**，开销远大于普通过程调用：

| 对比维度     | 普通过程调用              | 系统调用                                |
| -------- | ------------------- | ----------------------------------- |
| **运行模式** | 同一特权级，无模式切换         | **用户态→内核态**，涉及模式切换                  |
| **切换开销** | 仅 PC + 少量寄存器，**极小** | 完整上下文 + 栈切换（用户栈→内核栈），**大 2~3 个数量级** |
| **栈空间**  | 使用用户栈               | 切换到**内核栈**，与用户栈隔离                   |
| **入口确定** | 函数地址直接跳转，静态链接确定     | 通过**系统调用号**查内核系统调用表                 |
| **返回机制** | 总是直接返回原调用点          | 内核根据调度策略决定是否切换进程                    |
| **权限校验** | 无安全校验               | **校验参数合法性**（指针越界、地址空间检查等）           |

>[!note] 常见的系统调用类型
>| 类型 | 示例 |
>|------|------|
>| 进程控制 | `fork()`、`exit()`、`wait()` |
>| 文件操作 | `open()`、`read()`、`write()`、`close()` |
>| 设备管理 | `ioctl()`、`request()`、`release()` |
>| 信息维护 | `getpid()`、`alarm()`、`sleep()` |
>| 通信 | `pipe()`、`shmget()`、`mmap()` |

# 操作系统内核

**内核**是操作系统中最基本、最核心的核心组件包括与硬件紧密相关的模块（如中断处理程序、设备驱动程序）、运行频率较高的模块（如时钟管理、进程调度）、以及各模块共用的基础操作，集中置于紧靠硬件的层次并**常驻内存**，运行在**内核态**，负责管理系统的各种资源
这样设计的主要目的有：保护核心模块**免遭应用程序破坏**、提升操作系统的**运行效率**

## 支撑功能

支撑功能是确保操作系统**正常运行**所必需的基础机制，保障系统稳定且高效的运行

### 时钟管理

- 操作系统通过**硬件时钟**（如可编程间隔定时器）获取时间基准
- 主要功能：
	- **时间片轮转调度**：产生定时中断，当时钟中断发生时，检查当前进程的时间片是否用完，决定是否切换进程
	- **实时时钟**：维护系统日期和时间
	- **定时器管理**：为进程提供定时唤醒、超时检测等功能
	- **资源统计**：统计 CPU 使用时间、进程运行时间等

### 中断机制

中断是现代操作系统**并发**运行的基础，内核本身依赖中断来实现各种功能的调度和切换

- **保护现场和恢复现场**：中断发生时保存被中断程序的上下文，中断处理完毕后恢复
- **中断屏蔽和嵌套**：通过中断优先级控制，实现关键代码段的原子执行
- **中断向量表**：用于快速定位中断处理程序的入口地址

### 原语

**原语（Primitive）** 是由若干条指令组成的、具有**原子性**的操作序列，其执行必须**一气呵成、不可中断**

>[!important] 原语的核心特性：原子性
>原语的原子性通常通过**关中断**指令和**开中断**指令来实现——在原语执行期间屏蔽所有中断，保证不被其他进程打断

常见的原语包括：
- **进程同步原语**：`Test-and-Set`、`Swap`、信号量的 `wait()` / `signal()`
- **进程通信原语**：`send()`、`receive()`
- **队列操作原语**：入队、出队操作

## 系统资源管理功能

操作系统通过一系列**核心数据结构**（如进程控制块、设备控制块、内存分配表、缓冲区等）对系统资源统一管理

### 进程管理

- 进程的创建与撤销
- 进程的阻塞与唤醒
- 进程的挂起与激活
- 进程间的同步与通信
- 处理机的分配与调度

### 存储器管理

- 内存空间的分配与回收
- 逻辑地址到物理地址的转换（地址映射）
- 不同进程之间的内存保护（防止越界访问）
- 内存的共享机制
- 虚拟存储器的实现（内存扩充）

### 设备管理

- **缓冲管理**：设置各类缓冲区以匹配 CPU 与 I/O 设备的速度差异
- **设备分配**：根据请求为进程分配所需设备
- **设备驱动**：屏蔽硬件差异，向上提供统一的操作接口
- **设备独立性**：用户程序使用逻辑设备名，系统自动映射到物理设备
- 虚拟设备（SPOOLing 技术）：将独占设备改造为可共享的逻辑设备

# 硬件与操作系统的职责划分

在操作系统的运行环境中，中断/异常处理、系统调用、状态切换等过程由**硬件**和**操作系统**分工协作完成：

## 硬件自动完成（芯片级固化的行为）

| 职责                 | 涉及场景                                      |
| ------------------ | ----------------------------------------- |
| **检测中断/异常信号**      | 每条指令执行末尾，CPU 自动检查是否有中断请求或异常发生             |
| **关中断**            | 进入中断处理程序前，自动修改中断屏蔽状态，防止部分中断重入             |
| **保存断点（PC + PSW）** | 将当前程序的 PC 和 PSW **自动压入内核栈**，无需软件干预        |
| **模式切换**           | 修改 PSW 模式位，将 CPU 从**用户态切换为内核态**           |
| **中断向量寻址**         | 根据中断号/异常类型，从**中断向量表（IVT）** 中取出处理程序入口地址    |
| **执行陷入/访管指令**      | 执行 `int`、`syscall`、`trap` 等指令时完成上述状态保存与切换 |

## 操作系统完成（软件实现）

| 职责 | 涉及场景 |
|------|---------|
| **保存/恢复通用寄存器** | 中断/系统调用入口处，OS 将通用寄存器压栈保存，返回前恢复 |
| **中断优先级裁决** | 多个中断同时到达时，OS 根据优先级决定处理顺序 |
| **中断嵌套管理** | OS 通过选择性开/关中断控制是否允许更高优先级中断打断当前处理 |
| **中断服务程序主体逻辑** | 执行具体的中断处理（如设备驱动 I/O 完成处理） |
| **系统调用号查表分发** | OS 根据系统调用号查找 `sys_call_table`，调用对应内核函数 |
| **参数合法性校验** | 校验用户传入的指针、大小等参数是否在合法地址空间内 |
| **内核态→用户态返回** | 通过特权指令设置 PSW，将模式切换回用户态后返回 |
| **进程调度决策** | 中断/系统调用返回前，OS 可选择切换运行进程（而非返回原进程） |
| **中断向量表的初始化** | 系统启动时，OS 负责建立并加载中断向量表 |

## 典型流程中的分工示例

以一次**外中断**的处理流程为例，明确各步骤的执行者：

```
[硬件] 检测到中断信号 → [硬件] 关中断 → [硬件] 保存 PC+PSW（断点）
→ [硬件] 查中断向量表跳转 → [OS] 保存通用寄存器（现场）
→ [OS] 执行中断服务程序 → [OS] 恢复通用寄存器
→ [硬件/OS] 开中断 → [硬件] 恢复 PC+PSW（返回断点）
```

以一次**系统调用**的处理流程为例：

```
[用户程序] 传入系统调用号和参数 → [用户程序] 执行陷入指令
→ [硬件] 保存 PC+PSW，切换为内核态，查陷阱向量表跳转
→ [OS] 保存通用寄存器 → [OS] 参数合法性校验
→ [OS] 查系统调用表，调用内核函数 → [OS] 恢复通用寄存器
→ [硬件] 恢复 PC+PSW，切换回用户态，继续执行
```

>[!important] 软硬件分工
>**硬件**负责的是**通用**的、**芯片级固化**的操作——检测中断、保存断点、模式切换、向量跳转；**操作系统**负责的是**可编程**的、**策略相关**的操作——现场保存/恢复、中断服务逻辑、参数校验、调度决策。二者分工明确：硬件提供机制（Mechanism），操作系统提供策略（Policy）

>[!summary] 总结
>操作系统的运行环境围绕**处理器运行模式（用户态/内核态）**、**中断与异常机制**、**系统调用**以及**操作系统内核**四个方面展开。其中：
>- **中断**是实现 CPU 状态切换和并发执行的基础硬件机制
>- **系统调用**是用户程序访问内核服务的唯一合法途径
>- **内核**是 OS 的核心，其功能分为支撑功能（时钟管理、中断机制、原语）和资源管理功能（进程管理、存储器管理、设备管理）
