# 基本概念

**操作系统引导（Booting）** 是指计算机在开机时，通过一系列自举过程将操作系统从磁盘**加载到内存**并使之运行的过程。完成引导功能的程序称为**引导程序（Bootstrap Program）**，通常存放在**固件（如 ROM/EEPROM）** 或**磁盘的特定区域（引导扇区）** 中

>[!info] 引导程序的特点
>引导程序是计算机**开机后第一个执行的程序**，其核心任务是**将操作系统内核加载到内存**并将控制权移交给它

# 传统 BIOS + MBR 引导过程

## ① CPU 激活与 BIOS 初始化

- 计算机通电后，CPU 从 ROM 中读取并执行 **BIOS（Basic Input/Output System，基本输入/输出程序）**，初始化基本硬件，**建立中断向量表**，为引导阶段的设备访问提供支持
- **BIOS** 执行**加电自检（Power-On Self Test, POST）**，检测 CPU、内存、主板等关键硬件是否正常工作
- 若检测失败，发出蜂鸣声或显示错误信息

>[!warning] 建立中断向量表
>通电自检中需要使用中断向量表，在 CPU 监测到中断信号后，根据中断信号查询中断向量表，跳转到对应中断处理程序
## ② 硬件自检和 BIOS 启动引导

- POST 通过后，BIOS 读取**CMOS RAM**（主板上由电池供电的配置存储器，用于保存启动顺序、系统时间等参数）中存储的**启动顺序配置**（Boot Sequence），确定从哪个设备引导（如硬盘、U 盘、光驱等）
- 按顺序尝试各启动设备，直到找到一个**可引导的设备**

## ③ 读取主引导记录（MBR）

- BIOS 将可引导设备的**第一个物理扇区（0 号磁道、0 号柱面、1 号扇区，即 0 号扇区）** 读入内存
- 该扇区大小为 **512 字节**，称为**主引导记录（Master Boot Record, MBR）**
- MBR 的结构如下：
  - **引导代码（446 字节）**：**第 1 阶段引导程序**，负责扫描分区表、定位活动分区并加载其引导扇区
  - **分区表（DPT, Disk Partition Table，64 字节）**：记录磁盘的分区信息，每个分区项占 **16 字节**，最多支持 **4 个主分区**
  - **结束标志（2 字节）**：固定为 `0x55AA`，用于验证 MBR 是否有效

## ④ 查找活动分区并加载 PBR

- 引导程序扫描分区表，找到标记为**活动（Active）** 的分区（即引导分区，**装有操作系统的分区**）
- 将活动分区的**第一个扇区**——**分区引导记录（Partition Boot Record, PBR）** 或**卷引导记录（Volume Boot Record, VBR）**——读入内存
- PBR 中存放着**第 2 阶段引导程序**（如 `bootmgr`、`NTLDR`、`GRUB` 的启动代码）

## ⑤ 加载引导管理器

- PBR 中的代码负责加载**引导管理器（Boot Manager）**（如 Windows 的 `bootmgr`、Linux 的 `GRUB`）
- 引导管理器提供**操作系统选择菜单**（多系统环境下），加载对应的操作系统内核

## ⑥ 加载操作系统内核

- 引导管理器将操作系统内核文件（如 Windows 的 `ntoskrnl.exe`、Linux 的 `vmlinuz`）从磁盘加载到内存
- 将控制权移交给内核，重建**中断处理机制**，不再依赖 BIOS 提供的服务

## ⑦ 内核初始化与用户环境启动

- 内核完成核心子系统的**初始化**：
	- 内存管理模块构建内核页表及用户进程地址空间框架
	- 进程调度模块初始化进程控制块（PCB）链表、就绪队列与阻塞队列
	- 设备驱动模块建立设备控制块、设备分配表以及 I/O 缓冲区结构
- 初始化完成后，内核启动首个用户态进程（如 Linux 的 init），由此逐步构建完整的用户操作环境

>[!summary] BIOS + MBR 引导流程总结
>**CPU 激活与 BIOS 初始化** → **硬件自检（POST）** → **BIOS 读取 MBR** → **MBR 引导代码扫描分区表** → **加载活动分区的 PBR** → **加载引导管理器** → **加载操作系统内核**

# UEFI + GPT 引导方式

## 基本概念

- **UEFI（Unified Extensible Firmware Interface）** 是传统 BIOS 的替代方案，提供更现代化的固件接口
- **GPT（GUID Partition Table）** 是新一代磁盘分区表标准，替代 MBR

## 与传统 BIOS 的主要区别

| 对比项 | BIOS + MBR | UEFI + GPT |
|-------|-----------|------------|
| 寻址能力 | 仅支持 **2 TB** 以下磁盘 | 支持 **2 TB 以上**磁盘（最多 9.4 ZB） |
| 分区数量 | 最多 **4 个主分区** | 最多 **128 个主分区** |
| 引导方式 | 从 MBR 引导扇区启动 | 直接读取 **EFI 系统分区（ESP）** 中的引导文件 |
| 运行模式 | **16 位实模式**，寻址受限 | **32/64 位保护模式**，功能更强 |
| 启动速度 | 较慢（需逐阶段加载） | 较快（可直接加载引导管理器） |

## UEFI 引导过程

1. **加电**：CPU 执行 UEFI 固件代码
2. **硬件初始化**：UEFI 初始化所有硬件（包括 CPU、芯片组、内存、显卡等）
3. **读取 NVRAM 变量**：UEFI 从 NVRAM 中读取**引导项配置**，直接定位到 **EFI 系统分区（ESP）**
4. **加载引导程序**：从 ESP 分区（通常格式化为 FAT32）中读取 `.efi` 引导文件（如 `\EFI\Microsoft\Boot\bootmgfw.efi` 或 `\EFI\grubx64.efi`）
5. **加载操作系统内核**：引导管理器加载操作系统内核并移交控制权

> [!important] 安全启动（Secure Boot）
> UEFI 支持**安全启动**功能，可验证引导程序的**数字签名**，防止未经授权的恶意代码（如 Rootkit）在系统启动时加载，增强系统安全性

# 关键知识点

## MBR 与 PBR 的区别

| 项目 | MBR（主引导记录） | PBR（分区引导记录） |
|-----|-----------------|-------------------|
| 位置 | **整个磁盘的第一个扇区（0 号扇区）** | **活动分区的第一个扇区** |
| 作用 | 扫描分区表，定位活动分区 | 加载该分区上的引导管理器 |
| 是否依赖操作系统 | 与 OS 无关，由 BIOS 加载 | 与 OS 相关（不同 OS 的 PBR 内容不同） |

## 引导过程中需要多次"读入内存"的步骤

1. BIOS 将 **MBR** 读入内存
2. MBR 中的引导程序将 **PBR** 读入内存
3. PBR 中的代码将 **引导管理器** 读入内存
4. 引导管理器将 **操作系统内核** 读入内存

## 常见的引导管理器

- **Windows**：`bootmgr`（Boot Manager），配置文件为 `BCD（Boot Configuration Data）`
- **Linux**：`GRUB`（GRand Unified Bootloader），配置文件为 `/boot/grub/grub.cfg`
- **macOS**：`boot.efi`

## 引导过程中可能出现的常见问题

- **`Missing operating system`**：MBR 中未找到活动分区，或活动分区标记损坏
- **`Bootmgr is missing`**：引导管理器文件（bootmgr）丢失或损坏
- **`Invalid partition table`**：分区表信息损坏
- **`No bootable device`**：BIOS 在启动设备列表中未找到可引导设备

>[!summary] 408 常考要点
>- MBR 位于 **0 号扇区**，大小为 **512 字节**（引导代码 446 B + 分区表 64 B + 结束标志 2 B）
>- 引导过程的关键阶段：**ROM 中的引导程序** → **MBR** → **PBR** → **引导管理器** → **操作系统内核**
>- UEFI + GPT 与 BIOS + MBR 的对比，尤其是**分区数、磁盘容量、引导方式**的差异
>- **安全启动（Secure Boot）** 是 UEFI 特有的安全特性
