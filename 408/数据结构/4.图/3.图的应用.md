# 最小生成树
- **最小生成树(Minimum-Spanning-Tree, MST)**：在**带权连通无向图**中，权值之和最小的[[1.图#生成树、生成森林|生成树]]
- 当图中含有权值相同的边时，则 MST **可能不唯一**，当所有边的权值互不相通时，MST 唯一
- 最小生成树的权值之和是唯一的，且为最小值
- 边数 = 顶点数 - 1
## Prim（普里姆算法）
算法步骤：
1. 初始化：任选一起点，加入 MST 集合
2. 循环：更新所有未加入顶点到 MST 集合的距离，**选择距离最小的顶点**加入 MST
```c
void Prim(G, T){
	T = ∅;//初始化空树
	U = {w};//添加任意顶点
	while(V - U != ∅){//树中不含全部顶点
		设(u,v)是使 u∈U 与 v∈(V - U),且权值最小的边；
		T = T ∪ {(u,v)};
		U = U ∪ {v};
	}
}
```
### 算法特征
- 基于贪心策略
- 时间复杂度为 $O(|V|^2)$，与边数 $|E|$ 无关，适用于求解**稠密图**
## Kruskal（克鲁斯卡尔算法）
### 算法步骤
1. 初始化：将所有边按权值从小到大排序，每个顶点各自构成一个连通分量
2. 循环：从权值最小的边开始，若该边两端点属于**不同的连通分量**，则将此边加入 MST，否则舍去
```c
void Kruskal(G, T){
	T = V;//初始化树T，仅含顶点
	numS = n;// 连通分量数
	while(numS > 1){
		从 E 中选取当前权值最小的边(u,v);
		E = E - {(u,v)};
		if(u 和 v 不在同一连通分量中)
			T = T ∪ {(u,v)};
			numS--;
	}
}
```
### 算法特征
- 基于贪心策略
- 时间复杂度为 $O(|E|\log|E|)$，与顶点数 $|V|$ 无关，适用于求解**边稀疏但顶点较多**的图

## Dijkstra 求单源最短路径
算法需要维护三个辅助数组：
- `final[]`：标记各顶点是否已找到最短路径
- `dist[]`：记录从源点到各顶点的**最短路径长度**（尚未确定的为当前最短距离）
- `path[]`：记录路径上的**前驱顶点**，用于回溯完整路径
### 算法步骤
1. 初始化：设集合 $S=\{0\}$，`dist[]`的初始值为 `dist[i] = arcs[0][i], i = 1,2,···,n-1`
2. 选择最短路径顶点：从尚未确定最短路径的顶点集合 $V-S$ 中选出 `dist` 值最小的顶点 $v_j$，将 $v_j$ 加入 $S$ 中
3. 松弛操作：以 $v_j$ 为中间点，更新 $V-S$ 中所有与 $v_j$ 邻接的顶点 $v_k$ 的 `dist`：若 `dist[j] + arcs[j][k] < dist[k]`，则 `dist[k] = dist[j] + arcs[j][k]`
4. 重复步骤 2 和 3 共 $n-1$次，直到 $S$ 包含所有顶点（即 $S = V$）
### 算法特征
- 基于贪心策略
- 使用邻接矩阵的时间复杂度为 $O(|V|^2)$ ；若改用邻接表，虽然松弛操作的总代价可以降至 $O(|E|)$，但由于查找最小 `dist[]` 值仍需 $O(|V|)$ 时间，所以总时间复杂度仍为 $O(|V|^2)$ ；如果使用堆优化，时间复杂度则为 $O(E\log{|V|})$
- 适用于带权有向图和无向图
- 不适用于存在**负权边**的图，后续路径可能通过负权边绕回，造成错误
## Floyd 求各顶点之间最短路径
算法维护两个辅助矩阵：
- `Aᵏ[i][j]`：当前从顶点 $i$ 到顶点 $j$ 的**最短路径长度**（允许经过的顶点编号 $\le k$），初始 $A^{-1}[i][j] = arcs[i][j]$
- `path[i][j]`：记录从 $i$ 到 $j$ 的路径上**最后一个经过的中间顶点**，用于回溯完整路径
### 算法步骤
1. 初始化：将邻接矩阵 `arcs` 的值赋给 `A`，`path[i][j] = -1`（表示直接相连，无中间顶点）
2. 动态规划迭代：以每个顶点 $k$ 作为中间点，依次更新所有顶点对 $(i,j)$：
   - 若 `A[i][j] > A[i][k] + A[k][j]`，则更新 `A[i][j] = A[i][k] + A[k][j]`，并将 `path[i][j]` 置为 $k$
3. 重复步骤 2 共 $|V|$ 轮（$k = 0,1,\cdots,n-1$），最终 `A` 中即为各顶点间的最短路径长度
```c
vector<vector<long long>> shortestPathFloyd(int n, vector<vector<int>>& edges) {
    const long long INF = LLONG_MAX / 2; // 防止加法溢出
    vector f(n, vector<long long>(n, INF));
    for (int i = 0; i < n; i++) {
        f[i][i] = 0;
    }

    for (auto& e : edges) {
        int x = e[0], y = e[1];
        long long wt = e[2];
        f[x][y] = min(f[x][y], wt); // 如果有重边，取边权最小值
        f[y][x] = min(f[y][x], wt); // 无向图
    }

    for (int k = 0; k < n; k++) {
        for (int i = 0; i < n; i++) {
            if (f[i][k] == INF) { // 针对稀疏图的优化
                continue;
            }
            for (int j = 0; j < n; j++) {
                f[i][j] = min(f[i][j], f[i][k] + f[k][j]);
            }
        }
    }
    return f;
}
```
### 算法特征
- 基于**动态规划**思想
- 时间复杂度为 $O(|V|^3)$，空间复杂度为 $O(|V|^2)$
- 可处理**负权边**，但**不能有负权回路**（否则无最短路径）
- 代码简洁（三重循环），适合顶点数较少的图
> 在边权非负的条件下，也可以对每个顶点使用 Dijkstra 算法；Dijkstra 算法的时间复杂度为 $O(|V|^2)$，共需运行 $|V|$ 次，所以总时间复杂度为 $O(|V|^3)$，与 Floyd 算法相同
# 有向无环图描述表达式
- **有向无环图(Directed Acyclic Graph,DAG)**：不存在有向环的有向图
- 利用 DAG **共享子表达式**，优化表达式存储，**节省存储空间**，避免重复子表达式
- 在表达式的有向无环图表示中，**不会出现重复的操作数顶点**
> **示例**：表达式 $((a+b)*(b*(c+d))+(c+d)*e)*((c+d)*e)$![[二叉树和有向无环图描述表达式|900]]
## 算法步骤
1. 将操作数（变量/常量）作为叶子节点（不重复）
2. 按运算顺序逐步加入运算符节点，**合并相同的子表达式**
3. 最终得到无重复节点的 DAG
# 拓扑排序
- **AOV 网(Activity On Vertex)**：顶点表示活动，有向边表示活动之间的先后关系，**边没有权值**
- **拓扑排序**：将 AOV 网中所有顶点排成一个线性序列，满足：
	- 每个顶点恰好出现一次
	- 若图中存在边 `<vᵢ, vⱼ>`，则序列中 vᵢ 必须在 vⱼ 之前
## 算法步骤
1. 从 AOV 网中随机选择一个**没有前驱即入度为 0** 的顶点并输出
2. 从网中删除该顶点和所有以它为起点的有向边
3. 重复步骤 1 和 2，直到当前 AOV 网为空或无法找到新的入度为 0 的顶点为止，**后一种情况说明有向图中必有环**
```c
bool TopologicalSort(ALGraph G){
    InitStack(S);                         //初始化栈，存储入度为 0 的顶点
    int i;
    for (int i = 0; i < G.vexnum; i++)
        if (indegree[i] == 0)
            Push(S, i);                   // 将所有入度为 0 的顶点入栈
    int cnt = 0;
    while (!StackEmpty(S)){
        Pop(S, i);                        // 出队一个顶点 u
        print[cnt++] = i;                 // 输出 u
        for (p = G.vertices[i].firstarc; p; p = p->nextarc){
            int v = p->adjvex;
            if (!--indegree[v])
                Push(S, v);            // 入度变为 0 则入队
        }
    }
    if (cnt < G.vexnum)
        return false;                     // 存在环，拓扑排序失败
    else
        return true;                      // 拓扑排序成功
}
```
- 因为输出每个顶点的同时还要删除以它为起点的边，所以采用**邻接表**存储时，时间复杂度为 $O(|V|+|E|)$；采用**邻接矩阵**存储时，时间复杂度为 $O(|V|^2)$
## DFS 实现拓扑排序
- 利用 DFS 的**递归返回顺序**实现拓扑排序（本质是逆拓扑排序）
- 原理：当 DFS 访问到某个顶点时，先递归访问其所有后继，再将该顶点加入栈中；最终栈顶到栈底即为拓扑有序序列
> **注意**：DFS 实现拓扑排序的前提是图中**无环**；若图中存在环，DFS 将无法正确返回（可通过引入 `recStack[]` 标记递归栈来判断是否存在环）
## 逆拓扑排序
### 算法步骤
1. 从 AOV 网中选择一个**没有后继即出度为 0** 的顶点并输出
2. 从网中删除该顶点和所有以它为终点的有向边（对应邻接点的出边）
3. 重复步骤 1 和 2，直到所有顶点均已输出
## 唯一性判定
- 若拓扑排序的**每一步都恰好只有一个入度为 0 的顶点**，则拓扑排序结果**唯一**，反之，若某一步存在**多个入度为 0** 的顶点，则排序结果**不唯一**（选择不同的顶点输出会产生不同的序列）
- 为顶点人为添加**额外的偏序关系**（即增加有向边），减少入度为 0 的顶点同时出现的情况可以使拓扑排序结果唯一，
- **拓扑排序结果不能唯一确定图**，同一个拓扑序列可能对应**多个不同的有向无环图**拓扑序列只规定了顶点之间的前后关系（偏序），而图中具体有哪些边（即偏序关系的传递闭包中的哪些边被显式画出）可以有多种选择
- 因此，拓扑排序结果与图之间是**多对多**的关系：一个 DAG 可能有多组拓扑序列，一组拓扑序列也可能对应多个 DAG
## 算法特征
- **结果不唯一**：若拓扑排序的每一步都恰好只有一个入度为 0 的顶点，则拓扑排序结果唯一（否则不唯一）
- **判环**：若排序后输出的顶点数小于图中总顶点数则存在环；**拓扑排序是检测有向图是否有环的重要方法**
- **有向无环图即存在拓扑排序即可重排为三角矩阵**：任何有向无环图**至少存在（即不唯一）** 一个拓扑序列，也可根据拓扑排序结构对顶点重新编号，使得新的邻接矩阵称为上三角矩阵
- **时间复杂度**：采用邻接表存储时，拓扑排序和逆拓扑排序的时间复杂度均为 $O(|V|+|E|)$

# 关键路径
- **AOE 网(Activity On Edge)**：用**边**表示活动，**顶点**表示事件，边有权值代表活动及其持续时间
	- 只有某顶点代表事件发生后，从该顶点出发的各有向边所代表的活动才能开始
	- 只有当进入某顶点的各有向边所代表的活动都完成时，该顶点所代表的时间才能发生
	- 仅有一个源点（入度为 0，工程开始）和一个汇点（出度为 0，工程结束）
- **关键路径**：从源点到汇点**最长**的路径（决定工程的最短完成时间）
- **关键活动**：关键路径上的活动，其时间余量为 0，任一关键活动延迟，整个工程的完成时间将推迟
## 关键参量的定义
### 事件 $v_k$ 的最早发生时间 $v_e(k)$
- 从源点 $v_1$ 到顶点 $v_k$ 的**最长路径长度**决定了事件 $v_k$ 的最早发生时间
- 递推公式（按拓扑顺序计算）：
$$
\begin{cases}
v_e(源点) = 0 \\[2pt]
v_e(k) = \max\{v_e(j) + w(j,k) \mid \langle v_j, v_k \rangle \in E\}
\end{cases}
$$
### 事件 $v_k$ 的最迟发生时间 $v_l(k)$
- 在不推迟整个工程完成时间的前提下，事件 $v_k$ **最晚**必须发生的时间
- 递推公式（按**逆**拓扑顺序计算）：
$$
\begin{cases}
v_l(汇点) = v_e(汇点) \\[2pt]
v_l(k) = \min\{v_l(j) - w(k,j) \mid \langle v_k, v_j \rangle \in E\}
\end{cases}
$$
### 活动 $a_i$ 的最早开始时间 $e(i)$
- 活动 $a_i$ 由边 $\langle v_k, v_j \rangle$ 表示，其最早开始时间等于**起点事件 $v_k$ 的最早发生时间**
- 公式：$$e(i) = v_e(k)$$
### 活动 $a_i$ 的最迟开始时间 $l(i)$
- 活动 $a_i$ 由边 $\langle v_k, v_j \rangle$ 表示，其最迟开始时间等于**终点事件 $v_j$ 的最迟发生时间**减去该活动的持续时间
- 公式：$$l(i) = v_l(j) - w(k,j)$$
### 活动 $a_i$ 的时间余量 $d(i) = l(i) - e(i)$
- 活动 $a_i$ 在不影响整个工期的情况下**可以拖延的时间**
- 公式：$$d(i) = l(i) - e(i)$$
- 若 $d(i) = 0$，则该活动为**关键活动**；所有关键活动构成的路径即为**关键路径**
## 算法步骤
1. **拓扑排序求 $v_e$**：对 AOE 网进行拓扑排序，按拓扑顺序依次计算各事件的最早发生时间 $v_e(k)$
2. **逆拓扑排序求 $v_l$**：按逆拓扑顺序依次计算各事件的最迟发生时间 $vl(k)$（令 $v_l(汇点) = v_e(汇点)$ 逆推）
3. **求 $e(i)$**：遍历所有活动（边），每个活动的最早开始时间等于其起点事件的最早发生时间 $e(i) = v_e(k)$
4. **求 $l(i)$**：每个活动的最迟开始时间等于其终点事件的最迟发生时间减去权值 $l(i) = v_l(j) - w(k,j)$
5. **找关键活动**：找出满足 $e(i) = l(i)$ 的活动（即时间余量 $d(i)=0$ 的活动），由这些活动构成的从源点到汇点的路径即为**关键路径**
> **注意事项**：
> - 关键路径**不唯一**：有多条关键路径时，只压缩一条无法缩短工期，必须同时压缩**所有关键路径上共有**的关键活动
> - 关键活动的时间增加会**直接影响工期**，而非关键活动的时间余量可用于资源调配
> - 若关键路径上的某关键活动缩短到一定程度，可能变成非关键路径，因此不能无限制缩短
## 算法特征
- 基于**拓扑排序**和**动态规划**思想
- 时间复杂度为 $O(|V| + |E|)$（拓扑排序 + 逆拓扑排序各一次）
- 仅适用于 **AOE 网（有向无环图）**，且要求图**只有一个源点和一个汇点**
- 关键路径的长度决定了**整个工程的最短完成时间**
