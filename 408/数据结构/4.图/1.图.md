# 图的定义
图 $G$ 是由**非空**顶点集 $V$ 和边集 $E$ 组成，记为 $G=(V,E)$
若 $V=\{v_1,v_2,...,v_n\}$，则用 $|V|$ 表示图 $G$ 中边的数目； $E \subseteq \{\{v_i,v_j\} | v_i,v_j \in V, i \neq j\}$，用 $|E|$ 表示图 $G$ 中边的数目
> 注意：图不可以是空图，也就是说图中不能一个顶点也没有，但边可以为空
# 图的基本术语
## 有向图
若 $E$ 是有向边（也称弧）的有限集合，则称图 $G$ 为*有向图*；弧是顶点的有序对，记为 $<v,w>$，其中， $v$ 称为**弧尾或弧始**，$w$ 称为弧头，表示从 $v$ 到 $w$ 的有向边，也称 $v$ 邻接到 $w$
## 无向图
若 $E$ 是无向边的有限集合，则称图 $G$ 为*无向图*；边是顶点的无序对，记为 $(v,w)$ 或 $(w,v)$，表示 $v$ 和 $w$ 之间有一条边，称 $v$ 和 $w$ 互为邻接点，或称边 $(v,w)$ 和 $v,w$ 相关联
## 简单图和多重图
- **简单图**：不存在重复边且不存在顶点到自身的边的图
- **多重图**：存在重复边或存在顶点到自身的边的图；在数据结构中，若不特别说明，一般讨论的都是简单图
## 顶点的度、入度和出度
- **度 (Degree)**：在无向图中，顶点 $v$ 的度是指依附于 $v$ 的边的条数，记为 $\text{TD}(v)$
- **入度 (In-degree)**：在有向图中，顶点 $v$ 的入度是以 $v$ 为终点的有向边的数目，记为 $\text{ID}(v)$
- **出度 (Out-degree)**：在有向图中，顶点 $v$ 的出度是以 $v$ 为起点的有向边的数目，记为 $\text{OD}(v)$
- 有向图顶点 $v$ 的度等于入度与出度之和，即 $\text{TD}(v) = \text{ID}(v) + \text{OD}(v)$
- 所有顶点的度之和 $= 2|E|$（每条边贡献 2 度）
## 路径、路径长度和回路
- **路径**：从顶点 $v_p$ 到顶点 $v_q$ 的一个顶点序列 $v_p,v_0, v_1, v_2, ..., v_m,v_q$，其中 $(v_{i-1}, v_i) \in E$（无向图）或 $<v_{i-1}, v_i> \in E$（有向图）
- **路径长度**：路径上边的数目
- **回路（环）**：第一个顶点和最后一个顶点相同的路径，若一个图有 $n$ 个顶点，且有大于 $n-1$ 条边，则此图一定有环
## 简单路径、简单回路
- **简单路径**：路径中顶点不重复出现的路径，**回路不是简单路径**
- **简单回路**：除第一个顶点和最后一个顶点相同外，其余顶点不重复出现的回路
## 距离
从顶点 $u$ 到顶点 $v$ 的**最短路径**的长度（若存在）称为从 $u$ 到 $v$ 的距离；若从 $u$ 到 $v$ 不存在路径，则称距离为无穷大（$\infty$）
## 子图
设有图 $G=(V,E)$ 和图 $G'=(V',E')$，若 $V' \subseteq V$ 且 $E' \subseteq E$，则称 $G'$ 是 $G$ 的*子图*
若有 $V' \subset V$ 或 $E' \subset E$，则称 $G'$ 是 $G$ 的**真子图**
若有满足 $V(G') = V(G)$ 的子图 $G'$，则称其为 $G$ 的**生成子图**
> 注意：并非 $V$ 和 $E$ 的任意子集都能构成 $G$ 的子图，因为 $E'$ 中的边必须依附于 $V'$ 中的顶点
## 连通、连通图和连通分量（无向图）
- **连通**：在无向图中，若从顶点 $v$ 到 $w$ 存在路径，则称 $v$ 和 $w$ 是连通的
- **连通图**：图中任意两个顶点都是连通的，则称该图为连通图；否则称为**非连通图**
- **连通分量**：无向图中的**极大连通子图**称为连通分量
  - 连通图的连通分量只有一个，即其自身
  - 非连通图有多个连通分量
> 非连通图边最多的情况是有 $n-1$ 个顶点构成一个完全图，此时再加入一个顶点则变成非连通图；若一个图有 $n$ 个顶点，且边数小于 $n-1$，则此图必然非连通
## 强连通图、强连通分量（有向图）
- **强连通**：在有向图中，若从顶点 $v$ 到 $w$ 和从 $w$ 到 $v$ 之间都存在路径，则称 $v$ 和 $w$ 是强连通的
- **强连通图**：图中任意一对顶点之间都是**相互**强连通的，则称该图为强连通图
- **强连通分量**：有向图中的**极大强连通子图**称为强连通分量
> 强连通图至少需要 $n$ 条边，构成一个**回路**
## 生成树、生成森林
- **生成树**：**连通图**的**极小连通子图**，包含图中全部 $n$ 个顶点，但只有 $n-1$ 条边（构成一棵树）
  - 若在生成树中任意添加一条边，必形成回路；移除任意一条边，必变成非连通图
- **生成森林**：在**非连通图**中，每个连通分量都可以得到一个生成树，这些生成树构成该非连通图的生成森林
> 注意：生成树是连通图的概念包含全部节点
## 边的权、网和带权路径长度
- **权 (Weight)**：表示图中边具有的相关数值，通常表示距离、代价等
- **网 (Network)**：边上带有权值的图称为**带权图**，也称网
- **带权路径长度**：一条路径上所有边的权值之和
## 完全图（简单完全图）
- **无向完全图**：任意两个顶点之间都存在边的无向简单图。$n$ 个顶点的无向完全图的边数为 $\dfrac{n(n-1)}{2}$
- **有向完全图**：任意两个顶点之间都存在**方向相反**的两条弧的有向简单图。$n$ 个顶点的有向完全图的边数为 $n(n-1)$
## 稠密图、稀疏图
- **稀疏图**：边数很少的图（一般按 $|E| < |V|\log|V|$ 来大致衡量）
- **稠密图**：边数很多的图
- 两者之间没有明确的界限，是相对概念
## 有向树
一个顶点的入度为 0，其余顶点的入度均为 1 的有向图称为*有向树*
> 注意：有向树不一定是树形结构——"树"在图中和数据结构中的定义不同，在图中允许存在单独的有向树作为图的一种特殊形式
# 图的存储结构
## 邻接矩阵
用**一维数组**存储顶点信息和**二维数组**存储边的信息
```c
typedef struct{
	VertexType vex[MaxVertexNum]; //顶点表
	EdgeType edge[MaxVertexNum][MaxVertexNum];//边表
	int vexnum, arcnum;
}MGraph;
```
- 空间复杂度为 $O(|V|^2)$
- 无向图的邻接矩阵一定是**对称矩阵**，因此只需要保存上三角部分，在顶点编号固定的条件下，该邻接矩阵**唯一**
- 对于无向图，邻接矩阵第 $i$ 行或列的非零元素数量正好是顶点 $i$ 的度 $TD(v_i)$
- 对于有向图，邻接矩阵第 $i$ 行的非零元素数量是顶点 $i$ 的出度 $OD(v_i)$，第 $i$ 列的非零元素数量是顶点 $i$ 的入度 $ID(v_i)$
- 邻接矩阵容易判断任意两个点之间是否有边相连，但是要确定**总边数**需要遍历整个矩阵
- 删除边很方便，删除顶点需要移动大量数据
- **稠密图**适合邻接矩阵
- 设图 $G$ 的邻接矩阵为 $A$，$A^n$ 的元素 $A^n[i][j]$ 代表从顶点 $i$ 到顶点 $j$ 长度为 $n$ 的路径数量
## 邻接表
用**顺序存储（顶点表） + 链式存储（边表）** 的方式表示

```c
// 边/弧结点
typedef struct ArcNode{
    int adjvex;                 // 该弧指向的顶点位置（下标）
    struct ArcNode *nextarc;    // 指向下一条弧的指针
    // InfoType info;           // 权值等附加信息（可选项）
}ArcNode;
// 顶点结点
typedef struct VNode{
    VertexType data;            // 顶点信息
    ArcNode *firstarc;          // 指向第一条依附于该顶点的弧
}VNode, AdjList[MaxVertexNum];
// 邻接表（图）
typedef struct{
    AdjList vertices;           // 顶点数组
    int vexnum, arcnum;         // 顶点数和边/弧数
}ALGraph;
```
- 空间复杂度，无向图为 $O(|V| + 2|E|)$，因为一条边需要存储两次；有向图为 $O(|V| + |E|)$
- 适用于**稀疏图
- 查找指定顶点的所有邻接点时，只需要遍历对应边表，优于邻接矩阵；但是判断两顶点之间是否存在边，也需要遍历边表，不如邻接矩阵
- 无向图：遍历顶点 $v$ 的单链表即可得 $\text{TD}(v)$；有向图：出度遍历单链表，入度需遍历所有顶点的单链表（或另外建立**逆邻接表**）
- 删除边和顶点都非常不方便
- 邻接表不唯一（取决于边表的插入顺序）
## 十字链表
**有向图**的一种链式存储结构，将邻接表和逆邻接表合并在一个链表中，同时方便求出度和入度

|   **顶点结点**   |        （顺序存储）        |                      |                   |             |
| :----------: | :------------------: | :------------------: | :---------------: | :---------: |
|    `data`    |      `firstin`       |      `firstout`      |                   |             |
|     顶点数据     | 指向第一个以该顶点为**弧头**的弧结点 | 指向第一个以该顶点为**弧尾**的弧结点 |                   |             |
|   **弧结点**    |      **（链式存储）**      |                      |                   |             |
|  `tailvex`   |      `headvex`       |       `hlink`        |      `tlink`      |   `info`    |
| 弧尾顶点在顶点表中的下标 |     弧头顶点在顶点表中的下标     |  指向下一个**弧头相同**的弧结点   | 指向下一个**弧尾相同**的弧结点 | 权值等附加信息（可选） |
- 空间复杂度为 $O(|V| + |E|)$，每条弧只存储一次
- 求出度沿 `firstout -> tlink` 即可遍历所有以该顶点为弧尾的弧；求入度沿 `firstin -> hlink` 即可遍历所有以该顶点为弧头的弧
- 便于删除
- **只适用于有向图**
- 十字链表不唯一
## 邻接多重表
**无向图**的一种链式存储结构

|  **顶点结点**  |    （顺序存储）     |               |              |             |
| :--------: | :-----------: | :-----------: | :----------: | :---------: |
|   `data`   |  `firstedge`  |               |              |             |
|    顶点数据    | 指向依附于该顶点的第一条边 |               |              |             |
|  **弧结点**   |  **（链式存储）**   |               |              |             |
|   `ivex`   |    `ilink`    |    `jvex`     |   `jlink`    |   `info`    |
| 边依附于的 顶点编号 | 指向依附于该顶点的下一条  | 边依附于的 另一个顶点编号 | 指向依附于该顶点的下一条 | 权值等附加信息（可选） |
- 空间复杂度为 $O(|V| + |E|)$，每条弧只存储一次
- 便于删除
- **只适用于无向图**
- 十字链表不唯一
# 图的基本操作
- `Adjacent(G, x, y)`：判断图 $G$ 中是否存在边 $(x,y)$ 或 $<x,y>$
- `Neighbors(G, x)`：列出图 $G$ 中与顶点 $x$ 邻接的边
- `InsertVertex(G, x)`：在图 $G$ 中插入顶点 $x$
- `DeleteVertex(G, x)`：从图 $G$ 中删除顶点 $x$，及其相关的所有边
- `AddEdge(G, x, y)`：若边不存在，则向图 $G$ 中添加边 $(x,y)$ 或 $<x,y>$
- `RemoveEdge(G, x, y)`：若边存在，则从图 $G$ 中删除边 $(x,y)$ 或 $<x,y>$
- `FirstNeighbor(G, x)`：求图 $G$ 中顶点 $x$ 的第一个邻接点，若存在则返回顶点号，若 $x$ 没有邻接点则返回 $-1$
- `NextNeighbor(G, x, y)`：顶点 $y$ 是 $x$ 的一个邻接点，返回除 $y$ 外 $x$ 的下一个邻接点的顶点号，若没有则返回 $-1$
- `Get_edge_value(G, x, y)`：获取图 $G$ 中边 $(x,y)$ 或 $<x,y>$ 对应的权值
- `Set_edge_value(G, x, y, v)`：设置图 $G$ 中边 $(x,y)$ 或 $<x,y>$ 对应的权值为 $v$
