# 广度优先搜索(BFS)
BFS 类似于树的**层序遍历**，是一种**分层**的搜索过程。它以**广度**优先，即先访问离起始顶点最近的未被访问的顶点，再逐层向外扩展
> BFS 需要借助**队列**来实现
```c
// 访问标记数组
bool visited[MAX_VERTEX_NUM];

// 从顶点 v 出发进行广度优先遍历
void BFS(Graph G, int v) {
    visit(v);                 // 访问顶点 v
    visited[v] = true;        // 标记已访问
    EnQueue(Q, v);            // 顶点 v 入队

    while (!isEmpty(Q)) {
        DeQueue(Q, u);        // 队头顶点出队
        for (w = FirstNeighbor(G, u); w >= 0; w = NextNeighbor(G, u, w)) {
            if (!visited[w]) {
                visit(w);
                visited[w] = true;
                EnQueue(Q, w);
            }
        }
    }
}

// 对图进行广度优先遍历（处理非连通图）
void BFSTraverse(Graph G) {
    for (int i = 0; i < G.vexnum; i++)
        visited[i] = false;   // 初始化标记数组
    InitQueue(Q);             // 初始化辅助队列

    for (int i = 0; i < G.vexnum; i++)
        if (!visited[i])      // 每个未访问顶点作为起点
            BFS(G, i);
}
```
- **空间**：辅助队列最多容纳 $|V|$ 个顶点
- **邻接表时间**：每个顶点入队一次（$O(|V|)$），每条边在出队时被检查一次（无向图 $2|E|$ 次，$O(|E|)$），合计 $O(|V| + |E|)$
- **邻接矩阵时间**：每个顶点入队一次（$O(|V|)$），查找每个顶点的邻接点需遍历整行（$O(|V|)$），合计 $O(|V|^2)$
## BFS 生成树
- BFS 遍历过程中，通过访问关系得到的树称为 **BFS 生成树**
- 若图是**非连通**的，则得到 **BFS 生成森林**
- 生成树的深度取决于图的存储结构。**邻接矩阵**存储时，生成树**唯一**；**邻接表**存储时，由于邻接表中边的顺序可能不同，生成树**不唯一**
## 最短路径（无权图）
BFS 天然适合求解**无权图的单源最短路径**问题，因为 BFS 按层遍历，第一次到达目标顶点时的路径一定是最短的
```c
// BFS 求无权图单源最短路径
void BFS_MIN_Distance(Graph G, int v) {
    int dist[G.vexnum];
    for (int i = 0; i < G.vexnum; i++)
        dist[i] = INF;          // 初始化距离为无穷大
    visited[v] = true;
    dist[v] = 0;
    EnQueue(Q, v);

    while (!isEmpty(Q)) {
        DeQueue(Q, u);
        for (w = FirstNeighbor(G, u); w >= 0; w = NextNeighbor(G, u, w)) {
            if (!visited[w]) {
                visited[w] = true;
                dist[w] = dist[u] + 1;  // 路径长度 +1
                EnQueue(Q, w);
            }
        }
    }
}
```

# 深度优先搜索(DFS)
DFS 类似于树的**先序遍历**，尽可能**深**地搜索图的分支。当当前顶点的所有邻接顶点都被访问后，**回溯**到上一个顶点继续搜索
> DFS 通常借助**递归**（系统栈）实现，也可用显式栈模拟
```c
bool visited[MAX_VERTEX_NUM];

// 从顶点 v 出发进行深度优先遍历
void DFS(Graph G, int v) {
    visit(v);
    visited[v] = true;

    for (w = FirstNeighbor(G, v); w >= 0; w = NextNeighbor(G, v, w)) {
        if (!visited[w])
            DFS(G, w);          // 递归访问
    }
}

// 对图进行深度优先遍历（处理非连通图）
void DFSTraverse(Graph G) {
    for (int i = 0; i < G.vexnum; i++)
        visited[i] = false;

    for (int i = 0; i < G.vexnum; i++)
        if (!visited[i])
            DFS(G, i);
}
```
- **空间**：递归深度（系统栈）在最坏情况下为 $O(|V|)$（单链表）
- **时间**：与 BFS 相同，每个顶点和每条边都被访问一次
> DFS可以判断图中是否有环
## DFS 生成树 / 森林
- DFS 遍历过程中形成的树称为 **DFS 生成树**（深度优先生成树）
- 非连通图得到 **DFS 生成森林**
- 同样，邻接矩阵下唯一，邻接表下不唯一
> DFS 生成树的高度总是**大于或等于** BFS 生成树的高度
# 图的遍历与图的连通性
对于**无向图**，调用 BFS/DFS 的次数等于该图的**连通分量数**
- **连通图**：一次 BFS/DFS 即可访问所有顶点即连通分量数为 **1**
- **非连通图**：需要多次 BFS/DFS ，则调用次数 = 连通分量数
> 对于**有向图**，上述结论不直接成立。一次 BFS/DFS 能访问到的顶点是**从起点出发可达**的顶点，而非整个强连通分量
## 判断有向图强连通
**思路**：从一个顶点 $v$ 出发做 DFS，若能访问到所有顶点，再将所有边反向后从 $v$ 再做一次 DFS，若仍能访问所有顶点，则原图为**强连通图**。
> 此即 **Kosaraju 算法** 的核心思
## BFS/DFS 次数与图的属性

| 图类型     | 一次 BFS/DFS 能否遍历全部顶点 | 说明            |
| ------- | ------------------- | ------------- |
| 无向连通图   | ✅ 能                 | 一次即可          |
| 无向非连通图  | ❌ 不能                | 次数 = 连通分量数    |
| 有向强连通图  | ✅ 能                 | 一次即可          |
| 有向非强连通图 | ❌ 不能                | 次数取决于**起点**选择 |
