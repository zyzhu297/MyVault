# 顺序查找
## 一般线性表的顺序查找
### 算法思想
从线性表的一端开始，逐个检查每个元素是否满足条件，直到找到目标元素或扫描完整个表
### 算法实现
```c
typedef struct {
    ElemType *elem;  // 元素存储空间基址，0号单元留空
    int TableLen;    // 表长
} SSTable;

// 顺序查找（哨兵方式）
int Search_Seq(SSTable ST, ElemType key) {
    ST.elem[0] = key;  // 哨兵
    int i;
    for (i = ST.TableLen; ST.elem[i] != key; i--);
    return i;  // 若 i == 0，查找失败；否则返回元素下标
}
```
> **哨兵**：将 `ST.elem[0]` 设为关键字，从后往前查找；这样可以免去每次比较后判断下标是否越界，优化效率。
### 查找效率分析
- **平均查找长度（ASL）**：
$$ASL_{成功} = \sum_{i=1}^{n} P_i \cdot C_i = \frac{1}{n} \sum_{i=1}^{n} (n - i + 1) = \frac{n + 1}{2}$$
  其中 $P_i = \frac{1}{n}$（等概率），$C_i = n - i + 1$（比较次数）。
- **查找不成功**：需要比较 $n + 1$ 次（包括与哨兵的一次比较），即 $ASL_{失败} = n + 1$。
- **时间复杂度**：$O(n)$
## 有序线性表的顺序查找
### 算法思想
若线性表按关键字有序（递增或递减），则当查找到某个元素已大于（或小于）目标关键字时，**可提前判定查找失败**，无需继续向后查找。
### 查找效率分析
树中的圆形结点表示表中实际存在的元素；矩形结点称为*失败节点*（若有 $n$ 个结点，则相应的有 $n+1$ 个失败结点
![[有序表上的有序查找判定树.png]]
#### 查找成功的 ASL
与一般线性表相同：$ASL_{成功} = \frac{n + 1}{2}$
#### 查找失败的 ASL
对于有序表，查找失败时的比较次数取决于**判定树**的深度。失败位置共有 $n + 1$ 个（在每个元素之间的空隙处）。
- 等概率时，$ASL_{失败} = \frac{1}{n+1} \sum_{j=0}^{n} C_j$，其中 $C_j$ 为第 $j$ 个失败结点的比较次数。
- 一个典型的计算：$ASL_{失败} = \frac{1 + 2 + \cdots + n + n}{n + 1} = \frac{n}{2} + \frac{n}{n+1}$
> 在有序表中查找失败时，**比较次数** = 找到失败位置所需的比较次数，**不包含**与哨兵的比较。
# 折半查找（二分查找）
## 算法思想
仅适用于**有序**的顺序表。每次将待查区间中间位置的元素与关键字比较：
- 若相等，查找成功
- 若关键字 < 中间元素，则在左半区间继续查找
- 若关键字 > 中间元素，则在右半区间继续查找
## 算法实现
```c
int Binary_Search(SSTable L, ElemType key) {
    int low = 0, high = L.TableLen - 1, mid;
    while (low <= high) {
        mid = (low + high) / 2;   // 取中间位置
        if (L.elem[mid] == key)
            return mid;           // 查找成功
        else if (L.elem[mid] > key)
            high = mid - 1;       // 左半区间
        else
            low = mid + 1;        // 右半区间
    }
    return -1;  // 查找失败
}
```
每次查找必须使用**相同**的**取整**方式
## 判定树
折半查找的过程可以用**二叉判定树**表示，判定树是一棵[[2.二叉树#平衡二叉树|平衡二叉树]]
![[折半查找过程的判定树.png]]
### 构造判定树的关键性质
1. 树中结点数 = 表中元素个数 $n$
2. 树的深度（高度）为 $\lceil \log_2(n+1) \rceil$
3. **查找成功的比较次数**不超过树的深度
4. **查找失败的比较次数** = 从根结点到失败结点父结点的路径长度（即比较到叶结点仍未找到）
## 查找效率分析
- $ASL_{成功} = \frac{1}{n} \sum_{i=1}^{n} l_i$，其中 $l_i$ 为第 $i$ 个结点所在的层数
- $ASL_{失败} = \frac{1}{n+1} \sum_{j=0}^{n} (l_j' - 1)$，其中 $l_j'$ 为第 $j$ 个**失败结点父结点所在层数**
- **近似公式**：$ASL \approx \log_2(n + 1) - 1$
- **时间复杂度**：$O(\log n)$
## 折半查找的特点
查找效率高，时间复杂度 $O(\log n)$，仅适用于顺序存储结构（不适用于链式存储），要求表有序，排序本身有开销，插入/删除操作需要移动大量元素
# 分块查找
## 算法思想
分块查找（又称**索引顺序查找**）结合了顺序查找和折半查找的优点。
### 基本步骤
1. **分块**：将查找表分为若干块，块内元素可以无序，但**块间有序**（即第 $i$ 块的最大关键字 < 第 $i+1$ 块的最小关键字）
2. **建立索引表**：为每块建立一个索引项，包含块内最大关键字和块的首地址
3. **查找过程**：
   - 先在索引表中查找目标关键字所在的块（可用顺序查找或折半查找）
   - 再在块内进行顺序查找
## 查找效率分析
设长度为 $n$ 的查找表均匀分为 $b$ 块，每块有 $s$ 个元素（$n = b \times s$）。
### 索引表用顺序查找、块内用顺序查找
$$
ASL = ASL_{索引} + ASL_{块内} = \frac{b+1}{2} + \frac{s+1}{2} = \frac{b + s}{2} + 1 = \frac{s^2+2s+n}{2s}
$$

当 $s = \sqrt{n}$ 时，$ASL_{min} = \sqrt{n} + 1$
### 索引表用折半查找、块内用顺序查找
$$
ASL = ASL_{索引} + ASL_{块内} \approx \log_2(b + 1) - 1 + \frac{s + 1}{2}
$$
## 分块查找的特点

| 优点                    | 缺点                |
| --------------------- | ----------------- |
| 动态性强，插入/删除方便（只需在块内操作） | 需额外建立索引表，占用存储空间   |
| 块内无序，无需大量移动元素         | 查找效率介于顺序查找和折半查找之间 |
| 适合线性表要频繁插入/删除的情况      | —                 |

# 三种查找方法对比总结

| 算法   | 适用结构   | 时间复杂度         | ASL（成功，等概）              | 有序性要求 |
| ---- | ------ | ------------- | ----------------------- | ----- |
| 顺序查找 | 顺序表、链表 | $O(n)$        | $\frac{n+1}{2}$         | 无要求   |
| 折半查找 | 有序顺序表  | $O(\log n)$   | $\approx \log_2(n+1)-1$ | 有序    |
| 分块查找 | 分块有序表  | $O(\sqrt{n})$ | $\sqrt{n}+1$（最优情况）      | 块间有序  |
