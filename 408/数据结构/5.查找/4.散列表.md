# 散列表的基本概念
- 散列函数（哈希函数）：是一种将关键字映射到其对应存储地址的函数，记为$$Hash(key)=Addr \ \ or \ \ H(key)=Addr$$其中地址可以是数组下标、索引或内存地址等
- 散列函数可能会将两个或两个及以上的不同的关键字映射到同一地址，这种现象称为**冲突**，发生冲突的不同关键字称为**同义词**，冲突是不可以避免的
- 散列表（哈希表）：一种根据关键字直接访问的数据结构，建立了关键字到存储地址的直接映射，理想情况下，对散列表查找的时间复杂度为 $O(1)$
# 散列函数的构造方法
## 散列函数的要求
- **定义域**：必须包含全部需要存储的关键字
- **值域**：范围依赖于散列表的大小或地址范围
- **均匀性**：散列地址应尽可能均匀分布在整个地址空间中，减少冲突的发生
- **简单高效**：散列函数应尽可能简单，能在较短时间内计算出任一关键字的散列地址
## 直接定址法
$$H(key) = a \times key + b$$
- 取关键字本身或关键字加上某个线性函数值作为散列地址
- **优点**：简单，不会产生冲突
- **缺点**：适合关键字分布基本连续的情况；若关键字分布不连续，会造成大量存储空间的浪费
- **适用场景**：关键字分布范围小且连续的情况
## 除留余数法
$$H(key) = key \% p$$
- 模 $p$ 的选择是关键，$p$ 最好选**不大于散列表表长 $m$ 但最接近 $m$ 的质数**（或不包含小于 20 的质因子的合数）
- 若 $p$ 选得不好，容易产生冲突
- 这是最常用的散列函数构造方法
## 数字分析法
- 设关键字是 $r$ 进制数，$r$ 个数码在各数位上出现的频率不一定相同
- 选取数码分布**均匀**的若干位作为散列地址
- **适用场景**：事先清楚关键字的分布情况，且关键字的位数比散列地址位数多
## 平方取中法
- 取关键字平方的**中间几位**作为散列地址
- 中间几位与关键字中的每一位都相关，使散列地址分布更均匀
- **适用场景**：关键字的每一位取值都分布不均匀或关键字本身位数较少的情况
# 冲突处理方法
## 开放定址法
可存放新表项的空闲地址既向它的同义词表项开放，又向它的非同义词表项开放，其地址探测序列由如下递推公式给出：$$H_i = (H(key) + d_i) \% m$$其中 $m$ 为表长，$d_i$ 为增量序列
> **删除操作注意**：开放定址法下不能简单地物理删除表中已有元素，否则会截断其他同义词的查找路径，只能采用**懒惰删除**（逻辑删除标记）
### 线性探测法
增量序列：$d_i = 0, 1, 2, \dots, m-1$
- **优点**：只要散列表未满，总能找到不冲突的地址
- **缺点**：容易产生**堆积（聚集）** 现象，冲突后相邻位置被占用，导致本应散落在各处的关键字聚集在一起，大大降低查找效率
### 平方探测法
增量序列：$d_i = 0^2, 1^2, -1^2, 2^2, -2^2, \dots, k^2, -k^2$，$k \leq  m/2$
其中散列表长度 $m$ 必须是一个形如 $4k+3$ 的素数
- **优点**：可避免堆积，能探测到散列表上至少一半的单元
- **缺点**：不能探测到散列表上的所有单元
### 再散列法（双散列法）
增量序列：$d_i = i \times Hash_2(key)$
使用另一个散列函数 $Hash_2$ 计算增量
- **优点**：不易产生聚集
- **缺点**：增加了计算时间

## 拉链法（链接法，chaining）
将所有同义词存储在一个线性链表中，散列地址为 $i$ 的同义词构成一个单链表，散列表地址空间中存放各链表的头指针
- **优点**：
  1. 处理冲突简单，**无堆积现象**——非同义词不会冲突
  2. 删除操作方便——直接删除结点即可
  3. 结点空间动态申请，适合表长不确定的情况
- **适用场景**：经常进行插入和删除操作的情况
# 散列查找及性能分析

## 散列查找的过程
- 查找过程与构造过程基本一致：
  1. 给定关键字 $key$，通过散列函数计算散列地址 $H(key)$
  2. 若该地址为空，则查找失败
  3. 若该地址有关键字且等于 $key$，则查找成功
  4. 若该地址有关键字但不等于 $key$（发生冲突），按冲突处理方法计算下一个地址，直到找到或遇到空地址为止 
## 影响散列查找效率的因素
- **散列函数**：函数设计越好，冲突越少
- **冲突处理方法**：不同方法对查找性能影响不同
- **装填因子（负载因子）**：$$\alpha = \frac{表中已存记录数}{散列表长度}$$
  - $\alpha$ 越大，发生冲突的可能性越大，查找效率越低
  - $\alpha$ 越小，冲突可能性越小，但空间利用率越低
- **$\alpha$ 是影响平均查找长度的核心因素**——散列表的 ASL 仅依赖于 $\alpha$，与表长 $m$ 和记录数 $n$ **无直接关系**
## 平均查找长度(ASL)
通过计算每一各关键字在查找过程种经过几次哈希计算，然后求和除以关键字总数
查找失败的比较次数是从第一个哈希值的地址的关键字比较到**关键字为空的地址**，等于冲突次数加一
### 线性探测法
- 查找成功：$$ASL_{成功} \approx \frac{1}{2}\left(1 + \frac{1}{1-\alpha}\right)$$
- 查找失败：$$ASL_{失败} \approx \frac{1}{2}\left(1 + \frac{1}{(1-\alpha)^2}\right)$$
### 拉链法
- 查找成功：$$ASL_{成功} \approx 1 + \frac{\alpha}{2}$$
- 查找失败：$$ASL_{失败} \approx \alpha + e^{-\alpha} \ \ （或近似为 \ \alpha）$$
## 性能对比
| 特性   | 开放定址法            | 拉链法                 |
| ---- | ---------------- | ------------------- |
| 堆积现象 | 易产生              | 无                   |
| 删除操作 | 需懒惰删除            | 直接删除                |
| 空间利用 | 固定空间，$\alpha<1$  | 动态申请，$\alpha$ 可大于 1 |
| 查找效率 | 随 $\alpha$ 增大下降快 | 随 $\alpha$ 增大下降慢    |

## 工程实践建议
- 一般控制装填因子 $\alpha$ 在 **0.7~0.8** 之间
- $\alpha$ 过大时，可通过**动态扩容**（再散列）来降低 $\alpha$，即重新申请更大的散列表并将原有元素重新散列
