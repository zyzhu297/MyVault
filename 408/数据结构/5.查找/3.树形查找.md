# 二叉排序树(BST)
一棵空树，或者是具有以下特性的二叉树：
- 左子树上的所有结点的关键字**均小于**根节点的关键字
- 右子树上的所有结点的关键字**均大于**根节点的关键字
- 左子树和右子树本身也各是一棵二叉排序树
对二叉排序树进行**中序遍历**可以得到一个**递增**的有序关键字序列
## 二叉排序树的查找
```c
BSTNode *BST_Search(BiTree T, int key) {
    while (T != NULL && key != T->key) {
        if (key < T->key) T = T->lchild;
        else T = T->rchild;
    }
    return T;  // 找到返回结点，没找到返回 NULL
}
```
## 二叉排序树的插入
二叉排序树是一种动态树表，其结构是在查找过程中**动态构建**：当**查找失败**时，才执行插入操作，新插入的结点一定是**叶结点**
```c
bool BST_Insert(BiTree &T, int key) {
    if (T == NULL) {
        T = (BiTree)malloc(sizeof(BSTNode));
        T->key = key;
        T->lchild = T->rchild = NULL;
        return true;
    }
    if (key == T->key)
        return false;           // 关键字已存在，插入失败
    else if (key < T->key)
        return BST_Insert(T->lchild, key);
    else
        return BST_Insert(T->rchild, key);
}
```
## 二叉排序树的构造
```c
void Create_BST(BiTree &T, KeyType keys[], int n) {
    T = NULL;           // 初始为空树
    for (int i = 0; i < n; i++)
        BST_Insert(T, keys[i]);
}
```
> **注意**：不同次序的关键字序列构造出的 BST 形态可能不同，查找效率也不同。
## 二叉排序树的删除
删除一个结点时，必须将该节点从存储结构中摘下，并重新链接因删除操作而断裂的链表，同时保证整棵树仍满足二叉排序树的性质，具体有以下三种情况：
- 被删结点是**叶结点**，则直接删除
- 被删结点**只有左子树或只有右子树**，用其左/右子树替代该结点
- 被删结点**既有左子树又有右子树**，用**直接前驱**（左子树最右下）或**直接后继**（右子树最左下）替换，再删除该前驱/后继（转为前两种情况）
## 二叉排序树的查找效率分析
主要取决于**树的高度**
- $ASL_{成功}  = \text{ 各结点深度之和} / \text{结点数}$
- $ASL_{失败}  = \text{各外部结点（失败结点）的父结点深度之和} / \text{失败结点数}$
- **最好情况**：平衡二叉树形态，$h = \lceil \log_2(n+1) \rceil$，平均查找长度 $O(\log_2{n})$
- **最坏情况**：单支树形态（输入有序），$h = n$，平均查找长度 $O(n)$
就维护表的有序性而言，二叉排序树无需移动结点，时间复杂度为 $O(\log_2{n})$，而维护有序顺序表的时间复杂度为 $O(n)$
# 平衡二叉树(AVL)
在 BST 的基础上，任意结点的**左、右子树高度差的绝对值不超过 1**
**平衡因子(BF)** = 左子树高度 - 右子树高度，在平衡二叉树中所有节点的平衡因子只能取{-1, 0, 1}
## 平衡二叉树的插入
为了维持二叉树的平衡，在插入一个结点后，沿查找路径向上检查各祖先结点是否因该操作而失去平衡；若存在失衡因子的绝对值大于 1 的失衡结点 $A$，以 $A$ 为根的子树即为*最小不平衡子树*；随后在保持二叉排序树的特性的前提下，通过**旋转**调整子树来重新平衡
### LL 右单旋转
由于结点 $A$ 的**左孩子(L)的左子树(L)** 上插入了新结点，$A$ 的平衡因子由 1 增至 2，需要进行一次**向右**的旋转操作
- 将 $A$ 的左孩子 $B$ 向右上旋转，代替 $A$ 成为新的根节点
- 将 $A$ 向右下旋转，成为 $B$ 的右孩子
- $B$ 的原右子树则作为 $A$ 的左子树
### RR 左单旋转
由于结点 $A$ 的**右孩子(R)的右子树(R)** 上插入了新结点，$A$ 的平衡因子由 -1 减至 -2，需要进行一次**向左**的旋转操作
- 将 $A$ 的右孩子 $B$ 向左上旋转，代替 $A$ 成为新的根节点
- 将 $A$ 向左下旋转，成为 $B$ 的左孩子
- $B$ 的原左子树则作为 $A$ 的右子树
### LR 先左后右双旋转
由于结点 $A$ 的**左孩子(L)的右子树(R)** 上插入了新结点，$A$ 的平衡因子由 1 增至 2，需要进行**先左后右**两次旋转操作
- **第一步**：对 $A$ 的左孩子 $B$ 做 **RR 左单旋转**（$B$ 的右孩子 $C$ 向左上旋转代替 $B$）
- **第二步**：对 $A$ 做 **LL 右单旋转**（$C$ 向右上旋转代替 $A$）
- 最终 $C$ 成为新的根节点，$B$ 为 $C$ 的左孩子，$A$ 为 $C$ 的右孩子
- $C$ 的原左子树成为 $B$ 的右子树，$C$ 的原右子树成为 $A$ 的左子树
### RL 先右后左双旋转
由于结点 $A$ 的**右孩子(R)的左子树(L)** 上插入了新结点，$A$ 的平衡因子由 -1 减至 -2，需要进行**先右后左**两次旋转操作
- **第一步**：对 $A$ 的右孩子 $B$ 做 **LL 右单旋转**（$B$ 的左孩子 $C$ 向右上旋转代替 $B$）
- **第二步**：对 $A$ 做 **RR 左单旋转**（$C$ 向左上旋转代替 $A$）
- 最终 $C$ 成为新的根节点，$A$ 为 $C$ 的左孩子，$B$ 为 $C$ 的右孩子
- $C$ 的原左子树成为 $A$ 的右子树，$C$ 的原右子树成为 $B$ 的左子树
### 平衡二叉树的删除
AVL 的删除操作与 BST 删除类似，但删除后可能需要**从被删结点向上回溯**，检查祖先结点是否失衡，若失衡则调整。
**删除步骤**：
1. **删除结点**（同 BST 删除的三种情况）
2. **向上回溯**，找到第一个失衡的祖先结点 $x$（即最小不平衡子树的根）
3. **调整失衡**：根据 $x$ 的**较高孩子** $y$ 和 $y$ 的**较高孩子** $z$ 的位置关系，对应执行旋转
	- $y$ 是 $x$ 的**左孩子**，$z$ 是 $y$ 的**左孩子**，LL 右单旋转
	- $y$ 是 $x$ 的**右孩子**，$z$ 是 $y$ 的**右孩子**，RR 左单旋转
	- $y$ 是 $x$ 的**左孩子**，$z$ 是 $y$ 的**右孩子**，LR 先左后右双旋转
	- $y$ 是 $x$ 的**右孩子**，$z$ 是 $y$ 的**左孩子**，RL 先右后左双旋转
4. **继续向上回溯**，检查更上层的祖先是否因此次调整而失衡（与插入不同，删除可能引发**连锁失衡**，需要多次调整）
5. 重复步骤 2~4 直到根节点
> **插入 vs 删除调整的区别**：
> **插入**：最多只需调整**一次**（最小不平衡子树调整后局部高度恢复，上层不再失衡），且无 BF(y)=0 情况
> **删除**：BF(y) 可能为 0，旋转后**高度降低**，可能导致上层祖先**连锁失衡**，需要多次回溯调整
## 平衡二叉树的查找
- 平衡二叉树最大深度 $O(\log_2{n})$，平均查找长度  $O(\log_2{n})$
- $n_h$ 表示深度为 $h$ 的平衡二叉树的最少结点数：$n_0 = 0, n_1 = 1, n_2 = 2, n_h = n_{h-1} + n_{h-2} + 1$（类似斐波那契数列）
# 红黑树(Red-Black Tree)
红黑树是一种**自平衡的二叉排序树**，每个结点增加一个存储位表示颜色（红/黑）,满足以下 **5 条性质**：
1. **每个结点是红色或黑色**
2. **根结点是黑色的**
3. **叶结点（NULL / NIL）都是黑色的**
4. **红色结点的两个孩子都是黑色的**（即：不存在两个相邻的红色结点）
5. **对任一结点，到其所有后代叶结点的路径上，包含相同数量的黑色结点**（**黑高相同**）
红黑树引入了 $n+1$ 个**外部叶结点**，这些结点被视为**黑色**，以保证每个内部结点的左右子树非空
从某结点出发（不含该结点）到任意叶结点的*简单路径*上黑色结点的个数称为该结点的黑高（记为 bh)，根节点的黑高称为*红黑树的黑高*
## 重要结论
- 一棵有 $n$ 个内部结点的红黑树，高度 $h \le 2\log_2(n+1)$
- 即红黑树的查找、插入、删除时间复杂度均为 $O(\log_2 n)$
- 从根到叶结点的 **最长路径 ≤ 2 × 最短路径**
- 新插入的结点初始应为红色
## 红黑树的插入
新结点 z 按 BST 规则插入，初始为**红色**
- 若 z 的父结点为黑色则直接结束；
- 若 z 为根节点，则将其染为黑色
- 若 z 的父结点为红色（**双红冲突**），则根据叔父结点的颜色分情况调整：
	1. 叔父 $u$ 为**红色**，父节点和叔结点变黑，祖父结点变红（为了满足路径上黑结点数量相同），以祖父为新结点**继续向上检查**
	2. 叔父 $u$ 为**黑色**，新结点是**内插**（LR 或 RL 型），对父结点**旋转**，使新结点变为外插，转为情况 3
	3. 叔父 $u$ 为**黑色**，新结点是**外插**（LL 或 RR 型），对祖父**旋转**，父变黑、祖父变红，**调整结束**（先旋转再变色）
> **口诀**：叔红上移，叔黑外旋，内插先旋父
## 红黑树的删除
### 核心思想
1. **按 BST 规则删除**结点
2. 若被删结点为**红色** -> 直接删，不影响黑高，无需调整
3. 若被删结点为**黑色** -> 黑高被破坏，引入**双重黑**概念进行平衡调整
### 删除调整关键
- 有左右子树，直接后继代替它，删除直接后继
- 只有一个子结点，子节点代替它**变黑**
- 没有子节点，查看结点颜色
	- 结点是红色，直接删除
	- 结点是黑色，删除变**双黑**，查看双黑的兄弟颜色
		- 兄弟是红色：兄弟的父结点换色，向双黑方向旋转，保持双黑不变，继续调整
		- 兄弟是黑色：查看兄弟孩子的颜色（**都是先变色再旋转**）
			- 兄弟孩子全黑色：兄弟染红，双黑上移（继续消除双黑）**双黑遇到红色结点或根节点变单黑（唯一可能重复执行的操作，至多 $O(log_2{n}）$次）**
			- 兄弟孩子有红色：变色，旋转（双黑变单黑）
				- LL：交换兄结点和父节点的颜色，再将红子变黑，对父节点右旋
				- RR：交换兄结点和父节点的颜色，再将红子变黑，对父节点左旋
				- LR：红子和其父 $w$ 颜色互换，对 $w$ 左旋，即转为 LL 情况，再对待删结点的父结点右旋
				- RL：红子和其父 $w$ 颜色互换，对 $w$ 右旋，即转为 RR 情况，再对待删结点的父结点左旋
# B 树
$m$ 阶 B 树是一种**平衡的 $m$ 路查找树**，所有叶结点都在同一层上

| 性质         | 说明                                                               |
| ---------- | ---------------------------------------------------------------- |
| **子树个数**   | 每个结点**至多**有 $m$ 棵子树（即至多 $m-1$ 个关键字）                              |
| **根结点**    | 不是叶结点时，**至少**有 2 棵子树（至少 1 个关键字）                                  |
| **非叶根外**   | **至少**有 $\lceil m/2 \rceil$ 棵子树（至少 $\lceil m/2 \rceil - 1$ 个关键字） |
| **叶结点同层**  | 所有叶结点出现在**同一层**，且不携带信息（可视为查找失败的外部结点）                             |
| **关键字有序**  | 结点内关键字**递增排序**，左<中<右                                             |
| **外部节点数量** | 包含 $n$ 个关键字的 B 树，其外部节点恒为 $n+1$ （可以通过查找 $n$ 个关键字，失败的可能性 $n+1$ 种   |
- 终端节点：叶结点的上一层
- 失败节点：叶结点，不携带信息，表示查找失败
### 结点结构
$$
\underbrace{P_0}_{左指针}\;|\;K_1\;|\;P_1\;|\;K_2\;|\;P_2\;|\;\cdots\;|\;K_n\;|\;P_n
$$
- $K_i$ 为关键字，满足 $K_1 < K_2 < \cdots < K_n$
- $P_i$ 为指向子树根结点的指针，$P_{i-1}$ 所指子树中所有关键字 $< K_i$，$P_i$ 所指子树中所有关键字 $> K_i$
- $n$ 为结点中关键字个数（$\lceil m/2 \rceil - 1 \le n \le m-1$）
## B 树的高度（磁盘存取次数）
含 $n$ 个关键字、$m$ 阶 B 树的最小高度 $h_{\min}$ 和最大高度 $h_{\max}$：
- **最小高度**：每个结点关键字尽可能多（$m-1$ 个）$$n \le (m - 1)(1 + m + m^2 + \cdots + m^{h-1}) = m^h - 1$$$$
  h_{\min} = \lceil \log_m (n+1) \rceil
  $$
- **最大高度**：每个结点关键字尽可能少（$\lceil m/2 \rceil - 1$ 个）第一层至少有一个结点，第二层则至少有两个结点；除根节点外，每个非叶节点至少包含 $\lceil m/2 \rceil$ 棵子树，因此第 3 层至少有 $2\lceil m/2 \rceil$ 个结点……，第 $h+1$ 层至少有 $2(\lceil m/2 \rceil)^{h-1}$ 个结点，第 $h+1$ 层为外部结点，对于 $n$ 个结点的 B 树，有 $n+1$ 个外部结点，因此有  $n + 1 \ge 2(\lceil m/2 \rceil)^{h-1}$ 即：$$
  h_{\max} = \lceil \log_{\lceil m/2 \rceil} \frac{n+1}{2} \rceil + 1
  $$
> **B 树的高度不包含叶结点（失败结点）层**

## B 树的查找
在**磁盘**上找到目标结点后，先将结点读入**内存**，再在结点内**顺序查找**或**折半查找**关键字，若找到则成功；若找不到，则按对应指针到子树中继续查找，直到查到叶结点（失败结点）为止
> B 树的查找涉及**两次操作**：① 在结点内查找（内存） ② 沿指针到磁盘读取下一结点（I/O）
> 在磁盘上的查找次数（目标结点在 B 树上的层次数）是决定查找效率的关键因素
## B 树的插入
插入操作总是在**叶结点**（终端结点）上进行，步骤如下：
1. **定位**：查找找到插入的叶结点位置
2. **插入**：将关键字插入该结点
3. **判断分裂**：若结点关键字个数 $\le m-1$，结束；否则执行**分裂**：
	- 以中间位置 $\lceil m/2 \rceil$ 的关键字为分界点
	- 将关键字 $K_{\lceil m/2 \rceil}$ **上升**到父结点
	- 左半部分留在原结点，右半部分移到新结点
	- 分裂可能导致父结点也溢出，**连锁向上分裂**直到根结点（若根节点也被分裂，则 B 树高度加 1）
> 插入操作导致的 I/O 次数至多为 $3h+1$，产生的新结点最多为 $h+1$ 个
## B 树删除
### 删除非叶结点
从**直接前驱**（左子树最右下的关键字）或**直接后继**（右子树最左下的关键字）中选择一个替代它，转化为删除叶结点
### 删除叶结点
- 若被删除关键字所在结点删除前的关键字个数 $\ge \lceil m/2 \rceil$，则直接删除
- 若被删除关键字所在结点删除前的关键字个数 $= \lceil m/2 \rceil - 1$，且与该结点相邻的兄弟结点的关键字个数 $\ge \lceil m/2 \rceil$，则从兄弟**借**一个关键字（通过父结点旋转）
- 若被删除关键字所在结点删除前的关键字个数 $= \lceil m/2 \rceil - 1$，且与该结点相邻的兄弟结点的关键字个数也均为 $\lceil m/2 \rceil - 1$，则删除该关键字后，需将该结点、其中一个兄弟以及父结点中分隔它们的的关键字**合并为一个新结点**，可能导致父结点关键字减少，**连锁向上合并**（若其父结点是根节点且关键字个数减少至 0，则直接将根节点删除，合并后的新结点成为根）
# B+ 树
B+ 树是 B 树的变体，是**数据库索引**的常用结构。其定义与 B 树基本相同，但有以下不同：
## B+ 树和 B 树的差异

| 对比项 | B 树 | B+ 树 |
|--------|------|-------|
| **关键字与指针** | $n$ 个关键字对应 $n+1$ 棵子树 | $n$ 个关键字对应 $n$ 棵子树 |
| **关键字作用** | 结点中关键字**不重复**，每个关键字对应一条记录 | 叶结点包含全部关键字，非叶结点仅起**索引**作用 |
| **非叶结点** | 既做索引又存储数据（记录地址） | **只存索引**，不存数据，只存子树最大（或最小）关键字 |
| **叶结点** | 叶结点包含所有关键字，且包含指向记录的指针 | 叶结点包含全部关键字和指向记录的指针，叶结点按关键字大小**链接成链表** |
| **查找方式** | 查找可能停在非叶结点（成功） | 查找**必须到叶结点**才能得到数据 |
| **查找路径** | 查找路径不唯一（不同关键字深度可能不同） | 查找路径唯一（所有关键字都在叶结点） |
- B+ 树的非叶结点**不保存数据**，只保存索引，因此可以存放更多关键字，降低树高，从而**减少 I/O 次数**
- B+ 树的叶结点形成了**有序链表**，支持**范围查询**（如 $K_1 \le x \le K_2$），而 B 树的范围查询需要中序遍历
- B+ 树的**所有关键字都在叶结点**，查找次数稳定（树高）
- B+ 树的**插入和删除**操作与 B 树类似，同样需要分裂/合并
## B+ 树的典型结构
通常在 B+ 树中有两个头指针，一个指向根节点，另一个指向关键字最小的叶结点；因此支持两种查询方式：从最小关键字开始的顺序查找，从根节点开始的多路查找
```
         -->[ 50 ,  80 ]                    <- 索引层
	        /     |      \
   -->[10,30,50] [60,70,80] [90,100,120]    <- 叶结点层（数据）
       <------ 链表连接 ------>
```

# 五种树结构的综合对比

## 平衡二叉树家族对比（BST / AVL / 红黑树）

| 对比维度       | BST（二叉排序树）              | AVL（平衡二叉树）                | 红黑树                            |
| ---------- | ----------------------- | ------------------------- | ------------------------------ |
| **平衡机制**   | 无平衡要求                   | 平衡因子 BF $\in \{-1,0,1\}$  | 5 条颜色性质                        |
| **平衡程度**   | 不保证                     | 严格平衡（高度差 $\le 1$）         | 适度平衡（最长 $\le 2 \times$ 最短）     |
| **树高**     | $O(n)$（最坏）              | $h \approx 1.44 \log_2 n$ | $h \le 2 \log_2 n$             |
| **查找 ASL** | $O(\log_2 n) \sim O(n)$ | $O(\log_2 n)$             | $O(\log_2 n)$                  |
| **插入/删除**  | $O(h)$                  | 最多 $O(1)$ 次旋转             | 最多 $O(1)$ 次旋转 $+ O(\log n)$ 染色 |
| **插入调整次数** | 无需                      | 最多 1 次                    | 最多 2 次旋转                       |
| **删除调整次数** | 无需                      | 可能多次（连锁失衡）                | 最多 3 次旋转                       |
| **适用范围**   | 基本动态查找                  | 查多写少（如词典）                 | 读写均衡（如 STL map）                |
| **存储开销**   | 最小                      | 额外存 BF（int）               | 额外存颜色（1 bit）                   |

## 多路查找树对比（B 树 / B+ 树）

| 对比维度 | B 树 | B+ 树 |
|----------|------|-------|
| **本质** | 平衡的多路查找树 | B 树的变体，数据库索引常用 |
| **关键字与指针** | $n$ 个关键字 $\rightarrow$ $n+1$ 棵子树 | $n$ 个关键字 $\rightarrow$ $n$ 棵子树 |
| **非叶结点内容** | 存关键字 + 数据指针 | 只存索引（子树最大关键字） |
| **叶结点内容** | 存关键字 + 数据指针 | 存全部关键字 + 数据指针，且形成有序链表 |
| **查找终点** | 可能停在非叶结点（成功） | 必须到叶结点 |
| **查找稳定性** | 不同关键字深度可能不同 | 所有关键字深度相同 |
| **范围查询** | 需中序遍历，效率低 | 叶结点链表直接遍历，效率高 |
| **树高** | 相对较高 | 更矮（非叶结点存更多索引） |

## 全类型总览表

| 树类型      | 分支因子  | 平衡方式     | 树高                                                     | 查找时间                    | 适用场景         |
| -------- | ----- | -------- | ------------------------------------------------------ | ----------------------- | ------------ |
| **BST**  | 二叉    | 无        | $O(\log_2 n) \sim O(n)$                                | $O(\log_2 n) \sim O(n)$ | 简单动态查找       |
| **AVL**  | 二叉    | 严格平衡（BF） | $\approx 1.44 \log_2 n$                                | $O(\log_2 n)$           | 查多写少         |
| **红黑树**  | 二叉    | 适度平衡（颜色） | $\le 2 \log_2 n$                                       | $O(\log_2 n)$           | 读写均衡         |
| **B 树**  | $m$ 路 | 多路平衡     | $\log_m n \sim \log_{\lceil m/2 \rceil} \frac{n+1}{2}$ | $O(\log_m n)$           | 文件系统/数据库     |
| **B+ 树** | $m$ 路 | 多路平衡     | 比 B 树更矮                                                | $O(\log_m n)$           | 数据库索引（MySQL） |
