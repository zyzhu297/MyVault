# 算法的基本概念
需要具备以下五个特性：
- 有穷性：必须在有限步后结束，每一步在有限时间完成
- 确定性：每条指令含义明确，相同的输入一定得到相同的输出
- 可行性：所有操作都可以通过已实现的基本运算执行有限次实现
- 输入：0~n个输入
- 输出：至少一个输出

好的算法还需要以下特性：
- 正确性
- 可读性
- 健壮性
- 高效率和地存储量需求
# 算法效率的度量
## 时间复杂度
描述算法执行时间随问题规模 $n$增大而变化的趋势，有语句频度（某条语句的执行次数）来衡量，算法中所有的语句频度记为 $T(n)$ ,其执行次数数量级作为整个算法的时间复杂度，用**大O记号**表示
算法**实际**运行时间不仅依赖问题规模$n$还受**输入数据的初始状态**影响
推导时间复杂度的规则
- 加法规则：顺序执行的代码取高阶项
- 乘法规则：代码嵌套在另一端代码中，总时间复杂度为两者之积
时间复杂度阶次
$$ O(1) < O(\log n) < O(\sqrt{n}) < O(n) < O(n\log n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n) $$
# 空间复杂度
表示算法在运行过程中所需的**额外**存储空间，是问题规模的函数。
