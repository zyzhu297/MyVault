# 单链表
```c
typedef struct LNode{
	ElemType data;
	struct LNode *next;
}LNode, *LinkList;
```
单个结点内的数据和指针的地址一定连续，不同结点之间的地址可能连续
## 头指针和头结点的关系
头指针始终指向链表的第一个结点（有头结点则指向头结点，没有则指向第一个结点），头结点仅存在于带头结点的链表中，是链表的第一个结点，数据域通常不存储实际数据
引入头结点便于操作的统一性，空表和非空表处理统一
**所以需要注意链表是否有头结点**
## 单链表的基本操作
- 求链表长度：时间复杂度为 $O(n)$
- 按序号查找结点：时间复杂度为 $O(n)$
- 按值查找结点：时间复杂度为 $O(n)$
- 插入结点：为**防止断链**，需要确保插入结点先链接后续链表，再将前驱结点链接到插入结点，寻找插入位置的时间复杂度为 $O(n)$ ，插入结点的时间复杂度为 $O(1)$ ，总体时间复杂度为 $O(n)$
- 删除结点：将删除结点的前驱结点的指针指向删除结点的后继，然后释放删除结点的空间，时间复杂度类似插入结点
- 头插法建立单链表：创建的链表中元素顺序与输入顺序**相反**，时间复杂度为 $O(n)$
- 尾插法建立单链表：创建的链表中元素顺序与输入顺序**相同**，时间复杂度为 $O(n)$
# 双链表
```c
typedef struct DNode{
	ElemType data;
	struct LNode *prior, *next;
}DNode, *DLinkList;
```
插入和删除更加需要注意是否**断链**
# 循环链表
## 循环单链表
链表中尾结点的 next 域指向**头结点**
**判空条件**为检查表头结点的 next 是否指向自身 (`L->next == L`)
## 循环双链表
链表中尾结点的 next 域指向头结点，头结点的 prior 域指向尾结点
**判空条件**为检查表头结点的 next 和 prior 是否都指向自身 (`L->next == L && L->prior == L`)
# 静态链表
用数组来模拟线性表的链式存储结构
```c
#define MaxSize 50
typedef struct{
	ElemType data;
	int *next;
}SLinkList[MaxSize];
```
以`next == -1` 作为链表结束标志，静态链表在灵活性上不及动态链表，**一次性申请连续存储空间，查找时按链依次进行**
# 链表的优劣
- 存取方式：仅支持存取顺序存取
- 逻辑结构与物理结构：逻辑上相邻的元素在物理上未必相邻，逻辑关系由指针显式维护
- 查找、插入、删除操作：时间复杂度为 $O(n)$，主要用于查找元素位置（如果位置指针已知，操作时间复杂度为 $O(1)$
- 空间分配：动态分配结点，按需扩展，灵活性高，需要额外指针空间，空间效率低
- 适用情况：
存储方面：线性表长度难以估计
运算方面：以插入和删除为主的操作
环境方面：依赖指针操作，*有些开发语言不支持*
