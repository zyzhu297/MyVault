# 顺序表的定义
逻辑顺序与其物理存储顺序完全一致
- 优点：可以随机访问，存储密度高
- 缺点：插入和删除效率低需要移动大量元素，要求分配连续存储空间
# 顺序表上基本操作的实现
## 初始化
```c
void InitList(SqList &L){
	L.data = (ElemType *)malloc(InitSize*sizeof(ElemType));
	L.length = 0;
	L.MaxSize = InitSize;
}
```
## 插入操作
- 最好情况：在表尾插入($i = n + 1$)，无需移动元素，时间复杂度为 $O(1)$
- 最坏情况：在表头插入($i = 1$)，时间复杂度为 $O(n)$
- 平均情况：时间复杂度为 $O(n)$
## 删除操作
- 最好情况：删除表尾元素($i = n + 1$)，无需移动元素，时间复杂度为 $O(1)$
- 最坏情况：删除表尾元素($i = 1$)，时间复杂度为 $O(n)$
- 平均情况：时间复杂度为 $O(n)$
## 查找元素
- 按值查找：平均情况下时间复杂度为 $O(n)$
- 按序号查找：直接通过下标访问，时间复杂度为 $O(1)$
# 顺序表的优劣
- 存取方式：支持随机存取
- 逻辑结构与物理结构：逻辑上相邻的元素在物理内存中连续存放
- 查找、插入、删除操作：时间复杂度为 $O(n)$，其中按序号查找的时间复杂度为 $O(1)$
- 空间分配：预先设定容量，扩容需要申请新的连续内存块并复制全部原有数据
- 适用情况：
	- 存储方面：线性表长度确定
	- 运算方面：频繁进行按序号访问
	- 环境方面：基于数组实现，几乎所有高级语言都支持，实现简单
