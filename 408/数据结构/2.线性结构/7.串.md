# 串的定义
由零个或多个字符组成的有限序列
从一个原始序列中，通过删除零个或多个元素后，保持元素之间**相对顺序不变**而形成的新序列称为**子序列**
串中任意多个**连续**字符组成的**子序列**称为该串的**字串**，而包含该子串的串称为主串
某个字符在串中的序号称为该字符在串中的位置；字串在主串中的位置以其**第一个字符在主串中的位置**来标识
当两个串长度相等，且对应位置上的字符完全相同时，称这两个串相等
有一个或多个空格（空格是一种特殊字符）构成的串称为*空格串*，空格串不是空串，其长度等于空格字符的个数
# 串的基本操作
- `Str(&T, chars)`：赋值操作，将串 T 赋值为 chars
- `StrCopy(&T, S)`：复制操作，将串 S 复制给串 T
- `StrEmpty(S)`：判空操作，若 S 为空串返回 true，否则返回 false
- `StrCompare(S, T)`：比较操作，按字典序比较 S 和 T。若 S > T 返回值 > 0，S = T 返回值 = 0，S < T 返回值 < 0
- `StrLength(S)`：求串长，返回串 S 中字符的个数
- `SubString(&Sub, S, pos, len)`：求子串，用 Sub 返回串 S 中第 pos 个字符起长度为 len 的子串
- `Concat(&T, S1, S2)`：串联接，用 T 返回由 S1 和 S2 连接而成的新串
- `Index(S, T)`：定位操作（模式匹配），若主串 S 中存在与串 T 相同的子串，则返回其在 S 中第一次出现的位置，否则返回 0
- `ClearString(&S)`：清空操作，将 S 清为空串
- `DestroyString(&S)`：销毁操作，销毁串 S 并回收其存储空间
其中赋值、比较、求长、联接和求子串这五种操作构成了串类型的**最小操作子集**，其余操作除（清空和销毁外）均可基于最小操作子集实现
# 串的存储结构
## 定长顺序存储
使用一组地址连续的存储单元来存放串值的字符序列，为每个串变量**预先**分配一个**固定**长度的数组，串的实际长度不得超过**MAXLEN**，若超出，则多余部分被舍弃，称为**截断**
## 堆分配存储
仍以地址连续的存储单元来存放串值的字符序列，但其存储空间在程序运行过程中**动态申请获得**
存在一个称为**堆**的自由存储区，可以通过调用`malloce()`为新生成的串动态分配一块大小等于串长的连续存储空间
## 块链存储
类似于线性表的链式存储结构，每个链表结点可以存储一个或多个字符，每个结点称为一个**块**，整个链表称为块链结构/
# 串的匹配模式
- 模式匹配：在主串中查找于*模式串*（待搜索的字符串）完全相同的子串

## KMP算法
- 算法核心：若以匹配成功的序列中有后缀正好是模式串的前缀，则可直接将模式串右移到与这些字符串对齐的位置，而无需回溯主串指针 i 
- 部分匹配值(Partial Match, **PM**)：字符串前缀和后缀的**最长相等**先后缀长度，用于指导模式串在**失配**后的最优右移距离$$失配时所需右移位数 = 已匹配字符数 - 对应的部分匹配值$$
## next数组
next\[j] 的含义是，当模式串第 j 个字符失配时，下一轮应从模式串的第 next\[j] 个位置继续与主串**当前**字符比较
- next\[1] = 0 和 next\[2] = 1 是固定规则，该规则是串编号从1开始，若从零开始需要 next 数组整体减一
- **next 数组等于将 PM 表右移一位，再整体加一**，空出的第一个位置补零
## KMP算法的实现
```c
void get_next(SString T, int next[]){
	int i = 1, j = 0;  
	next[1] = 0;
	while(i < T.length){
		if(j == 0 || T.ch[i] == T.ch[j]){
			i++; j++;
			next[i] = j;//若 p_i=p_j ，则 next[j+1] = next[j] + 1
		}
		else
			j = next[j];//否则令 j = next[j]，继续循环 
	}
}
int KMP(SString S, SString T, int next[]){
	int i = 1, j = 1;
	whlie(i <= S.length && j <= T.length){
		if (j == 0 || S.ch[i] == T.ch[j]){
			i++; j++;
		}
		else
			j = next[j];
	}
	if (j > T.length) 
		return i - T.length;
	else
		return 0;
}
```
时间复杂度为 $O(m+n)$
## KMP算法的进一步优化——nextval数组
- 不应出现$p_j=p_{next[j]}$，当$p_j \ne s_i$时，下一轮将用$P_{next[j]}$与$s_i$比较，若相同则相当于用与$p_j$相同的字符去匹配已知不等的字符$s_i$
- 所以当**模式串中正在比较的值等于失配时下一个要比较的值**时，应不断将 next\[j] 替换为 next\[next\[j]]
```c
void get_nextval(SString T, int nextval[]){
	int i = 1, j = 0;  
	nextval[1] = 0;
	while(i < T.length){
		if(j == 0 || T.ch[i] == T.ch[j]){
			i++; j++;
			if(T.ch[i] != T.ch[j])
				nextval[i] = j;
			else
				nextval[i] = nextval[j];
		}
		else
			j = nextval[j];//否则令 j = nextval[j]，继续循环 
	}
}
```
