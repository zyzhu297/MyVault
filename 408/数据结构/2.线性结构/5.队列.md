# 队列的基本概念
仅允许在表的一端进行插入，另一端进行删除
- 队首(Front)：允许删除的一端
- 队尾(Rear)：允许插入的一端
操作特性为**先进先出(First In First Out, #FIFO)**
# 队列的基本操作
- `InitQueue(&Q)`
- `QueueEmpty(Q)`：队列为空则返回 true
- `EnQueue(&Q, x)`：入队操作，**若队列未满**，将 x 加入队尾
- `DeQueue(&Q, &x)`：出队操作，**若队列非空**，则删除队首元素并通过 x 返回
- `GetHead(&Q, &x)`：**若队列非空**，**读**出队首元素并通过 x 返回读出值
- `DestroyList(&S)`
不允许直接访问或修改队列中间的元素
# 队列的顺序存储结构
## 顺序队列
```c
typedef struct{
	ElemType data[MaxSize];
	int front, rear;
}SqQueue;
```
当 rear 以达到 MaxSize 若此时继续入队则会出现**上溢**，但是若 front 不为 0 实际上数组中仍有空闲单元，此现象称为*假溢出*
## 循环队列
为了解决*假溢出*问题，将顺序存储空间在逻辑上组织为环状结构，通过取模运算实现指针到达数组末尾时自动回到起始位置
- 初始状态：`Q.front == Q.rear`
- 指针进一：`Q.ptr == (Q.ptr + 1) % MaxSize`
- 队列长度：`(Q.rear + MaxSize - Q.front) % MaxSize`
### 判断初始指针情况
根据入队操作要执行`Q.rear == (Q.rear + 1) % MaxSize`以及该队列 rear 指针指向队尾还是队尾后的元素得出前一个状态 rear 指针的值
### 循环队列判满和判空条件
#### 牺牲一个存储单元
入队时少用一个存储单元，**以队尾的下一个位置为队首**作为队满标志
- 队空条件：`Q.front == Q.rear`
- 队满条件：`(Q.rear + 1) % MaxSize == Q.front`
#### 设置size成员
记录当前元素个数，入队成功则`size++` ，出队成功则`size--`
- 队空条件：`Q.size == 0`
- 队满条件：`Q.size == MaxSize`
#### 增设tag标志
- 出队后置`tag = 0`，若此时`Q.front == Q.rear`，则队空
- 入队后置`tag = 1`，若此时`Q.front == Q.rear`，则队满
# 队列的链式存储结构
```c
typedef struct LinkNode{
	ElemType data;
	struct LinkNode *next;
}LinkNode;
typedef struct{
	LinkNode *front, *rear;
}LinkQueue;
```
- 出队时，需要单独判断队列中是否只剩一个元素
- 适用于元素**数量频繁变化**的场景，不存在队满或溢出的问题
