# 栈的基本概念
仅允许在一端进行插入和删除的[[1.线性表的定义和基本操作|线性表]]
- 栈顶(Top)：允许进行插入和删除操作的一端
- 栈底(Bottom)：固定不变，不允许操作
操作特性为**后进先出(Last In First Out, #LIFO)**
# 栈的基本操作
- `InitStack(&S)`
- `StackEmpty(S)`：栈为空则返回 true
- `Push(&S, x)`：**若栈未满**，将 x 压入栈顶
- `Pop(&S, &x)`：**若栈非空**，则弹出栈顶元素并通过 x 返回弹出值
- `GetTop(&S, &x)`：**若栈非空**，**读**出栈顶元素并通过 x 返回读出值
- `DestroyList(&S)`
不允许访问或修改除栈顶外的元素
# 栈的数学性质
当 n 个不同元素按固定次序入栈时，可能的出栈序列总数为[[卡特兰(Catalan)数]]
# 栈的顺序存储结构
## 顺序栈
```c
typedef struct{
	ElemType data[MaxSize];
	int top;
}SqStack;
```
- 栈顶指针：`S.top`，初始可以设为-1（或者0，表示目前栈顶下一个位置的指针）
- 栈顶元素：`S.data[S.top]`
- 入栈操作：`S.data[++S.top] = x`
- 出栈操作：`x = S.data[S.top--]`
- 栈空条件：`S.top == -1`
- 栈满条件：`S.top == MaxSize - 1`
- 栈长：`S.top + 1`
## 共享栈
利用栈底相对固定的特性，**解决上溢的基础上节省存储空间**（当一个栈满时可以使用另一个栈的空闲空间），让两个栈共享一段一维数组空间，将两个栈的栈底分别置于数组两端，栈顶向中间延伸
**栈满条件**：top1 - top2 == 1（两栈顶相邻）

# 栈的链式存储结构
采用链式存储，所有操作均在表头进行，约定不带头结点，便于动态分配存储空间

## 算数表达式
- 中缀表达式：如 3+4
- 后缀表达式：如 34+
- 前缀表达式：如 +34
中缀表达式依赖于括号表明运算次序，前缀和后缀表达式隐含了运算顺序，便于计算机求值
### 中缀表达式转后缀表达式
使用一个**运算符栈**
1. 遇到**操作数**：直接加入后缀表达式
2. 遇到**界限符**：若为 "(" ，直接入栈；若为 ")" ，不入栈，且不断弹出栈顶运算符加入后缀表达式，直到遇到 "(" ，将其弹出并丢弃
3. 遇到**运算符**：
	1. 若运算符优先级**高于**栈顶运算符，则入栈
	2. 若运算符优先级**低于或等于**栈顶运算符，则依次弹出并加入后缀表达式，直到栈空或者栈顶为 "(" 或栈顶的运算符优先级更低，**再将当前运算符入栈**
4. 扫描完所有字符后，将栈中剩余运算符依次弹出并加入后缀表达式
### 后缀表达式求值
运用一个**操作数栈**
1. 遇到**操作数**：入栈
2. 遇到**运算符**：弹出栈中的两个操作数进行运算，然后将结果入栈
## 递归
+ 栈可以模拟递归过程，以此来消除递归
+ 对于单向递归（一个递归函数在一次调用中，只递归调用自己一次）和尾递归（递归调用是函数执行的最后一步），可以用迭代（即循环）的方式来消除递归
