# 归并排序的基本思想
**基本思想**：归并排序基于**分治法(Divide and Conquer)**，**将两个或两个以上的有序表合并成一个新的有序表**
- **分(Divide)**：将含有 $n$ 个元素的待排序表分成两个大小大致相等的子表（即从中间一分为二）
- **治(Conquer)**：递归地对两个子表进行归并排序
- **合(Combine)**：将两个已排好序的子表合并成一个有序表
> 与快速排序的对比：快速排序是"先划分，再递归"（重在分），归并排序是"先递归，再合并"（重在合）
> 归并排序是**稳定**的 $O(n \log n)$ 排序算法，但需要 **$O(n)$ 额外空间**
# 二路归并(2-Way Merge)
## 合并算法思想
将两个**各自有序**的子序列 `A[low...mid]` 和 `A[mid+1...high]` 合并成一个有序序列
- 设定三个指针：$i$ 指向左子表首元素，$j$ 指向右子表首元素，$k$ 指向辅助数组 `B` 的当前位置
- 比较 `A[i]` 和 `A[j]`，将**较小者**复制到 `B[k]` 中，对应指针后移
- 重复直到某一子表全部复制完，然后将另一子表的剩余元素直接复制到 `B` 中
- 最后将辅助数组 `B` 中合并好的结果**复制回**原数组 `A`
## 合并算法实现
```c
// 将有序的 A[low...mid] 和 A[mid+1...high] 合并到 A[low...high] 中
void Merge(int A[], int low, int mid, int high) {
    // 需要辅助数组 B 存放合并结果
    int *B = (int *)malloc((high - low + 1) * sizeof(int));
    int i, j, k;
    for (k = low; k <= high; k++)
        B[k - low] = A[k];              // 将 A 中元素复制到 B（偏移量 low）
    i = low;         // i 指向左子表在 B 中的起始位置
    j = mid + 1;     // j 指向右子表在 B 中的起始位置
    k = low;         // k 指向 A 中当前填入位置
    while (i <= mid && j <= high) {
        if (B[i - low] <= B[j - low])   // 注意：<= 保证稳定性（相等时取左边）
            A[k++] = B[i++ - low];
        else
            A[k++] = B[j++ - low];
    }
    // 左侧剩余元素直接复制
    while (i <= mid)
        A[k++] = B[i++ - low];
    // 右侧剩余元素直接复制
    while (j <= high)
        A[k++] = B[j++ - low];
    
    free(B);
}
```
> **稳定性关键**：比较时使用 `<=` 而非 `<`，当左右元素相等时**优先取左边的**，保证相等元素的相对顺序不变
> **不使用偏移量的简化版**（开辟 $n$ 大小的全局辅助数组，一次性分配）也是常见写法，考试中更简洁
## 合并过程图解示例
合并有序子表 `[13, 38, 49, 76, 97]` 和 `[27, 49*, 65]`：

| 步骤 | 左指针(i) | 右指针(j) | 比较 | 输出 | 结果 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| 1 | 13 | 27 | 13 < 27 | 13 | [13] |
| 2 | 38 | 27 | 38 > 27 | 27 | [13, 27] |
| 3 | 38 | 49* | 38 < 49* | 38 | [13, 27, 38] |
| 4 | 49 | 49* | 49 ≤ 49* | 49 | [13, 27, 38, 49] |
| 5 | 76 | 49* | 76 > 49* | 49* | [13, 27, 38, 49, 49*] |
| 6 | 76 | 65 | 76 > 65 | 65 | [13, 27, 38, 49, 49*, 65] |
| 7 | 76 | — | 右子表空 | 76, 97 | [13, 27, 38, 49, 49*, 65, 76, 97] |

> 注意步骤 4：`49 ≤ 49*`，优先取左边的 49，保证**稳定** ✅
# 递归实现（自顶向下）
## 算法实现
```c
// 归并排序（递归版）
void MergeSort(int A[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;          // 从中间划分两个子序列
        MergeSort(A, low, mid);              // 递归对左半排序
        MergeSort(A, mid + 1, high);         // 递归对右半排序
        Merge(A, low, mid, high);            // 合并两个有序子表
    }
}
```
> 递归过程类似**二叉树的后序遍历**：先处理左右子树，再处理根（合并）
> 当 `low == high` 时，子表长度为 1，自然有序，递归终止
# 非递归实现（自底向上）
**算法思想**：不通过递归分解，而是直接从长度为 1 的子表开始，**逐趟**将相邻的两个有序子表合并
- 第 1 趟：将长度为 1 的相邻子表两两合并，得到长度为 2 的有序子表
- 第 2 趟：将长度为 2 的相邻子表两两合并，得到长度为 4 的有序子表
- ......
- 第 $\lceil \log_2 n \rceil$ 趟：最终合并成一个长度为 $n$ 的有序表
## 算法实现
```c
// 归并排序（非递归版）
void MergeSortNonRecursive(int A[], int n) {
    int step;  // 步长（当前子表长度）
    
    for (step = 1; step < n; step *= 2) {
        // 每趟：将相邻的长度为 step 的子表两两合并
        for (int low = 0; low < n; low += 2 * step) {
            int mid = low + step - 1;
            int high = low + 2 * step - 1;
            if (mid >= n)          // 只有一个子表，无需合并
                break;
            if (high >= n)         // 第二个子表不完整，修正上界
                high = n - 1;
            Merge(A, low, mid, high);
        }
    }
}
```
> 注意边界处理：最后一个子表可能长度不足 `step`，第二个子表也可能不完整，需要修正 `high`
# 性能分析

| 指标 | 值 |
|:---:|:---:|
| **最好时间复杂度** | $O(n \log_2 n)$ — 与初始序列无关，分解 + 合并次数固定 |
| **最坏时间复杂度** | $O(n \log_2 n)$ — 无论序列状态如何，都需要全部分解和合并 |
| **平均时间复杂度** | $O(n \log_2 n)$ |
| **空间复杂度** | $O(n)$ — 需要大小为 $n$ 的辅助数组（**非原地排序**） |
| **稳定性** | **稳定** ✅ — `Merge` 中相等元素优先取左侧子表的 |
| **适用性** | 适用于顺序和链式存储的线性表 |

## 时间复杂度详细分析
- 归并排序的递归树是一棵**二叉树**，高度为 $\lceil \log_2 n \rceil$
- 每层递归中，`Merge` 操作需要遍历当前层的所有元素，时间为 $O(n)$
- 总时间复杂度：$O(n) \times O(\log n) = O(n \log n)$（**与初始序列无关**）
> 对 $N$ 个元素进行 $k$ 路归并排序，排序的趟数 $m$ 满足 $k^m = N$，从而 $m =\lceil \log_k{N} \rceil$ 
> **$O(n \log n)$ 的同级对比**：

| 排序算法 | 最好 | 最坏 | 平均 | 空间 | 稳定 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **归并排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | ✅ 稳定 |
| **堆排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | ❌ 不稳定 |
| **快速排序** | $O(n \log n)$ | $O(n^2)$ | $O(n \log n)$ | $O(\log n)$ | ❌ 不稳定 |

# 特点总结
## 优点
- **时间复杂度稳定**：任何情况下都是 $O(n \log n)$，不会退化（与快速排序的最大区别）
- **稳定排序**：`Merge` 过程天然保证相等元素的相对顺序
- **适用于外部排序**：归并排序是**外部排序**的核心算法（多路归并 + 败者树）
	- 详见 [[9.外部排序]]
## 缺点
- **空间复杂度 $O(n)$**：需要与原始数组同等大小的辅助空间（最大痛点）
- **实际运行效率**：递归调用有额外开销，`Merge` 中的复制操作增加了常数因子
- 实际中，当内存受限时堆排序（$O(1)$ 空间）更优；当需要稳定 $O(n \log n)$ 时归并排序唯一选择
## 适用场景
- 数据量较大且**要求稳定排序**的场景
- **外部排序**（多路归并）的基础
- **链表排序**最佳选择：链表归并排序空间复杂度可降至 $O(1)$（原地修改指针），且时间复杂度仍为 $O(n \log n)$
- 并行计算友好：分治结构天然适合并行化
> **优劣总结**：✅ 任何情况 $O(n \log n)$ 不退化、稳定、适合外部排序和链表；❌ 空间 $O(n)$ 开销大、常数因子较大（比快速排序慢）
# 关键考点（408 高频）

1. **归并排序的分治思想**：先递归分解，再合并（与快速排序"先划分再递归"对比）
2. **`Merge` 函数的实现**：两个有序表合并为一个有序表的算法（手写代码重点）
3. **稳定性**：归并排序是**稳定**的 ✅（`<=` 比较 + 优先取左侧元素）
4. **时间复杂度**：始终 $O(n \log n)$，**与初始序列无关**（常与快速排序对比考查）
5. **空间复杂度**：$O(n)$（常与堆排序 $O(1)$、快排 $O(\log n)$ 对比）
6. **递归深度 / 趟数**：$\lceil \log_2 n \rceil$ 趟
7. **比较次数**：每趟合并时最多比较 $n-1$ 次（当两个子表元素交替出现时），最少比较 $\frac{n}{2}$ 次（一个子表元素全小于另一个）
8. **非递归归并**：自底向上，从 `step=1` 开始，每趟 `step *= 2`，边界处理
9. **与外部排序的关系**：归并排序是外部排序的核心 → [[9.外部排序]]
10. **链表归并排序**：链式存储下空间复杂度降为 $O(1)$，是链表排序的最优解
# 归并排序与其他排序的对比

## 二路归并排序 vs 快速排序

| 比较维度 | 归并排序 | 快速排序 |
|:---:|:---:|:---:|
| 思想核心 | **先递归，后合并**（重在合） | **先划分，后递归**（重在分） |
| 递归顺序 | 类似**后序遍历** | 类似**先序遍历** |
| 时间复杂度 | 始终 $O(n \log n)$ | 最好/平均 $O(n \log n)$，最坏 $O(n^2)$ |
| 空间复杂度 | $O(n)$ | 最好 $O(\log n)$，最坏 $O(n)$ |
| 稳定性 | **稳定** ✅ | 不稳定 ❌ |
| 子问题是否有序 | 合并前子表**各自有序** | 划分后子表**内部无序**，但子表间已分大小 |
| 每趟结果 | 子表合并后有序 | 枢轴到达最终位置 |

## 二路归并 vs 堆排序

| 比较维度 | 归并排序 | 堆排序 |
|:---:|:---:|:---:|
| 时间复杂度 | $O(n \log n)$ | $O(n \log n)$ |
| 空间复杂度 | $O(n)$ ❌ | $O(1)$ ✅（原地排序） |
| 稳定性 | ✅ 稳定 | ❌ 不稳定 |
| 链表适用 | ✅（空间可优化为 $O(1)$） | ❌（依赖随机访问） |
| 外部排序 | ✅ 核心算法 | ❌ 不适用 |
