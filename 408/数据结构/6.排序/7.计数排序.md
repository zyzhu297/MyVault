# 计数排序的基本思想
**基本思想**：计数排序(Counting Sort)是一种**非比较型**排序算法，核心是通过统计每个元素出现的**次数**来确定元素位置
- **不基于比较**：不通过元素间的比较决定顺序，而是利用元素值直接计算其在有序序列中的位置
- **统计计数**：用一个辅助数组 $count[k+1]$ 统计每个值出现的次数
- **累加确定位置**：对计数数组做累加，得到每个值在有序序列中的**最终位置**
# 算法步骤
1. **统计**：遍历原数组，统计各元素值出现的次数，填入计数数组 $count$
2. **累积**：对 $count$ 做前缀和，$count[i]$ 表示值 $\le i$ 的元素个数
3. **归位**：从后向前遍历原数组，根据 $count$ 确定每个元素在输出数组中的位置
> **为什么要从后往前遍历？** 为了保证**稳定性**：值相同的元素，后出现的放在更后面的位置，保持原相对顺序
# 算法实现
```c
// 计数排序（升序，稳定版本）
// 假设元素值在 [0, k] 范围内
void CountingSort(int A[], int n, int k) {
    int *count = (int *)calloc(k + 1, sizeof(int));  // 计数数组，初始为 0
    int *output = (int *)malloc(n * sizeof(int));     // 输出数组
    int i;
    
    // 1. 统计各值出现次数
    for (i = 0; i < n; i++)
        count[A[i]]++;
    
    // 2. 对 count 做前缀和：count[i] = 值 ≤ i 的元素个数
    for (i = 1; i <= k; i++)
        count[i] += count[i - 1];
    
    // 3. 从后向前遍历原数组，确定每个元素位置（保证稳定性）
    for (i = n - 1; i >= 0; i--) {
        output[count[A[i]] - 1] = A[i];  // count[A[i]] 是 ≤ A[i] 的元素个数，即排名
        count[A[i]]--;                   // 值相同的下一个元素向前放
    }
    
    // 4. 复制回原数组
    for (i = 0; i < n; i++)
        A[i] = output[i];
    
    free(count);
    free(output);
}
```
> `count[A[i]] - 1`：因为有 $count[A[i]]$ 个元素 $\le A[i]$，所以 $A[i]$ 应放在下标 $count[A[i]] - 1$ 的位置（下标从 0 开始）
# 性能分析

| 指标 | 值 |
|:---:|:---:|
| **时间复杂度** | $O(n + k)$ |
| — 最好 / 最坏 / 平均 | **三者相同** ✅（与初始序列无关） |
| **空间复杂度** | $O(n + k)$ — 输出数组 $n$ + 计数数组 $k$ |
| **稳定性** | **稳定** ✅（从后向前遍历保证了稳定性） |
| **是否为比较排序** | ❌ 不基于比较 |

- $n$：待排序元素个数
- $k$：元素取值范围（最大值）
- 当 $k = O(n)$ 时，时间复杂度退化为 $O(n)$ — **线性时间排序**
- 当 $k$ 远远大于 $n$（如 $k = n^2$）时，效率急剧下降
# 特点总结
## 优点
- **线性时间 $O(n)$**：当 $k$ 较小时极快，突破比较排序下界 $O(n \log n)$
- **稳定排序** ✅
- **算法简单**：只需统计和归位两步
## 缺点
- **空间开销大**：需要 $O(n+k)$ 额外空间，$k$ 大时不可接受
- **适用面窄**：只能排序**整数**（因为数组下标必须是整数），且值域不能太大
- **不能原地排序**
## 适用场景
- 整数排序，且取值范围不大（如：年龄、成绩、考试分数等）
- 元素取值密集，$k = O(n)$
- 需要稳定排序且数据范围可控
# 关键考点（408 高频）
1. **计数排序的基本思想**：统计计数 → 累积 → 归位
2. **时间复杂度**：$O(n+k)$，最好/最坏/平均三者相同，与初始序列无关
3. **空间复杂度**：$O(n+k)$
4. **稳定性**：稳定 ✅（从后向前遍历是关键）
5. **不基于比较**：能突破 $O(n \log n)$ 的比较排序下界
6. **适用条件**：① 整数；② 取值范围 $k$ 不能太大
7. **与基数排序的区别**：计数排序一次性统计整个值，基数排序按位分趟
