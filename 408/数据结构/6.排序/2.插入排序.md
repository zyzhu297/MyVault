# 插入排序的基本思想
**基本思想**：将待排序记录逐个插入到已排好序的有序序列中，直到全部记录插入完毕
- 将序列分为**有序区**和**无序区**，固定有序区在前，无序区在后
- 初始时，有序区通常只包含第一个元素
- 每趟从无序区取一个元素，插入到有序区的合适位置
- 每插入一个元素，有序区长度 +1，无序区长度 -1
- 当无序区为空时排序结束
> 类比：打扑克牌时，每摸一张牌就将其插入到手牌中正确的位置
# 直接插入排序
## 算法思想
将待排序元素依次与有序区中元素从右向左比较，找到合适位置后插入
- $L[1...n]$ 为待排序序列（下标从 1 开始）
- 初始：$L[1]$ 为有序区，$L[2...n]$ 为无序区
- 第 $i$ 趟：将 $L[i]$ 暂存到哨兵 $L[0]$，依次与 $L[i-1], L[i-2], ...$ 比较，若待插元素更小则后移，直到找到插入位置
## 算法实现
```c
// 直接插入排序（升序）
void InsertSort(int A[], int n) {
    int i, j, temp;
    for (i = 1; i < n; i++) {              // 将各元素插入已排好序的序列中
        temp = A[i];                        // 取出待插入元素
        j = i - 1;
        while (j >= 0 && temp < A[j]) {    // 从右向左扫描，找插入位置
            A[j + 1] = A[j];               // 元素后移
            j--;
        }
        A[j + 1] = temp;                   // 插入元素
    }
}
```
## 手工执行示例
初始序列：`[49, 38, 65, 97, 76, 13, 27]`

| 趟数 | 有序区 | 操作 | 结果 |
|:---:|:-----:|:---:|:----:|
| 初始 | [49] | — | 49, 38, 65, 97, 76, 13, 27 |
| 1 | [38, 49] | 38 与 49 比较，49 后移，38 插入 | **38, 49**, 65, 97, 76, 13, 27 |
| 2 | [38, 49, 65] | 65 > 49，直接插入末尾 | **38, 49, 65**, 97, 76, 13, 27 |
| 3 | [38, 49, 65, 97] | 97 > 65，直接插入末尾 | **38, 49, 65, 97**, 76, 13, 27 |
| 4 | [38, 49, 65, 76, 97] | 76 依次比较，插入 65 和 97 之间 | **38, 49, 65, 76, 97**, 13, 27 |
| 5 | [13, 38, 49, 65, 76, 97] | 13 最小，所有元素后移，插入最前 | **13, 38, 49, 65, 76, 97**, 27 |
| 6 | [13, 27, 38, 49, 65, 76, 97] | 27 插入 13 和 38 之间 | **13, 27, 38, 49, 65, 76, 97** |

## 性能分析

|       指标        |                     值                      |
| :-------------: | :----------------------------------------: |
| **最好情况**（序列已有序） |       每趟只需比较 1 次，移动 0 次，时间复杂度 $O(n)$       |
| **最坏情况**（序列逆序）  | 第 $i$ 趟比较 $i$ 次、移动 $i+1$ 次，时间复杂度 $O(n^2)$  |
|    **平均情况**     | 比较和移动次数均约为 $\dfrac{n^2}{4}$，时间复杂度 $O(n^2)$ |
|    **空间复杂度**    |         仅需常数个辅助空间，$O(1)$（**原地排序**）         |
|     **稳定性**     |   **稳定** ✅（当待插元素与有序区元素相等时，插入到其后方，相对位置不变）   |
|     **适用性**     |       适用于顺序和链式存储的线性表，采用链式存储时不需要移动元素        |

## 特点总结
- 最坏比较次数为 $\frac{n(n-1)}{2}$ 次
- 数据量小（$n \le 50$）或基本有序时效率较高
- 数据量大且无序时性能较差
- 每趟排序**一定**能使有序区多一个元素，但**不能保证**某个元素到达最终位置
> [!info] 与冒泡排序区别：
> 直接插入排序每趟只是**扩大有序区**，而冒泡排序每趟能确定一个元素的最终位置（沉底）
# 折半插入排序
## 算法思想
直接插入排序中，查找插入位置采用顺序查找，比较次数较多。由于有序区已有序，可以用**折半查找**（二分查找）来优化查找过程，减少比较次数
- 在有序区用折半查找找到待插元素的插入位置
- 找到位置后，统一移动后续元素
- 插入元素
## 算法实现
```C
// 折半插入排序（升序）
void BinaryInsertSort(int A[], int n) {
    int i, j, low, high, mid, temp;
    for (i = 1; i < n; i++) {
        temp = A[i];                        // 取出待插入元素
        low = 0;high = i - 1;
        // 折半查找插入位置
        while (low <= high) {
            mid = (low + high) / 2;
            if (A[mid] > temp) high = mid - 1;             // 插入位置在左半区
            else low = mid + 1;              // 插入位置在右半区（含相等情况）
        }
        // 此时 low 即为插入位置，high = low - 1
        // 统一后移元素
        for (j = i - 1; j >= low; j--)
            A[j + 1] = A[j];
        A[low] = temp;                      // 插入元素
    }
}
```
> 关键点：当 `A[mid] == temp` 时，继续在右半区查找（`low = mid + 1`），保证**稳定性**
## 性能分析

|     指标      |                              值                              |
| :---------: | :---------------------------------------------------------: |
|  **比较次数**   | 折半查找每趟比较 $\lceil \log_2 i \rceil + 1$ 次，总比较次数约 $O(n\log n)$ |
|  **移动次数**   |            仍为 $O(n^2)$，与直接插入排序相同（**因为移动方式未改变**）             |
| **平均时间复杂度** |               $O(n^2)$（比较次数减少但移动次数不变，整体量级不变）                |
|  **空间复杂度**  |                      $O(1)$（**原地排序**）                       |
|   **稳定性**   |                          **稳定** ✅                           |
|  **适用性 **   |                  仅适用于顺序存储的线性表 — 要求能够随机访问元素                  |

## 与直接插入排序的对比

| 对比项 | 直接插入排序 | 折半插入排序 |
|:-----:|:----------:|:----------:|
| 查找方式 | 顺序查找 | 折半查找 |
| 比较次数 | $O(n^2)$ | $O(n\log n)$ ✅ 减少 |
| 移动次数 | $O(n^2)$ | $O(n^2)$（不变） |
| 总时间复杂度 | $O(n^2)$ | $O(n^2)$（量级未变） |

> [!warning] 折半插入时间复杂度不变
> 折半插入排序减少了比较次数，但移动次数并未改变，因此总的时间复杂度量级仍然是 $O(n^2)$。当数据量较大时，仍然不适用
# 希尔排序
## 算法思想
希尔排序(Shell Sort)是直接插入排序的改进版本，也称为**缩小增量排序**(Diminishing Increment Sort)
**核心思想**：将待排序列分割成若干子序列，对子序列分别进行**直接插入排序**；待整个序列基本有序后，再对全体记录进行一次直接插入排序。
- 选择一个**增量序列** $d_1, d_2, ..., d_k$，其中 $d_1 = \lfloor n/2 \rfloor$，$d_{i+1} = \lfloor d_i/2 \rfloor$，$d_k = 1$
- 按增量 $d_k$ 将序列分成 $d_k$ 组，每组中相邻元素下标相差 $d_k$
- 对每组分别进行直接插入排序
- 逐渐缩小增量，重复上述过程
- 当增量 $d_k = 1$ 时，相当于对整个基本有序的序列做一次直接插入排序
> 常用的增量序列（Shell 增量）：$d_1 = \lfloor n/2 \rfloor,\ d_{i+1} = \lfloor d_i/2 \rfloor$，最后 $d_k = 1$
## 手工执行示例
初始序列：`[49, 38, 65, 97, 76, 13, 27, 49]`

|  增量   |                分组情况                | 每组排序后结果 |                整体序列                |
| :---: | :--------------------------------: | :-----: | :--------------------------------: |
| $d=4$ | (49,76), (38,13), (65,27), (97,49) | 各组插入排序  | **49, 13, 27, 49, 76, 38, 65, 97** |
| $d=2$ |    (49,27,76,65), (13,49,38,97)    | 各组插入排序  | **27, 13, 49, 38, 65, 49, 76, 97** |
| $d=1$ |                 全体                 | 直接插入排序  | **13, 27, 38, 49, 49, 65, 76, 97** |

## 算法实现

```c
// 希尔排序（升序）
void ShellSort(int A[], int n) {
    int d, i, j, temp;
    // 步长变化
    for (d = n / 2; d >= 1; d = d / 2) {
        // 对每个分组进行插入排序
        for (i = d; i < n; i++) {
            temp = A[i];
            j = i - d;
            while (j >= 0 && temp < A[j]) {
                A[j + d] = A[j];          // 组内元素后移
                j = j - d;
            }
            A[j + d] = temp;              // 插入元素
        }
    }
}
```
> 注意：希尔排序的代码是**从 $d$ 到 $n-1$ 遍历**，每组交替处理，而不是严格按组分开处理
## 性能分析

|                  指标                  |                 值                  |
| :----------------------------------: | :--------------------------------: |
|              **时间复杂度**               |        依赖于增量序列的选择，数学上尚未完全解决        |
| — Shell 增量 $(\lfloor n/2^k \rfloor)$ |           最坏情况 $O(n^2)$            |
|       — Hibbard 增量 $(2^k - 1)$       | 最坏情况 $O(n^{3/2})$，平均约 $O(n^{5/4})$ |
|      — Knuth 增量 $((3^k - 1)/2)$      |           约 $O(n^{1.3})$           |
|            — Sedgewick 增量            |   约 $O(n^{4/3})$，最坏 $O(n^{7/6})$   |
|              **空间复杂度**               |          $O(1)$（**原地排序**）          |
|               **稳定性**                |    **不稳定** ❌（分组排序可能打乱相等元素的相对顺序）    |
|               **适用性 **               |     仅适用于顺序存储的线性表 — 要求能够随机访问元素      |

## 特点总结
- **优点**：突破了 $O(n^2)$ 的时间复杂度，当 $n$ 较大时效率优于直接插入排序
- **缺点**：不稳定；时间复杂度依赖于增量序列的选择，没有最好情况
- 适用于中等规模（$n \le 10000$）的数据排序
- **最后一个增量必须为 1**
- **增量序列应互质**（无公因子），以提高排序效率
> 证明不稳定的反例：$[49, 49^*, 1]$，$d=2$ 时两组分别插入，排序后 $49^*$ 可能跑到 $49$ 前面
# 三种插入排序对比总结
## 总体对比

| 排序算法 | 平均时间复杂度 | 最坏时间复杂度 | 最好时间复杂度 | 空间复杂度 | 稳定性 | 适用场景 |
|:-------:|:-----------:|:-----------:|:-----------:|:--------:|:----:|:-------:|
| **直接插入排序** | $O(n^2)$ | $O(n^2)$ | $O(n)$ | $O(1)$ | ✅ 稳定 | 数据量小或基本有序 |
| **折半插入排序** | $O(n^2)$ | $O(n^2)$ | $O(n\log n)$ | $O(1)$ | ✅ 稳定 | 数据量小，优化比较次数 |
| **希尔排序** | $O(n^{1.3-2})$ | $O(n^2)$ | $O(n)$ | $O(1)$ | ❌ 不稳定 | 中等规模数据 |

## 关键考点（408 高频）
1. **直接插入排序**的趟数 = 插入次数 = $n-1$，每趟比较次数和移动次数要会分析
2. **折半插入排序**虽然减少了比较次数（$O(n\log n)$），但移动次数仍是 $O(n^2)$，总时间复杂度量级**未变**
3. **希尔排序**的增量序列选取直接影响性能，Shell 增量 $d = \lfloor d/2 \rfloor$ 是最常见的考法
4. 希尔排序的**稳定性**判断：不稳定 ❌（举反例：$[49, 49^*, 1]$）
5. 希尔排序的**一趟排序结果**：会给初始序列和增量 $d$，要求写出该趟排序后的结果
6. 三种插入排序的**空间复杂度均为 $O(1)$**，都是原地排序
