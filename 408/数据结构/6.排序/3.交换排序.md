# 交换排序的基本思想
两两比较待排序元素的关键字，发现逆序（即排序次序相反的两个元素）时进行交换，直到整个序列中没有逆序的元素为止
# 冒泡排序
## 算法思想
通过相邻元素之间的比较和交换，使关键字较小的元素逐渐从底部移向顶部（"冒泡"），或者使关键字较大的元素逐渐沉向底部。
## 算法步骤
1. 从后往前（或从前往后）两两比较相邻元素的值，若为逆序（即 $A[j-1] > A[j]$），则交换它们
2. 每一趟排序都会将当前未排好序部分中**最大**的元素"沉"到末尾
3. 对剩下的未排序序列重复上述过程，直到某一趟没有发生交换为止
## 代码实现
```c
void BubbleSort(int A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false;                 // 本趟是否有交换的标志
        for (int j = n - 1; j > i; j--) { // 从后往前比较
            if (A[j - 1] > A[j]) {
                swap(A[j - 1], A[j]);
                flag = true;
            }
        }
        if (!flag)                         // 本趟没有交换，说明已经有序
            return;
    }
}
```
## 性能分析

|     指标      |                                  值                                   |
| :---------: | :------------------------------------------------------------------: |
| **最好时间复杂度** |                     $O(n)$ — 初始序列已有序，只需一趟比较，无交换                      |
| **最坏时间复杂度** | $O(n^2)$ — 初始序列逆序，比较次数 $= \sum_{i=1}^{n-1} (n-i) = \frac{n(n-1)}{2}$ |
| **平均时间复杂度** |                               $O(n^2)$                               |
|  **空间复杂度**  |                         $O(1)$ — 仅使用常数个辅助单元                          |
|   **稳定性**   |               **稳定** — 当 $A[j-1] == A[j]$ 时不交换，相对次序不变                |
|   **适用性**   |                            适用于顺序和链式存储的线性表                            |

## 特点
- **每趟排序都会确定一个元素的最终位置（当前最大值的位置）**
- 设置 `flag` 标志位可提前结束排序
- 冒泡排序产生的有序子序列一定是全局有序的（即每次确定的元素都放在了最终位置上）
# 快速排序
## 算法思想
基于**分治**策略。在待排序表 $L[1...n]$ 中任取一个元素 `pivot` 作为**枢轴**（基准），通过一趟排序将待排序表划分为独立的两部分：
- $L[1...k-1]$ 中的所有元素 $< pivot$
- $L[k+1...n]$ 中的所有元素 $\ge pivot$
则 `pivot` 放在了其最终位置 $L[k]$ 上。然后分别递归地对两个子表重复上述过程。
## 划分过程(Partition)
### 算法步骤
```c
int Partition(int A[], int low, int high) {
    int pivot = A[low];                    // 以第一个元素为枢轴
    while (low < high) {
        while (low < high && A[high] >= pivot)
            high--;                        // 从右向左找比 pivot 小的
        A[low] = A[high];                  // 移到左边
        while (low < high && A[low] <= pivot)
            low++;                         // 从左向右找比 pivot 大的
        A[high] = A[low];                  // 移到右边
    }
    A[low] = pivot;                        // 枢轴归位
    return low;                            // 返回枢轴最终位置
}
```
### 递归主体

```c
void QuickSort(int A[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(A, low, high); // 划分
        QuickSort(A, low, pivotpos - 1);        // 递归左半
        QuickSort(A, pivotpos + 1, high);       // 递归右半
    }
}
```
## 性能分析

|     指标      |                   值                   |
| :---------: | :-----------------------------------: |
| **最好时间复杂度** |  $O(n \log_2 n)$ — 每次划分都均匀，即枢轴在中间位置   |
| **最坏时间复杂度** |    $O(n^2)$ — 初始序列基本有序或逆序，每次划分极不平衡    |
| **平均时间复杂度** |            $O(n \log_2 n)$            |
| **最好空间复杂度** |        $O(\log_2 n)$ — 递归调用栈深度        |
| **最坏空间复杂度** |           $O(n)$ — 递归退化为单支树           |
|   **稳定性**   | **不稳定** — 因为枢轴元素与远处元素交换时可能打乱相同元素的相对次序 |
|   **适用性**   |       仅适用于顺序存储的线性表 — 要求能够随机访问元素       |

## 递归深度与树形
- 若每次划分均匀，递归树高度为 $\lceil \log_2(n+1) \rceil$
- 若每次划分极不均匀，递归树高度为 $n$
- 递归深度直接影响空间复杂度
## 优化策略
- **随机选取枢轴**：随机选 `A[low...high]` 中的一个元素与 `A[low]` 交换，降低最坏情况概率
- **三数取中法**：取 `low`、`mid`、`high` 三个位置的中位数作为枢轴
- **小规模子表改用插入排序**：当子表长度小于某个阈值（如 5~20）时，直接使用插入排序，减少递归开销
## 快速排序的特点
- 快速排序是所有内部排序算法中**平均性能最优**的排序算法
- 在 $n$ 较大时，快速排序优势明显
- **每趟排序会将一个元素（枢轴）放到其最终位置上**，
	- 两趟排序若确定两个元素的最终位置，则至少有一个元素在数组的首端或尾端
	- 若基准元素不在首端或尾端，则两趟排序至少确定三个元素的最终位置
- 快速排序**不产生有序子序列**——每次划分得到的两个子表内部仍然无序，**但是子表之间有序**，只有枢轴到达最终位置
