# 基数排序的基本思想
**基本思想**：基数排序(Radix Sort)是一种**非比较型**整数排序算法，其核心思想是**多关键字排序**——将单关键字按位数切割成不同的"位"，然后对每一位分别进行排序
- **不基于比较**：基数排序不通过元素间的相互比较来决定顺序，而是采用**分配(distribute** + **收集(collect)** 的策略
- **多关键字思想**：将待排序元素视为多关键字，对每个关键字依次进行一趟排序
- **稳定排序是前提**：每一趟分配与收集所采用的子排序算法必须是**稳定**的，否则基数排序整体无法保证正确性
> 类比：对扑克牌排序时，可以先按花色分堆，再按点数排序（高位优先）；也可以先按点数分堆，再按花色排序（低位优先）
# 核心概念
## 关键字优先级
 **LSD(Least Significant Digit First)最低位优先**：从最低位开始，逐步向高位进行，例如:个位，十位，百位，...
**MSD(Most Significant Digit First)最高位优先**：从最高位开始，逐步向低位进行，例如:
百位，十位，个位，...
> LSD 优先，因为 LSD 可以通过链式基数排序统一处理，算法简洁；MSD 通常需要递归实现，更为复杂
## 链式基数排序（LSD 实现）
**核心思想**：利用**队列**（或链表）完成分配与收集，**低关键字先排，高关键字后排**
- **分配( Distribute )**：根据当前位上的值，将元素分配到对应的队列中（0 ~ 9）
- **收集( Collect )**：按从小到大的顺序依次将各队列中的元素首尾相连，形成新的序列
- **逐位递推**：从最低位开始，重复分配与收集，直到最高位处理完毕
> 每次收集后，元素按照当前位（及更低位）有序；当处理完最高位后，整个序列有序
> **关键理解**：高位关键字排序时，低位已经有序；高位相同的元素，其相对顺序由已排好的低位决定；这要求每趟分配收集所用的排序方法必须是**稳定的** ✅

# 简化版实现（整数排序）
```c
// 简化版 LSD 基数排序（对 int 数组排序）
// d 为最大位数，radix 为基数（通常为 10）
void RadixSortSimple(int A[], int n, int d) {
    int *bucket = (int *)malloc(n * sizeof(int));        // 辅助桶
    int count[RADIX];                                    // 计数器
    int i, j, k;
    int radixPower = 1;  // 当前位的权重（1, 10, 100, ...）
    
    for (i = 0; i < d; i++) {  // 进行 d 趟分配和收集
        // 清零计数器
        for (j = 0; j < RADIX; j++)
            count[j] = 0;
        
        // 统计当前位各数字的出现次数（分配）
        for (j = 0; j < n; j++) {
            k = (A[j] / radixPower) % RADIX;  // 取当前位的值
            count[k]++;
        }
        
        // 将 count[j] 转化为累积计数（计算各桶的结束位置）
        // 这一步使得多次出现的关键字有各自的存放区间
        for (j = 1; j < RADIX; j++)
            count[j] += count[j - 1];
        
        // 从后向前遍历，将元素放入桶中对应位置（保证稳定性）
        for (j = n - 1; j >= 0; j--) {
            k = (A[j] / radixPower) % RADIX;  // 取当前位的值
            bucket[--count[k]] = A[j];
        }
        
        // 将桶中元素复制回原数组（收集）
        for (j = 0; j < n; j++)
            A[j] = bucket[j];
        
        radixPower *= RADIX;  // 处理下一位（10 → 100 → 1000...）
    }
    
    free(bucket);
}
```
> **为什么从后向前遍历？** 为了保证**稳定性**：因为代码中 `count` 保存的是每个位的**结束位置**，而相同当前位的元素，先出现的应该排在前面的位置；从后往前遍历再通过 `--count` 递减索引，能保证同桶中后扫描的元素放在更后面的位置，因此保持了原相对顺序
# 性能分析

| 指标 | 值 |
|:---:|:---:|
| **时间复杂度** | $O(d \cdot (n + r))$ |
| — 最好 / 最坏 / 平均 | **三者相同** ✅（与初始序列无关） |
| **空间复杂度** | $O(n + r)$ — 需要 $n$ 个元素的辅助空间 + $r$ 个队列的头尾指针 |
| **稳定性** | **稳定** ✅ — 前提是每趟分配收集稳定 |
| **适用性** | ① 数据可以分割成独立的"位"；② 每位取值范围不大（$r$ 较小） |
| **是否为比较排序** | ❌ 不基于比较 — 突破了 $O(n \log n)$ 的比较排序下界 |

## 时间复杂度详细分析
- $d$：关键字位数，即**趟数**
- $n$：待排序元素个数
- $r$：基数（每位的取值范围，十进制 $r = 10$）
- 每趟分配：$O(n)$ — 遍历所有元素
- 每趟收集：$O(r)$ — 遍历所有桶队列
- **总时间复杂度**：$O(d \cdot (n + r))$
> 当 $d$ 为常数（如 $d = 3$ 对于三位数），$r$ 为常数（$r = 10$），则时间复杂度退化为 $O(n)$ —— **线性时间排序**
> **关键理解**：基数排序不是基于比较的排序，因此可以突破 $\Omega(n \log n)$ 的比较排序理论下界
# 特点总结
## 优点
- **线性时间复杂度**：当 $d$ 和 $r$ 为常数时，时间复杂度 $O(n)$，快于所有比较排序
- **稳定排序**：天然稳定 ✅（前提：每趟分配收集使用稳定方法）
- **适合整数/字符串**：对可以分割成独立"位"的关键字有效
- **无须关键字比较**：不依赖元素间的比较操作
- **适用于多关键字排序**：链式基数排序是多关键字排序的经典实现
## 缺点
- **空间开销大**：需要额外的桶空间 + 队列指针
- **适用范围窄**：关键字必须能分割成独立的"位"，且每位取值范围不能太大
  - 例如对 `float`、`double` 排序较复杂，需要特殊转换
  - 对取值范围极大的关键字（如 64 位整数 $r = 2^{64}$），不适用
- **不能原地排序**：需要 $O(n)$ 辅助空间
- **依赖稳定子排序**：若子排序不稳定，基数排序整体失效
## 适用场景
- 整数排序（位数固定，基数 10）
- 字符串排序（固定长度的 ASCII 字符串）
- 多关键字排序（如：先按年龄，再按姓名排序）
- 需要稳定且数据量大的整数排序场景
> **优劣总结**：✅ 线性时间 $O(n)$、稳定、突破比较下界；❌ 空间开销大、适用范围窄、不能原地排序
# 基数排序 vs 其他排序的全面对比

## 与计数排序对比

| 比较维度 | 基数排序 | 计数排序 |
|:---:|:---:|:---:|
| **思想** | 按位分多趟分配收集 | 统计个数的累加计数 |
| **时间复杂度** | $O(d \cdot (n+r))$ | $O(n+k)$ |
| **空间复杂度** | $O(n+r)$ | $O(n+k)$ |
| **适用范围** | 关键字可分位 | 关键字取值范围小（$k$ 小） |
| **稳定性** | ✅ 稳定 | ✅ 稳定 |
| **趟数** | $d$ 趟 | 1 趟 |

> 计数排序是基数排序的特殊情况吗？不是。计数排序一次性按整个关键字值统计，而基数排序按位分趟排序。当 $d=1$ 时基数排序退化为桶排序的变体而非计数排序。
## 与桶排序对比

| 比较维度 | 基数排序 | 桶排序 |
|:---:|:---:|:---:|
| **思想** | 多次分配收集（按位） | 一次分配收集（按值域区间） |
| **桶的处理** | 每个位值对应一个桶 | 每个区间对应一个桶 |
| **排序范围** | 整数/字符串 | 均匀分布的浮点数 |
| **稳定性** | ✅ 稳定 | 取决于桶内排序算法 |

## 与归并 / 快速 / 堆排序对比

| 排序算法 | 平均时间 | 最坏时间 | 空间 | 稳定 | 是否比较 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **基数排序** | $O(d \cdot (n+r))$ | $O(d \cdot (n+r))$ | $O(n+r)$ | ✅ 稳定 | ❌ 非比较 |
| **归并排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | ✅ 稳定 | ✅ 比较 |
| **堆排序** | $O(n \log n)$ | $O(n \log n)$ | $O(1)$ | ❌ 不稳定 | ✅ 比较 |
| **快速排序** | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | ❌ 不稳定 | ✅ 比较 |
| **计数排序** | $O(n+k)$ | $O(n+k)$ | $O(n+k)$ | ✅ 稳定 | ❌ 非比较 |

> 当 $d$ 和 $r$ 为常数时，基数排序的 $O(n)$ 是**理论最优**的基于整数的排序算法（线性时间，突破 $\Omega(n \log n)$）

# 补充：MSD 最高位优先
**MSD 算法思想**：
1. 先按**最高位**进行分配与收集
2. 对于最高位相同的子序列，**递归**地按次高位进行排序
3. 直到所有位都处理完毕
**MSD vs LSD 对比**：

|  维度   |   LSD   |       MSD        |
| :---: | :-----: | :--------------: |
| 排序顺序  | 个位 → 高位 |     高位 → 个位      |
| 实现方式  |  循环迭代   |       通常递归       |
|  稳定性  |  ✅ 稳定的  | ❌ 不一定稳定（取决于递归实现） |
| 代码复杂度 |   简单    |       较复杂        |
| 考研考查  |  ✅ 重点   |       了解即可       |
