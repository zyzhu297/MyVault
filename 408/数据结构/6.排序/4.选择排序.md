# 选择排序的基本思想
**基本思想**：每一趟在待排序元素中选取关键字最小（或最大）的元素，加入到有序序列中，直到全部记录排序完毕
- 将序列分为**有序区**和**无序区**，固定有序区在前，无序区在后
- 初始时，有序区为空，无序区包含所有元素
- 每趟从无序区选出关键字**最小**的元素，与无序区的**第一个**元素交换（放入有序区末尾）
- 每选出一个最小元素，有序区长度 $+1$，无序区长度 $-1$
- 经过 $n-1$ 趟后，序列变为有序
> 与插入排序的区别：插入排序是"摸一张牌插入手牌"，选择排序是"从桌上挑最小的牌放到手牌末尾"
> 与冒泡排序的区别：冒泡排序通过相邻交换"沉底"，选择排序直接与当前无序区首元素交换
# 简单选择排序
## 算法思想
每一趟在剩余未排序元素中找到关键字最小的元素，将其与当前无序区的第一个元素交换，有序区逐步扩大
- $L[1...n]$ 为待排序序列（下标从 0 开始或从 1 开始均可）
- 初始：$L[0...n-1]$ 均为无序区
- 第 $i$ 趟（$i = 0, 1, ..., n-2$）：从 $L[i...n-1]$ 中选出最小元素，与 $L[i]$ 交换
- 经过 $n-1$ 趟后排序结束
## 算法实现
```c
// 简单选择排序（升序）
void SelectSort(int A[], int n) {
    int i, j, minIndex, temp;
    for (i = 0; i < n - 1; i++) {           // 进行 n-1 趟
        minIndex = i;                        // 假设当前无序区第一个元素最小
        for (j = i + 1; j < n; j++) {        // 在无序区中寻找真正的最小值
            if (A[j] < A[minIndex])
                minIndex = j;
        }
        if (minIndex != i) {                // 若找到更小的，交换
            temp = A[i];
            A[i] = A[minIndex];
            A[minIndex] = temp;
        }
    }
}
```
> `if (minIndex != i)` 可省略，但加上可以避免不必要的自身交换
## 手工执行示例
初始序列：`[49, 38, 65, 97, 76, 13, 27]`

| 趟数 | 有序区 | 操作（选出最小并交换） | 结果 |
|:---:|:-----:|:---:|:----:|
| 初始 | [ ] | — | 49, 38, 65, 97, 76, 13, 27 |
| 1 | [13] | 选出 13，与 49 交换 | **13**, 38, 65, 97, 76, 49, 27 |
| 2 | [13, 27] | 选出 27，与 38 交换 | **13, 27**, 65, 97, 76, 49, 38 |
| 3 | [13, 27, 38] | 选出 38，与 65 交换 | **13, 27, 38**, 97, 76, 49, 65 |
| 4 | [13, 27, 38, 49] | 选出 49，与 97 交换 | **13, 27, 38, 49**, 76, 97, 65 |
| 5 | [13, 27, 38, 49, 65] | 选出 65，与 76 交换 | **13, 27, 38, 49, 65**, 97, 76 |
| 6 | [13, 27, 38, 49, 65, 76] | 选出 76，与 97 交换 | **13, 27, 38, 49, 65, 76**, 97 |

## 性能分析

|       指标       |                            值                            |
| :------------: | :-----------------------------------------------------: |
| **最好情况**（任何序列） |          每趟必须遍历整个无序区找最小值，比较次数不变，时间复杂度 $O(n^2)$          |
| **最坏情况**（任何序列） |                    同上，时间复杂度 $O(n^2)$                    |
|    **平均情况**    | 比较次数 $\dfrac{n(n-1)}{2}$，移动次数至多 $3(n-1)$，时间复杂度 $O(n^2)$ |
|   **空间复杂度**    |               仅需常数个辅助空间，$O(1)$（**原地排序**）                |
|    **稳定性**     |            **不稳定** ❌（交换可能打乱相等元素的相对顺序，详见下方反例）            |
|    **适用性**     |              适用于顺序和链式存储的线性表（链式存储时交换结点而非数据）              |

## 特点总结
- 比较次数恒为 $\dfrac{n(n-1)}{2}$，与初始序列**无关**（最好 = 最坏 = 平均 $O(n^2)$）
- 移动次数至多 $3(n-1)$ 次（每次交换 3 次移动），最少为 0 次（已有序）
- **每趟一定能使一个元素到达最终位置**（与冒泡排序相同，与直接插入排序不同）
- 简单选择排序是所有 $O(n^2)$ 排序算法中**移动次数最少**的算法
- **Top-K 应用**：只需进行 $K$ 趟选择即可得到前 $K$ 小（或前 $K$ 大）的元素，时间复杂度 $O(Kn)$，当 $K \ll n$ 时明显优于完整排序
> **优劣总结**：✅ 实现最简单、所有 $O(n^2)$ 排序中移动次数最少、每趟确定最终位置、可提前终止做 Top-K；❌ 比较次数恒为 $\frac{n(n-1)}{2}$ 与初始序列无关（即使已有序也要比满）、不稳定、大数据量性能差
> **证明不稳定反例**：$[2, 2^*, 1]$，第一趟选出最小元素 $1$，与 $A[0]=2$ 交换 → $[1, 2^*, 2]$，此时 $2$ 跑到 $2^*$ 后面，相对位置改变 ❌
> 简单选择排序可以是**稳定的**：用**链表存储**可以实现稳定，但数组实现的交换会破坏稳定性
# 堆排序
## 堆的定义
**堆(Heap)** 是一棵[[2.二叉树#完全二叉树|完全二叉树]]，且满足以下性质：
- **大根堆（大顶堆）**：每个结点的值都 $\ge$ 其左右孩子结点的值 → 根结点最大
  - $L(i) \ge L(2i+1)$ 且 $L(i) \ge L(2i+2)$（下标从 0 开始）
- **小根堆（小顶堆）**：每个结点的值都 $\le$ 其左右孩子结点的值 → 根结点最小
  - $L(i) \le L(2i+1)$ 且 $L(i) \le L(2i+2)$（下标从 0 开始）
> 堆是用**数组**存储的**完全二叉树**，逻辑上是树，物理上是数组
> 从 $0$ 开始编号时：结点 $i$ 的左孩子为 $2i+1$，右孩子为 $2i+2$，父结点为 $\lfloor (i-1)/2 \rfloor$
> 从 $1$ 开始编号时：结点 $i$ 的左孩子为 $2i$，右孩子为 $2i+1$，父结点为 $\lfloor i/2 \rfloor$
> **堆是用于排序的数据结构，不适合查找**
## 算法思想
利用大根堆（升序排序）或小根堆（降序排序）的特性，每次输出堆顶元素，然后调整剩余元素重建堆，重复直到全部元素输出
- **建堆**：将初始无序序列调整为大根堆
- **排序**：反复将堆顶（最大值）与堆的最后一个元素交换，堆大小 $-1$，再向下调整堆顶
- 每趟确定一个元素的最终位置
## 堆的删除操作：向下调整(HeadAdjust / Heapify)
将以 $k$ 为根的子树调整为大根堆，前提：$k$ 的左右子树已经是大根堆
```c
// 向下调整：将以 k 为根的子树调整为大根堆（下标从 0 开始）
void HeadAdjust(int A[], int k, int len) {
    int temp = A[k];                        // 暂存根结点（当前待"下坠"的结点）
    for (int i = 2 * k + 1; i < len; i = 2 * i + 1) {
        // i 指向左孩子，比较左右孩子，选较大的
        if (i + 1 < len && A[i] < A[i + 1])
            i++;                            // i 指向右孩子（较大的）
        if (temp >= A[i]) break;            // 根 ≥ 较大的孩子，满足大根堆，结束
        else {
            A[k] = A[i];                    // 较大的孩子上移
            k = i;                          // 继续向下调整
        }
    }
    A[k] = temp;                            // 原根结点归位
}
```
> 调整过程形象理解：根结点不断与较大（**若有两个孩子，先比较两个孩子取较大值**）的孩子比较，若自己较小则"下坠"，较大的孩子"上浮"
## 建堆（BuildMaxHeap）
从最后一个非叶结点开始，**自下而上**依次对每个非叶结点做**向下调整**
```c
// 建立大根堆
void BuildMaxHeap(int A[], int len) {
    // 最后一个非叶结点下标为 len/2 - 1（下标从 0 开始）
    for (int i = len / 2 - 1; i >= 0; i--)
        HeadAdjust(A, i, len);
}
```
> 最后一个非叶结点编号：$\lfloor n/2 \rfloor - 1$（下标从 0 开始）或 $\lfloor n/2 \rfloor$（下标从 1 开始）
## 堆排序主函数
```c
// 堆排序（升序）
void HeapSort(int A[], int n) {
    BuildMaxHeap(A, n);                     // 1. 建堆：O(n)
    for (int i = n - 1; i > 0; i--) {
        // 2. 堆顶（最大值）与堆尾交换
        swap(A[0], A[i]);
        // 3. 堆大小减 1，从根向下调整，重新成堆
        HeadAdjust(A, 0, i);               // 对根做向下调整，当前堆长为 i
    }
}
```
> 升序排序用**大根堆**，降序排序用**小根堆**（仅改变比较方向）
## 堆的插入操作：向上调整
将新元素置于堆的末端，然后**自下而上**与其父结点比较，若违反堆性质，则交换
```c
// 在堆尾插入新元素后，向上调整
void HeadAdjustUp(int A[], int k) {
    // k 为新插入元素的下标
    int temp = A[k];
    int parent = (k - 1) / 2;              // 父结点
    while (k > 0 && A[parent] < temp) {    // 大根堆：父 < 子则上浮
        A[k] = A[parent];
        k = parent;
        parent = (k - 1) / 2;
    }
    A[k] = temp;
}
```
## 性能分析

|      指标       |                          值                          |
| :-----------: | :-------------------------------------------------: |
|  **建堆时间复杂度**  |      $O(n)$（虽然每个结点调整复杂度不同，但**级数求和**后为 $O(n)$）       |
| **每次调整时间复杂度** |       $O(\log n)$（堆的高度，向下最多比较/移动 $\log n$ 层）        |
|  **总时间复杂度**   | $O(n \log n)$（建堆 $O(n)$ + $n-1$ 次调整，每次 $O(\log n)$） |
|  **最好时间复杂度**  |         $O(n \log n)$（与初始序列无关，建堆 + 调整次数固定）          |
|  **最坏时间复杂度**  |                    $O(n \log n)$                    |
|  **平均时间复杂度**  |                    $O(n \log n)$                    |
|   **空间复杂度**   |         $O(1)$（**原地排序**，仅需常数个辅助空间，不借助额外数组）          |
|    **稳定性**    |        **不稳定** ❌（建堆和调整过程中结点"下坠"会打乱相等元素的相对顺序）        |
|    **适用性**    |       仅适用于顺序存储的线性表 — 需要随机访问（完全二叉树的父子关系依赖数组下标）       |

## 特点总结
- **优点**：时间复杂度 $O(n \log n)$ 在所有情况下都稳定，且是**原地排序**，综合性能优异
- **缺点**：不稳定；不能用于链式存储；实际常数因子比快速排序大
- 堆排序是 $O(n \log n)$ 级别中**唯一**同时满足原地排序（$O(1)$ 空间）的算法
- 适合**大规模数据**排序，且在内存受限场景下比归并排序有优势
- **堆的插入**：新元素放在堆尾，然后**向上调整**（上浮）
- **堆的删除**：用堆尾元素替换堆顶，堆大小 $-1$，然后**向下调整**（下坠）
- **主要应用**：Top-K 问题和优先队列
> **Top-K 详细分析**：求前 $K$ **大**的元素 → 维护一个大小为 $K$ 的**小根堆**（堆顶即当前第 $K$ 大）；遍历 $n$ 个元素，若新元素 $>$ 堆顶，则替换堆顶并向下调整。时间复杂度 $O(n \log K)$，空间复杂度 $O(K)$。当 $K$ 较大时优于**简单选择排序或冒泡排序**的 $O(Kn)$
> **优劣总结**：✅ 任何情况下都是 $O(n \log n)$ 无退化风险、原地排序（$O(1)$ 空间）、适合大数据量和内存受限场景；❌ 不稳定、实际常数因子大（访问不连续导致缓存不友好，通常慢于快速排序）、不能用于链式存储

## 关键考点（408 高频）
1. **简单选择排序**的比较次数恒为 $\dfrac{n(n-1)}{2}$，**与初始序列无关**（最好 = 最坏 = 平均）
2. **简单选择排序**的移动次数至多 $3(n-1)$，是最少的 $O(n^2)$ 排序算法
3. 简单选择排序**每趟确定一个元素的最终位置**（与冒泡、堆排序相同，与直接插入排序不同，后者只是扩大有序区）
4. **堆的结构性质**：完全二叉树 + 大/小根堆定义，给定序列要求画出初始堆的树形
5. **建堆过程**：从 $\lfloor n/2 \rfloor - 1$ 开始向后往前做向下调整，建堆时间复杂度 $O(n)$
6. **堆排序过程**：输出堆顶 → 堆尾元素替换 → 向下调整，共 $n-1$ 次，每次 $O(\log n)$
7. 堆排序中**空间复杂度 $O(1)$** 是常考点（与快速排序 $O(\log n)$、归并排序 $O(n)$ 对比）
8. **向下调整（HeadAdjust）** 的算法实现和应用（删除堆顶、建堆、堆排序的核心操作）
9. 向堆中**插入元素**：放入堆尾，做**向上调整**
10. 堆排序的**稳定性**判断：不稳定 ❌
11. 简单选择排序同样是**不稳定**的 ❌（记住反例）
