# 森林的定义
森林：$m(m \ge 0)$ 棵互不相交的树的集合
# 树、森林与二叉树的转换
- 二叉树转换为树或森林的结果是唯一的
- 二叉树中**右指针为空**的节点有 $n+1$ 个：无论是树转换的二叉树还是森林转换的二叉树，最后一个棵树转换后的右指针为空，然后 $n$ 为树或森林中所有**非终端结点（分支节点）数**，每个非终端结点转化后**最后一个孩子**的右指针也为空
- 原来结构中的**叶结点**转换后为二叉树中**左指针为空**的结点
## 树转换为二叉树
每个结点的左指针指向其第一个孩子；右指针指向其在原树中的下一个右兄弟；由树转换的二叉树的根节点**必无右子树**
转换手工画法：
1. 在所有兄弟结点间添加连线
2. 对每个结点，仅保留其与第一个孩子的连线，删除与其他孩子的连线
3. 以树根为轴心，顺时针旋转 45°
## 森林转换为二叉树
转换手工画法：
1. 将森林中的每棵树转换为相对应的二叉树
2. **每棵树的根也可视为兄弟关系**，在各树的根之间依次添加连线
3. 以第一棵树的根为轴心，顺时针旋转 45°
## 二叉树转化为森林
转换规则：
1. 二叉树的根及其左子树对应第一棵树的二叉树表示
2. 将根的右指针断开，其右子树即代表剩余森林转换后的二叉树
3. 对右子树递归应用相同规则，直至右子树为空
4. 最后将分离出的二叉树还原成普通树

# 森林的遍历
## 先序遍历
1. 访问森林中的第一棵树的根节点
2. 先序遍历第一棵树中根节点的子树森林
3. 先序遍历除去第一棵树后的剩余森林
先序遍历序列与这个**树对应的二叉树的先序序列**相同
## 中序遍历（也称后序遍历）
1. 中序遍历第一棵树中根节点的子树森林
2. 访问森林中的第一棵树的根节点
3. 中序遍历除去第一棵树后的剩余森林
中序遍历序列与这个**树对应的二叉树的中序序列**相同

# 森林的应用
## 并查集
处理不相交集合和**合并**与**查询**问题的树状结构
### 并查集的存储结构
采用[[1.树#双亲表示法|树的双亲表示法]]作为其存储结构，每个子集合用一棵树表示，所有子集合对应的树构成一个森林，并统一存储在一个数组中
在该数组中，数组元素下标代表集合中的元素编号，而每个元素所存储的值为其双亲结点的下标，对于根节点，其双亲域设为负数——通常取该子集合的元素个数的相反数
> 注意：`find`操作总返回**当前根节点**作为集合标志，而不是根节点的双亲域
### 并查集的基本实现
```c
class UnionFind {
    vector<int> fa; // 代表元
    vector<int> sz; // 集合大小

public:
    int cc; // 连通块个数

    UnionFind(int n) : fa(n), sz(n, 1), cc(n) {
        // 一开始有 n 个集合 {0}, {1}, ..., {n-1}
        // 集合 i 的代表元是自己，大小为 1
        ranges::iota(fa, 0); // iota(fa.begin(), fa.end(), 0);
    }

    // 返回 x 所在集合的代表元
    // 同时做路径压缩，也就是把 x 所在集合中的所有元素的 fa 都改成代表元
    int find(int x) {
        // 如果 fa[x] == x，则表示 x 是代表元
        if (fa[x] != x) {
            fa[x] = find(fa[x]); // fa 改成代表元
        }
        return fa[x];
    }

    // 判断 x 和 y 是否在同一个集合
    bool is_same(int x, int y) {
        // 如果 x 的代表元和 y 的代表元相同，那么 x 和 y 就在同一个集合
        // 这就是代表元（根节点）的作用：用来快速判断两个元素是否在同一个集合
        return find(x) == find(y);
    }

    // 把 from 所在集合合并到 to 所在集合中
    // 返回是否合并成功
    bool merge(int from, int to) {
        int x = find(from), y = find(to);
        if (x == y) { // from 和 to 在同一个集合，不做合并
            return false;
        }
        fa[x] = y; // 合并集合。修改后就可以认为 from 和 to 在同一个集合了
        sz[y] += sz[x]; // 更新集合大小（注意集合大小保存在代表元上）
        // 无需更新 sz[x]，因为我们不用 sz[x] 而是用 sz[find(x)] 获取集合大小，但 find(x) == y，我们不会再访问 sz[x]
        cc--; // 成功合并，连通块个数减一
        return true;
    }

    // 返回 x 所在集合的大小
    int get_size(int x) {
        return sz[find(x)]; // 集合大小保存在代表元上
    }
};
```
并查集的时间复杂度：$O(\alpha(n))$，其中 $\alpha(n)$ 是反阿克曼函数，增长极其缓慢，在实际应用中可以视为常数通常小于等于 4
