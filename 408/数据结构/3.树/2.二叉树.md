# 二叉树的定义
- 每个结点至多拥有两棵子树
- 子树具有明确的左右之分，属于**有序树**
二叉树和度为 2 的有序树的本质区别：
- 度为 2 的树至少包含 3 个结点，而二叉树可以为空
- 度为 2 的有序树若结点只有一个孩子，则无需区分左右，而在二叉树中一定要区分左右
## 特殊二叉树
### 满二叉树
一棵高度为 $h$ 且包含 $2^h-1$ 个结点的二叉树，即**每一层都含有该层能容纳的最大节点数**，叶节点均位于最下一层，且除叶节点外，其余结点的度均为 2
可对满二叉树按层序进行编号：约定根节点编号为 $1$，自上而下，自左往右依次编号；对于编号为 $i$ 结点，若以下结点存在时，双亲编号为 $\lfloor i/2 \rfloor$，左孩子编号为 $2i$，右孩子编号为 $2i+1$
### 完全二叉树
一棵高度为 $h$ 且包含 $n$ 个结点的二叉树，当且仅当其每个结点与高度为 $h$ 的满二叉树中结点的编号一一对应，称为完全二叉树
### 二叉排序树
其左子树上的所有结点的关键字均小于根节点的关键字；右子树上的所有结点的关键字均大于根节点的关键字，且左子树和右子树本身也各是一棵二叉排序树
### 平衡二叉树
树中任意一个结点的左子树和右子树的高度之差的绝对值不超过 1
### 正则二叉树（严格二叉树）
树中每个分支结点均有 2 个孩子，即树中仅有度为 0 或 2 的结点
# 二叉树的性质
- ⭐非空二叉树的叶节点数等于度为 2 的节点数加 1，即 $n_0=n_2+1$
- 非空二叉树的第 $k$ 层最多有 $2^{k-1}$ 个结点 $(k \ge 1)$
- 高度为 $h$ 的二叉树至多有 $2^h-1$ 个结点$(k \ge 1)$
- 对于依次编号的完全二叉树
	- 最后一个分支节点的编号为 $\lfloor n/2 \rfloor$
	- 叶节点只可能出现在最后两层
	- 若存在度为 1 的结点，则最多只有一个，且该节点只有左孩子而无右孩子，且是最后一个分支节点
	- 若 $n$ 为奇数，则每个分支结点都有左右孩子；若 $n$ 为偶数，则编号最大的分支节点只有左孩子，其余分支结点都有左右孩子
	- 结点 $i$ 所在层次（深度）为 $\lfloor \log_2{i} \rfloor + 1$
- 具有 $n$ 个 $(n>0)$ 结点的完全二叉树的高度为 $\left\lceil\log_2\bigl(n+1\bigr)\right\rceil$ 或 $\left\lfloor\log_2{n}\right\rfloor + 1$
# 二叉树的存储结构
## 二叉树的顺序存储
使用一组连续存储单元，按照自上而下，自左往右的层序，依次存储
满二叉树和完全二叉树非常适合顺序存储，一般二叉树，则需插入若干“空结点”，会浪费空间
## 二叉树的链式存储
```c
typedef struct BiTNode{
	ElemType data;
	struct BiTNode *lchild, *rchild;
}BiTNode, *BiTree;
```
> 在含有 $n$ 个结点的二叉链表中，共有 $n+1$ 个**空链域**
# 二叉树的遍历
关键在于确定对根节点（ N ），左子树（ L ）和右子树（ R ）的访问顺序
先序、中序和后序遍历算法中，递归遍历左右子树的顺序是固定的，区别在于访问根节点的时机不同；**每个结点都被访问一次且仅一次**，时间复杂度均为 $O(n)$ ；在递归实现中，**系统工作栈的深度等于树的高度 $k$ ，空间复杂度为 $O(k)$**，最坏情况下（如单支树），空间复杂度为 $O(n)$

## 中序遍历(InOrder, LNR)
## 后序遍历(PostOrder, LRN)
```c
void PreOrder(BiTree T){
	if (T != NULL){
		//先序遍历
		PreOrder(T->lchild);
		//中序遍历
		PreOrder(T->rchild);
		visit(T);//后序遍历
	}
}
```
## 层次遍历
层次遍历需要借助一个**队列**
```c
void LevelOrder(BiTree T){
	InitQueue(Q);
	BiTree P;
	EnQueue(Q, T);
	while (!IsEmpty(Q)){
		DeQueue(Q, p);
		visit(p);
		if (p->lchild != NULL)
			EnQueue(Q, p->lchild)
		if (p->rchild != NULL)
			EnQueue(Q, p->rchild)
	}
}
```
## 由遍历序列构造二叉树
若已知中序序列，并辅以其他三种遍历序列中的任意一种，则可以**唯一**的重构出该二叉树
若仅凭先序、后序和层序序列中的任意两种组合，通常**无法唯一**确定一棵二叉树
单一序列构造出的不同二叉树有[[卡特兰(Catalan)数]]种，根据遍历递归算法中**递归工作栈**的状态变化得出，相当于以一种遍历序列入栈次序，以中序序列为出栈次序，根据这两个序列可以确定唯一一棵二叉树
# 线索二叉树
在遍历序列中，除首尾结点外，每个结点都有唯一的直接前驱和直接后继
线索数量即为**空链域**数量（总节点数+1）
## 线索二叉树的存储结构
- 若某节点无左子树，则将 lchild 指向其指定遍历序列中的前驱结点
- 若某节点无右子树，则将 rchild 指向其指定遍历序列中的后继结点
为区分指针域指向的是孩子还是线索，需在结点结构中增设两个标志域
- ltag=0 表示 lchild 指向左孩子，ltag=1 表示 lchild 指向前驱
- rtag=0 表示 rchild 指向右孩子，ltag=1 表示 rchild 指向后继
```c
typedef struct ThreadNode{
	ElemType data;
	struct ThreadNode *lchild, *rchild;
	int ltag, rtag;
}ThreadNode, *ThreadTree;
```
## 中序线索二叉树的构造
线索化的本质上就是对二叉树进行一次遍历
```c
void InThread(ThreadTree &p, ThreadTree &pre){
	if (p != NULL){
		InThread(p->lchild, pre);
		if (p->lchild == NULL){
			p->lchild = pre;
			p->ltag = 1;
		}
		if (pre->rchild == NULL){
			pre->rchild = p;
			pre->rtag = 1;
		}
		pre = p;
		InThread(p->rchild, pre);
	}
}
//建立中序线索树的主过程
void CreateInThread(ThreadTree T){
	ThreadTree pre = NULL;
	if (T != NULL){
		InThread(T, pre);
		pre -> rchild = NULL;//处理遍历的最后一个结点
		pre -> rtag = 1;
	}
}
```
为方便遍历，可以在线索链表上增设一个**头结点**：其 lchild 指向根节点；rchild 指向中序序列的最后一个结点；中序序列的第一个结点的 lchild 和最后一个结点的 rchild 指向头节点；如此便形成一个**双向循环线索链表**，从而支持正向和反向的高效遍历
## 中序线索二叉树的遍历
由于已隐含结点的前驱和后继信息，因此遍历时无需借助栈或递归
```c
//求中序序列中的第一个结点
ThreadNode *Firstnode(ThreadNode *p){
	while (p->ltag==0) p = p->lchild;//沿左孩子练走到最左下结点
	return p;
}
//求结点 p 在中序序列的后继
ThreadNode *NextNode(ThreadNode *p){
	if (p->rtag == 0) return Firstnode(p->rchild);//右子树中最左下结点
	else return p->rchild;//rtag = 1 则后孩子指针是后继线索
}
//非递归的中序遍历
void InOrder(ThreadNode *T){
	for (ThreadNode *p = FirstNode(T); p != NULL; p = NextNode(p))
		visit(p);
}
```
> 寻找最后一个结点和结点 p 的前驱的算法，只需将算法中的 l 和 r 替换即可
## 先序线索二叉树和后序线索二叉树
- 先序线索树找后继：若有左孩子，则后继是左孩子，否则若有右孩子，则后继是右孩子，否则沿线索寻找
- 先序线索树找前驱：**由于一个结点的前驱可能是父节点或者某子树的最右下结点**，所以不容易找前驱，通常需要**带父指针的三叉链表**来提供父节点信息
- 后序线索树找后继：**由于访问完一个结点后，下一个结点往往是它的父结点**，所以不容易找前驱，也需要**父指针**
- 后序线索树找前驱：若有右孩子，则前驱是右孩子，否则若有左孩子，则前驱是右孩子，否则沿线索寻找
根据遍历序列孩子在根的位置判断是否容易寻找，先序遍历先根后孩子，所以后继好找，后序遍历先孩子后根，所以前驱好找，**中序遍历前驱后继都好找**。
# 二叉树的应用
## 哈夫曼树
### 带权路径长度(WPL)
树中所有**叶结点**的带权路径长度之和称为该树的**带权路径长度(WPL)**，记为$$WPL = \sum_{i=1}^{n} w_il_i$$
式中，$w_i$ 为第 $i$ 个**叶结点**的权值，$l_i$ 为该叶结点到根结点的[[1.树#树的基本术语#路径和路径长度|路径长度]]
或者记为$$WPL = \sum_{i=1}^{m} w_i$$式中，$w_i$ 为第 $i$ 个**非叶结点**的权值，$m$ 为非叶节点的数量
若将各叶结点的权值视为**其出现的频度或概率**，则每个叶结点的路径长度以其权值为权重做计算出的加权平均值称为**加权平均长度**，记为$$ \text{加权平均长度}=WPL/\text{各叶结点的权值之和}$$
### 哈夫曼树的定义
在含有 $n$ 个带权叶结点的二叉树中，**WPL 最小的二叉树**称为*哈夫曼树*，也称为*最优二叉树*
### 哈夫曼树的构造
给定 $n$ 个权值分别为 $w_1, w_2, \cdots, w_n$ 的结点，构造哈夫曼树的步骤如下：
1. 将这 $n$ 个结点分别视为 $n$ 只包含一个结点的二叉树，构成森林 $F$
2. 构造一个新结点，从 $F$ 中**选取两棵根值结点权值最小的树**作为其左右子树，并将新结点设为左右子树根节点的权值之和
3. 从 $F$ 中删除所选两个树，并将新生成的树加入 $F$
4. 重复步骤 1 和 2，直至 $F$ 中仅剩一棵树为止
### 哈夫曼树的性质
- 每个初始结点最终都成为叶结点，且权值越小的结点，其到根节点的路径长度越大
- 构造过程中**共新建了 $n-1$ 个双分支结点**，因此哈夫曼树的**总结点数为 $2n-1$**
- 不存在度为 1 的结点，为[[2.二叉树#特殊二叉树#正则二叉树（严格二叉树）|严格二叉树]]
### 哈夫曼编码
可利用二叉树设计二进制[[编码#前缀编码|前缀编码]]，叶结点对应字符，并约定左分支标记为 0，右分支标记为 1；将根到各叶结点路径上的分支标记拼接成的字符串作为对应字符编码
利用哈夫曼树可以设计出总长度最短的二进制前缀编码
# 二叉树算法
## 最近公共祖先
 ```c
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
	if (root == nullptr || root == p || root == q) {
		return root; // 找到 p 或 q 就不往下递归了
	}
	TreeNode* left = lowestCommonAncestor(root->left, p, q);
	TreeNode* right = lowestCommonAncestor(root->right, p, q);
	if (left && right) { // 左右都找到
		return root; // 当前节点是最近公共祖先
	}
	// 如果只有左子树找到，就返回左子树的返回值
	// 如果只有右子树找到，就返回右子树的返回值
	// 如果左右子树都没有找到，就返回 nullptr（注意此时 right = nullptr）
	return left ? left : right;
}
```
