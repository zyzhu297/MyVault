# I/O 接口的功能

**I/O 接口** 是主机与外部设备之间的桥梁，位于系统总线和外设之间。由于主机（CPU/内存）与外设之间存在巨大差异，接口需完成以下核心功能：

- **地址译码和设备选择**：接口根据地址线上的设备码/端口号进行译码，判断 CPU 是否正在访问本接口所连接的外设，选通指定设备并建立通信连接
- **通信联络与时序协调**：CPU 与外设工作速度差异巨大，接口需通过握手信号（如 READY、BUSY、DONE、STB 等）协调双方时序，确保数据在正确的时刻被发送和接收
- **数据缓冲**：CPU 为 ns 级，外设常为 ms 级；接口通过数据缓冲寄存器（DBR）暂存数据，实现速度匹配，避免因速度不匹配导致数据丢失
- **信号格式转换**：CPU 使用并行、TTL 电平信号，外设可能使用串行、不同电平标准甚至模拟信号。接口需完成电平转换（TTL ↔ RS-232）、格式转换（串 ↔ 并）、数/模或模/数转换（A/D、D/A）等
- **控制命令与状态信息传递**：CPU 通过接口向设备发送控制命令（读/写、启/停等）；接口通过状态寄存器记录设备当前状态（忙/闲、就绪、出错等），供 CPU 查询或通过中断上报

# I/O 接口的基本结构

| 组成部分               | 功能                                              |
| ------------------ | ----------------------------------------------- |
| **数据缓冲寄存器 (DBR)**  | 暂存 CPU 与外设之间传送的数据，实现速度缓冲                        |
| **控制寄存器 (CR)**     | 存放 CPU 发来的控制命令（如读/写、启动/停止），控制设备操作，**仅被 CPU 写入** |
| **状态寄存器 (SR/PSW)** | 记录外设当前状态（忙/闲、就绪、出错等），**仅被 CPU 读取**              |
| **设备选择电路（地址译码器）**  | 对地址线上的端口地址进行译码，选中对应接口                           |
| **控制逻辑电路**         | 产生接口内部所需的控制信号，协调各部分工作                           |
| **中断控制逻辑**         | 实现中断请求、中断屏蔽、中断优先级判别、中断向量生成等功能                   |
| **数据线/地址线/控制线驱动**  | 总线接口电路，负责信号驱动和接收，匹配总线电气特性                       |
控制寄存器和状态寄存器的访问方向相反且访问时间错开，因此在某些设计中可**复用同一端口地址**：读操作访问状态，写操作访问控制状态；对上述寄存器的访问通过专用指令完成，这类指令称为 **I/O 指令**，通常为**特权指令**，仅限操作系统内核使用，也是及机器指令的一种
- **数据线**传送的是读/写数据、状态信息、控制信息以及中断类型号（在向量中断中）
- **地址线**指定要访问的 I/O 接口内部寄存器的端口地址
- **控制线**传送的是读/写控制信号（用于区分寄存器的访问方向），以及中断请求、总线仲裁信号和设备握手信号

>[!note] I/O 控制逻辑的功能
>I/O 控制逻辑是接口的核心控制部件，根据 CPU 通过控制总线发来的命令信号（如 `IOR` / `IOW` / `MEMR` / `MEMW`），结合地址译码器的选通结果，产生接口内部所需的全部控制信号，包括：
> - 对控制寄存器中的命令字进行译码，并将生成的控制信号经外设界面控制逻辑送至外设
> - 输出时，将数据缓冲寄存器的内容发送给外设；输入时，将外设数据写入数据缓冲寄存器
> - 实时采集外设状态，并更新至状态寄存器

> **CPU 视角**：接口中的每个寄存器都被赋予一个 **端口地址（Port Address）**。CPU 通过端口地址访问对应的寄存器，实现与外设的数据交换和状态查询


# I/O 接口的类型

## 按数据传送方式分类

| 类型 | 数据线 | 特点 | 典型接口/应用 |
|------|--------|------|-------------|
| **并行接口** | 多根数据线，一次传送一个字节/字 | 速度快、距离短，线缆成本高 | Intel 8255（并行接口芯片）、打印机并口（IEEE 1284） |
| **串行接口** | 一根数据线，逐位传送 | 速度较慢、距离远，线缆成本低 | Intel 8251（串行接口芯片）、USB、RS-232C |

## 按功能灵活性分类

| 类型 | 特点 | 示例 |
|------|------|------|
| **可编程接口** | 工作方式和功能可通过编程设定，灵活通用 | 8255（可编程并行接口）、8251（可编程串行接口）、8259（中断控制器） |
| **不可编程接口** | 功能固定，硬件连线决定，成本低 | 简单的锁存器、缓冲器构成的接口 |

## 按通用性分类

| 类型 | 特点 | 示例 |
|------|------|------|
| **通用接口** | 可连接多种不同类型外设 | USB、PCIe、Thunderbolt |
| **专用接口** | 专为某一类设备设计 | 显卡接口（HDMI/DP）、硬盘接口（SATA/SAS）、键盘/鼠标接口（PS/2） |

## 按传送控制方式分类

| 类型         | 说明                              |
| ---------- | ------------------------------- |
| **程序查询接口** | CPU 不断轮询状态寄存器，条件满足时传送数据         |
| **中断接口**   | 外设准备好时主动向 CPU 发中断请求，CPU 响应后传送数据 |
| **DMA 接口** | 专用的 DMA 控制器接管总线，直接在内存和外设间传送数据   |
| **通道接口**   | 使用专用的 I/O 处理器（通道），可独立执行通道程序     |

# I/O 端口及其编址

## I/O 端口

**I/O 端口** 是指 I/O 接口中可由 CPU 直接读/写的寄存器。每个端口被赋予一个地址，称为 **端口地址**。

| 端口类型     | 简称  | 功能                                 |
| -------- | --- | ---------------------------------- |
| **数据端口** | DBR | 暂存 CPU 与外设交换的数据，支持读写               |
| **控制端口** | CR  | 存放 CPU 对外设的控制命令，仅支持写               |
| **状态端口** | SR  | 记录外设的当前状态（READY、BUSY、ERROR 等），仅支持读 |

> 多个端口可复用同一端口地址（如数据输入寄存器和数据输出寄存器分时使用同一地址），具体由读/写信号区分。

## 端口编址方式

| 项目 | 统一编址（存储器映射 I/O） | 独立编址（I/O 映射 I/O） |
|------|--------------------------|--------------------------|
| **地址空间** | I/O 端口和主存共享同一地址空间 | I/O 端口有独立地址空间 |
| **访问指令** | 用访存指令访问 I/O（`LOAD`/`STORE`） | 用专用 I/O 指令（`IN`/`OUT`） |
| **地址线** | 充分利用地址线，全部参与译码 | 通常只用部分低位地址线（I/O 端口数远小于内存） |
| **优点** | 指令丰富，编程灵活；不需额外控制信号区分访存/访 I/O | 不占用主存空间；译码电路简单；I/O 指令短，执行快 |
| **缺点** | 占用主存地址空间；译码电路复杂；访 I/O 指令较长 | I/O 指令功能有限；编程灵活性差 |
| **控制信号** | 只需 `MEMR`/`MEMW` | 需额外 `IOR`/`IOW` 控制信号 |
| **典型代表** | ARM、MIPS 等 RISC 架构 | x86（Intel 8086 系列） |

## 端口地址译码

接口中的设备选择电路根据地址线和控制信号译码，判断 CPU 是否正在访问本接口：

- **统一编址**：通过地址总线高位地址参与片选，`MEMR`/`MEMW` 配合使能
- **独立编址**：通过 `IOR`/`IOW` 信号配合地址译码

## 两种编址方式的典型考题辨析

| 辨析点 | 统一编址 | 独立编址 |
|--------|---------|---------|
| 是否需要专门的 I/O 指令 | ✗ 不需要 | ✓ 需要 |
| 访问 I/O 端口是否可用全部访存指令 | ✓ 可以 | ✗ 只能使用少数 I/O 指令 |
| 译码时是否需要 I/O 读/写控制线 | ✗（用 MEMR/MEMW） | ✓（用 IOR/IOW） |
| 外设地址是否占用主存地址 | ✓ 占用 | ✗ 不占用 |
