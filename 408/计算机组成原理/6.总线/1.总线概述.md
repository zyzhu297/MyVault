# 总线概述

## 总线的基本概念

- **总线 (Bus)**：一组能为多个部件**分时共享**的公共信息传送线路。
- **分时**：同一时刻只允许一个部件向总线发送信息（发送端需分时）。
- **共享**：总线上的所有部件都可以接收总线上的信息（接收端可共享）。
- 总线是连接计算机各部件（CPU、主存、I/O 设备）的**信息传输干线**。

# 总线的分类

## 按连接部件分类

| 分类         | 连接对象                          | 说明                                    |
| ---------- | ----------------------------- | ------------------------------------- |
| **内部总线**   | CPU 芯片内部各部件                   | ALU、寄存器、控制器之间，在**芯片内部**               |
| **系统总线**   | 计算机系统内各功能部件                   | CPU、主存、I/O 接口之间，即通常说的"总线"             |
| **I/O 总线** | 主机与**内部**各类 I/O 控制器           | 连接各类外设（键盘、显示器、磁盘等），通常通过 I/O 接口与系统总线相连 |
| **通信总线**   | 主机与**外部** I/O  设备之间或不同计算机系统之间 | 也称外部总线，用于系统间通信                        |

## 按传输内容分类（系统总线的进一步划分）

| 分类            | 传输内容            | 方向                    | 特点                                 |
| ------------- | --------------- | --------------------- | ---------------------------------- |
| **数据总线 (DB)** | 数据信息            | 双向，三态                 | 宽度决定 CPU 字长 / 系统吞吐量                |
| **地址总线 (AB)** | 主存单元 / I/O 端口地址 | 单向（CPU → 主存/I/O）      | 宽度决定可寻址范围（如 32 根 → $2^{32}$ = 4GB） |
| **控制总线 (CB)** | 控制信号和时序信号       | 每根线方向固定，由若干单向或双向信号线组成 | 包括读/写、中断请求/响应、时钟、复位等               |

>[!note] 三态门
>总线上的器件通过三态门与总线连接。三态：高电平("1")、低电平("0")、**高阻态**（相当于断开）

>[!note] 总线上的信息传输方向
>
>| 类型 | 方向 | 承载总线 | 说明 |
>|------|------|---------|------|
>| **数据** | 双向 | 数据总线 | CPU ↔ 主存、CPU ↔ I/O 之间传输的数据信息 |
>| **地址** | 单向（CPU 发出） | 地址总线 | CPU 访问主存单元或 I/O 端口的地址编码 |
>| **控制** | 单向（CPU 发出） | 控制总线 | 读/写命令、中断响应、总线请求/允许等 |
>| **状态** | 单向（发往 CPU） | 控制总线 | 设备就绪、中断请求、总线请求等反馈信号 |
>| **时序** | — | 控制总线 | 时钟脉冲、同步信号，为总线操作提供时间基准 |
>
>> [!tip] 记忆要点
>> 地址总是 CPU 发出（CPU 知道要去哪）；状态总是反馈给 CPU（CPU 需要知道外设怎么样了）。**控制总线上的信号方向是零散的——有的 CPU 输出，有的 CPU 输入，不能笼统说是单向的。**
## 按传输方式分类

| 分类       | 说明                                |
| -------- | --------------------------------- |
| **并行总线** | 多条数据线同时传输多位数据，速度快但成本高、同步困难、易受干扰   |
| **串行总线** | 一条数据线逐位传输，速度较慢(早期)但成本低、抗干扰强、适合远距离 |

## 按时序控制方式分类

| 分类        | 说明                   | 例子                |
| --------- | -------------------- | ----------------- |
| **同步总线**  | 统一时钟控制，所有部件按同一时钟节拍工作 | PCI（并行型）、SDRAM 总线 |
| **异步总线**  | 无统一时钟，采用应答（握手）信号协调   | RS-232            |
| **半同步总线** | 同步基础上允许插入等待周期        | 大多数现代总线           |
| **分离式总线** | 将总线事务分为请求和应答两个独立操作   | AXI 总线（部分特性）      |

# 常见的总线标准

## 常见总线标准一览

| 标准             | 全称                                       | 中文名          | 类别   | 传输     | 功能特点                                                   |
| -------------- | ---------------------------------------- | ------------ | ---- | ------ | ------------------------------------------------------ |
| **ISA**        | Industry Standard Architecture           | 工业标准体系结构     | 系统总线 | 并行     | 最早的 PC 扩展总线，支持即插即用（后期 ISA PnP），已淘汰                     |
| **EISA**       | Extended ISA                             | 扩展工业标准体系结构   | 系统总线 | 并行     | ISA 的 32 位扩展版，**向下兼容** ISA 插槽和板卡，支持多主控器和突发传输，已淘汰       |
| **VESA** (VLB) | Video Electronics Standards Association  | 视频电子标准协会     | 局部总线 | 并行     | 直接挂 CPU 局部总线，专为显卡高速传输设计，支持突发传送，已被 AGP 取代               |
| **PCI**        | Peripheral Component Interconnect        | 外设部件互连标准     | 局部总线 | 并行     | 即插即用，与 CPU 无关（通过 PCI 桥隔离），支持总线仲裁和自动配置，支持猝发传送           |
| **AGP**        | Accelerated Graphics Port                | 加速图形端口       | 局部总线 | 并行     | 显卡专用高速端口，支持直接内存访问（DIME），×1/×2/×4/×8 模式，已被 PCIe ×16 取代  |
| **PCIe**       | PCI Express                              | 高速外设部件互连     | 局部总线 | **串行** | 点到点**全双工**，可灵活组合 Lane（×1/×4/×8/×16），支持热插拔、QoS 和高级错误报告  |
| **RS-232C**    | Recommended Standard 232C                | 推荐标准 232C    | 设备总线 | 串行     | 异步串行通信，全双工，DTE 与 DCE 之间串行二进制通信的标准接口                    |
| **USB**        | Universal Serial Bus                     | 通用串行总线       | 设备总线 | 串行     | 热插拔、即插即用、总线供电，支持菊花链拓扑（最多 127 设备），支持多种传输类型（控制/批量/中断/同步） |
| **PCMCIA**     | PC Memory Card International Association | 个人计算机存储卡国际协会 | 设备总线 | 并行     | 笔记本扩展卡标准，支持热插拔和即插即用，已淘汰                                |
| **IDE** (ATA)  | Integrated Drive Electronics             | 集成驱动电子设备     | 设备总线 | 并行     | 连接主板与硬盘/光驱的传统并行接口，已淘汰                                  |
| **SATA**       | Serial ATA                               | 串行高级技术附件     | 设备总线 | 串行     | 串行点对点连接，7 针数据线，支持热插拔和 NCQ（原生命令队列）                      |
| **SCSI**       | Small Computer System Interface          | 小型计算机系统接口    | 设备总线 | 并行     | 可挂多个设备（7~15 个），支持总线主控和断开/重连，支持命令排队，服务器/工作站用，现代已转向 SAS  |

# 总线的性能指标

1. **总线传输周期（总线周期）**
   - 完成一次完整总线操作（如一次读/写）所需的时钟周期数或时间。

2. **总线时钟频率**
   - 总线控制器的基准时钟频率，即每秒的时钟周期数，单位：**Hz / MHz / GHz**。
   - 时钟周期 $T$ 与时钟频率 $f$ 的关系：$f = 1/T$
   - 现在总线的时钟通常独立于 CPU 时钟

3. **总线工作频率**
   - 总线上实际数据传输的频率，单位：**Hz / MHz**。
   - 在**同步总线**中，工作频率通常等于时钟频率或为其分频；在**双倍数据速率 (DDR)** 总线中，时钟上、下沿各传一次，工作频率 = 2× 时钟频率（如 DDR SDRAM）。

4. **总线宽度**
   - 数据总线的根数，即**一次能并行传输的数据位数**，单位：**位 (bit)**。

1. **总线带宽**
   - 也称总线数据传输率，即**单位时间内总线能传输的最大数据量**。
   - 公式（一次传输占一个时钟周期）：
     $$\text{总线带宽} = \text{总线宽度} \times \text{总线工作频率}$$
   - 若需多个时钟周期完成一次传输，则：
     $$\text{带宽} = \frac{\text{总线宽度} \times \text{总线工作频率}}{\text{传输所需时钟周期数}}$$
   - 单位：**B/s, MB/s, GB/s**
>[!warning] 计算带宽（最大数据传输率）
>应依据总线的物理参数（宽度、时钟频率、每周期传输的次数），反映其**理论峰值传输能力**，无需考虑具体总线事务中的地址/命令开销或突发传输细节；仅在求平均数据传输率时，才需要分析事物时序

6. **总线复用**
   - **多路复用**：地址线与数据线共用同一组物理线路（如 PCI 先传地址再传数据，USB 以包为单位交替传输），节省引脚数但需分时使用，增加传输延迟。
   - **独立线**：地址线和数据线各自独立，并行性好但引脚数多。

7. **总线寻址能力**
   - 由**地址总线宽度**决定：$n$ 根地址线 → $2^n$ 个地址单元。
   - 以字节编址为例：32 根 → $2^{32}$ = 4GB；36 根 → $2^{36}$ = 64GB。
   - 寻址能力与数据总线宽度无关，两者是独立参数。

# 总线的结构

## 早期共享总线结构

所有部件挂接在同一组系统总线上，分时共享。

### 单总线结构

- **特点**：CPU、主存、I/O 全部通过同一组总线通信。
- **优点**：结构简单，成本低。
- **缺点**：带宽瓶颈严重——高速的 CPU-主存传输与低速 I/O 争用同一总线。

### 双总线结构

- **特点**：CPU-主存之间有独立的**存储总线**，I/O 设备通过**I/O 总线**经 I/O 通道管理与主存交换数据。
- **优点**：缓解了 CPU-主存与 I/O 的带宽冲突。
- **缺点**：**I/O 与主存的数据交换仍需经 CPU 控制**，CPU 负担仍较重。

### 三总线结构

- **组成**：**存储总线** + **I/O 总线** + **DMA 总线**。
- **核心改进**：DMA 总线使高速外设可直接与主存交换数据，无需 CPU 干预，进一步释放 CPU

## 传统分层总线：南北桥结构

早期共享总线无法满足日益增长的带宽需求，于是将高速设备与低速设备分离，形成了**南北桥分层的总线架构**

| 芯片 | 全称 | 功能 |
|------|------|------|
| **北桥 (MCH)** | Memory Controller Hub | 连接 CPU、主存、显卡等高速设备，速度要求高 |
| **南桥 (ICH)** | I/O Controller Hub | 连接 USB、SATA、PCI、网卡、声卡等低速外设 |

- **设计思想**：将速度差异大的设备分层接入不同桥片，高速归北桥、低速归南桥，避免互相拖慢。
- **局限性**：北桥成为 CPU 与主存之间的瓶颈（前端总线 FSB 带宽受限）；南北桥之间互连总线也是瓶颈。

## 现代集成化总线结构

为消除北桥瓶颈，将内存控制器和高速 I/O 控制器**直接集成进 CPU**：

- **关键变化**：
  - 内存控制器 (IMC) 集成进 CPU → CPU 直接访问主存，消除 FSB 瓶颈，延迟大幅降低。
  - PCIe 控制器集成进 CPU → 显卡和高速 SSD 直连 CPU，不再经北桥中转。
  - 北桥消失，南桥简化为 **PCH（Platform Controller Hub）**，经 DMI/UPI 与 CPU 相连。

- **三大优势**：

  1. **内存访问路径缩短**：CPU 通过片内 IMC 直接访问主存，不再经北桥中转。传统南北桥架构中，CPU → 前端总线 → 北桥 → 内存的访存路径被简化为 CPU → 内存，延迟大幅降低，带宽利用率更高。
  2. **处理器间高速互联**：多路 CPU 之间通过 **QPI**（Intel）或 **HyperTransport**（AMD）等高速点对点互连总线直接通信，无需经过芯片组中转，支持多处理器系统的低延迟数据共享和缓存一致性协议。
  3. **I/O 通路直连**：PCIe 控制器集成进 CPU 后，显卡、NVMe SSD 等高速外设直接使用 CPU 提供的 PCIe Lane，不再经北桥或南桥中转。传统路径（设备 → 南桥 → 北桥 → CPU）被简化为（设备 → CPU），高速 I/O 延迟显著降低，吞吐量大幅提升。
