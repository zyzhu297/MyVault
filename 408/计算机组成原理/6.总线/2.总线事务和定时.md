# 总线事务

## 总线事务的基本概念

- **总线事务 (Bus Transaction)**：*主设备*与*从设备*之间完成一次完整的信息交换过程
- 总线事务是总线上**最基本的操作单位**，典型的总线事务有：**读事务**、**写事务**、**块传送事务**等。

>[!note] 主设备 vs 从设备
>- **主设备 (Master)**：获得总线控制权并发起总线事务的设备（如 CPU、DMA 控制器）。
>- **从设备 (Slave)**：被主设备访问、响应总线事务的设备（如主存、I/O 接口）。
>- 一次总线事务中，**主设备只有一个**，从设备可以有一个或多个。

## 总线事务的三个基本阶段

| 阶段 | 名称 | 说明 |
|------|------|------|
| ① | **地址传送阶段** | 获得总线权的主设备通过地址总线发出从设备地址和读/写命令 |
| ② | **从设备响应阶段** | 从设备收到地址和命令后，进行译码和准备（选中对应单元、准备数据或等待接收） |
| ③ | **数据传送阶段** | 主设备与从设备之间通过数据总线完成数据交换（读或写） |

>[!note] 总线事务的本质
>一次总线事务的核心是**寻址 → 响应 → 传输**。总线仲裁（请求和释放）属于总线使用权管理，严格来说不属于一次事务本身，而是为事务的执行做准备和环境清理。

> [!note] 数据传送方向
> - 读事务：主设备发送地址和读命令，从设备将被选中单元的数据放到数据总线上，主设备从数据总线上采样数据。
> - 写事务：主设备发送地址和写命令，从设备选中单元，主设备将数据放到数据总线上，从设备从数据总线上采样数据接受数据。

## 非突发传输和突发传输

### 非突发传输 (Non-Burst Transfer)

- 每次总线事务只传送**一个数据单元**（一个字节/一个字），地址传送阶段和数据传送阶段**交错进行**。
- **流程**：发地址 → 传一个数据 → 发下一个地址 → 传下一个数据 → ……
- 每传送一个数据都需要一次完整的寻址过程，**地址开销大**

### 突发传输 (Burst Transfer)

- 在一次总线事务中，只给出**首地址**，随后连续传送**多个数据单元**（一个数据块）。
- **流程**：发首地址 → 传数据₁ → 传数据₂ → 传数据₃ → ……（地址由硬件自动递增）
- **优势**：只需一次地址传送阶段，后续数据按约定节拍连续传输，地址开销被分摊，**总线利用率高**。

>[!important] 突发传输的核心特征
>只需一个地址段，后续是连续的数据段。地址开销只发生一次，因此**传送的数据块越长，平均传输效率越高**。

### 对比总结

| 特性 | 非突发传输 | 突发传输 |
|------|-----------|---------|
| 每次传送数据量 | 一个单元 | 多个单元（一个块） |
| 地址阶段次数 | 每个数据一次 | 仅首地址一次 |
| 总线利用率 | 低 | 高 |
| 典型应用 | 简单 I/O 操作 | Cache 行填充、DMA 块传送、PCIe 事务 |

>[!note] 突发传输 ≠ 分离式通信
>突发传输解决的是**一次传输中地址开销大的问题**（连续传数据），分离式通信解决的是**从设备准备数据期间总线空等的问题**（释放总线）。两者可以结合使用。

## 串行传输和并行传输

### 串行传输 (Serial Transmission)

- 数据在**一条**数据线上**逐位 (bit by bit)** 依次传送。
- 传输方向可以是单工、半双工或全双工。

串行传输按**时钟同步方式**分为同步串行和异步串行：

**优点**：
- 信号线少，成本低，适合远距离传输
- 无并行线间串扰问题，信号完整性好，适合高速传输
- 可采用差分信号（如 LVDS），抗干扰能力强

**缺点**：
- 单条线速率受限时，总带宽受限（需通过提高频率或多 Lane 绑定弥补）

**典型应用**：USB、SATA、PCIe、RS-232、以太网
#### 同步串行

- **时钟**：由发送方时钟控制接收方时钟
- **工作机制**：收发双方按同一时钟节拍工作，数据**连续不间断**传送，效率高。
- **典型应用**：SPI、I²C、PCIe、USB

**优点**：
- 数据连续传输，无起始/停止位开销，有效带宽高
- 时钟由发送方提供，接收方无需高精度本地时钟
- 适合高速、大块数据传输

**缺点**：
- 至少需要一根时钟线（或编码/解码电路），**硬件稍复杂**
- 时钟信号在高速时同样面临偏斜和抖动问题

#### 异步串行

- **时钟**：无共用时钟线，收发双方**各自使用本地时钟**。
- **工作机制**：双方事先约定**波特率**，数据以**字符帧**为单位传送，每帧由起始位 + 5~8 数据位 + **可选**校验位 + 停止位组成，帧之间可插入空闲位
- **每帧开销**：⭐**至少 2 个额外位（1 起始位 + 1 停止位）**
- **典型应用**：RS-232（UART）

**优点**：
- 无时钟线，硬件实现极简（只需 TX/RX 两根线）
- 帧间可有任意长度空闲，适合间歇性、不定时通信
- 成本极低，几乎所有 MCU 内置 UART 外设

**缺点**：
- 每帧有起始/停止位开销，有效带宽低于同波特率的同步串行
- 波特率必须事先约定，两端时钟漂移会导致采样错误
- 不适合极高速度（通常 ≤ 几 Mbps），无法连续大块传输


### 并行传输 (Parallel Transmission)

- 数据在**多条**数据线上**同时（同一时钟沿）** 传送多位，只适合短距离通信

**优点**：
- 一个时钟周期传送多位，同等频率下带宽更高

**缺点**：
- 信号线多，成本高，占用引脚多
- 线间串扰和信号偏斜（Skew）问题严重，频率难以提升
- 各条线到达时间不一致导致采样错误，限制了工作频率

**典型应用**：早期 IDE (PATA)、PCI、SCSI（并口版本）、DDR 内存总线（芯片级）

### 对比总结

| 特性 | 串行传输 | 并行传输 |
|------|---------|---------|
| 数据通路 | 1 条（或少量 Lane） | 多条（如 8/16/32/64 条） |
| 传输方式 | 逐位串行 | 同时并行 |
| 成本 | 低（引脚少） | 高（引脚多） |
| 远距离 | 适合 | 不适合（串扰和偏斜） |
| 高频能力 | **强**（差分信号，无偏斜） | 弱（偏斜和串扰限制频率） |
| 现代趋势 | **主流**（PCIe, USB, SATA） | 芯片内部和存储器总线仍用并行 |

>[!tip] 现代总线为什么"串行化"
>传统并行总线通过加宽数据线来提升带宽，但频率提升时并行线间的信号偏斜成为瓶颈。现代高速总线（如 PCIe、USB 3.x）采用**多条独立串行 Lane 绑定**（×1/×4/×8/×16）替代传统并行传输——每个 Lane 是一条高速串行通道，互不干扰，既获得了串行的高频优势，又通过多 Lane 绑定了总带宽。

# 总线定时

总线定时是指总线操作中各信号在时间上的配合关系，决定总线上各部件何时发送信号、何时采样信号。

## 同步定时 (Synchronous Timing)

### 基本概念

- 所有总线操作由**统一的时钟信号 (Clock)** 控制。
- 所有部件在**固定的时钟沿**（上升沿/下降沿）上发送或采样信号。
- 总线周期长度固定，每个操作占确定数量的时钟周期。

### 优缺点

| 优点              | 缺点                                |
| --------------- | --------------------------------- |
| 控制简单，易于实现和测试    | **所有部件必须按最慢设备设计时钟周期**，快设备被慢设备拖慢   |
| 速度快（信号不需握手来回等待） | 缺乏应答或握手机制，无法根据从设备实际状态动态调整时序，可靠性较差 |
| 时序关系确定，可靠性高     | 时钟偏斜（Clock Skew）问题随频率升高而严重        |

>[!note] 同步总线为何"按最慢设计"
>同步总线上所有操作必须在统一时钟周期内完成。如果某个慢速设备在一个时钟周期内无法完成操作，会导致采样到无效数据。为避免此问题，时钟周期必须 ≥ 所有设备中**最长的传输延迟**。

## 异步定时 (Asynchronous Timing)

### 基本概念

- **无统一时钟**，总线操作由**应答信号（握手信号）** 来协调，一次握手可传递多位数据
- 通信双方通过请求/应答的**互锁机制**来保证数据可靠传输。
- 总线周期长度**不固定**，取决于实际交互速度。
- **使用场景**：适用于连接速度差异大、对可靠性要求较高而对速率要求不苛刻的系统

### 握手方式

#### 不互锁方式

- 主设备发出请求信号（维持一段时间），时间到后**自行撤销**，不等待从设备任何应答。
- 从设备是否收到、是否正确执行，主设备**完全不管**。
- **特点**：主设备只管发，从设备自己处理，无任何反馈。可靠性最差，一般仅用于对可靠性要求极低的场合。

>[!warning] 不互锁 vs 无握手
>不互锁是异步定时的极端简化——虽有信号交换但无确认机制。严格来说这不算真正的"握手"
#### 非全互锁方式（半互锁/部分互锁）

- 主设备发出请求后，等待从设备的应答信号到来，但**不判断应答是否正确**。
- 即：主设备等到应答就把请求撤销，从设备等到请求撤销就把应答撤销。
- 只有**主到从、从到主各一次握手信号**，没有二次确认。

>[!note] 非全互锁 vs 全互锁
>全互锁是四步闭环（请求 → 应答 → 撤销请求 → 撤销应答），而非全互锁只有两步（请求 → 应答），双方各自在收到对方信号后自行撤销己方信号，不再等待对方二次确认。

#### 全互锁方式

所有信号变化均需对方的应答确认，形成完整的握手链：

**握手信号对**（以读操作为例）：

| 步骤 | 信号方向 | 信号 | 含义 |
|------|---------|------|------|
| 1 | 主 → 从 | 地址 + 读请求 | 主设备请求读数据 |
| 2 | 从 → 主 | 数据 + 就绪 (Ready/Ack) | 从设备已将数据放上总线 |
| 3 | 主 → 从 | 撤销读请求 | 主设备已读走数据 |
| 4 | 从 → 主 | 撤销就绪 | 从设备确认释放总线 |

>[!tip] 全互锁的本质
>每一步信号的改变都依赖于对方的响应信号。**请求 → 应答 → 撤销请求 → 撤销应答**，形成闭环。这种方式**最可靠**，不会因为速度不匹配导致数据丢失。

### 优缺点

| 优点                         | 缺点                   |
| -------------------------- | -------------------- |
| **自适应快慢设备**：周期长度根据设备速度动态调整 | 控制复杂，需额外握手信号线        |
| 总线上可同时挂接速度差异大的设备           | 每次传输需多次握手来回，**延迟较大** |
|                            | 实现成本高                |

## 半同步定时 (Semi-Synchronous Timing)

### 基本概念

- 在**同步定时**的基础上，允许从设备通过"等待信号"插入**等待周期 ($T_w$)**。
- 统一时钟仍然存在，但总线周期长度可变为不固定。
- 地址、命令、数据的发送严格参照系统时钟前沿；接收方通常在时钟后沿进行识别；同时增设一条 Wait 信号线，允许慢速从设备反馈准备状态；主设备在时钟上升沿检测 Wait 信号状态；若 Wait 无效（高电平），表示数据未就绪，主设备插入等待周期；直到 Wait 有效（低电平），才从数据先读取数据
- 是同步与异步的折中方案。
- **使用场景**：包含多种速度差异较大的设备、但性能要求不高的简单系统

### 优缺点

| 优点              | 缺点                   |
| --------------- | -------------------- |
| 兼具同步的简单性和异步的灵活性 | 比纯同步复杂               |
| 在同一时钟下工作，可靠性高   | 仍有统一的时钟信号，时钟偏斜问题存在   |
| **现代总线最常用的方式**  | 系统时钟周期受限于最慢设备，整体速度不高 |

## 分离式通信 (Split Transaction)

### 基本概念

- 将一个完整的总线事务**拆分为两个独立的子事务**：**请求事务**和**应答事务**。
- 在请求和应答之间，**总线被释放**，可供其他设备使用。
- 也称为**分裂事务 (Split Transaction)**
- **适用场景**：从设备响应慢、总线负载高的系统（如磁盘 I/O、远程 DMA），以及需要高效复用总线带宽的多主设备系统（如服务器、SoC 总线）

### 工作流程

以读操作为例，一次分离式通信分为两个独立的总线占用：

**请求阶段（主设备 → 从设备）**：

① 主设备申请并获得总线控制权
② 主设备通过总线发出**地址**和**读命令**
③ 从设备**锁存**地址和命令（记录下"谁要读哪里"）
④ 主设备**立即释放总线**（关键！不再空等）

**应答阶段（从设备 → 主设备）**：

⑤ 从设备在后台**准备数据**（如读取磁盘、DRAM 刷新等）
⑥ 数据就绪后，从设备申请并获得总线控制权（**反客为主**）
⑦ 从设备将**数据 + 原主设备标识**放上总线
⑧ 原主设备接收数据，完成事务，释放总线

>[!tip] 分离式通信与三个阶段的关系
>分离式通信的**每个子事务**仍然遵循"地址传送 → 从设备响应 → 数据传送"三个基本阶段。请求子事务完成地址传送；应答子事务完成数据传送。两者的从设备响应阶段被放到了总线释放后的后台等待期中。

>[!important] 分离式通信的核心价值
>传统总线事务中，从设备准备数据期间**总线被空占**（"死等"），造成带宽浪费。分离式通信使总线在等待期间被释放给其他设备使用，**大幅提高总线利用率**。

### 与其他定时方式的对比

| 特性 | 同步/异步/半同步 | 分离式通信 |
|------|---------------|------------|
| 事务完整性 | 一次占用连续完成 | 拆分为两次独立占用 |
| 等待期间总线状态 | **被占用（空闲等待）** | **释放给其他设备** |
| 总线利用率 | 较低（有空闲等待） | 高 |
| 实现复杂度 | 简单 | 复杂（需总线仲裁支持从设备主动发起传输） |
| 典型应用 | 传统系统总线 | AXI 总线、PCIe 的非转发式事务 |

>[!note] 408 考试要点
>- 分离式通信的关键特征：**一次事务拆为两次总线占用**。
>- 从设备在应答阶段需要"反客为主"取得总线控制权，因此要求支持分离式通信的总线具备**多主设备仲裁能力**。
>- 常与猝发传输结合考察总线带宽/利用率计算。

# 四种定时方式总结对比

| 定时方式 | 时钟 | 周期 | 灵活性 | 复杂度 | 总线利用率 | 典型应用 |
|---------|------|------|--------|--------|-----------|---------|
| **同步** | 统一时钟 | 固定 | 低（按最慢设计） | 低 | 一般 | 早期系统总线、SDRAM |
| **异步** | 无统一时钟 | 可变 | 高 | 高 | 一般（握手开销大） | RS-232 |
| **半同步** | 统一时钟+等待 | 可变 | 中 | 中 | 较高 | 现代大多数总线 |
| **分离式** | 可结合以上任一种 | 事务拆分 | 高 | 高 | **最高** | AXI、PCIe |
