# SISD、SIMD、MISD、MIMD 的基本概念

基于指令流和数据流的**弗林分类法 (Flynn's Taxonomy)**，将计算机体系结构分为四类。

| 分类   | 指令流 | 数据流 | 典型代表       |
| ---- | --- | --- | ---------- |
| SISD | 1   | 1   | 单核 CPU     |
| SIMD | 1   | 多   | 向量处理器、GPU  |
| MISD | 多   | 1   | 脉动阵列(少见)   |
| MIMD | 多   | 多   | 多核处理器、多计算机 |

## SISD (单指令流单数据流)

**Single Instruction stream, Single Data stream**

- **定义**：传统串行计算机结构。每个时钟周期只有一条指令在执行，操作一份数据。
- **特点**：
  - 单一控制单元，单一处理单元
  - 指令顺序执行 (即使采用指令流水线，逻辑上仍是串行)
  - 可采用**流水线**来提高性能，但从 Flynn 分类角度，流水线只是时间上的重叠，指令流和数据流在逻辑上仍是单一的
- **典型代表**：早期单核 CPU (如 Intel 8086)、冯·诺依曼结构

## SIMD (单指令流多数据流)

**Single Instruction stream, Multiple Data streams**

- **定义**：一条指令同时控制多个处理单元，在不同数据上执行相同操作，**数据级并行**
- **特点**：
  - 一个控制单元，多个执行单元
  - 所有执行单元执行**同一条指令**，但处理**不同的数据**
  - 特别适合**数据级并行** (Data-Level Parallelism)
  - 所有处理单元同步执行，共享同一个控制器
- **典型应用**：
  - **向量处理器**：一条向量指令同时处理多个数据元素
  - **GPU (图形处理器)**：大规模并行处理像素/顶点
  - **多媒体扩展指令集**：MMX、SSE、AVX 等
- **并行性来源**：数据级并行 (同一操作作用于大量数据)

## MISD (多指令流单数据流)

**Multiple Instruction streams, Single Data stream**

- **定义**：多条指令同时操作同一份数据流。这是 Flynn 分类中**最罕见**的类型。
- **几乎没有实际应用**

## MIMD (多指令流多数据流)

**Multiple Instruction streams, Multiple Data streams**

- **定义**：多个处理器在不同数据上执行不同的指令流。这是**最通用**的并行计算模型。
- **特点**：
  - 每个处理器有独立的控制单元和执行单元
  - 各处理器**独立**执行不同程序或同一程序的不同部分
  - 支持**线程级并行** (Thread-Level Parallelism)或**任务集并行**
- **子分类**：
  1. **共享内存多处理器 (Shared Memory Multiprocessor)**：
     - 所有处理器**共享**同一个物理地址空间
     - 通过共享变量进行通信
     - 例：多核 CPU (Intel Core i7, AMD Ryzen)
     - 又分为 **UMA** (统一存储访问) 和 **NUMA** (非统一存储访问)
  2. **消息传递多计算机 (Message-Passing Multicomputer)**：
     - 各处理器有**独立**的内存空间
     - 通过显式消息传递(如 MPI)进行通信
     - 例：大规模集群、分布式系统
- **典型代表**：多核处理器、对称多处理器 (SMP)、集群

# 硬件多线程基本概念

为了减少传统 CPU 中，线程切换设计保存和恢复寄存器上下文等操作的开销，**硬件多线程**技术的处理器为**每个线程配备独立的通用寄存器组、程序计数器（IP）等关键状态部件**；线程切换时，只需激活对应线程的寄存器组，无需将上下文写入或从内存读出，，从而大幅降低切换延迟

## 细粒度多线程 (Fine-Grained Multithreading)

- **别名**：交错多线程 (Interleaved Multithreading)
- **切换方式**：**每个时钟周期**轮流切换线程
- **切换频率**：极高，每个周期可切换
- **代价**：切换开销为零或极小（硬件支持）
- **特点**：
  - 多个线程以**轮转 (Round-Robin)** 方式交替执行，每个周期执行不同线程的指令
  - **能隐藏所有停顿**(包括短期和长期停顿)
  - 但由于每个线程执行速度慢，**单个线程性能较低**
  - 需要多套寄存器组，PC 等，但可在流水级间共享
- **适用场景**：**吞吐率优先**的场景，如服务器

## 粗粒度多线程 (Coarse-Grained Multithreading)

- **别名**：块多线程 (Block Multithreading)
- **切换方式**：仅在当前线程遇到**长时间停顿**（如 L2 Cache Miss）时才切换
- **切换频率**：较低，仅在触发停顿时
- **代价**：切换时有开销，需**排空流水线**或**刷新流水线**
- **特点**：
  - 流水线被一个线程持续占用，直到发生高延迟事件
  - 切换后流水线需要重新填充 → **有切换开销**
  - **不能隐藏短期停顿**（如流水线数据冲突），只能隐藏长停顿
  - 由于不频繁切换，单个线程性能相对较好
- **适用场景**：单线程性能仍重要，但同时希望利用长延迟隐藏的场景

## 同时多线程 (Simultaneous Multithreading, SMT)

- **别名**：同时多线程、超线程 (Hyper-Threading，Intel 的商标名称)
- **基本思想**：在同一时钟周期内，**同时发射**来自多个线程的指令到不同功能单元执行，在一个物理核心中维护两套完整的线程状态部件，而高速缓存、ALU 等执行资源则有两个逻辑核心共享，从而在单核上实现接近双核的并发处理能力
- **核心机制**：
  - 在超标量处理器的基础上实现
  - 多个线程**共享**同一套流水线资源 (功能单元、Cache、TLB 等)
  - 每个线程有独立的寄存器组和 PC
  - 一个时钟周期可以发射来自不同线程的多条指令
- **与细粒度/粗粒度的本质区别**：
  - 细粒度和粗粒度是**时间上的多线程**（分时使用处理器）
  - SMT 是**空间上的多线程**（同时使用处理器不同功能单元）
- **优势**：
  - 充分利用超标量处理器中闲置的功能单元
  - 一个线程在等待时，另一个线程可占用执行单元
  - 对操作系统透明，一个物理核表现为多个逻辑核
- **适用场景**：通用计算，提升多任务性能和处理器利用率

### 三种硬件多线程对比

细粒度和粗粒度多线程均在统一时刻仅执行一个线程的指令，**不支持真正的指令级并行**，其核心是通过线程切换隐藏长延迟操作，提升资源利用率

| 特性 | 细粒度 | 粗粒度 | 同时多线程 |
|------|--------|--------|------------|
| 切换频率 | 每个周期 | 仅长停顿 | 每个周期 |
| 多线程指令发射 | 分时 | 分时 | **同时** |
| 隐藏短停顿 | ✅ | ❌ | ✅ |
| 隐藏长停顿 | ✅ | ✅ | ✅ |
| 切换开销 | 极小 | 较大(刷新流水线) | 极小 |
| 单线程性能 | 较差 | 较好 | 一般 |
| 总吞吐量 | 高 | 中等 | **最高** |
| 硬件复杂度 | 中 | 低 | 高 |

# 多核处理器的基本概念

**多核处理器 (Multi-Core Processor)** 是指在一颗芯片上集成多个处理核心 (Core) 的处理器

## 定义与特征

- **定义**：将两个或更多独立的处理器核心集成在**同一个物理芯片**上的处理器
- **本质**：一种 MIMD 结构的实现形式
- **核心特征**：
  - 每个核心有独立的运算部件、控制部件和 L1 Cache
  - 多个核心通常**共享最后一级 Cache (LLC，即 L3 Cache)** 和内存控制器
  - 多个核心通过**片上互联网络 (On-Chip Interconnect)** 进行通信
  - 必须采用多线程（或多进程）模型，支持**物理上同时进行**
  - 对操作系统呈现为多个可独立调度的逻辑处理器

## 多核与单核的关键区别

| 对比维度 | 单核处理器 | 多核处理器 |
|----------|-----------|------------|
| 并行性来源 | 指令级并行 (ILP) | 线程级并行 (TLP) |
| 性能提升方式 | 提高主频、加深流水线 | 增加核心数量 |
| 功耗效率 | 高主频 → 高功耗 | 多核低频 → 更优能效 |
| 程序要求 | 单线程优化 | 需多线程编程 |

## 为什么需要多核？—— 功耗墙与 ILP 墙

1. **功耗墙 (Power Wall)**：
   - 提升主频导致功耗呈超线性增长（$P \propto f \cdot V^{2}$）
   - 散热能力有限，单核主频无法持续提升
   - **解决方案**：以多个低频核心代替单个高频核心，在相同功耗下获得更高吞吐量

2. **ILP 墙 (ILP Wall)**：
   - 单线程内部的指令级并行性有限（ILP 极限约 3~4）
   - 继续加深流水线、加宽发射宽度带来的收益递减
   - **解决方案**：转向线程级并行，利用多线程/多任务的自然并行性

## 多核的关键技术问题

1. **Cache 一致性 (Cache Coherence)**：
   - 问题：不同核心的私有 Cache 中，同一内存块可能有不同副本
   - 解决：硬件一致性协议（窥探协议、目录协议）
   - 详见：[[../6.总线/总线与Cache一致性]]（待补充）

2. **内存一致性 (Memory Consistency)**：
   - 问题：多个核心看到的访存操作顺序如何保证
   - 涉及乱序执行、写缓冲区等带来的可见性问题

3. **片上互联 (On-Chip Interconnect)**：
   - 总线 (Bus) → 交叉开关 (Crossbar) → 片上网络 (NoC / Network on Chip, 环形/网格拓扑)

# 共享内存多处理器的基本概念

**共享内存多处理器 (Shared Memory Multiprocessor, SMP)** 指多个处理器（或多核处理器中的多个核心）共享同一个物理地址空间的计算机系统。

## 定义与特征

- **定义**：多个处理器通过某种互联方式**共享同一主存**，所有处理器均可通过常规的访存指令访问任意内存位置
- **通信方式**：通过**共享变量**（Load / Store 指令）进行处理器间通信
- **与消息传递系统的区别**：
  - 共享内存：隐式通信，程序员只需读写变量，硬件负责数据传输
  - 消息传递：显式通信，程序员调用 send / recv 等接口

## UMA (统一存储访问)

**Uniform Memory Access**，又称**对称多处理器 (Symmetric Multiprocessor, SMP)**

- **特征**：
  - 所有处理器访问主存的**延迟相同**
  - 所有处理器在地位上**完全平等**（对称）
  - 任何处理器均可处理 I/O 和中断
- **架构**：
  - 多个处理器通过**总线**或**交叉开关**连接到共享主存
  - 每个处理器通常有私有 Cache
- **优点**：
  - 编程模型简单（所有内存对任何处理器都一样近）
  - 负载均衡性好（OS 可将任务分配到任意处理器）
- **缺点**：
  - **可扩展性差**：随着处理器数量增加，总线/互联成为瓶颈
  - 通常支持 2~32 个处理器（受限于互联带宽）
- **典型代表**：早期多路服务器、多核 CPU（核数较少时）

> [!info] 北桥 (North Bridge) 与 SMP 架构
> 
> **北桥**是传统 PC/服务器主板芯片组中的核心部件，在早期 SMP 系统中扮演处理器与内存、高速外设之间的**枢纽**角色。
> 
> **功能**：
> - 集成**内存控制器 (Memory Controller)**，负责 CPU 与主存之间的数据交换
> - 提供与显卡的高速通道（AGP / PCIe ×16）
> - 通过**前端总线 (FSB, Front-Side Bus)** 与所有 CPU 通信
> - 与南桥相连，间接管理低速外设
> 
> **在 SMP 中的作用**：
> - 多个 CPU 共用一个北桥和同一套主存 → 形成 **UMA 结构**
> - 所有 CPU 通过 FSB 访问北桥中的内存控制器，访存延迟一致
> - **瓶颈**：随着 CPU 数量增加，FSB 带宽成为限制（"总线墙"）
> 
> **演进——北桥的消失**：
> - 现代处理器将**内存控制器集成到 CPU 芯片内部**（如 AMD 从 K8 开始，Intel 从 Nehalem 开始）
> - CPU 直接连接内存，不再需要北桥中转 → 访存延迟大幅降低
> - 多路系统中，各 CPU 通过高速点对点互联（QPI/UPI/Infinity Fabric）直连 → 自然演化为 **NUMA 结构**
> - 北桥功能被拆分：内存控制器进 CPU、PCIe 控制器进 CPU（部分），剩余少量功能并入南桥（升级为 PCH / FCH）
> 
> **408 考点关联**：北桥与 SMP/UMA 的对应关系、FSB 的瓶颈效应、内存控制器集成进 CPU 是 NUMA 架构的前提。

## NUMA (非统一存储访问)

**Non-Uniform Memory Access**

- **特征**：
  - 每个处理器有**本地内存**，访问本地快、访问远程慢
  - 主存被划分为多个区域，分别连接到不同 CPU
  - 物理上分布式存储，逻辑上统一编址（单一地址空间）
  - 处理器地位不对等
- **架构**：
  - 系统划分为多个节点，每个节点 = 处理器 + 本地内存 + I/O
  - 节点间通过高速互联（如 QPI、UPI、Infinity Fabric）连接
- **优点**：
  - **可扩展性好**：增加节点即可扩展，不受单一总线限制
  - 本地访问快，带宽高
- **缺点**：
  - 编程模型复杂（需考虑数据放置，远程访问代价高）
  - 操作系统需要 NUMA 感知的调度策略
- **典型代表**：多路 AMD EPYC 服务器、大规模多核系统

### UMA vs NUMA 对比

| 特性 | UMA (SMP) | NUMA |
|------|-----------|------|
| 访存延迟 | 所有处理器相同 | 本地快，远程慢 |
| 可扩展性 | 差 (通常 ≤ 32 核) | 好 (可达数百至数千核) |
| 地址空间 | 统一物理内存 | 单一地址空间，物理分布 |
| 编程模型 | 简单 | 需考虑数据局部性 |
| 互联方式 | 总线/交叉开关 | 点对点互联 |
| 典型规模 | 中小规模 | 大规模 |

## 共享内存系统中的 Cache 一致性问题

共享内存多处理器系统的核心挑战是 **Cache 一致性**：

- **问题来源**：每个处理器有私有 Cache，同一内存块的多份副本可能不一致
- **示例**：
  1. 处理器 A 读取变量 X = 0（缓存到 A 的 Cache）
  2. 处理器 B 写入变量 X = 1（写入 B 的 Cache，可能尚未写回主存）
  3. 处理器 A 再次读取 X，从自己 Cache 读到 0（**脏数据！**）
- **解决方案**：Cache 一致性协议
  - **窥探协议 (Snooping Protocol)**：基于总线广播，每个 Cache 控制器监听总线事务（如 MESI 协议）
  - **目录协议 (Directory Protocol)**：维护目录记录每个内存块被哪些 Cache 持有，点对点通信（适合大规模 NUMA 系统）
