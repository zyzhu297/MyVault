# CPU 的功能

**中央处理器**（Central Processing Unit, CPU）是计算机的核心部件，由**运算器**和**控制器**两部分组成，负责指令的执行和数据的处理。

- **取指令并译码**：根据 PC 从主存取出指令送入 IR，操作码经译码器翻译为控制信号
- **更新程序计数器**：取指后 PC 自动递增指向下条指令，转移/调用/中断时被修改为目标地址
- **执行算术与逻辑运算**：ALU 完成算术（加减乘除）、逻辑（与或非异或）及移位运算，并设置标志位
- **取操作数或写结果**：访问主存或 I/O 接口，读取操作数或将运算结果写回
- **处理异常或中断**：响应内部异常（缺页、除零）或外部中断（I/O 请求），保存断点并转入服务程序
- **时序控制**：通过时钟信号产生节拍，确保各微操作在正确时刻有序执行

# CPU 的基本结构

CPU 主要由**运算器**、**控制器**、**总线控制逻辑**和**中断机构**等组成。

## 运算器（数据通路）

运算器负责数据的**算术运算**和**逻辑运算**，主要包括：

| 组成部分         | 缩写        | 功能                                               |
| ------------ | --------- | ------------------------------------------------ |
| **算术逻辑单元**   | ALU       | 执行算术运算（+、-、×、÷）和逻辑运算（与、或、非、异或、移位）                |
| **暂存寄存器**    | -         | 暂存从主存读出的数据或 ALU 运算的中间结果，避免总线冲突                   |
| **累加寄存器**    | ACC       | 存放 ALU 运算的操作数和运算结果（通用寄存器型 CPU 中 ACC 被通用寄存器组替代）   |
| **通用寄存器组**   | GPRs      | 存放操作数和中间结果，减少访存次数，提高运算速度                         |
| **程序状态字寄存器** | PSW/FLAGS | 存放运算结果的状态标志（进位 CF、零 ZF、符号 SF、溢出 OF 等），由**触发器实现** |
| **移位器**      | Shifter   | 完成移位操作（逻辑移位、算术移位、循环移位），可与 ALU 协同工作               |

## 控制器

控制器是指挥中心，负责**取指令、分析指令、执行指令**，主要包括：

| 组成部分 | 缩写 | 功能 |
|----------|------|------|
| **程序计数器** | PC | 存放**下一条要执行指令的地址**；取指后自动递增（+1 或 +指令字长），遇到转移指令时被修改 |
| **指令寄存器** | IR | 存放**当前正在执行的指令**；操作码送译码器，地址码送 MAR 或地址形成部件 |
| **指令译码器** | ID | 对 IR 中的操作码进行译码，产生相应控制电位信号 |
| **地址形成部件** | - | 根据寻址方式计算操作数有效地址，处理变址、间址等 |
| **时序产生器** | - | 产生时序信号（节拍电位、脉冲），控制操作的时间顺序 |
| **微操作信号发生器** | - | 根据译码结果 + 时序信号 + 状态条件，产生具体的微操作控制信号（组合逻辑/微程序） |
| **存储器地址寄存器** | MAR | 存放**访存地址**；读操作时送地址到地址总线，写操作同理 |
| **存储器数据寄存器** | MDR | 存放**与主存交换的数据**；读操作时接收数据，写操作时送出数据 |

> [!important] MAR 和 MDR 的归属
> 在 408 考试中，MAR 和 MDR 在**逻辑上属于 CPU（控制器部分）**，但在物理实现上可能位于 CPU 芯片内部或靠近主存接口处。做题时，默认 MAR/MDR 在 CPU 内部。

## 总线控制逻辑

总线控制逻辑负责管理和协调 CPU 内部及系统总线的使用，确保多个部件**分时共享总线**时不发生冲突。

- **总线仲裁**：当多个主设备同时请求总线时，按优先级分配总线使用权（集中式仲裁：链式查询、计数器定时查询、独立请求；分布式仲裁）
- **总线驱动**：控制数据何时送上总线，通常通过三态门实现——不使用时输出高阻态，避免电平冲突
- **总线周期管理**：控制一次总线传输的时序，包括地址周期、读/写数据周期
- **总线握手**：协调主从设备的数据传输节奏（同步通信 / 异步通信 / 半同步通信 / 分离式通信）

## 中断机构

中断机构用于处理**异常**（CPU 内部）和**中断**（CPU 外部），实现 CPU 对突发事件的响应能力。

| 组成部分 | 功能 |
|----------|------|
| **中断请求寄存器**（IRR） | 记录各中断源是否发出请求 |
| **中断屏蔽寄存器**（IMR） | 控制哪些中断可被响应（1 表示屏蔽） |
| **中断判优逻辑** | 多个中断同时请求时，按优先级选出最高者（软件查询 / 硬件链式 / 向量中断） |
| **中断向量形成部件** | 根据中断类型号生成中断向量地址，指向对应的中断服务程序入口 |
| **中断响应逻辑** | 保存断点（PC）和 PSW → 关中断 → 加载中断向量 → 转入服务程序 |
| **中断返回逻辑** | 服务程序结束后恢复现场，执行中断返回指令（IRET）回到原程序断点 |

> [!note] 中断机构在 408 中的归属
> 中断机构逻辑上属于**控制器**的一部分（中断响应和执行需要控制器发出微操作序列），但在 CPU 结构框图中常作为独立功能模块画出。详细内容见 [[5.异常和中断机制|异常和中断机制]]。

## 中央处理器的内部总线

CPU 内部各部件通过**内部总线**或**专用数据通路**连接：

| 连接方式 | 特点 | 典型代表 |
|----------|------|----------|
| **单总线结构** | 所有部件挂在一组总线上，结构简单但容易冲突，需要分时使用 | 早期 CPU |
| **多总线结构** | 多条总线并行，不同总线传输不同数据，效率高但结构复杂 | 现代 CPU |
| **专用数据通路** | 各部件之间通过专用线路直连，并行度高但连线多，面积大 | 高性能 CPU 关键路径 |

# CPU 中的寄存器

CPU 中的寄存器按其功能可分为**用户可见寄存器**和**控制/状态寄存器**两大类。

## 用户可见寄存器

供程序员（或编译器）在编程时直接使用的寄存器。

### 通用寄存器（GPRs）

- **功能**：存放操作数、运算结果、地址指针等
- **特点**：可被程序员自由使用，功能通用
- **数量**：RISC 架构通常有 32 个以上（如 MIPS 有 32 个 32 位 GPR，ARM 有 16 个 32 位 GPR），CISC 架构较少（x86-32 有 8 个，x86-64 有 16 个）

### 数据寄存器

- 专门用于存放操作数和运算结果
- 有些架构中数据寄存器与通用寄存器是同一组（x86 的 EAX/EBX/ECX/EDX）

### 地址寄存器

- 存放访存地址，用于各种寻址方式
- 包括：
  - **段寄存器**（CS/DS/SS/ES）：x86 中存放段基址
  - **变址寄存器**（SI/DI）：x86 中存放源/目的变址
  - **基址寄存器**（BX/BP）：x86 中存放基地址
  - **堆栈指针**（SP）：指向栈顶
  > 在 RISC 架构中，地址寄存器通常由通用寄存器兼任

### 程序计数器

- 存放**下一条要执行指令的地址**

### 条件码寄存器（标志寄存器）

是**只读型用户可见寄存器**，不能直接修改其值，存放运算结果的状态特征，供条件转移指令判断：

| 标志位 | 名称 | 置位条件 |
|--------|------|----------|
| **CF**（Carry Flag） | 进位/借位标志 | 无符号数运算产生进位或借位 |
| **ZF**（Zero Flag） | 零标志 | 运算结果为 0 |
| **SF**（Sign Flag） | 符号标志 | 运算结果为负（最高位为 1） |
| **OF**（Overflow Flag） | 溢出标志 | 有符号数运算产生溢出 |
| **PF**（Parity Flag） | 奇偶标志 | 结果低 8 位中 1 的个数为偶数（x86） |
| **AF**（Auxiliary Flag） | 辅助进位标志 | 低 4 位向高 4 位产生进位（x86，用于 BCD 调整） |

> [!note] CF vs OF
> - **CF** = 无符号数进位（进位/借位的 C）：最高位向外的进位 ⊕ sub（减法时取反）
> - **OF** = 有符号数溢出（Overflow）：最高位进位 ⊕ 次高位进位（即进位到符号位和符号位向外的进位不同时为 1）
> - 同一个加法，CPU 同时设置 CF 和 OF，由程序员选择使用哪个标志

## 用户不可见寄存器

CPU 内部使用，控制指令执行流程。

### 指令寄存器（IR）

- 存放**当前正在执行的指令**
- 操作码字段 → 指令译码器
- 地址码字段 → 地址形成部件/MAR
- 对程序员透明（不可见）

### 存储器地址寄存器（MAR）

- 存放**访存的地址**
- 位数决定最大寻址空间：MAR 有 n 位 → 最大寻址 $2^n$ 个存储单元
- 连接地址总线

### 存储器数据寄存器（MDR）

- 存放**与主存交换的数据**
- 位数通常等于存储字长
- 连接数据总线
- 读操作时从数据总线接收数据，写操作时向数据总线发送数据

### 页表基址寄存器

- 存放当前进程**页表的起始物理地址**
- 地址转换时：将虚拟页号作为索引，结合页表基址找到对应页表项，得到物理页框号
- 进程切换时由 OS 更新，指向新进程的页表
## 各寄存器的字长决定因素

| 寄存器       | 字长决定因素   | 说明                                                                 |
| --------- | -------- | ------------------------------------------------------------------ |
| **PC**    | 主存地址空间   | 存放指令地址，与存储器容量和指令字长有关，至少为 $\log_2{\frac{\text{主存容量}}{\text{指令字长}}}$ |
| **IR**    | 指令字长     | 必须能完整容纳一条指令的所有位（操作码 + 地址码）                                         |
| **MAR**   | 主存地址空间   | n 位可表示 $2^n$ 个地址，其位数决定了 CPU 的最大寻址能力                                |
| **MDR**   | 存储字长     | 位数等于一次能从主存读出/写入的二进制位数，通常与数据总线宽度一致                                  |
| **通用寄存器** | 机器字长     | 与 CPU 一次能处理的数据位数相同（32 位 CPU → 32 位 GPR）                            |
| **PSW**   | 需记录的状态位数 | 由条件码位数 + 控制标志位数决定，不直接与机器字长挂钩                                       |
| **ACC**   | 机器字长     | 累加器宽度与机器字长一致                                                       |

> [!important] 408 常见考点
> 1. **MAR 和 PC 位数相同**（都存放地址，由寻址空间决定），但 MAR 送地址总线、PC 指向指令地址
> 2. **MDR 和 IR 位数不一定相同**：MDR 由存储字长决定，IR 由指令字长决定（指令可能跨多个存储字）
> 3. **机器字长、存储字长、指令字长的关系**：
>    - **机器字长** = CPU 一次处理的二进制位数（决定通用寄存器和 ALU 宽度）
>    - **存储字长** = 一个存储单元存放的二进制位数（决定 MDR 宽度）
>    - **指令字长** = 一条指令的二进制位数（决定 IR 宽度）
>    - 三者不一定相等。例如早期 x86 机器字长 32 位，但指令字长可变（1~15 字节），存储字长按字节

# 总结

> [!summary] 本章重点
> 1. CPU 由**运算器**和**控制器**组成，完成五大功能：**指令控制、操作控制、时间控制、数据加工、中断处理**
> 2. CPU 的基本工作循环：**取指令 → 分析指令 → 执行指令 → 中断检测**，体现了冯·诺依曼**存储程序**的核心思想
> 3. **运算器**（数据通路）包括 ALU、暂存寄存器、累加器 ACC、通用寄存器组 GPRs、PSW 和移位器，负责算术/逻辑运算
> 4. **控制器**包括 PC、IR、指令译码器 ID、时序产生器、微操作信号发生器、MAR 和 MDR，负责取指/译码/执行
> 5. **PC（程序计数器）** 存放**下一条指令地址**，取指后自动递增，转移指令时被修改——是 CPU 指令控制的核心
> 6. **IR（指令寄存器）** 存放**当前指令**，操作码送译码器，地址码送地址形成部件——对程序员完全透明
> 7. **MAR 和 MDR** 是 CPU 与主存的桥梁：MAR 送地址，MDR 传数据；MAR 位数决定最大寻址空间（n 位 → $2^n$ 个单元）
> 8. CPU 内部连接方式：**单总线**（简单但冲突）→ **多总线**（效率高）→ **专用数据通路**（并行度最高但面积大）
> 9. CPU 寄存器分为两类：
>    - **用户可见**：通用寄存器 GPRs、数据/地址寄存器、条件码（PSW 中可见部分）
>    - **控制/状态**：PC、IR、MAR、MDR、PSW（对程序员部分或完全透明）
> 10. 条件码标志的核心区分：
>     - **CF**（进位标志）服务于**无符号数**运算
>     - **OF**（溢出标志）服务于**有符号数**运算
>     - **ZF**（零标志）和 **SF**（符号标志）对两者都有效
> 11. **寄存器是存储层次中的最顶层**，速度最快但容量最小；寄存器数量影响指令格式（寄存器字段位数）和访存次数
