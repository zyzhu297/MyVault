# 数据通路的功能

**数据通路**是指令执行过程中**数据所经过的路径**，包括路径上的各种功能部件（ALU、寄存器、多路选择器等）以及这些部件之间的**连接方式**

## 数据通路的基本功能

- **数据传送**：在寄存器之间、寄存器与 ALU 之间、寄存器与主存之间传递数据
- **数据加工**：通过 ALU 等组合逻辑部件对数据进行算术/逻辑运算
- **数据暂存**：通过寄存器暂存中间结果、操作数、地址等信息

## 数据通路与控制器的关系

| 维度 | 数据通路 | 控制器 |
|------|---------|--------|
| **性质** | **执行部件**（"怎么做"） | **指挥部件**（"什么时候做"） |
| **组成** | 组合逻辑 + 时序逻辑（寄存器） | 组合逻辑/微程序 + 时序产生器 |
| **输出** | 运算结果、状态标志 | 控制信号（使能、选择、读写） |
| **工作方式** | 受控制信号驱动，被动执行 | 根据指令译码主动产生控制信号 |
| **对应 CPU 部件** | **运算器** | **控制器** |

> [!important] 数据通路 = 运算器
> 在 408 考试中，"数据通路"和"运算器"在功能范畴上基本等价。数据通路侧重描述**数据流动的路径和部件**，运算器侧重描述**数据处理的功能**，两者是同一事物的不同表述角度。

# 数据通路的组成

数据通路由**组合逻辑元件**和**时序逻辑元件**两大类部件构成。

## 组合逻辑元件（操作元件）

**组合逻辑元件**的特点是：输出**仅取决于当前输入**，不受时钟控制，无记忆功能，且无输出到输入的反馈通路，因此具有**确定性**和**及时响应性**

| 元件                        | 功能                       | 在数据通路中的作用                  |
| ------------------------- | ------------------------ | -------------------------- |
| **ALU**（算术逻辑单元）           | 完成算术运算、逻辑运算、移位运算         | 数据加工的核心部件                  |
| **加法器**（Adder）            | 完成加法运算                   | PC 递增、地址计算、分支目标计算          |
| **多路选择器**（MUX）            | 从多个输入中选择一个输出             | 选择数据来源（如寄存器号、ALU 输入、写回数据等） |
| **译码器**（Decoder）          | 将 n 位输入转换为 $2^n$ 位独热输出   | 寄存器号的译码选择、控制信号译码           |
| **符号扩展单元**                | 将短立即数扩展为机器字长             | I 型指令立即数的符号扩展              |
| **移位器**                   | 完成移位操作                   | 逻辑/算术/循环移位                 |
| **零检测单元**                 | 检测数据是否为 0                | 分支指令的条件判断（beq 等）           |
| **三态门**（Tri-State Buffer） | 控制输出端的通断，输出可为 0/1**高阻态** | 控制部件是否向总线发送数据，实现**总线共享**   |

> [!note] 组合逻辑的关键特征
> - 无状态、无记忆——输入一旦改变，输出立即改变（仅受门延迟影响）
> - 不能构成存储单元（无法保存数据跨越时钟周期）
> - 在数据通路中负责**数据处理**和**路径选择**，不负责数据暂存

## 时序逻辑元件（状态元件）

**时序逻辑元件**的特点是：输出**取决于当前输入和之前的输入**（有记忆功能），通常由**时钟边沿**触发状态更新。

### 核心状态元件

| 元件                      | 功能                 | 归属  |
| ----------------------- | ------------------ | --- |
| **程序计数器**（PC）           | 存放下一条指令的地址；时钟上升沿更新 | 控制器 |
| **指令寄存器**（IR）           | 存放当前指令             | 控制器 |
| **寄存器堆**（Register File） | 存放操作数和中间结果；时钟上升沿写入 | 运算器 |
| **存储器地址寄存器**（MAR）       | 存放访存地址             | 控制器 |
| **存储器数据寄存器**（MDR）       | 存放与主存交换的数据         | 控制器 |
| **暂存寄存器**（A/B/Temp）     | 暂存 ALU 输入操作数或中间结果  | 运算器 |
| **ALU 输出寄存器**（ALUOut）   | 暂存 ALU 计算结果或有效地址   | 运算器 |
| **程序状态字**（PSW/FLAGS）    | 存放运算结果的状态标志        | 运算器 |

### 寄存器堆的内部结构

- **读操作**：组合逻辑行为——给定读寄存器号，数据**立即**读出（无需等时钟）
- **写操作**：时序逻辑行为——需要**写使能信号 + 时钟上升沿**才会写入
- **同时读写同一寄存器**：先读出旧值，再在时钟上升沿写入新值

> [!important] 408 考点：寄存器堆的读写特性
> - 读是**组合逻辑**，不需要时钟；写是**时序逻辑**，需要时钟沿
> - 这意味着：在同一个时钟周期内，可以先读出一个寄存器的值，再在同一周期的时钟上升沿写入该寄存器——读到的仍是旧值
> - 此特性是实现**写后读转发**的基础

### 状态元件的时钟周期

在同步数字系统中，所有状态单元（寄存器）由**同一个时钟信号**驱动，在时钟边沿统一更新；这条路径上的三个关键时间参数共同决定了**最小时钟周期**：

| 参数          | 符号                    | 含义                                  | 决定因素                          |
| ----------- | --------------------- | ----------------------------------- | ----------------------------- |
| **寄存器延迟**   | $T_{\text{clk-to-Q}}$ | 时钟沿到达后，寄存器输出稳定为新值所需的时间              | 寄存器内部电路（触发器的传播延迟）             |
| **关键路径延迟**  | $T_{\text{ max}}$     | 信号从源寄存器输出经过**组合逻辑**到达目的寄存器输入的最长传播时间 | 数据通路中最慢的组合逻辑路径（如 ALU 运算、访存路径） |
| **寄存器建立时间** | $T_{\text{setup}}$    | 时钟沿到来**之前**，寄存器输入数据必须稳定的最小时间        | 寄存器内部电路（触发器对输入稳定性的要求）         |

#### 时钟周期的约束

$$
T_{\text{clock}} \geq T_{\text{clk-to-Q}} + T_{\text{max}} + T_{\text{setup}} 
$$

即一个时钟周期内，信号必须完成：**从源寄存器输出 → 穿越组合逻辑 → 在目的寄存器输入端稳定并满足建立时间**。

> [!important] 408 考点：三个延迟决定主频
> - **寄存器延迟** $T_{\text{clk-to-Q}}$：输出慢 → 留给组合逻辑的时间减少
> - **关键路径延迟** $T_{\text{max}}$：**最主要的限制因素**——数据通路中时序路径越长（如 lw 指令的取指→译码→ALU计算地址→访存→写回），$T_{\text{max}}$ 越大，时钟周期被迫拉长
> - **建立时间** $T_{\text{setup}}$：寄存器自身的"硬性要求"，不满足则数据无法可靠锁存
> - 三者之和的**最大值**决定了 CPU 的主频上限；流水线的核心思想正是将一条长路径拆成多段短路径，每段的 $T_{\text{max}}$ 大幅减小，从而允许更高的时钟频率

# 数据通路的组织与分类

数据通路可以按照**部件连接方式**和**时序组织方式**两个维度进行分类。

## 按部件连接方式划分

根据 CPU 内部各功能部件的互联方式，可分为三种基本结构：

### 1. 单总线结构

- **特点**：所有寄存器、ALU、MAR/MDR 都挂在一组公共的 CPU 内部总线上
- **工作方式**：任一时刻只能有一个部件向总线发送数据，**分时**传送
- **数据传送**：通过**暂存寄存器** Y 和 Z（通常 Y 在 ALU 输入端，Z 在 ALU 输出端）解决总线冲突
- **优缺点**：
  - 优点：结构简单，连线少，控制信号数量少
  - 缺点：数据传送必须串行，速度慢，存在竞争，并发度低

### 2. 多总线结构

- **特点**：CPU 内部设有多组总线，不同数据传送可以**并行进行**
- **常见形式**：
  - **双总线结构**：一条总线供寄存器间传送，另一条总线供 ALU 输入
  - **三总线结构**：两条源操作数总线 + 一条目的总线，ALU 运算可在一个时钟周期内完成：源1→A 总线、源2→B 总线、结果→C 总线同时进行
- **优缺点**：
  - 优点：并行度高，数据传送快
  - 缺点：连线增多，控制复杂，芯片面积和功耗增加

### 3. 专用数据通路

- **特点**：各部件之间通过**专用线路**直接连接，无需共享总线仲裁
- **工作方式**：根据指令需求，控制信号选择不同的专用路径
- **优缺点**：
  - 优点：并行度最高，没有总线冲突
  - 缺点：连线数量极多，布线复杂，面积大，扩展性差，成本较高

### 三种连接方式的对比

| 维度         | 单总线                                    | 多总线                                 | 专用数据通路                                 |
| ---------- | -------------------------------------- | ----------------------------------- | -------------------------------------- |
| **结构复杂度**  | **低**——一组总线 + 三态门控制，结构规整简单             | **中**——多组总线需分别布线，需总线仲裁逻辑            | **高**——各部件间专用连线 + 大量 MUX，布线密集复杂        |
| **扩展灵活性**  | **高**——新增部件只需挂到总线上，基本不改动原有结构           | **中**——新增部件需连接到合适的某一组总线，可能需调整仲裁     | **低**——新增部件需铺设大量新专用连线，改动范围大            |
| **数据传输效率** | **低**——同一时刻只能传一组数据，操作串行化严重（取一条指令需 6 步） | **较高**——多条总线可并行传送不同数据，减少总线冲突        | **最高**——各路径互不干扰，一个时钟周期内可同时完成多组数据传送     |
| **硬件成本**   | **低**——连线少，芯片面积小，功耗低                   | **中**——连线增多，需仲裁器，面积和功耗增加            | **高**——连线数量极大，MUX 开销大，芯片面积和功耗最大        |
| **性能瓶颈**   | **总线独占**——所有部件争用同一总线，瓶颈在总线带宽；最慢的传送拖累整体 | **仲裁延迟**——多主设备同时请求时需排队仲裁，高负载下仲裁成为瓶颈 | **面积与功耗**——连线密度达到物理极限，散热困难，设计验证复杂度指数增长 |

> [!important] 408 考点：三种结构的取舍
> - **单总线**："简单但慢"——以时间换面积，适用于低成本、低性能场景，在 408 中主要用于**分析微操作时序**
> - **多总线**："折中方案"——在性能和成本间取得平衡，是**现代通用 CPU 的主流方案**
> - **专用数据通路**："快但贵"——以面积换时间，用于高性能 CPU 的**关键路径**（如寄存器堆到 ALU 的直连），但不会全芯片使用
> - **实际 CPU 是混合结构**：关键路径用专用通路，非关键路径用总线，不存在"纯"单总线或"纯"专用通路的现代处理器

## 按时序组织方式划分

根据指令执行的时序安排，数据通路可分为三种类型，详见 [[2.指令执行过程#处理器指令执行模型|指令执行过程]]：

| 类型      | 时钟周期         | CPI        | 硬件特点               |
| ------- | ------------ | ---------- | ------------------ |
| **单周期** | 最长指令时间       | = 1        | 部件**不能复用**，存储器必须分开 |
| **多周期** | 最长阶段时间       | > 1（各指令不同） | 部件可复用，存储器可统一       |
| **流水线** | 最长阶段时间+寄存器延迟 | ≈ 1（理想）    | 各阶段独立硬件，流水线寄存器隔离   |
>[!warning]
>单周期处理器（CPI = 1）不能采用单总线结构的数据通路，因为单总线将所有寄存器连接到一条公共总线上，一个时钟周期内只允许一次操作，无法完成一条指令的所有操作
# 单总线结构的数据通路

单总线结构的 CPU 数据通路中，所有部件通过**各自的三态门**与总线连接，以控制其与总线之间的通断，避免多个输出同时驱动总线造成冲突；在单总线结构中，**一次只能传送一个数据**，寄存器间的传送需要借助总线；

## 加法指令（ADD rs, rd）

- **指令功能**：将 rs 和 rd 寄存器中的值相加，结果写入 rd 寄存器，即 `R[rd] ← (R[rs]) + (R[rd])`
- **指令格式**：二地址指令，操作码 + rs 字段 + rd 字段，隐含目的操作数在 rd
- **本例取值**：`rs = 0001`（R₁），`rd = 0010`（R₂）——两个操作数分别来自 R₁ 和 R₂，结果写回 R₂

### 取指阶段

取指是所有指令**公共操作**，与本指令具体功能无关：

| 时钟     | 微操作             | 有效的控制信号                                                              | 说明                                                 |
| ------ | --------------- | -------------------------------------------------------------------- | -------------------------------------------------- |
| C1     | `MAR ← (PC)`    | **PC_out, MAR_in**                                                   | PC 驱动内部总线（三态门使能），MAR 在时钟沿锁存总线数据                    |
| C2~C6  | `MDR ← M(MAR)`  | **MEM_R, MDR_in**                                                    | MAR→地址总线（C1 已锁存地址）；主存读出→数据总线；MDR 在 C6 时钟沿锁存        |
| C7     | `IR ← (MDR)`    | **MDR_out, IR_in**                                                   | MDR 驱动内部总线，IR 在时钟沿锁存；此后操作码送译码器                     |
| C8~C10 | `PC ← (PC) + 1` | **C8:** PC_out, Y_in → **C9:** ALU_add, Z_in → **C10:** Z_out, PC_in | C8: PC→Y; C9: ALU加1→Z; C10: Z→PC（跨 3 拍，每拍 1 次总线传送） |

> [!note] C2~C6 的 5 个时钟周期分解——"以最慢者为准"
> 
> | 时钟 | 主存内部操作 | CPU 侧可见动作 |
> |------|------------|---------------|
> | **C2** | MAR 中地址送上**地址总线**，主存芯片接收地址 → 片内地址译码器锁存 | CPU 侧：地址总线驱动，MAR 使能输出 |
> | **C3** | 主存内部**行/列译码**，选中存储单元所在的行线（字线）和列线（位线） | CPU 侧：等待，总线空闲 |
> | **C4** | **读出放大**——存储电容/触发器的微弱信号经读出放大器放大为逻辑电平 | CPU 侧：等待，总线空闲（此时为最慢的模拟电路操作） |
> | **C5** | 放大后的数据驱动到**数据总线**，信号沿 PCB 走线返回 CPU | CPU 侧：数据总线由主存驱动，等待信号稳定 |
> | **C6** | 数据总线上的电平已稳定，MDR 内部锁存器**建立时间**满足 | CPU 侧：MDR 时钟沿采样，锁存总线数据 |
> 
> **为什么跨 5 拍？**
> - **C4（读出放大）**和 **C5（总线驱动 + PCB 传输）**是瓶颈，属于纯物理延迟，无法通过电路结构优化消除
> - 5 拍 = **信号穿越芯片引脚、PCB 走线、再返回芯片**的最小往返时间 ÷ 时钟周期
> - 如果主存更快（如使用 Cache 或 SRAM），这个拍数可以减少；如果主存更慢（如早期的 DRAM），甚至需要更多拍
> - 这也恰好说明：**一个机器周期（以一次访存为界）通常由多个时钟周期组成**，具体数量取决于主存速度

### 执行阶段

| 时钟  | 微操作                 | 有效的控制信号                                     | 说明                                                                      |
| --- | ------------------- | ------------------------------------------- | ----------------------------------------------------------------------- |
| C11 | `Y ← (R[rs])`       | **rs = 0001, GPRout, Y_in**                 | IR 的 rs 字段 → 寄存器堆读地址，R₁ 输出到总线，Y 锁存                                      |
| C12 | `Z ← (Y) + (R[rd])` | **rd = 0010, GPRout, ALU_add, Z_in** | IR 的 rd 字段 → 寄存器堆读地址，R₂ 输出到总线→ALU B 端；Y(锁存 R₁) 在 ALU A 端；ALU 做加法，Z 锁存结果 |
| C13 | `R[rd] ← (Z)`       | **Z_out, rd = 0010, GPRin, R_write** | Z 驱动总线，IR 的 rd 字段 → 寄存器堆写地址，R₂ 时钟沿打入                                    |

**执行阶段共需 3 拍**（3 次总线数据传送：R₁→Y, R₂+ALU→Z, Z→R₂）。

> [!note] 单总线结构下的 ALU 与总线——C12 的并行性与输出暂存
> 以下机制是**单总线数据通路特有的设计约束**，与专用数据通路（MUX 选路直连）截然不同。
> - **输入的并行**：由于 ALU 是无存储能力的组合逻辑电路，运算时需要两个输入同时有效，因此需要先将一个操作数经总线送入暂存寄存器，寄存器 rd 的读出和 ALU 运算可以**在同一总线周期内完成**——rd 通过总线直接送到 ALU 的 B 输入端（不经 Y 暂存），而 A 输入端已在上一周期锁存了 Y 的值，因此加法本身不需要额外占用总线周期
> - **输出必须经由 Z 暂存**：**ALU 输出端不直连总线**，运算结果必须先打入暂存器 Z（`Z_in` 在同一拍有效），再由 Z 在 C13 驱动总线写回 rd。原因是：
>   - ALU 是纯**组合逻辑**，无锁存能力——输出随输入变化，无法跨时钟周期保持稳定
>   - 若 ALU 直连总线，C12 中 rd 一到 ALU B 端，结果立即出现在总线上，会与 rd 正在驱动总线的信号**冲突**
>   - Z 作为**时序元件**隔离了 ALU 输出与总线：C12 锁存结果，C13 再由 Z 独占总线输出
> - **与专用数据通路的对比**：专用通路中 ALU 结果通过 MUX 直连寄存器堆写数据端，**不需要 Z 暂存和额外一拍**——这正是单总线"串行化"代价的体现


## 取数指令（LOAD rd, rs）

- **指令功能**：以 rs 寄存器中的值为**主存地址**，将该地址单元的内容取出，送入 rd 寄存器，即 `R[rd] ← M[(R[rs])]`
- **指令格式**：二地址指令，rs 提供访存基地址，rd 为加载目标
- **本例取值**：`rs = 0001`（R₁ 存访存地址），`rd = 0010`（R₂ 为加载目标）

### 取指阶段

与加法指令完全相同：C1 `MAR←(PC)` → C2~C6 `MDR←M(MAR)` → C7 `IR←(MDR)` → C8~C10 `PC←(PC)+1`，控制信号不变。

### 执行阶段

| 时钟 | 微操作 | 有效的控制信号 | 说明 |
|------|--------|-------------|------|
| C11 | `MAR ← (R[rs])` | **rs = 0001, GPRout, MAR_in** | IR 的 rs 字段 → 寄存器堆读地址，R₁（存有访存地址）输出到总线，MAR 锁存 |
| C12~C16 | `MDR ← M(MAR)` | **MEM_R, MDR_in** | MAR→地址总线（已在 C11 锁存）；主存读出→数据总线；MDR 在 C16 时钟沿锁存（跨 5 拍，同 C2~C6） |
| C17 | `R[rd] ← (MDR)` | **MDR_out, rd = 0010, GPRin, R_write** | MDR 驱动内部总线，IR 的 rd 字段 → 寄存器堆写地址，R₂ 时钟沿打入 |

**执行阶段共需 7 拍**（设地址 1 拍 + 访存 5 拍 + 写回 1 拍）

## ADD vs LOAD vs STORE 单总线执行对比

| 对比维度 | ADD（加法） | LOAD（取数） | STORE（存数） |
|----------|-----------|-------------|-------------|
| 取指时钟周期 | C1~C10（10 拍） | C1~C10（10 拍） | C1~C10（10 拍） |
| 执行时钟周期 | C11~C13（3 拍） | C11~C17（7 拍） | C11~C17（7 拍） |
| 总时钟周期 | **13** | **17** | **17** |
| 取指中访存次数 | 1 次（读指令，5 拍） | 1 次（读指令，5 拍） | 1 次（读指令，5 拍） |
| 执行中访存次数 | 0 | 1 次（读数据，5 拍） | 1 次（写数据，5 拍） |
| 总访存次数 | 1 | 2 | 2 |
| ALU 使用 | 取指（PC+1）+ 执行（ADD） | 仅取指（PC+1） | 仅取指（PC+1） |
| 数据传输方向 | 寄存器↔寄存器 | 主存→寄存器 | 寄存器→主存 |

> [!important] 单总线结构指令执行的核心规律
> 1. **取指阶段完全一致**（C1~C10），与指令类型无关——控制器设计时 FE 周期微操作序列固化
> 2. **执行阶段的拍数 = 内部传送拍数 + 访存拍数**，访存每次 5 拍是瓶颈
> 3. **机器周期以一次访存为界**：ADD 的指令周期含 1 个机器周期（FE=EX 合为 1 次访存），LOAD/STORE 含 2 个机器周期（FE 1 次 + EX 1 次）
> 4. **总线冲突**须三态门控制：每拍精确控制哪个部件输出、哪个部件打入
> 5. 单总线为**多周期处理器**的典型数据通路——每个时钟周期只完成一次总线传送，周期短、部件可复用

## 单总线数据通路的控制信号汇总

每条微操作对应一组**输出使能**（决定谁驱动总线）和**输入打入**（决定谁锁存总线数据）信号。

### 控制信号分类

| 类别            | 信号名        | 作用                | 对应硬件                   |
| ------------- | ---------- | ----------------- | ---------------------- |
| **输出使能**（三态门） | PC_out     | PC 驱动内部总线         | PC 输出端三态门              |
|               | MDR_out    | MDR 驱动内部总线        | MDR 输出端三态门             |
|               | GPR_out(n) | 寄存器 n 驱动内部总线      | 寄存器堆输出端三态门 + 译码器       |
|               | Z_out      | 暂存器 Z 驱动内部总线      | Z 输出端三态门               |
| **输入打入**（锁存）  | MAR_in     | 总线数据打入 MAR        | MAR 时钟使能               |
|               | MDR_in     | 总线数据打入 MDR（CPU 侧） | MDR 时钟使能               |
|               | IR_in      | 总线数据打入 IR         | IR 时钟使能                |
|               | PC_in      | 总线数据打入 PC         | PC 时钟使能                |
|               | Y_in       | 总线数据打入暂存器 Y       | Y 时钟使能                 |
|               | GPR_in(n)  | 总线数据打入寄存器 n       | 寄存器堆写使能 + 写地址译码        |
|               | Z_in       | ALU 结果打入暂存器 Z     | Z 时钟使能                 |

|               | MEM_W      | 主存写使能             | 主存写控制线                 |
| **ALU 控制**    | ALU_add    | ALU 做加法           | ALU 操作选择线              |
|               | ALU_pass   | ALU 直通（A 端原样输出）   | ALU 操作选择线（用于 PC 自增第一步） |

### ADD / LOAD / STORE 控制信号逐拍对照

| 时钟 | 微操作 | ADD | LOAD | STORE |
|------|--------|-----|------|-------|
| **C1** | MAR←(PC) | PC_out, MAR_in | PC_out, MAR_in | PC_out, MAR_in |
| **C2~C6** | MDR←M(MAR) | MEM_R, MDR_in | MEM_R, MDR_in | MEM_R, MDR_in |
| **C7** | IR←(MDR) | MDR_out, IR_in | MDR_out, IR_in | MDR_out, IR_in |
| **C8~C10** | PC←(PC)+1 | C8: PC_out,Y_in → C9: ALU_add,Z_in → C10: Z_out,PC_in | ← 同 | ← 同 |
| **C11** | 执行第一步 | rs=0001, GPRout, Y_in | rs=0001, GPRout, MAR_in | rs=0001, GPRout, MAR_in |
| **C12** | 执行第二步 | rd=0010, GPRout, ALU_add, Z_in | MEM_R, MDR_in (跨 C12~C16) | rd=0010, GPRout, MDR_in |
| **C12~C16** | 执行第二步（访存） | — | (跨 5 拍，同 C2~C6) | — |
| **C13** | 执行第三步 | Z_out, rd=0010, GPRin, R_write | — | MAR_out, MEM_W (跨 C13~C17) |
| **C13~C17** | 执行第三步（访存） | — | — | (跨 5 拍) |
| **C17** | 执行第三步 | — | MDR_out, rd=0010, GPRin, R_write | — |

> [!important] 408 考点：控制信号的规律
> 1. **每条微操作 = 1 个输出使能 + 1 个输入打入**（单总线同一时刻只允许一个源输出）
> 2. **ALU 运算与总线传送可重叠**：源操作数走总线→ALU B 端的同时，ALU A 端已锁存 Y 值，运算结果自动入 Z（Z_in 可与总线传送同一拍）
> 3. **访存类微操作不占用内部总线**：`MDR←M(MAR)` 用的是外部地址/数据总线，MDR_in 仅锁存数据总线上的值
> 4. **MEM_R / MEM_W 与 MAR_out 配合**：MAR 输出到地址总线后才能启动访存，两者在时序上相邻（通常 MAR 在上一拍锁存）
> 5. 控制信号的产生方式决定了控制器的类型：**硬布线控制器**（组合逻辑直接产生）或 **微程序控制器**（查控存），详见 [[4.控制器的功能和工作原理]]
> 6. 与专用通路的差异：专用数据通路用 **MUX 选择信号** 替代三态门使能，用 **RegWrite 等统一写使能** 替代分散的打入脉冲

## 单总线结构的优缺点总结

| 优点 | 缺点 |
|------|------|
| 硬件简单，互联线少 | 数据传送效率低，串行操作多 |
| 控制信号少 | 需要额外的暂存寄存器（Y、Z） |
| 扩展方便（新部件挂总线上即可） | 同一时刻只有一组数据在总线上 |
| 芯片面积和功耗低 | 不适合高性能 CPU |

# 专用结构的数据通路

**专用数据通路**是根据指令执行的需要，在各部件之间设置**专用连接路径**，并通过**多路选择器（MUX）** 在不同路径间切换的结构。相比单总线，专用数据通路**并行度高、控制结构清晰**，是单周期处理器和多周期处理器的标准实现方式。

下面以 **MIPS 架构**的单周期处理器为例，逐步构建各类指令的数据通路。

## 取指令部件的设计

取指令是每条指令都必须完成的公共操作，取指部件是所有指令**共用**的数据通路部分
- **程序计数器**（PC）：32 位寄存器，存放下一条指令地址
- **加法器**：计算 PC + 4（下一条指令的地址，因为 MIPS 指令定长 4 字节）
- **指令存储器**（Instruction Memory）：存放指令，输入地址，输出指令
- **多路选择器**（MUX）：在"PC + 4"和"分支目标地址"之间选择下一个 PC 值
  - **顺序执行**：当指令为普通顺序指令时，直接计算 PC + 4 得到下一条指令的地址
  - **转移执行**：对于分支或转移指令，需要根据指令类型计算转移目标地址
>[!warning] PC 更新无需写使能信号
>在每个周期无条件更新为下一条指令地址


## R 型运算类指令数据通路

**R 型指令**完成**两个寄存器之间**的运算，结果写回寄存器，以加法指令为例：
```
add rd, rs, rt           #R[rs] + R[rt] → R[rd]
```
![[R 型指令相关的数据通路.png]]
该数据通路能够对两个寄存器 Rs 和 Rt 的内容进行运算，并将结果写回 Rd 寄存器。例如 add 和 sub 等指令还需判断运算结果是否溢出，仅当不溢出时才将结果写回 Rd 寄存器；发生溢出时，会触发异常处理
指令中 Rs 和 Rt 是两个源操作数寄存器的编号，Rd 是目的寄存器的编号；因此：
- 寄存器堆的两个读地址端 Ra 和 Rb 分别与 Rs 和 Rt 相连
- 写地址端 Rw 应与 Rd 相连
- ALU 的运算结果通过总线 busW 连至寄存器堆的写数据端
控制信号 RegWr 作为寄存器堆的写使能信号，在 RegWr 为 1 且无溢出的情况下，运算结果才被写入寄存器堆
ALU 的操作类型由控制信号 ALUctr 决定，假设 ALU 支持 $N$ 中不同的运算，则 ALUctr 至少需要 $\lceil\log_2{N}\rceil$ 位来选择具体操作类型
由于单周期处理器必须在一个时钟周期内完成指令，其**数据通路中不设置指令寄存器（IR）** 而直接从**指令存储器**中取指令并解析执行，否则仅取指令到 IR 就需一个时钟周期

## I 型运算指令的数据通路

**I 型运算指令**完成**一个寄存器与一个立即数**之间的运算，结果写回寄存器，以有符号加法指令为例：

```
addi rt, rs, imm16        # R[rs] + SignExt(imm16) → R[rt]
```
![[I 型指令相关的数据通路.png]]
该通路在 R 型指令数据通路的基础上，增加支持 I 型指令的功能模块后形成的专用数据通路；该通路可同时执行 R 型和 I 型运算指令
基于 R 型数据通路，I 型需要做以下**三处修改**：

1. **ALU 第二操作数来源**：从寄存器 Rb 输出切换为**符号扩展后的立即数**——在 ALU 第二输入端前增加一个 **MUX**，由 `ALUSrc` 信号选择 0=Rb / 1=立即数
2. **目的寄存器的选择**：在写地址端 Rw 前增加一个 **MUX**，由 `RegDst` 信号选择 0=rd / 1=rt
3. **立即数的处理**：新增一个立即数扩展器，其输入为指令的 imm16 字段；由控制信号 `ExtOP` 决定扩展方式：**对于算术类指令，采用符号扩展；对于按位逻辑操作指令，采用零扩展**

> [!important] I 型的两个 MUX——与 R 型共用数据通路的关键
> I 型与 R 型指令**共用同一套数据通路**，差异仅由两个 MUX 切换：
> - **ALUSrc MUX**：R 型选 Rt 输出，I 型选立即数
> - **RegDst MUX**：R 型选 rd，I 型选 rt
> 
> 这种"一套通路 + MUX 切换"的思想贯穿整个单周期处理器设计——取指、R 型、I 型、访存、分支全都共用同一套寄存器堆、同一套 ALU，通过控制信号在不同指令类型间切换路径。

## 访存指令的数据通路

MIPS 访存指令属于 **I 型**（需要对立即数做地址偏移运算），在 R/I 型数据通路基础上增加**数据存储器**和**写回数据 MUX**后形成。

```
lw rt, imm16(rs)        # M[R[rs] + SignExt(imm16)] → R[rt]
sw rt, imm16(rs)        # R[rt] → M[R[rs] + SignExt(imm16)]
```

![[支持 LOAD 和 STORE 指令功能的数据通路.png]]
LOAD 和 STORE 指令的访存地址计算逻辑完全一致：首先对指令中的 16 为立即数 imm16 进行符号扩展，然后将其与寄存器 Rs 的内容相加，得到访存地址；区别在与两者的数据传输方向：
- LOAD 指令从该地址读取 32 位数据，并将数据写入寄存器 Rt
- STORE 指令则将寄存器 Rt 中的 32 位数据写入该地址对应的存储单元

| 区别点 | R 型 / I 型（运算） | lw | sw |
|--------|------------------|----|----|
| 需要数据存储器 | 否 | **是**（读） | **是**（写） |
| EX 阶段 ALU 作用 | 做算术/逻辑运算 | **计算访存地址** EA=rs+offset | **计算访存地址** EA=rs+offset |
| MEM 阶段 | 无（跳过） | 读主存 MDR←M[EA] | 写主存 M[EA]←MDR |
| WB 阶段 | ALU 结果→寄存器 | **主存数据**→寄存器 | 无 |
| 写回数据来源（MemtoReg） | 0（ALU） | **1**（主存） | ×（不写寄存器） |

基于 R/I 型通路，主要有以下两处改动，以支持 LOAD/STORE 指令的执行：
- **写回数据的选择（新增 MemtoReg MUX）**：为了处理写回寄存器数据来源的冲突——运算类指令（R/I 型）写回 ALU 结果，LOAD 指令写回主存读出的数据——在寄存器堆写数据端 busW 前新增一个二路 MUX：
  - MemtoReg = 0（运算类指令）时，选择 ALU 结果通过 busW 送入寄存器堆写数据端
  - MemtoReg = 1（LOAD 指令）时，选择数据存储器读出的数据通过 busW 送入寄存器堆写数据端
- **新增数据存储器模块**：
  - 数据存储器的地址端 Adr 直接与 **ALU 输出端**相连——ALU 计算出访存地址 EA = R\[rs] + SignExt(imm16)，不需要额外暂存
  - STORE 指令需要将 **Rt 的值写入数据存储器**——因此寄存器堆的**第二个读数据端口 busB（对应寄存器 Rt）** 直接连接到数据存储器的**写数据端**（DataIn）
  - 数据存储器读出的数据经 **busW** 送入 MemtoReg MUX（而非单独走线），与 ALU 结果共享同一写回路径
  - 控制信号 **MemWr** 控制数据存储器的写使能；控制数据存储器的读/写操作
  - LOAD/STORE 指令计算访存地址时，需要对 16 位立即数进行符号扩展，并通过 ALU 执行**不判溢出的加法（addu）** ，此时控制信号 ALUSrc 被配置为对应的操作码

> [!important] 访存指令的三个新增模块
> 在 R/I 型通路基础上，支持访存需要增加：
> 1. **数据存储器**——与指令存储器分离（哈佛结构），输入端接 ALU 结果（地址）+ 寄存器 Rb 输出（sw 写数据），输出端提供读出数据
> 2. **MemtoReg MUX**——选择写回数据来源（ALU vs 主存），是单周期通路中的第三个 MUX（前两个是 ALUSrc 和 RegDst）
> 3. **控制信号扩展**——新增 MemWr、MemtoReg 两个个信号
> 
> 单周期处理器中，指令存储器和数据存储器**必须分开**（哈佛结构），否则取指与 lw/sw 访存将争用同一存储器造成结构冲突。

> 下图展示了 MIPS 单周期处理器的完整数据通路，整合了取指、R 型、I 型、访存和分支指令的所有数据路径。通过 8 个控制信号（由主控制器 + ALU 控制器产生），在不同指令类型之间切换。

![[单周期数据通路.png]]
其中取指令部件，还需接受三类关键外部控制信号：
- 控制信号 Branch 在执行分支指令时置 1
- 控制信号 Jump 在执行无条件转移指令时置 1
- 标志位 Zero 用于判断分支条件是否成立，该信号由 ALU 输出；例如，执行 beq 指令（相等位移）时，ALU 执行减法操作，若结果为 0，则 Zero 置 1，表示满足转移条件

> [!important] 408 高频考点
> 1. **lw 是唯一同时使 MemRead=1 和 RegWrite=1 的指令**（既要读主存又要写寄存器）
> 2. **sw 是唯一使 MemWrite=1 的指令**（且 RegWrite=0，不写寄存器）
> 3. **R 型指令 ALUSrc=0**（第二操作数来自寄存器），**I 型/lw/sw ALUSrc=1**（第二操作数来自立即数）
> 4. **RegDst** 区分写回目标：R 型=rd，I 型/lw=rt
> 5. **MemtoReg** 区分写回数据来源：ALU=0，主存=1
> 6. **单周期处理的弊端**：每个部件在每条指令中最多只能用一次（如 ALU 不能既算地址又算 PC+4），这导致需要**两个加法器**（一个算 PC+4，一个算分支目标/地址）

## 单周期处理器的特点

- **存储器分离设计**：指令存储器和数据存储器**必须分开**（哈佛结构），因为取指和 lw/sw 访存在**同一个时钟周期**内发生——若共用同一存储器，两者将产生结构冲突。相比之下，多周期处理器可以将两者合并（冯·诺依曼结构），因为取指和访存发生在不同周期
- **无指令寄存器**：单周期通路中**不设 IR**——指令从指令存储器取出后，其各字段（opcode/rs/rt/rd/imm/funct）直接通过组合逻辑连线分发到译码器和寄存器堆地址端，无需寄存器暂存。若加入 IR，仅取指→IR 就需一个时钟周期，无法实现单周期
- **时钟周期由最慢指令决定**：时钟周期必须 ≥ 所有指令中**最长执行路径**的延迟（通常为 lw 指令：取指 + 译码 + ALU算地址 + 访存 + 写回），导致 ADD 等简单指令被迫等待同样长的时间——这是单周期处理器性能低下的根本原因
- **数据通路资源专用**：每个功能部件在一条指令中**最多只能使用一次**——如 ALU 不能既算 PC+4 又算分支目标地址，因此需要**独立的 PC 加法器和分支/访存地址加法器**，硬件利用率低
- **控制信号单周期有效**：所有控制信号在一个时钟周期开始时由 opcode 直接译码产生，**维持整个周期不变**——无需状态机、无需多周期分阶段切换，控制器设计最简单，是硬布线控制器的最简化形式

# 总结

> [!summary] 本章重点
> 1. **数据通路**是运算器的具体实现，描述数据在 CPU 内部的流动路径和加工过程；**控制器**产生控制信号指导数据通路工作
> 2. 数据通路由**组合逻辑元件**（ALU、MUX、译码器、符号扩展、零检测）和**时序逻辑元件**（PC、IR、寄存器堆、MAR、MDR、暂存寄存器、PSW）组成
> 3. 寄存器堆的**读是组合逻辑**（立即输出），**写是时序逻辑**（需时钟沿 + 写使能）
> 4. 按连接方式，数据通路分为**单总线**（简单低效）、**多总线**（平衡）、**专用数据通路**（高速复杂）
> 5. 按时序组织，数据通路分为**单周期**（CPI=1，最慢指令定周期）、**多周期**（不同指令不同周期，部件可复用）、**流水线**（指令重叠执行，需处理冒险）
> 6. 单总线结构下，一次只能传送一个数据，取指需要多步微操作，效率低但结构简单
> 7. MIPS 单周期专用数据通路中，**取指部件**是所有指令公用的基础模块
> 8. **R 型指令**：数据来自两个寄存器（ALUSrc=0），结果写回 rd（RegDst=1），操作由 funct 字段决定
> 9. **I 型指令**：第二操作数是立即数（ALUSrc=1），结果写回 rt（RegDst=0），立即数需符号扩展/零扩展
> 10. **lw 指令**：EX 阶段计算地址，MEM 阶段真正访存（MemRead=1），WB 阶段写回寄存器（MemtoReg=1, RegWrite=1）——是唯一同时读写主存和写寄存器的指令
> 11. **sw 指令**：EX 阶段计算地址，MEM 阶段写主存（MemWrite=1），无 WB 阶段（RegWrite=0）
> 12. **beq 指令**：ALU 做减法比较两寄存器，Zero=1 且 Branch=1 时 PCSrc 有效，PC 更新为分支目标
> 13. 完整数据通路由**主控制器**产生 10 个控制信号（RegDst/RegWrite/ALUSrc/MemtoReg/MemRead/MemWrite/Branch/Jump/ExtOP/ALUOp），加上 **ALU 控制器**根据 ALUOp+funct 生成 ALUctr（共 3~4 位）
> 14. 控制信号真值表是 408 高频考点：需根据指令类型快速确定各信号取值
