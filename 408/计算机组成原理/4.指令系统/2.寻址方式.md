# 指令寻址和数据寻址

**寻址方式**（Addressing Mode）是指确定**本条指令的数据地址**以及**下一条要执行指令的地址**的方法；分为**指令寻址**和**数据寻址**两大类

## 指令寻址

### 顺序寻址

通过**程序计数器（PC）** 自动增量实现：

- 执行完当前指令后，`PC + "1" → PC`
- 这里的 "1" 指一个指令字的长度（对于定长指令字结构，就是指令字长对应的字节数；对于变长指令字结构，PC 增量取决于当前指令长度）
- 若主存按字节编址，定长指令字为 4 字节，则 `PC + 4 → PC`
### 跳跃寻址

通过**转移类指令**（JMP、CALL、RET 等）修改 PC 的值：

- 下条指令地址不是 PC 自动增量产生，而是由指令本身给出
- 实现程序的**分支、循环、调用、返回**等控制流
- 跳跃后按新的 PC 值继续顺序执行

跳跃寻址按目标地址的给出方式，分为两种：

| 类型       | 目标地址            | 指令形式                  | 特点                                       |
| :------- | :-------------- | :-------------------- | :--------------------------------------- |
| **绝对寻址** | PC ← 目标地址       | 指令中直接给出跳转目标的绝对地址      | 可跳转到内存任意位置，但指令较长、不支持位置无关代码               |
| **相对寻址** | PC ← (PC) + 偏移量 | 指令中给出相对当前 PC 的偏移量（补码） | 偏移量位数少、指令短；**支持位置无关代码**（程序整体浮动后跳转目标随之调整） |

> [!note] 与数据寻址中"相对寻址"的关系
> 数据寻址中的**相对寻址**（EA = (PC) + A）与跳跃寻址中的**相对寻址**（PC ← (PC) + offset）原理相同，都是相对 PC 偏移；前者用于**取数据**（如取跳转表、常量池），后者用于**改控制流**（转移指令）；二者共同构成**位置无关代码**的基础

> [!important] 两种寻址的区别
> **顺序寻址**由硬件自动完成（PC 增量器），**跳跃寻址**由转移指令显式指定目标地址。现代 CPU 中还有**分支预测**技术来提前猜测跳跃目标，减少流水线停顿。

## 数据寻址

数据寻址解决"操作数在哪里"的问题。

### 指令格式中的寻址特征

为了支持多种寻址方式，指令中通常设有**寻址特征字段（Mode）**：

```
| 操作码 OP | 寻址特征 Mode | 形式地址 A |
```

- **形式地址**（A）：指令字中给出的地址字段
- **有效地址**（EA）：操作数在主存中的真实地址
- 寻址方式的本质是：**EA = f(Mode, A)**
	- 若为*立即寻址*：则形式地址的位数决定了操作数的位数
	- 若为*直接寻址*：则形式地址的位数决定了直接寻址的范围
	- 若为*寄存器寻址*：则寄存器编号的位数决定了可访问的寄存器数量
	- 若为*寄存器间接寻址*：则寄存器的位数决定了可寻址的内存空间大小
	
### 数据寻址的意义

1. **缩短指令字长**：通过寄存器间接寻址、变址寻址等，指令中只需给出少量位数即可访问大地址空间
2. **扩大访存空间**：16 位地址码直接寻址只能访问 64KB，通过基址/变址寻址可访问更大空间
3. **支持程序重定位**：基址寻址允许程序装入内存的任意位置
4. **支持数据结构和数组**：变址寻址、基址变址寻址可高效访问数组、结构体等

# 常见的数据寻址方式

各类寻址方式的核心公式汇总：

| 寻址方式 | 有效地址 EA | 操作数位置 | 访存次数（取指后） |
|:---:|:---|:---|:---:|
| 立即寻址 | 无（操作数 = A） | 指令字中 | 0 |
| 直接寻址 | EA = A | 主存 | 1 |
| 间接寻址 | EA = (A) | 主存 | ≥2 |
| 寄存器寻址 | 无（操作数在 Ri） | 寄存器 | 0 |
| 寄存器间接寻址 | EA = (Ri) | 主存 | 1 |
| 相对寻址 | EA = (PC) + A | 主存 | 1 |
| 基址寻址 | EA = (BR) + A | 主存 | 1 |
| 变址寻址 | EA = (IX) + A | 主存 | 1 |
| 堆栈寻址 | EA = (SP) | 栈顶 | 0~1 |
| 隐含寻址 | 由指令隐式指定 | 约定位置 | 不定 |

> **注意**：访存次数指取指阶段之后。立即寻址和寄存器寻址操作数已在指令/寄存器中，不需要额外访存。

## 1. 立即寻址（Immediate Addressing）

**操作数直接包含在指令中**，指令中的地址码字段不是地址，而是操作数本身（称为立即数）

```
| OP | Mode | #立即数 |
```

- 特征：指令中地址码字段存放的是**数据本身**，通常用 `#` 前缀表示
- 例：`ADD R1, #5` → `R1 ← R1 + 5`
- 取指令时操作数同时被取出，**不访存取操作数**

| 优点                   | 缺点            |
| -------------------- | ------------- |
| 取指同时取出操作数，**执行速度最快** | 操作数大小受地址码位数限制 |
| 无需额外访存               | 操作数固定，灵活性差    |

> **用途**：常用于初始化寄存器、提供常数（如循环计数器初值、地址偏移量等）

## 2. 直接寻址（Direct Addressing）

指令中**直接给出操作数的有效地址**。

```
| OP | Mode | A（有效地址） |
```
$$EA = A$$
- 取指后，根据 A 再访存一次即可取出操作数
- **访存 1 次**（取操作数）

| 优点           | 缺点              |
| ------------ | --------------- |
| 简单直观，执行时一次访存 | A 的位数限制了直接寻址的范围 |
| 无需额外计算 EA    | 地址固定，不利于程序重定位   |

## 3. 间接寻址（Indirect Addressing）

指令中给出的是**操作数地址的地址**。

```
| OP | Mode | A |
```
- $$EA = (A)$$，即先根据 A 访问一次存储器，读出操作数的有效地址，再根据该地址访存取操作数
- **至少访存 2 次**：第一次取有效地址，第二次取操作数

> 若一次间接仍不能得到操作数，还可**多次间接**：用存储字的最高位标记是否为间接地址（如最高位为 1 表示仍是间址，为 0 表示是有效地址）。

| 优点                          | 缺点         |
| --------------------------- | ---------- |
| 扩大了寻址范围（EA 可放在一个完整存储字中）     | 多次访存，执行速度慢 |
| 便于**实现程序返回和子程序嵌套**，支持地址动态生成 | 多次间接时速度更慢  |

## 4. 寄存器寻址（Register Addressing）

操作数**存放在 CPU 的通用寄存器中**。

```
| OP | Mode | Ri |
```

- 指令中的地址码字段给出**寄存器编号**
- 操作数已在寄存器中，**不需要访存**
- 例：`ADD R1, R2` → `R1 ← R1 + R2`

| 优点                   | 缺点                  |
| -------------------- | ------------------- |
| 操作数在 CPU 内部，无需访问，速度快 | 寄存器数量有限（通常 16~32 个） |
| 指令字长短（只需寄存器编号）       | 编程时需关注寄存器分配         |

> 现代 RISC 架构大量使用寄存器寻址，**运算类指令的操作数都在寄存器中**（Load/Store 架构的核心理念）。

## 5. 寄存器间接寻址（Register Indirect Addressing）

寄存器中存放的是**操作数在主存中的有效地址**

```
| OP | Mode | Ri |
```
$$EA = (Ri)$$
- 先读寄存器获取 EA，再访存取操作数
- **访存 1 次**

| 优点                         | 缺点             |
| -------------------------- | -------------- |
| 比间接寻址少访存 1 次（EA 在寄存器中而非主存） | 需占用寄存器         |
| 扩大寻址范围（取决于寄存器位数）           | 执行前需将 EA 装入寄存器 |

> 与间接寻址对比：**间接寻址**的 EA 存在主存中（多一次访存），**寄存器间接寻址**的 EA 存在寄存器中（更快）

## 6. 隐含寻址（Implied Addressing）

操作数的位置**隐含在指令操作码中**，指令字中不显式给出。

- 例：累加器型机器中 `ADD M` 隐含目的操作数为 ACC：`(ACC) + (M) → ACC`
- 例：`PUSH R1` 隐含栈指针 SP 所指位置
- 例：8086 的 `MUL BL` 隐含被乘数在 AL 中，结果在 AX 中

| 优点      | 缺点           |
| ------- | ------------ |
| 缩短指令字长  | 依赖存储隐含操作数的硬件 |

## 7. 相对寻址（Relative Addressing）

有效地址 = **程序计数器（PC）的内容** + 指令中的**形式地址 A**。

$$
EA = (PC) + A
$$

```
| OP | Mode | A（偏移量，一般用补码表示） |
```

- A 为**相对位移量**，用补码表示，可正可负
- ⭐指令执行时，PC 的值为**当前指令地址 + 指令长度**（**即下一条指令的地址**）
- **访存 1 次**

| 优点 | 缺点 |
|------|------|
| 指令中只需偏移量，指令短 | 寻址范围受偏移量位数限制 |
| **程序可在内存中浮动（位置无关代码）** | 只能相对 PC 寻址 |

> [!important] 核心用途
> 相对寻址广泛用于**转移指令**（条件/无条件跳转）。转移目标地址 = (PC) + 偏移量，由于大部分转移的目标都在附近，偏移量位数不大即可覆盖。

## 8. 基址寻址（Base Addressing）

有效地址 = **基址寄存器（BR）的内容** + 指令中的**形式地址 A**。

$$
EA = (BR) + A
$$

- BR 可由**专用基址寄存器**或**通用寄存器**充当
- 基址寄存器的内容由操作系统设定，在程序执行期间保持不变
- 当使用通用寄存器作为基址寄存器时，尽管用户可以选择哪个寄存器，但其内容仍由操作系统控制
- **访存 1 次**

| 优点                             | 缺点           |
| ------------------------------ | ------------ |
| 扩大寻址范围（BR 位数 ≥ 地址码 A 位数）       | 需额外寄存器       |
| **便于程序动态重定位**（OS 只需修改 BR），简化编程 | 形式地址 A 的位数较短 |
| 支持多道程序                         |              |

> [!important] 基址寻址与重定位
> 操作系统为每个程序分配基址寄存器值。程序中的地址均为相对于基址的偏移量，装入不同内存位置时只需修改 BR，无需修改程序本身。这是**动态重定位**的硬件基础。

## 9. 变址寻址（Indexed Addressing）

有效地址 = **变址寄存器（IX）的内容** + 指令中的**形式地址 A**。

$$
EA = (IX) + A
$$

- IX 可由**专用变址寄存器**或**通用寄存器**充当
- 变址寄存器**面向用户**，其内容可在程序执行过程中**由用户动态修改**，而形式地址 A 保持不变
- **访存 1 次**

| 优点 | 缺点 |
|------|------|
| 便于**数组/向量操作** | 需额外寄存器 |
| 循环中只需修改 IX 即可遍历整个数组 | |

> [!example] 数组访问示例
> 设数组 `arr[0]` 的起始地址为 `ARR_BASE`：
> 1. 将 `ARR_BASE` 作为形式地址 A 放在指令中
> 2. 变址寄存器 IX 依次设为 `0, 4, 8, ...`（设元素占 4 字节）
> 3. 每次 `EA = (IX) + ARR_BASE`，即可顺序访问 `arr[0], arr[1], arr[2], ...`
>
> 循环体中**指令不变，只改 IX**，极大简化了数组遍历的代码。


## 基址寻址 vs. 变址寻址

| 对比维度 | 基址寻址 | 变址寻址 |
|:---|:---|:---|
| **面向对象** | 面向**系统**（OS） | 面向**用户**（程序员） |
| **寄存器内容含义** | 基地址（程序中所有地址的基准） | 偏移量/索引（随循环变化） |
| **谁变谁不变** | BR 不变（或 OS 才改），A 变 | A（基准地址）不变，IX 变 |
| **主要用途** | 程序重定位、多道程序 | 数组遍历、字符串处理、向量运算 |
| **修改权限** | 一般由特权指令修改 | 用户程序可自由修改 IX |
| **典型使用** | 程序逻辑地址 → 物理地址 | `for i=0 to N: arr[i] ← ...` |

## 10. 堆栈寻址（Stack Addressing）

操作数隐含在**栈顶**，通过**栈指针（SP）** 访问。

- 堆栈是主存中一块**后进先出（LIFO）** 的存储区
- SP 始终指向栈顶元素
- `PUSH` 操作：`SP ← SP - 1`（向下生长），然后写入
- `POP` 操作：读出后 `SP ← SP + 1`

根据堆栈的物理实现方式，分为**硬堆栈**和**软堆栈**：

| 类型 | 实现方式 | 速度 | 容量 | 成本 |
|:---|:---|:---:|:---:|:---:|
| **硬堆栈** | CPU 内部的**寄存器组**（移位寄存器）实现，栈顶数据保存在寄存器中 | 极快（无需访存） | 极小（受寄存器数量限制） | 高 |
| **软堆栈** | **主存**中划出一块区域实现，SP 寄存器仅记录栈顶地址 | 较慢（需访存） | 大（取决于主存大小） | 低 |

> 堆栈寻址是一种**隐含寻址**（操作数位置由 SP 隐含指定），常出现在**零地址指令**中。

| 优点 | 缺点 |
|------|------|
| 指令短（操作数隐含） | 应用范围有限 |
| 支持子程序调用/返回 | 堆栈操作较慢（需访存） |

> [!note] 堆栈生长方向
> 不同架构规定不同：x86 堆栈向**低地址**生长（满递减栈），ARM 支持四种堆栈类型（满递增/满递减/空递增/空递减）。


## 11. 复合寻址方式

现代计算机常将多种基本寻址方式组合使用，以实现灵活高效的数据访问。

### 基址变址寻址

$$
EA = (BR) + (IX) + A
$$

- 综合基址和变址双重优势：**重定位 + 数组遍历**
- 需要**两次加法**计算 EA

### 相对变址寻址（x86 常见）

$$
EA = (PC) + (IX) + A
$$

- 先相对、后变址，用于位置无关的数据访问

### 变址间接 / 间接变址

| 方式 | EA 计算 | 说明 |
|:---|:---|:---|
| **先变址后间址** | EA = ((IX) + A) | 先计算地址，再取该地址中的内容作为最终地址 |
| **先间址后变址** | EA = (IX) + (A) | 先取 A 中的内容作为基址，再加 IX |

## 寻址方式综合对比表

| 寻址方式    | EA 公式         | 操作数在 | 访存  | 主要用途     | 灵活性 |
| :------ | :------------ | :--- | :-: | :------- | :-: |
| 立即寻址    | 操作数 = A       | 指令中  |  0  | 常数       |  低  |
| 直接寻址    | EA = A        | 主存   |  1  | 简单变量     |  低  |
| 间接寻址    | EA = (A)      | 主存   | ≥2  | 指针、子程序返回 |  中  |
| 寄存器寻址   | 操作数在 Ri       | 寄存器  |  0  | 运算       |  —  |
| 寄存器间接寻址 | EA = (Ri)     | 主存   |  1  | 指针操作     |  高  |
| 相对寻址    | EA = (PC) + A | 主存   |  1  | 转移指令     |  高  |
| 基址寻址    | EA = (BR) + A | 主存   |  1  | 程序重定位    |  高  |
| 变址寻址    | EA = (IX) + A | 主存   |  1  | 数组处理     |  高  |
| 堆栈寻址    | EA = (SP)     | 主存   | 0~1 | 子程序调用    |  中  |


## x86 典型寻址方式示例

以 x86 为例，展示各类寻址方式的汇编写法：

| 寻址方式 | x86 示例 | 含义 |
|:---|:---|:---|
| 立即寻址 | `MOV AX, 1234H` | AX ← 1234H |
| 寄存器寻址 | `MOV AX, BX` | AX ← BX |
| 直接寻址 | `MOV AX, [1000H]` | AX ← [DS:1000H] |
| 寄存器间接寻址 | `MOV AX, [BX]` | AX ← [DS:BX] |
| 基址寻址 | `MOV AX, [BX+10H]` | AX ← [DS:BX+10H] |
| 变址寻址 | `MOV AX, [SI+10H]` | AX ← [DS:SI+10H] |
| 基址变址寻址 | `MOV AX, [BX+SI]` | AX ← [DS:BX+SI] |
| 相对基址变址寻址 | `MOV AX, [BX+SI+10H]` | AX ← [DS:BX+SI+10H] |

> x86 实模式下：`BX/BP` 为基址寄存器，`SI/DI` 为变址寄存器，段地址默认由 `DS`（数据段）或 `SS`（堆栈段）提供。


# 总结

> [!summary] 本章重点
> 1. **寻址方式** = 指令寻址 + 数据寻址。指令寻址解决"下一条指令在哪"（顺序/跳跃），数据寻址解决"操作数在哪"
> 2. **立即寻址**：操作数在指令中，不访存；速度快但操作数大小受限
> 3. **直接寻址**：EA = A，访存 1 次；简单直观但寻址范围受限
> 4. **间接寻址**：EA = (A)，至少访存 2 次；扩大寻址范围但速度慢
> 5. **寄存器寻址**：操作数在寄存器，不访存；速度最快但寄存器有限——RISC 架构核心寻址方式
> 6. **寄存器间接寻址**：EA = (Ri)，访存 1 次；兼顾速度与灵活性
> 7. **相对寻址**：EA = (PC) + A，访存 1 次；**核心用途是转移指令**，支持位置无关代码（可在内存中任意位置执行）
> 8. **基址寻址 vs. 变址寻址**（408 高频考点）：
>    - **基址寻址**：EA = (BR) + A，面向 OS，BR 是**基准地址**（不变），A 是偏移；用途为**程序重定位**
>    - **变址寻址**：EA = (IX) + A，面向用户，A 是**基准地址**（不变），IX 是变化的**索引**；用途为**数组遍历**
>    - 口诀：**基址变A，变址变IX**
> 9. **堆栈寻址**：隐含寻址的一种，EA = (SP)，操作数在栈顶；支持子程序调用（PUSH/POP、CALL/RET）
> 10. **隐含寻址**：操作数位置由操作码隐含决定（如 ACC、SP），指令字长短但灵活性差
> 11. **访存次数对比**（取指后）：
>
> | 寻址方式 | 立即 | 寄存器 | 直接 | 寄存器间接 | 相对/基址/变址 | 间接 |
> |:---:|:---:|:---:|:---:|:---:|:---:|:---:|
> | 访存次数 | 0 | 0 | 1 | 1 | 1 | ≥2 |
