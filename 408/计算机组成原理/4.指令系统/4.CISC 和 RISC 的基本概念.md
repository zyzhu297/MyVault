# 复杂指令系统计算机（CISC）

**CISC**：Complex Instruction Set Computer

## 主要特点

1. **指令系统复杂庞大**：指令数量多（通常 200 条以上），功能强
2. **指令长度不固定**：变长指令格式，指令字长因指令而异
3. **寻址方式多样**：支持多种寻址方式（可多达十几种）
4. 多数指令可以直接访问内存
5. **指令执行时间差异大**：简单指令和复杂指令的执行周期数相差悬殊
6. **指令使用频率悬殊**：80% 的功能语句只用 20% 的指令
7. **硬件复杂度高**：控制器多采用微程序控制方式，设计复杂
8. **难以实现流水线**：指令格式不规整，流水线效率受限
9. 难以通过优化编译生成高效的目标代码

## 代表机型

- Intel x86 / x86-64 系列
- AMD x86-64 系列
- IBM System/360、VAX-11

# 精简指令系统计算机（RISC）

**RISC**：Reduced Instruction Set Computer

## 主要特点

1. **指令数量少**：只包含使用频率最高的简单指令（通常不超过 100 条）
2. **指令长度固定**：定长指令格式（通常 32 位），便于取指和译码
3. **寻址方式简单**：仅保留最常用的少数几种寻址方式
4. **指令执行时间相近**：绝大多数指令在一个时钟周期内完成
5. CPU 中配备大量寄存器
6. **只有 Load/Store 指令访存**：运算类指令的操作数均来自**寄存器**
7. **以硬布线控制器为主**：控制逻辑简单，执行速度快
8. **易于实现流水线**：指令格式规整，非常适合指令流水线
9. **强调编译器优化**：通过编译优化来提高程序执行效率

## 代表机型 / 架构

- MIPS
- ARM（嵌入式 / 移动端主流）
- RISC-V（开源指令集架构）
- PowerPC、SPARC、Alpha

# CISC 与 RISC 的比较

与 CISC 相比，RISC 的优势有：

- **更高效利用芯片面积**
	- CISC 采用微程序控制器，其**控制存储器占据了 CPU 芯片面积的 50% 以上**
	- RISC 采用硬布线控制器，逻辑电路只占芯片面积的 **10% 左右**
	- 节省出的面积可用于放置更多**通用寄存器**和**高速缓存（Cache）**，进一步提升性能

- **更高的运算速度**
	- RISC 指令格式规整、定长，**取指和译码速度更快**
	- 绝大多数指令**单周期**完成，CPI 接近 1
	- 易于实现**指令流水线**，大幅提高指令吞吐率
	- 大量寄存器减少了访存次数，而访存本身就是性能瓶颈

- **更易设计与维护**
	- 指令数量少、格式统一，**设计周期短**，便于验证
	- 硬布线控制器的逻辑相对固定，**易于调试和排错**
	- 指令集的向前兼容压力小

- **更利于编译优化**
	- 指令格式规整，编译器生成目标代码时**选择空间大、优化策略明确**
	- 大量寄存器为**寄存器分配优化**提供了充足资源
	- Load/Store 结构使编译器可以更精确地控制数据流向，更好地进行**指令调度**

| 比较维度      | CISC               | RISC                 |
| --------- | ------------------ | -------------------- |
| **指令数量**  | 多（>200 条）          | 少（<100 条）            |
| **指令字长**  | 变长                 | 定长（通常 32 位）          |
| **寻址方式**  | 多种（十几种）            | 少（几种）                |
| **访存方式**  | 运算指令可直接访存          | 仅 Load/Store 访存      |
| **执行时间**  | 差异大（多周期）           | 趋近相同（单周期为主）          |
| **控制器**   | 多为微程序控制            | 多为硬布线控制              |
| **流水线**   | 可以通过一定方式实现         | **必须实现**             |
| **控制方式**  | 绝大多数为微程序控制         | 绝大多数为组合逻辑控制          |
| **编译优化**  | 硬件承担更多复杂操作，编译器压力较小 | 指令简单，需要编译器更好安排寄存器和指令 |
| **硬件复杂度** | 高                  | 低（但寄存器数量多）           |
| **软件复杂度** | 低（汇编编程较方便）         | 高（指令简单，编程量大）         |
| **应用场景**  | PC、服务器（x86）        | 嵌入式、移动设备（ARM）、IoT    |
>[!warning] CISC 的兼容性远好于 RISC
> **x86（CISC）是兼容性的天花板**：Intel 从 1978 年的 8086 到现在，始终保持**二进制级向后兼容**——四十多年前的程序理论上仍能在最新 CPU 上运行。这是靠微程序控制器实现的：通过更新微程序来支持新指令，同时保留旧指令的微程序不变。
> **RISC 不把兼容性放在首位**：RISC 追求精简和高效，新版本架构可能裁剪旧指令、改变编码格式。比如 MIPS 从 MIPS I 到 MIPS V 经历了多次不兼容变更，ARM 从 ARM32 到 AArch64 也是全新指令编码。
## 发展趋势

- **相互借鉴、走向融合**：
  - 现代 CISC 处理器（如 x86）内部将复杂指令**翻译为类 RISC 微操作**执行
  - 现代 RISC 处理器也引入了一些复杂指令（如多媒体指令）
- **CISC 不代表落后，RISC 不代表先进**：各有适用场景
- 当代高性能 CPU 往往采用 **CISC 外壳 + RISC 内核** 的混合设计

# 408 考点提示

1. **选择题高频考点**：区分 CISC 和 RISC 的特点，尤其是**指令长度、访存方式、控制器类型、流水线适应性**
2. **常见陷阱**：
   - ❌ "RISC 的指令一定比 CISC 少"（趋势上如此，但不是绝对）
   - ❌ "CISC 不能流水线"（只是难以高效流水线，现代 x86 也实现了流水线）
   - ❌ "RISC 速度快就一定比 CISC 好"（取决于应用场景）
3. **重点记忆 RISC 的核心特征**：定长指令、Load/Store 结构、硬布线、适合流水线
