# 半导体随机存取存储器

**随机存取存储器**（Random Access Memory, RAM）是构成主存储器的最核心部件。根据存储原理的不同，半导体 RAM 分为 **SRAM**（静态 RAM）和 **DRAM**（动态 RAM）两大类，**均为易失性存储器**

## SRAM（静态随机存取存储器）

SRAM 使用**双稳态触发器**（通常由 6 晶体 MOS 管构成）来存储一位二进制信息。

### 存储原理

- 每个存储单元是一个**双稳态触发器**电路，具有两个稳定状态，分别表示 0 和 1
- 只要不断电，触发器的状态就**稳定保持**，信息不会丢失
- 读出时不会破坏原有信息（**非破坏性读出**）

> [!note] 为什么 SRAM 快？
> SRAM 本质上是触发器电路，状态翻转极快；读出时只需检测触发器状态（或通过位线差分放大），无需像 DRAM 那样先读出再重写。

## DRAM（动态随机存取存储器）

DRAM 使用**电容**上存储的电荷来存储一位二进制信息。

### 存储原理

- 每个存储单元由一个 **MOS 管 + 一个电容**（1T1C 结构）构成
- 电容上有电荷 → 表示 "1"；电容上无电荷 → 表示 "0"
- 由于电容存在**漏电**，电荷会逐渐泄放，因此信息只能保持很短时间（通常 **2ms ~ 64ms**）

### 破坏性读出与刷新

DRAM 的读出是**破坏性读出**：读出时电容上的电荷会发生变化（被检测电路分享/放掉），读出后原存储信息被破坏。

因此 DRAM 必须在每次读出后立即**重写**（再生），并且在未访问期间也必须定期**刷新**（Refresh）。

> [!important] 刷新（Refresh）的本质
> 刷新就是周期性地按**行**对存储单元进行一次**读-重写**操作，补充电容电荷。刷新按行进行，一次刷新一行中的所有存储单元。

### 刷新方式

| 刷新方式 | 操作时机 | 特点 |
|----------|----------|------|
| **集中刷新** | 在固定的刷新周期内，**集中一段时间**连续对所有行刷新 | 刷新期间存**取死区时间**较长，但控制简单 |
| **分散刷新** | 将刷新操作**分散**到每个存取周期之后 | 存取周期变长（= 读写时间 + 刷新时间），但无死区 |
| **异步刷新** | 在规定的刷新间隔内，**均匀分散**安排各行刷新 | 兼顾死区时间和存取周期，最常用 |

**刷新周期计算**：若 DRAM 需要在 2ms 内完成全部 $N$ 行的刷新，则平均每 $\frac{2 \text{ms}}{N}$ 刷新一行。

## SRAM 与 DRAM 的对比

| 对比维度       | SRAM                      | DRAM                                     |
| ---------- | ------------------------- | ---------------------------------------- |
| **存储原理**   | 双稳态触发器（6 晶体 MOS）          | 栅极电容存储电荷（1 MOS 管 + 1 电容/位，1T1C 结构）       |
| **存储单元结构** | 6MOS 管双稳态触发器              | 1T1C（1 管 + 1 电容）                         |
| **集成度**    | **低**（每 bit 需多个晶体管，占用面积大） | **高**（每 bit 只用 1T1C，面积小），约为 SRAM 的 4~6 倍 |
| **速度**     | **快**（存取时间几 ns ~ 十几 ns）   | **较慢**（存取时间几十 ns）                        |
| **功耗**     | 较大（始终有电流维持状态）             | 较小                                       |
| **送地址**    | 行列同时送                     | 分两次送（复用）                                 |
| **是否需要刷新** | **不需要**（只要不断电，信息自动保持）     | **需要**（电容漏电，必须定期刷新，刷新间隔通常 2ms~64ms）      |
| **读出方式**   | **非破坏性读出**                | **破坏性读出**（读出后需重写/再生）                     |
| **单位成本**   | **高**                     | **低**                                    |
| **典型用途**   | **Cache**（高速缓冲存储器）        | **主存储器**（内存条）                            |
| **芯片容量**   | 较小（KB~MB 级）               | 较大（GB 级）                                 |

## 存储芯片的组成

| 电路                         | 功能                             | 说明                                 |
| -------------------------- | ------------------------------ | ---------------------------------- |
| **存储矩阵**（Memory Array）     | 以矩阵形式排列的大量存储单元，是真正存放二进制数据的物理实体 | 如 $1024 \times 1024$ 矩阵 = 1M 个存储单元 |
| **地址译码器**（Address Decoder） | 将地址信号译码为字线选通信号，**选中唯一**对应的存储单元 | 分为行译码和列译码两部分                       |
| **I/O 电路**（I/O Buffer）     | 控制数据在内部存储矩阵与外部数据引脚之间的输入/输出缓冲 | 读出时经**灵敏放大器**→ 输出三态门驱动到数据总线；写入时经**输入缓冲器**→ 写入驱动送入存储矩阵 |
| **片选控制电路**（Chip Select Logic） | 根据片选信号决定芯片是否被选中工作，并参与读写控制的使能逻辑 | $\overline{\text{CS}}$ 有效时芯片正常工作；$\overline{\text{CS}}$ 无效时芯片进入**低功耗待机**，数据引脚呈**高阻态** |
| **读写控制电路**（R/W Control）    | 根据片选和读写使能信号，控制数据的读出或写入         | 包含三态门输出缓冲器和写入驱动电路                  |

### 地址译码方式

地址译码器的核心任务是将 $n$ 位地址信号译码为 $2^n$ 根字线中的**唯一一根**选通信；根据译码电路的结构，分为**单译码法**和**双译码法**两种。

| 对比维度          | 单译码法             | 双译码法                            |
| ------------- | ---------------- | ------------------------------- |
| **译码器数量**     | 1 个（$n$ → $2^n$） | 2 个（行 + 列，各译一半）                 |
| **选择线总数**     | $2^n$            | $2^{n/2} + 2^{n/2}$（行 + 列）      |
| **门电路规模**     | 随地址位数**指数增长**    | 随地址位数**线性增长**（对比单译码大幅缩小）        |
| **适用容量**      | 小容量              | **大容量**（几乎所有现代存储芯片）             |
| **DRAM 中的体现** | —                | DRAM 的**地址复用**正是建立在双译码的行列分离基础之上 |

## DRAM 芯片的关键技术
### 地址引脚复用技术

DRAM 芯片容量大、地址线多，但芯片封装**引脚数量有限**。为在有限的引脚数下支持大容量寻址，DRAM 采用**地址引脚复用**（Address Multiplexing）技术：将地址分为**行地址**和**列地址**两部分，**分时**通过同一组地址引脚送入芯片。

**工作流程**：
1. 芯片先接收**行地址**，由 **$\overline{\text{RAS}}$**（Row Address Strobe，行地址选通）信号锁存
2. 芯片再接收**列地址**，由 **$\overline{\text{CAS}}$**（Column Address Strobe，列地址选通）信号锁存
3. 行、列地址经译码后选中存储矩阵中唯一的交叉点，交叉点上的所有位平面上的数据被同时读和写

| 特性 | 说明 |
|------|------|
| **引脚节省** | 若容量为 $2^n$ 单元，传统需 $n$ 根地址线；复用后仅需 $n/2$ 根，引脚数**减半** |
| **本质前提** | DRAM 的**双译码结构**（行译码 + 列译码）为地址复用提供了硬件基础 |
| **代价** | 地址分两次送入，**地址建立时间变长**，但相对于 DRAM 本身较长的存取时间而言影响不大 |

> [!note] 为什么 SRAM 不用地址复用？
> SRAM 追求**速度**，一次送出完整地址可以省去行列分时传送的等待时间；且 SRAM 容量较小，地址引脚数本就不多，没有复用必要。

> [!important] 考试辨析
> DRAM 采用地址复用技术的**根本原因**是：芯片引脚数有限，而 DRAM 容量大、所需地址线多。**双译码结构**是地址复用的实现基础（行、列分时送入），而不是根本原因。

### 刷新机制
DRAM 的刷新不仅是**维持数据不丢失**的必要操作，其效率也与存储阵列的物理设计密切相关。

**刷新与阵列结构的关系**：

- DRAM 存储阵列按**行 × 列**矩阵排列，刷新以**行**为单位
- 一次刷新操作：选中一行 → 将整行数据读入**灵敏放大器**（Sense Amplifier）→ 放大后**自动写回**该行的所有存储单元
- 一行中的所有电容**同时**被刷新，无需逐位进行

**灵敏放大器的双重角色**：

| 工作模式     | 灵敏放大器的作用                              |
| -------- | ------------------------------------- |
| **正常读写** | 检测位线上微弱的电压变化，放大后输出到数据总线               |
| **刷新**   | 读出行数据后，利用自身的驱动能力将放大的信号**写回**电容，完成电荷补充 |

> [!note] 为什么灵敏放大器能"顺便"做刷新？
> 在 DRAM 读出过程中，灵敏放大器将电容上的微小电压差放大为全摆幅逻辑电平（0/1），这些放大了的电平信号会**自动驱动回位线**，重新对电容充电——这就是"读后再生"的物理基础。刷新本质上就是一次**只读不输出**的"假读"操作。

DRAM 芯片内部集成一个**刷新行地址计数器**，用于在刷新操作时自动提供**待刷新的行地址**，其位数与行地址位数相同；行地址缓冲器与刷新计数器**通过一个多路选择器（MUX）共享**通往行译码器的地址通路；**刷新周期**定义为对某一特定行完成一次刷新后，到下一次对该行再次刷新的时间间隔

> [!warning] 刷新对性能的影响
> 刷新操作需要**占用存储体的访问周期**（$t_{\text{RFC}}$，Row Refresh Cycle Time），在此期间该 Bank **不能响应读写请求**。当刷新频率较高时，刷新开销会降低有效存储带宽。因此 DRAM 控制器需要精心调度刷新与正常访问的时机。

### 阵列设计优化

DRAM 采用地址引脚复用技术，引脚数量由行列地址位数中的较大者决定，为最小化地址引脚数，通常设计为行数略小于或等于列数的近似正方形结构

### 缓存机制和突发传输

现代 DRAM（SDRAM 及后续 DDR 系列）芯片内部引入了一系列**加速数据传输**的机制，核心思想是：**一次行激活 → 多次列访问** + **连续地址自动传输**。

**行缓冲（Row Buffer）机制**：

DRAM 芯片内部设有一个**行缓冲器**（**通常由 SRAM 实现**），用于缓存被选中行中所有列的数据；其容量等于一行中所有存储单元的数据总量，即**列数 × 每个存储单元的位数**；当某一行被选中后，该行的全部数据被**一次性**加载到行缓冲器中

**突发传输（Burst Transfer）**：

突发传输是核心提速机制：提供**起始地址**和**突发长度**后，芯片内部自动生成后续地址序列，**连续输出/读入**多个数据，无需逐次提供地址，在每个时钟周期连续输出一个存储单元的数据

| 参数 | 含义 |
|------|------|
| **突发长度**（Burst Length, BL） | 一次突发传输的数据个数，典型值 BL = 4 或 BL = 8 |
| **突发类型** | **顺序**（Sequential）：地址线性递增；**交织**（Interleaved）：地址按位异或顺序变化 |

> [!tip] 行缓冲 + 突发传输 = 高带宽
> 当访问模式为连续地址（如数组遍历）时，一次行激活即可覆盖多个突发传输的所有数据——全部命中 Page Hit，配合突发传输以流水线方式持续输出数据，DRAM 的实际带宽可**接近理论峰值**。这正是低位交叉编址 + DRAM 行缓冲 + 突发传输的协同效果。

## 同步 DRAM

**SDRAM**（Synchronous DRAM）是相对于传统**异步 DRAM** 而言的。其核心特征是：芯片的所有操作（读写、刷新、预充电等）均与**系统时钟（CLK）的上升沿同步**，能够以 CPU-主存总线的较高速率运行；在连续访问同一行（页）内的数据时，可实现突发传输，**显著减少甚至避免插入等待状态**；SDRAM 在系统时钟驱动下工作，它将 CPU 发出的地址和控制信号锁存，并在预设的若干时钟周期后返回数据并完成写入，使得 CPU 无需等待，可在延迟期间执行其他指令，显著提升系统性能

# 非易失性存储器

**非易失性存储器**（Non-Volatile Memory）指断电后信息不丢失的存储器。其结构简单，位密度高于 SRAM 等可读/写存储器；主存储器中通常使用 **ROM 芯片**来存放固化程序（如 BIOS / UEFI），外部存储器中广泛使用 **Flash 闪存** 和新兴的 **SSD**；

| 类型 | 存储单元结构 | 写入方式 |
|------|-------------|----------|
| **MROM** | 制造时用掩膜工艺直接固化（有/无 MOS 管） | 制造时写入，不可更改 |
| **PROM** | 存储单元串有熔丝（fuse） | 用户一次性烧断熔丝编程 |
| **EPROM** | 浮栅 MOS 管（FAMOS），紫外线擦除 | 紫外线照射擦除后重新编程 |
| **EEPROM** | 浮栅 MOS 管（FLOTOX），电擦除 | 在线电擦除，可逐字节改写 |
| **Flash** | 改进的浮栅 MOS 管 | 在线电擦除，按**块**擦写 |

> [!important] Flash 与 EEPROM 的关键区别
> - EEPROM 可以**按字节**擦除和改写
> - Flash 只能**按块（Block）** 擦除，擦除粒度大但速度快、成本低
> - Flash 广泛用于 U 盘、SSD、存储卡

## Flash 存储器

Flash 存储器（又称闪存）使用**浮栅 MOS 管**（Floating Gate MOSFET）存储信息，是一种在 EPROM 基础上发展而来的非易失性存储器，现代主板普遍采用 Flash 存储器存储 BIOS；兼具 ROM 和 RAM 的部分优点：断电后长期保存；支持电擦除和在在线重写；无需紫外线照射等特殊设备；读取速度接近 RAM，但写入速度显著较慢，**读/写性能不对称**

## 固态硬盘（SSD）

**固态硬盘**（Solid State Drive）是以 Flash 为存储介质的大容量外部存储器，由控制单元和存储单元组成；继承了 Flash 存储器的重要特性，相比于传统硬盘，SSD 具有读/写速度快、**功耗低**、抗震性强等优势，缺点是价格高，且写入寿命受限于 Flash 存储器的擦写次数，详见[[3.磁盘和固态硬盘#固态硬盘 (SSD)]]

# 多模块存储器

单个体存储器的存取周期限制了存储系统的整体速度。当 CPU 速度远高于主存速度时，主存成为系统瓶颈。
**多模块存储器**（Multi-Module Memory）通过将存储体划分为多个**独立模块**，使多个模块可以**并行或交叉工作**，从而提高存储系统的等效带宽；是一种**空间并行技术**。
根据编址方式不同，多模块存储器分为**连续编址**和**交叉编址**两大类

## 连续编址

**连续编址**的核心特征是：**相邻地址存放于同一模块内**，地址空间在各模块间按顺序分段排列。包括**单体多字**和**高位交叉编址**两种实现。

### 单体多字存储器

**单体多字**（Single-Bank Multi-Word）结构中，存储体只有一个，但每个存储字包含多个字（如 4 个字），CPU 一次访存可同时读出/写入多个连续字。本质上是一种**空间并行**方式——以加宽数据位宽换取等效带宽。

| 特点 | 说明 |
|------|------|
| **优点** | 一次访存取出多条指令/数据，提高指令和数据带宽；结构简单，只需一套译码和控制电路 |
| **缺点** | 要求指令和数据**连续存放**；遇分支跳转 → 已取出的后续指令全部浪费；访存粒度粗、灵活性差 |

> [!question] 单体多字解决了什么问题？
> **核心矛盾**：CPU 执行速度快，主存存取周期长——若每次只取一条指令/一个数据，CPU 频繁**等待主存**，严重拖慢系统性能（即"存储墙"问题）。
> 
> **解决思路**：利用程序的**局部性原理**（指令顺序执行、数据连续存放），在一次存取周期 $T$ 内**多取几个连续的字**。这样：
> - 单次访存延迟 $T$ 不变，但取回了 $k$ 个字 → 等效带宽提升 **$k$ 倍**
> - 取指令场景：一次取出多条连续指令，后续几条指令**无需再访存**，CPU 可直接从指令缓冲中取走
> - 本质是**用空间（加宽总线/存储字）换时间（减少访存次数）**

### 高位交叉编址（顺序编址）

高位交叉编址中，用**高位地址**来选择存储体，**低位地址**作为体内地址。

$$
\text{地址格式：}\quad \underbrace{\text{体号（高位）}}_{k \text{ 位}}\;\underbrace{\text{体内地址（低位）}}_{n \text{ 位}}
$$

| 特性 | 说明 |
|------|------|
| **地址分布** | 相邻地址位于**同一个**存储体内（顺序排列） |
| **并行性** | 各存储体本质上还是**串行**工作（连续地址访问集中在同一体） |
| **适合场景** | 多道程序环境，不同程序使用不同存储体 |
| **别名** | 顺序编址 |

> [!note] 高位交叉的局限
> 当 CPU 连续访问一片地址时（如顺序取指令），所有访问都落在同一个存储体内，需要等当前存取周期结束后才能进行下一次访问，**无法利用多体并行优势**。

## 交叉编址

**交叉编址**的核心特征是：**相邻地址分布于不同模块**，多个模块可以**流水化轮流工作**。主要实现是**低位交叉编址**。

### 低位交叉编址（模 m 编址）

低位交叉编址中，用**低位地址**来选择存储体，**高位地址**作为体内地址。

$$
\text{地址格式：}\quad \underbrace{\text{体内地址（高位）}}_{n \text{ 位}}\;\underbrace{\text{体号（低位）}}_{k \text{ 位}}
$$

| 特性       | 说明                       |
| -------- | ------------------------ |
| **地址分布** | 相邻地址分布在**不同**的存储体内（交叉排列） |
| **并行性**  | 连续地址访问可以**流水化/并行**进行     |
| **适合场景** | 顺序取指令、数组遍历等连续地址访问        |
| **别名**   | 交叉编址、模 m 编址（m = 存储体个数）   |

> [!note] 交叉编址的底层协同
> 交叉编址的高带宽效果并非孤立的地址编排技巧，而是与之前章节的多项 DRAM 技术形成**层层叠加**的协同效应：DRAM 的**行缓冲**（一次激活覆盖连续地址）→ **突发传输**（连续拍发出数据）→ **低位交叉编址**（跨存储体流水化访问）。三者叠加时，实际带宽可**接近理论峰值**。

### 流水线分析

设有 $m$ 个存储体，每个存储体的存取周期为 $T$（含恢复时间），则在低位交叉编址下，启动各存储体的时间间隔为：

$$
\tau = \frac{T}{m} \quad \text{（理想情况，各体轮流启动）}
$$

- 在一个存取周期 $T$ 内，可以依次启动全部 $m$ 个存储体的访问
- 等效**存储带宽**可提升到单体的 **$m$ 倍**（理想情况）

> [!important] 考试计算要点
> **连续访存参数的流水线分析**：
> - 设存储体数 $m$，存取周期 $T$，总线传输周期 $\tau$
> - 连续读出 $n$ 个字的总时间：$T + (n-1)\tau$
> - 若 $\tau = T/m$，则交叉访问可以无缝衔接
> - 若 $n \gg m$，等效带宽接近单体带宽的 $m$ 倍

### 轮流启动 vs 同时启动

多模块存储器的核心性能差异取决于模块的**启动方式**：是**轮流错开**启动（流水化），还是**同时**启动（集中输出）。

**轮流启动**（交叉编址 / 低位交叉）：

- 各模块以间隔 $\tau = T/m$ **依次错开**启动
- 单个模块的存取周期仍是 $T$，但各模块的 $T$ 在时间轴上**重叠覆盖**
- 从 CPU/总线的视角：等效存取周期缩短为 $\tau$，**每隔 $\tau$ 就能获得一个数据**

**同时启动**（连续编址 / 高位交叉 或 单体多字）：

- 所有模块（或所有字）在**同一时刻**启动访问
- 经过 $T$ 后，$m$ 个数据**同时到达**
- 然后必须等待**完整的下一个** $T$ 才能发起下一批访问
- **内存访问要求数据对齐**，否则一个数据需要两次访问

| 对比维度 | 轮流启动 | 同时启动 |
|----------|---------|---------|
| **编址方式** | 低位交叉编址 | 高位交叉编址 / 单体多字 |
| **单模块存取周期** | $T$（不变） | $T$（不变） |
| **等效存取周期** | $\tau = T/m$（缩短） | $T$（不变，甚至因等待更差） |
| **数据输出节奏** | **连续均匀**，每 $\tau$ 一个数据 | **突发集中**，等待 $T$ 后一次性输出 $m$ 个 |
| **总线利用率** | **高**（接近 100%） | 低（数据突发后长时间空闲） |
| **并行本质** | **时间并行**（流水重叠） | **空间并行**（同时读出多字） |
| **带宽公式** | $B = m / T$（等效） | $B = m / T$（瞬时，但不可持续） |

> [!important] 存取周期 $T$ 的核心结论
> 无论在哪种启动方式下，**单个存储模块的物理存取周期 $T$ 都不会改变**——它由存储介质的物理特性决定。
> 
> 两种方式的本质区别在于**对 $T$ 的利用策略**：
> - **轮流启动**：将各模块的 $T$ 在时间轴上**错位叠加**，隐藏了单个模块的等待时间，等效于"榨干"了总线带宽
> - **同时启动**：各模块的 $T$ 完全同步对齐，等待和忙碌同时发生，总线利用率低
> 
> 考试中"存取周期"通常指单模块的 $T$，而"等效存取周期"或"总线传输周期"才是 $\tau$，**二者不可混淆**。

## 连续编址 vs 交叉编址

| 对比维度       | 连续编址          | 交叉编址          |
| ---------- | ------------- | ------------- |
| **核心特征**   | 相邻地址在同一模块     | 相邻地址在不同模块     |
| **体号位置**   | 在地址**高位**     | 在地址**低位**     |
| **具体实现**   | 连续编址（高位交叉编址）  | 低位交叉编址        |
| **并行方式**   | 串行（高位交叉）      | 时间并行（流水化访问）   |
| **连续访问效率** | 低（串行等待）       | **高**（流水化并行）  |
| **灵活性**    | 要求数据连续存放      | 可分别访问各体中的任意地址 |
| **指令流场景**  | 遇分支指令浪费严重     | 遇分支可只取所需      |
| **主要用途**   | 扩大存储容量、多道程序隔离 | **提高存储带宽**    |
