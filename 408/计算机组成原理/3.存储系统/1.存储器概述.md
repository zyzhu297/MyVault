# 存储器的分类

存储器是计算机系统中的**记忆设备**，用来存放程序和数据。计算机中的全部信息，包括输入的原始数据、计算机程序、中间运行结果和最终运行结果都保存在存储器中。

> [!important] 存储器的核心地位
> 存储器是计算机**五大部件**之一，其性能直接影响整机的性能。在现代计算机中，存储系统往往是最大的性能瓶颈，即所谓的「**存储墙**」（Memory Wall）问题。

## 按存储介质分类

| 类型 | 存储原理 | 典型实例 | 特点 |
|------|----------|----------|------|
| **半导体存储器** | 利用半导体器件（如双稳态触发器、电容）存储信息 | DRAM、SRAM、Flash | 体积小、功耗低、存取速度快 |
| **磁表面存储器** | 利用磁层表面的剩磁状态存储信息 | 磁盘、磁带 | 非易失、容量大、速度较慢 |
| **光盘存储器** | 利用激光在盘片上烧制凹凸点存储信息 | CD-ROM、DVD、蓝光 | 非易失、容量大、便于携带 |

## 按存取方式分类

| 类型 | 存取方式 | 特点 | 典型实例 |
|------|----------|------|----------|
| **随机存取存储器**（RAM） | 可按地址随机访问任一存储单元，存取时间与物理位置**无关** | 读写方便、使用灵活，但断电后信息丢失（易失性） | DRAM、SRAM |
| **顺序存取存储器**（SAM） | 按记录顺序从存储介质的始端开始读写，存取时间与物理位置**有关** | 存取速度慢，适合批量数据存储 | 磁带 |
| **直接存取存储器**（DAM） | 先直接定位到一个小区域（磁道），再在该区域内顺序查找 | 既有随机存取的定位能力，又有顺序存取的查找过程 | 磁盘（机械硬盘） |
| **只读存储器**（ROM） | 正常工作时只能随机读出，不能随机写入 | 非易失性，断电后信息不丢失 | ROM、PROM、EPROM、EEPROM |
| **相联存储器**（CAM） | 按**内容**而非地址进行存取 | 可并行比较，速度快但成本高 | TLB、部分 Cache |

> [!note] RAM 与 ROM 的本质区别
> RAM 和 ROM 都是随机存取（均可按地址访问任一单元），区别在于**可写性**：RAM 可随机读写，ROM 只能随机读、写入需特殊操作（甚至不可写入）。

### ROM 的类型细分

| 类型 | 全称 | 编程方式 | 可擦除性 |
|------|------|----------|----------|
| **MROM** | Mask ROM（掩膜式 ROM） | 制造时掩膜写入 | 不可擦除 |
| **PROM** | Programmable ROM | 用户一次性编程（烧断熔丝） | 不可擦除 |
| **EPROM** | Erasable PROM | 用户可多次编程 | 紫外线擦除 |
| **EEPROM / E²PROM** | Electrically Erasable PROM | 用户可多次编程 | 电擦除，可按字节擦写 |
| **Flash Memory**（闪存） | 闪速存储器 | 用户可多次编程 | 电擦除，按块擦写 |

> [!note] Flash 与 EEPROM 的关系
> Flash 也是一种 EEPROM，但擦除以**块**为单位（而非字节），擦写速度更快，成本更低。现代 SSD、U 盘均使用 NAND Flash。

## 按信息的可保存性分类

| 类型 | 断电后 | 典型实例 |
|------|--------|----------|
| **易失性存储器**（Volatile） | 信息丢失 | RAM（DRAM、SRAM） |
| **非易失性存储器**（Non-volatile） | 信息保留 | ROM、Flash、磁盘、光盘 |

> [!warning] 注意区分
> - **破坏性读出** vs **非破坏性读出**：属于读出方式的分类。DRAM 是破坏性读出（读出后需要重写/再生），SRAM 是非破坏性读出。
> - **易失性** vs **非易失性**：属于信息保存性的分类。不要将二者混淆。

## 按在计算机中的作用（层次）分类

| 层次 | 存储器类型 | 作用 |
|------|-----------|------|
| **高速缓冲** | Cache（通常由 SRAM 构成） | 缓和 CPU 与主存之间的速度矛盾 |
| **主存储器** | 主存 / 内存（通常由 DRAM 构成） | 存放当前正在运行的程序和数据 |
| **辅助存储器** | 辅存 / 外存（磁盘、SSD、光盘等） | 存放大量暂不使用的程序和数据 |

# 主存储器的组成和基本操作

## 主存储器的基本组成

主存储器（Main Memory, MM）通常由**存储体**（Memory Array / Bank）、**地址寄存器**（MAR）、**数据寄存器**（MDR）和**读写控制电路**组成。

### 各部件功能

| 部件          | 英文                           | 功能                            | 位宽          |
| ----------- | ---------------------------- | ----------------------------- | ----------- |
| **存储体**     | Memory Array / Bank          | 存储信息的物理实体，由大量存储单元构成           | —           |
| **存储地址寄存器** | MAR（Memory Address Register） | 存放待访问的存储单元**地址**              | **等于地址线位数** |
| **存储数据寄存器** | MDR（Memory Data Register）    | 存放从存储体读出或即将写入存储体的**数据**       | **等于数据线宽度** |
| **读写控制电路**  | Read/Write Control           | 根据 CPU 发出的读/写控制信号，控制存储器的读/写操作 | —           |

> [!important] MAR 与 MDR 的位置
> 在现代计算机中，MAR 和 MDR 通常集成在 **CPU 芯片内部**，而非存储芯片内部。但逻辑上它们属于主存储器与 CPU 之间的接口。

### 存储体的基本结构

- **记忆单元**（存储元）：存放**一位**二进制信息（0 或 1）的基本电路单元
- **存储单元**：一个记忆单元集合，通常以**字节（Byte）** 或**字（Word）** 为单位，是一组拥有相同地址的记忆单元
- **存储字长**：每个存储字包含的位数，通常为 8 位（1 字节）或其整数倍
- **地址**：每个存储字的唯一标识编号，从 0 开始顺序编号

存储体的组织形式：**存储单元数 × 存储字长**

- 如 $1\text{M} \times 8$ 位 = 1MB，表示有 $2^{20}$ 个存储单元，每个存储单元存储 8 位数据

### 译码驱动方式

| 方式 | 原理 | 特点 |
|------|------|------|
| **线选法**（Linear Selection） | 用一根字线直接选中一个存储字的所有位 | 简单，但需要 $2^n$ 根选择线（$n$ 为地址位数），适用于小容量存储器 |
| **重合法**（Coincident Selection） | 用行地址和列地址两根选择线交叉选中存储单元 | 选择线数大幅减少为 $2^{n/2} + 2^{n/2}$，适用于大容量存储器 |

## 主存储器的基本操作

主存储器与 CPU 之间的基本操作有两种：**读操作**（Read）和**写操作**（Write）。

### 读操作流程

1. CPU 将待读取单元的**地址**送入 **MAR**
2. CPU 通过**地址总线**将地址发送给主存
3. CPU 发出**读控制信号**（$\overline{\text{RD}}$ / Read Enable）
4. 主存根据地址译码，选中对应存储单元
5. 主存将选中单元的数据读出到 **MDR**
6. CPU 通过**数据总线**从 MDR 取走数据

### 写操作流程

1. CPU 将待写入单元的**地址**送入 **MAR**
2. CPU 将待写入的**数据**送入 **MDR**
3. CPU 发出**写控制信号**（$\overline{\text{WR}}$ / Write Enable）
4. 主存根据地址译码，选中对应存储单元
5. 主存将 MDR 中的数据写入选中单元

> [!note] 读/写周期的关键信号
> - **片选信号**（$\overline{\text{CS}}$ / Chip Select）：选中该存储芯片
> - **读使能**（$\overline{\text{OE}}$ / Output Enable 或 $\overline{\text{RD}}$）：控制读出
> - **写使能**（$\overline{\text{WE}}$ / Write Enable 或 $\overline{\text{WR}}$）：控制写入
> - 读操作时 $\overline{\text{CS}} + \overline{\text{OE}}$ 有效；写操作时 $\overline{\text{CS}} + \overline{\text{WE}}$ 有效

# 存储器的层次化结构

存储器的**速度**、**容量**和**价格/每位**三个指标之间存在着根本矛盾：

| 存储器类型 | 速度 | 容量 | 价格/每位 |
|-----------|------|------|-----------|
| 寄存器 | 最快 | 极小（几十~几百字节） | 极高 |
| Cache（SRAM） | 很快 | 小（KB~MB 级） | 高 |
| 主存（DRAM） | 较快 | 中等（GB 级） | 中等 |
| 辅存（磁盘/SSD） | 慢 | 极大（TB 级） | 低 |

> [!important] 核心矛盾
> 速度越快的存储器，单位价格越高、容量越小。单一类型的存储器无法同时满足**高速、大容量、低成本**三个要求。

### 程序访问的局部性原理

层次化结构之所以有效，根本原因在于**程序访问的局部性原理**（Principle of Locality）：

| 类型 | 含义 | 典型表现 |
|------|------|----------|
| **时间局部性**（Temporal Locality） | 最近被访问的信息，在不久的将来**很可能再次被访问** | 循环中的指令和数据反复访问 |
| **空间局部性**（Spatial Locality） | 某地址被访问后，其**邻近地址**在不久的将来很可能被访问 | 顺序指令执行、数组顺序遍历 |

> [!note] 局部性原理是体系结构的基石
> 正是因为程序具有局部性，才使得将少量高速存储器（Cache）+ 中量中速存储器（主存）+ 大量低速存储器（辅存）组合成一个逻辑上的存储系统成为可能——大多数访存在高层即可命中，整体表现出接近高层存储器的速度和接近底层存储器的容量。

## 三级层次结构

现代计算机普遍采用 **Cache — 主存 — 辅存** 三级层次结构，**上一层的内容始终是下一层内容的子集副本**，上一层存储器作为下一层的高速缓存

### 两个重要层次

| 层次               | 速度       | 解决的问题                 | 实现方式                      | 对用户/程序员                 |
| ---------------- | -------- | --------------------- | ------------------------- | ----------------------- |
| **Cache — 主存层次** | 接近于Cache | 解决 CPU 与主存之间的**速度矛盾** | 完全由**硬件**实现               | **透明**（不可见）             |
| **主存 — 辅存层次**    | 接近于主存    | 解决主存**容量不足**的问题       | 由**硬件 + 操作系统**共同实现（虚拟存储器） | **不透明**（程序逻辑地址到物理地址的映射） |

> [!important] 两个层次的区别
> - Cache — 主存层次：速度的匹配，对**所有程序员透明**
> - 主存 — 辅存层次：容量的扩展，对**应用程序员透明**，但对**系统程序员不透明**

### 层次化结构的有效性

层次化结构之所以有效，必须满足以下条件：

1. **程序具有局部性**：确保大部分访存集中在高层（Cache / 主存）
2. **层次间存在包含关系**：高层内容是低层内容的子集（Cache 中的数据一定是主存中某块的副本）
3. **数据一致性维护**：当高层数据被修改时，需通过**写策略**（写直达/写回）同步到底层

# 存储器的主要指标

## 存储容量

**存储容量**（Storage Capacity）指存储器能够存放的二进制信息总量。

### 表示方式

| 表示方法             | 含义        | 示例                                             |
| ---------------- | --------- | ---------------------------------------------- |
| **存储单元数 × 存储字长** | 直接表示物理结构  | $1\text{M} \times 8$ 位，表示 $2^{20}$ 个单元，每单元 8 位 |
| **字节数**（Bytes）   | 以字节为单位的容量 | 1MB、4GB、1TB                                    |

**地址线数量与容量的关系**：若 MAR 有 $n$ 位，则最多能寻址 $2^n$ 个存储字。

$$
\text{可寻址最大存储字数} = 2^n
$$

## 存取时间

**存取时间**（Access Time），又称**存储器访问时间**，指从**启动一次存储器操作**到**完成该操作**所经历的时间

| 操作 | 存取时间含义 |
|------|-------------|
| **读出时间** | 从给出有效地址到数据出现在数据总线上并稳定可用的时间 |
| **写入时间** | 从给出有效地址和待写入数据到数据被正确写入存储单元的时间 |

- 存取时间是反映存储器**速度**的最根本指标
- 记作 $T_A$，单位通常为 ns（纳秒）

## 存取周期

**存取周期**（Memory Cycle Time）指存储器做**两次连续独立的存取操作**所需的最小时间间隔。

> [!important] 存取时间 ≠ 存取周期
> 存取周期通常**大于**存取时间。原因是：
> - 破坏性读出的存储器（如 DRAM），读出后需要时间进行**重写/再生**
> - 地址线、数据线、控制线可能存在**信号稳定时间**要求
> 
> 即：存取周期 = 存取时间 + 恢复时间

记作 $T_M$（或 $T_C$）。

## 存储带宽

**存储器带宽**（Memory Bandwidth）指存储器在单位时间内能传输的**最大信息量**，反映数据传输速率。

### 公式

$$
\text{存储器带宽} = \frac{\text{数据总线宽度（位）}}{\text{存取周期（秒）}} \quad \text{（单位：位/秒 或 字节/秒）}
$$

### 提高存储带宽的方法

| 方法            | 原理               |
| ------------- | ---------------- |
| **缩短存取周期**    | 使用更快的存储芯片        |
| **增加数据总线宽度**  | 一次读出/写入更多位       |
| **采用多体交叉存储器** | 多个存储体轮流工作，流水化访问  |
| **增加 Cache**  | 减少 CPU 直接访问主存的次数 |
