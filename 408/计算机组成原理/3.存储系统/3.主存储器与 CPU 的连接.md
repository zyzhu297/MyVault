# 连接原理

主存储器通过**三组总线**与 CPU 连接：

- **地址总线 (Address Bus)**：CPU → 主存，单向。传送访存地址，位数 $n$ 决定**最大寻址范围** $2^n$。
- **数据总线 (Data Bus)**：CPU ↔ 主存，双向。传送数据，位数 $m$ 决定了**一次能传送的位数**（机器字长 / 存储字长）。
- **控制总线 (Control Bus)**：CPU → 主存，部分单向/双向。包括 $\overline{\text{MREQ}}$（访存请求）、$\text{R}/\overline{\text{W}}$（读/写控制）、$\overline{\text{MFC}}$（存储器完成信号）等。

# 主存容量的扩展

当单片存储芯片的容量不满足要求时，需用多片芯片**组合扩展**。扩展方式取决于不满足的维度：

| 不满足的维度 | 扩展方式 |
|:-----------:|:--------:|
| 数据线位数不够 | **位扩展** |
| 地址线个数不够（单元数不够） | **字扩展** |
| 两者都不够 | **字位同时扩展** |

## 总芯片数计算

$$N = \frac{\text{目标存储器容量}}{\text{单片芯片容量}}$$

其中容量 = 单元数 × 数据位数，分别对应字和位方向上的倍数。

## 位扩展

**适用场景**：单片芯片的数据位数 < 系统数据线宽度，但地址单元数够用。

**方法**：多片芯片**并联**，各片的地址线、片选线和 R/W 控制线**同名端并联**，同时工作，数据线**各自独立**连接到数据总线的不同位。

**原则**：**并联逐位分担** —— 各片负责数据总线的不同位段。

> 位扩展的特点是：**同时选中所有芯片**（片选连在一起），各片贡献一部分数据位。

## 字扩展

**适用场景**：单片芯片的地址单元数 < 系统地址空间的单元数，但数据位数够用。

**方法**：多片芯片**串联**（通过片选信号区分），各片的数据线与**低位**地址线**同名端并联**，**高位**地址线经**译码器**产生片选信号来选中不同芯片。

**原则**：**字向由片选扩充** —— 通过片选逻辑区分不同地址范围。

> 字扩展的特点是：**任一时刻只有一片被选中**，通过译码器 + 片选实现地址范围划分。

## 字位同时扩展

**适用场景**：单片芯片的地址单元数和数据位数都不满足要求。

**方法**：先做**位扩展**形成"芯片组"（数据位数满足要求），再做**字扩展**将"芯片组"叠加（地址单元数满足要求）。

**原则**：**组内并联（位扩展），组间串联（字扩展）**。

# 连接设计的通用步骤

1. **分析需求**：确定目标存储器的容量（单元数 × 字长）和可用芯片规格。
2. **计算芯片数**：
   - 数据方向倍数 $m = \frac{\text{目标字长}}{\text{芯片字长}}$
   - 字方向倍数 $n = \frac{\text{目标单元数}}{\text{芯片单元数}}$
   - 总芯片数 $= m \times n$
3. **地址线分配**：
   - 芯片内部地址线：接系统低位地址（位数 = $\log_2(\text{芯片单元数})$）
   - 剩余高位地址：送入译码器产生片选信号
4. **译码器设计**（若需要字扩展）：$k$ 条高位地址线 → $2^k$-条输出的译码器
5. **画连接图**：标明地址线、数据线、控制线的接法
