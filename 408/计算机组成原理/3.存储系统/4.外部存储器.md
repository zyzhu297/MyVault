# 磁盘存储器

**磁盘存储器**（Hard Disk Drive, HDD）是以**磁表面**为存储介质的直接存取存储器（DAM），是现代计算机系统中最主要的大容量、非易失性外部存储设备。

> [!important] 磁盘的数据组织层次
> 磁盘的数据组织从大到小依次为：**盘片组 → 柱面 → 磁道 → 扇区**。理解这个层次关系是掌握磁盘地址和访存过程的基础。

## 磁盘设备的组成

### 磁盘存储器的组成

磁盘存储器由**磁盘驱动器**、**磁盘控制器**和**盘片**三大部分组成，此外还包含**磁盘高速缓存**（Disk Cache）用于缓冲数据。

- 磁盘驱动器是**机械/物理执行部件**，驱动磁盘旋转并通过磁头在盘面上执行读/写操作
- 磁盘控制器是主机与磁盘驱动器之间的**接口逻辑电路**，负接受并解析来自 CPU 的命令，向磁盘驱动器发送控制信号，同时监听其运行状态

### 存储区域

磁盘的存储空间由**记录面**、**磁道/柱面**和**扇区**三个层次构成。

- 记录面数：即**盘面数**，等于**磁头数**，一片盘片有上下两个记录面或只有一面
- 柱面数：即**磁道数**，所有盘面中同一半径位置的磁道构成一个柱面
- 扇区数：每条磁道上的弧形记录区段数量，是磁盘**最小的独立读写单位**

### 磁盘高速缓存

**磁盘高速缓存**（Disk Cache）是在**内存中**开辟的一块空间，用于暂存待写入的数据；优点是：磁盘以**簇**为单位进行写操作，缓存可以减少频繁的小块写入；同时，中间结果若在写回前被再次使用，可直接从缓存读取，提升效率

## 磁记录原理

当磁头和磁性记录介质发生相对运动时，通过电磁转换实现数据的读/写操作

## 磁盘的性能指标

### 记录密度

磁盘的记录密度分为**道密度**和**位密度**：

- 道密度（Track Density）：沿磁盘**半径方向**单位长度内的磁道数，单位为 **TPI**（Tracks Per Inch）
- 位密度（Bit Density）：沿磁道**圆周方向**单位长度内记录的二进制位数，单位为 **BPI**（Bits Per Inch）
- 面密度：是位密度和道密度的乘积，反应单位面积的存储能力
### 磁盘容量

磁盘容量分为**非格式化容量**和**格式化容量**：

- 非格式化容量 = 记录面数 × 柱面数 × 每磁道磁化单元数
- 格式化容量 = 记录面数 × 柱面数 × 每道扇区数 × 每扇区字节数（是按特定格式组织后实际可用的存储容量）

> [!note] 格式化容量小于非格式化容量
> 因为每条磁道需要留出扇区间隙、ECC 校验字段、同步字段等开销。

### 响应时间和存取时间

- 响应时间：从发出读/写请求到**数据传输完成**的总时间，包括**排队延迟**、**控制器时间**和**存取时间**
- 存取时间 = **寻道时间** + **旋转延迟** + **传输时间**（是衡量磁盘性能的核心指标）
  - 寻道时间：磁头臂移动到目标柱面所需时间，通常取最大寻道时间的一半（从最外道到最内道时间的二分之一）
  - 旋转延迟：目标扇区旋转到磁头下方所需时间，平均为转半圈的时间
  - 传输时间：实际读出/写入数据所需时间，取决于磁盘转速和数据密度
### 数据传输速率

数据传输速率指磁盘在单位时间内能向主机传送的字节数；若磁盘转速为 $r$ 转/秒。单磁道容量为 $N$ 字节，则**最大数据传输速率**为$$ D_r = rN $$
## 磁盘地址

磁盘访问时，主机需要向磁盘控制器提供完整的**磁盘地址**。磁盘地址由以下字段组成：

### 地址格式

$$
\text{磁盘地址的组成：}\quad \underbrace{\text{驱动器号}}_{\text{多磁盘时}} \; + \; \underbrace{\text{柱面号（磁道号）}}_{\text{高位}} \; + \; \underbrace{\text{盘面号（磁头号）}}_{\text{中位}} \; + \; \underbrace{\text{扇区号}}_{\text{低位}}
$$

| 地址字段              | 含义              | 作用                    |
| ----------------- | --------------- | --------------------- |
| **驱动器号**          | 标识具体的物理磁盘设备     | 系统中有多块硬盘时区分目标磁盘       |
| **柱面号**（Cylinder） | 标识目标柱面          | 控制磁头臂**寻道**，移动到对应柱面   |
| **盘面号**（Head）     | 标识目标盘面（即选择哪个磁头） | 通过电子开关**切换磁头**，无需机械移动 |
| **扇区号**（Sector）   | 标识目标磁道中的具体扇区    | 等待扇区旋转到磁头下方后开始读写      |

### 磁盘地址的计算

设磁盘参数：$C$ 个柱面、$H$ 个盘面（磁头）、$S$ 个扇区/道，则：

- 总扇区数 = $C \times H \times S$
- 逻辑扇区号 → 物理地址的转换关系：
  - 柱面号 = $\lfloor$ 逻辑扇区号 $/ (H \times S)$ $\rfloor$
  - 盘面号 = $\lfloor$ (逻辑扇区号 $\bmod$ $(H \times S)$) $/ S$ $\rfloor$
  - 扇区号 = 逻辑扇区号 $\bmod \; S$

> [!note] 地址编排策略
> 同一柱面内的扇区**先按盘面变化、再按扇区变化**进行连续编号，这样连续的逻辑扇区不会触发寻道操作（只切换磁头 / 等待旋转），可有效减少平均存取时间。

## 磁盘工作过程

磁盘的主要操作包括寻址、读盘和写盘，每种操作对应一个控制字；磁盘工作时，先读取控制字，然后执行该控制字；由于磁盘是机械式部件，因此其读/写操作为**串行执行**

## 磁盘阵列（RAID）

**RAID**（Redundant Array of Independent Disks，独立冗余磁盘阵列）是将多块独立的物理磁盘按一定方式组合，形成逻辑上的**一个大容量、高性能、高可靠的**磁盘系统。

> [!important] RAID 的核心目标
> RAID 通过**数据分块**（Striping）提高并行性和传输速率，通过**冗余信息**（镜像或奇偶校验）提高可靠性。不同 RAID 级别在这两个目标之间权衡取舍。

### RAID 0 — 无冗余条带化

| 特性        | 说明                                  |
| --------- | ----------------------------------- |
| **原理**    | 数据按**条带（Stripe）分块**，**交替均匀分布**到多块磁盘 |
| **优点**    | 读写**完全并行**，容量利用率为 **100%**，传输速率大幅提升 |
| **缺点**    | **无冗余**，任何一块磁盘故障即导致全部数据丢失           |
| **最少磁盘数** | 2 块                                 |
| **适用场景**  | 对速度要求极高、但数据可重新生成（如视频编辑暂存）           |

### RAID 1 — 镜像

| 特性 | 说明 |
|------|------|
| **原理** | 每块数据盘对应一块**镜像盘**，数据同时写入两块磁盘（镜像复制） |
| **优点** | 读速度提升（可从任意副本读取），可靠性极高（任何一块故障不影响） |
| **缺点** | 容量利用率仅 **50%**，写入速度无提升（需同时写两份） |
| **最少磁盘数** | 2 块 |
| **适用场景** | 对可靠性要求极高的系统（操作系统盘、关键数据库） |

### RAID 2 — 位级条带 + 海明码纠错

| 特性 | 说明 |
|------|------|
| **原理** | 数据按**位**分散到多块数据盘，用多块校验盘存放**海明码** |
| **特点** | 可纠正单错、检测双错；但校验盘数量随数据盘对数增长，磁盘利用率低 |
| **现状** | **已被淘汰**：现代磁盘自带 ECC，无需在 RAID 层面做位级纠错 |

### RAID 3 — 字节级条带 + 专用奇偶校验盘

| 特性 | 说明 |
|------|------|
| **原理** | 数据按**字节**条带化到多块数据盘，单独一块**专用校验盘**存放奇偶校验信息 |
| **优点** | 大块连续读写性能好；单盘故障可恢复 |
| **缺点** | 校验盘成为**写瓶颈**（每次写入都要更新校验盘）；小数据量随机写性能差 |
| **最少磁盘数** | 3 块 |
| **现状** | 较少使用 |

### RAID 4 — 块级条带 + 专用奇偶校验盘

| 特性 | 说明 |
|------|------|
| **原理** | 与 RAID 3 类似，但数据按**块**（Block）条带化，块大小适中 |
| **优点** | 支持一定的并发读取 |
| **缺点** | 校验盘仍是**写瓶颈**（所有写操作都要经过同一块校验盘） |
| **最少磁盘数** | 3 块 |
| **现状** | 较少使用，被 RAID 5 取代 |

### RAID 5 — 块级条带 + 分布式奇偶校验

| 特性 | 说明 |
|------|------|
| **原理** | 数据按块条带化，奇偶校验信息**均匀分布在所有磁盘上**（无专用校验盘） |
| **优点** | 消除 RAID 4 的校验盘瓶颈；读写性能均衡；允许**1 块盘故障** |
| **缺点** | 写入时有"读-改-写"开销（每写一个块需读取旧数据和旧校验、计算新校验、写入新数据和新校验）；重建过程慢 |
| **最少磁盘数** | 3 块 |
| **适用场景** | 文件服务器、Web 服务器等读多写少场景 |

> [!note] RAID 5 的"读-改-写"惩罚
> 每当向 RAID 5 写入一个数据块时：
> 1. 读出旧数据块和旧校验块
> 2. 计算新校验块：新校验 = 旧数据 XOR 新数据 XOR 旧校验
> 3. 写入新数据块和新校验块
>
> 一次写入涉及 **4 次磁盘访问（2 读 + 2 写）**，写放大显著。

### RAID 6 — 块级条带 + 双分布式奇偶校验

| 特性 | 说明 |
|------|------|
| **原理** | 类似 RAID 5，但使用**两种独立的奇偶校验算法**（如 P + Q），分布到所有磁盘 |
| **优点** | 允许**同时 2 块磁盘故障**而不丢失数据 |
| **缺点** | 奇偶校验计算复杂，写放大更大；容量利用率比 RAID 5 低 |
| **最少磁盘数** | 4 块 |
| **适用场景** | 大型存储阵列、归档存储等可靠性要求非常高的场景 |

### RAID 组合级别

| 组合级别 | 原理 | 特点 |
|----------|------|------|
| **RAID 10**（RAID 1+0） | 先镜像（RAID 1），再条带化（RAID 0） | 兼具高性能和高可靠性；最少 4 块盘；容量利用率 50% |
| **RAID 01**（RAID 0+1） | 先条带化（RAID 0），再镜像（RAID 1） | 与 RAID 10 容量相同但可靠性略差（一组条带中的任意盘故障 → 整组失效） |
| **RAID 50**（RAID 5+0） | 先 RAID 5 组，再 RAID 0 条带化 | 容量利用率和性能的折中；最少 6 块盘 |

> [!important] RAID 10 vs RAID 01 的关键区别
> 两者物理结构不同，容错能力不同：
> - **RAID 10**：每组镜像内只要不超过 1 块盘故障，整阵列可用
> - **RAID 01**：任一条带组内 1 块盘故障即导致该组失效，需全组重建

### RAID 级别对比汇总

| 级别 | 最少盘数 | 容错能力 | 容量利用率 | 读性能 | 写性能 | 适用场景 |
|------|----------|----------|------------|--------|--------|----------|
| RAID 0 | 2 | 无 | 100% | ★★★ | ★★★ | 高性能暂存 |
| RAID 1 | 2 | 1 块 | 50% | ★★ | ★ | 系统盘、关键数据 |
| RAID 3 | 3 | 1 块 | $(n-1)/n$ | ★★ | ★ | 大块顺序读写 |
| RAID 5 | 3 | 1 块 | $(n-1)/n$ | ★★ | ★ | 文件/Web 服务器 |
| RAID 6 | 4 | 2 块 | $(n-2)/n$ | ★★ | ★ | 归档存储 |
| RAID 10 | 4 | 每组 1 块 | 50% | ★★★ | ★★ | 数据库 |

# 固态硬盘（SSD）

**固态硬盘**（Solid State Drive, SSD）是以 **NAND Flash** 为存储介质的大容量外部存储器，由**控制单元**（控制器 + 固件）和**存储单元**（Flash 芯片阵列）组成。

> [!important] SSD 与 HDD 的本质区别
> SSD 完全没有机械运动部件，所有操作均为电子操作。这一根本差异导致 SSD 在速度、功耗、抗震性等方面对 HDD 形成代差优势。

## 固态硬盘的特性

### 核心特性

| 特性 | 说明 |
|------|------|
| **随机访问快** | 无寻道和旋转延迟，随机读取延迟通常在 **几十微秒**（HDD 为毫秒级） |
| **读写不对称** | 读取速度快（接近 DRAM），写入速度较慢（需先擦除再写入） |
| **先擦后写** | NAND Flash 写入前必须先**擦除**（Erase），擦除以**块**为单位 |
| **有限擦写寿命** | 每个存储单元的擦写次数有限（通常几千到几万次），超出后单元失效 |
| **低功耗** | 无机械运动部件，功耗远低于 HDD |
| **抗震性强** | 无磁头、盘片等精密机械部件，抗冲击、抗振动 |
| **静音** | 无机械噪音 |
| **写放大**（Write Amplification） | 实际写入闪存的物理数据量大于主机发送的逻辑数据量 |

> [!warning] 读/写/擦除的粒度差异
> NAND Flash 的三种基本操作的粒度不同：
> - **读/写（Program）**：以**页**（Page）为单位（通常 4KB~16KB）
> - **擦除（Erase）**：以**块**（Block）为单位（通常包含 64~256 页，即几 MB）
>
> 这是造成"写放大"和"垃圾回收"开销的根本原因——即使只修改页中几个字节，也需要将整块内容读出 → 修改 → 擦除整块 → 写回。

## 磨损均衡（Wear Leveling）

由于 NAND Flash 每个存储单元的擦写次数有限，磨损均衡是 SSD 中**延长整体寿命**的关键技术。

### 磨损不均衡问题

如果某些存储块（如存放操作系统的固定区域）被反复擦写，而其他块很少被使用，会导致这些"热点"块过早失效，进而导致整个 SSD 报废。

### 磨损均衡策略

| 策略 | 原理 | 特点 |
|------|------|------|
| **动态磨损均衡**（Dynamic Wear Leveling） | 每次写入时，从**空闲块池**中选择擦写次数最少的块使用 | 仅对空闲块做均衡；对已占用且长期不修改的"冷数据"无效 |
| **静态磨损均衡**（Static Wear Leveling） | 主动将**冷数据**（长期不变的数据）从擦写次数少的块**搬运**到擦写次数多的块，释放出低磨损块用于后续写入 | 全盘均衡效果更好，但引入额外数据搬运开销 |

> [!important] 动态 vs 静态磨损均衡
> - **动态**：只均衡"当下写入"的目标选择，开销小但效果不彻底
> - **静态**：主动"搬家"——把冷数据搬到磨损多的块，把高频写入引导到磨损少的块，确保全盘均匀老化
> - 现代 SSD 普遍采用两种策略结合的方式



> [!note] 复习建议
> 1. **磁盘性能计算**：寻道时间 + 旋转延迟 + 传输时间的分步计算是必考题型，务必熟练
> 2. **磁盘地址结构**：柱面号—盘面号—扇区号的层次关系，以及逻辑扇区号与物理地址的转换
> 3. **RAID 各级别对比**：408 常考 RAID 0/1/5/6/10 的特点、最少磁盘数、容错能力、容量利用率
> 4. **SSD 的关键特性**：读/写/擦除粒度、写放大、FTL 功能、磨损均衡策略（动态/静态）
> 5. **SSD vs HDD 综合对比**：从存储介质、性能、功耗、可靠性等多个维度掌握
