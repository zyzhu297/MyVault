# 虚拟存储器的基本概念

**虚拟存储器**（Virtual Memory）是计算机系统中用于解决主存**容量不足**的核心技术。它将**主存**和**辅存**（如磁盘、SSD）统一编址，构成一个逻辑上远大于实际主存的地址空间

> [!important] 虚拟存储器的核心思想
> 将**一部分暂时不用的程序和数据**放在辅存中，仅在需要时调入主存。以此用辅存的大容量来弥补主存容量的不足，同时隐藏主存与辅存之间的数据交换细节，对应用程序员透明，而对系统程序员不透明

## 虚拟存储器的基本工作原理

- 程序给出**逻辑地址**（Logical Address，又称虚拟地址 VA），由硬件（MMU 等）和 OS 共同将其转换为**物理地址**（Physical Address，PA）
- 程序运行时，只将**当前需要的**页面/段调入主存，其余部分保留在辅存中
- 当访问的信息不在主存中时，产生**缺页中断**（Page Fault），由 OS 负责将所需页面从辅存调入主存


## 虚拟存储器与 Cache 的对比

虚拟存储器和 Cache 都基于**程序访问的局部性原理**，均将底层存储中频繁访问的数据缓存到上层更快的存储介质中。

### 相同之处

- 最终目标都是提高系统性能，体现了容量、速度、价格的梯度
- 都利用**程序的局部性原理**（时间局部性 + 空间局部性）实现层次化存储
- 都将地址划分为**两部分**：高位标识块/页、低位标识块内/页内偏移
- 都需要进行**地址映射**：Cache 用 Tag 比较判断命中，虚拟存储器用页表/段表实现映射
- 都存在**替换策略**：Cache 用硬件 LRU，虚拟存储器用软件 LRU/Clock 等
- 都需要维护**数据一致性**：Cache 有写直达/写回策略，虚拟存储器同样采用**类似写回策略**
- 都引入**高速缓冲加速**：Cache 本身就是缓冲，虚拟存储器用 TLB 加速地址转换
- 都需要**有效位**标记数据是否有效，需要**脏位**标记是否被修改

### 不同之处

| 对比项目   | Cache               | 虚拟存储器                 |
| ------ | ------------------- | --------------------- |
| 目标     | 弥补 CPU 与主存的**速度**差距 | 弥补主存与辅存的**容量**差距      |
| 实现方式   | 纯硬件                 | 硬件 + OS               |
| 映射方式   | 直接/组相联/全相联          | 全相联（通过页表）             |
| 未命中处理  | 硬件控制调入块             | OS 缺页中断处理             |
| 未命中代价  | 较小（几十~几百时钟周期）       | 极大（磁盘访问，百万周期量级）       |
| 地址转换   | 硬件比较 Tag            | 硬件查页表 + TLB           |
| 对程序员透明 | 对**所有**程序员透明        | 对**应用程序员**透明，对系统程序员可见 |

# 页式虚拟存储器

**页式虚拟存储器**（Paged Virtual Memory）是目前**最常用**的虚拟存储器实现方式。它将虚拟地址空间和物理主存空间都划分为**大小固定**的块，分别称为**页**（Page）和**页框**（Page Frame / 物理页）

## 页表

**页表**（Page Table）是实现虚实地址映射的核心数据结构。每个**进程**拥有自己的页表，页表存放在**主存**中，由 OS 维护。

## 页表的结构

页表本质上是一个数组，以**虚页号**为索引：

- 每个页表项（Page Table Entry，PTE）对应一个虚页
- PTE 中存放该虚页对应的**实页号**（若已在主存）和**控制信息**

### 页表项的组成

| 字段                                    | 功能                            | 说明                                                        |
| ------------------------------------- | ----------------------------- | --------------------------------------------------------- |
| **有效位**（Valid / Present Bit）          | 标识该虚页是否已在主存中                  | $V=1$ 表示已在主存，PTE 中的实页号有效；$V=0$ 表示不在主存（在辅存），访问时将触发**缺页中断** |
| **修改位**（Dirty Bit / Modified Bit）     | 表示该页被调入后是否被修改过                | $D=1$ 表示被修改过，被替换时必须写回辅存；$D=0$ 表示未被修改，可直接丢弃                |
| **访问位**（Reference Bit / Accessed Bit） | 记录该页最近是否被访问过                  | 用于 OS 的页置换算法（如 Clock 算法）                                  |
| **实页号 / 外存地址**（PPN / Disk Address）    | 存放该虚页对应的主存页框号 **或** 该页在辅存中的地址 | **双用途字段**：$V=1$ 时存实页号 PPN，$V=0$ 时存**外存（磁盘）地址**，供缺页中断时定位页面 |

> [!important] 页表项字段的双重用途
> 当 $V=0$ 时，该页不在主存中，存放"实页号"的那些位被重新用于存放**该页在辅存（磁盘/SSD）中的地址**。

> [!important] 页表与 Cache 表项的对比
> | 相似点 | 差异 |
> |--------|------|
> | 都有有效位（V）、脏位（D） | Cache 有 **Tag** 字段做匹配，页表用虚页号做**索引**直接定位 |
> | 都需要替换信息 | Cache 替换由**硬件**完成，页置换由 **OS 软件**完成 |
> | — | 页表存于**主存**，由 OS 管理；Cache 表在**Cache 硬件**内部 |

## 地址转换

虚拟地址到物理地址的转换由 **MMU** 硬件完成：

1. CPU 发出**虚拟地址** VA，将其拆分为**虚页号** VPN 和**页内偏移** Offset
2. 以 VPN 为索引查**页表**（PTE 地址 = **页表基址寄存器** PTBR + VPN × PTE 大小）
3. 检查 PTE 的**有效位**：
   - $V=1$：取 PTE 中的**实页号** PPN，**物理地址 PA = PPN ∥ Offset**（页内偏移直接复用，不转换）
   - $V=0$：触发**缺页中断**，OS 从 PTE 中取出该页的**外存地址**，从辅存调入页面，更新 PTE（V 置 1、写入 PPN），重新执行该指令

## 缺页中断处理

当 $V=0$ 时触发的缺页中断（Page Fault）处理流程：

1. **保护现场**：保存当前指令的 PC 和其他寄存器
2. OS 在**主存中寻找空闲页框**，若无空闲则执行**页置换**（Page Replacement）
3. 从 PTE 中取出**外存地址**，据此从辅存中调入所需页面到分配的页框中
4. **更新页表**：将 PTE 的 V 置 1，写入实页号（覆写原来的外存地址）
5. **恢复现场**，重新执行触发缺页的指令（这次将命中）

## 快表（TLB）

**快表**（Translation Lookaside Buffer，TLB），又称**地址转换后备缓冲器**，由**高速 SRAM 构成**，是专门用于**加速地址转换**的高速缓存。

### TLB 的基本原理

TLB 是 **页表的 Cache**，将近期使用过的页表项缓存起来：

- TLB 位于 **MMU 内部**（或靠近 MMU），速度极快（1~2 个时钟周期）
- TLB 保存**虚页号 → 实页号**的映射关系（即 PTE 的子集）
- TLB 命中时，**无需访问主存中的页表**，直接获得实页号

### TLB 的结构

| 字段 | 功能 | 说明 |
|------|------|------|
| **Tag（标记）** | 存放**虚页号** | 与 VA 中的 VPN 比较以判断命中 |
| **Data（数据）** | 存放**实页号**（及部分控制位） | 即 PPN + V、D、保护位等 |
| **有效位**（V） | 标识该 TLB 表项是否有效 | 进程切换时需要刷新（或加 ASID） |

> [!important] TLB 与普通 Cache 的关键区别
> - TLB 缓存的是**页表项**（虚→实地址映射），普通 Cache 缓存的是**数据/指令**
> - TLB 通常采用**全相联**映射（容量极小，速度要求极高）

### TLB + 页表的地址转换流程

1. CPU 发出虚拟地址 VA
2. MMU 以 VPN 并行查询 **TLB**：
   - **TLB 命中**（TLB Hit）：直接从 TLB 获取实页号 PPN → 拼接得到物理地址 PA
   - **TLB 未命中**（TLB Miss）：访问主存中的**页表**，获得 PTE
     - 若 PTE 的 $V=1$：将映射关系**填入 TLB**，然后继续转换
     - 若 PTE 的 $V=0$：触发**缺页中断**

### 进程切换与 TLB

- 进程切换时，虚拟地址空间改变，TLB 中旧的映射项失效
- 处理方法：
  - **TLB 刷新**（Flush）：简单但开销大
  - **ASID**（Address Space ID）：每个 TLB 表项带一个进程标识，避免刷新

### 各种命中/未命中的组合

| TLB  | Page      | Cache | 含义                   | 结果                                    |
| ---- | --------- | ----- | -------------------- | ------------------------------------- |
| Hit  | 快表命中则一定命中 | Hit   | 最佳情况，无需访问主存          | TLB → PA → Cache 命中，**数据返回最快**        |
| Hit  | 快表命中则一定命中 | Miss  | 访问一次主存               | TLB → PA → 访问主存取数据填入 Cache            |
| Miss | Hit       | Hit   | 访问一次主存               | 查页表得 PA → 更新 TLB → Cache 命中（需多 1 次访存） |
| Miss | Hit       | Miss  | 访问两次主存               | 查页表得 PA → 更新 TLB → 访问主存取数据            |
| Miss | Miss      | —     | **缺页**，需要访问磁盘和至少两次主存 | 触发缺页中断，从辅存调页                          |
>[!note] 缺失处理
>Cache 的缺失由硬件自动完成；缺页处理由操作系统通过“缺页异常处理程序”实现；TLB 缺失既可用硬件处理，也可用软件处理

# 段式虚拟存储器

**段式虚拟存储器**（Segmented Virtual Memory）以**段**（Segment）为单位管理存储空间。段是**可变长度**的逻辑单元，通常对应程序的逻辑划分（如代码段、数据段、堆栈段）。

## 段式管理的特点

| 特点 | 说明 |
|------|------|
| **段长可变** | 每个段的长度可以不同，由程序逻辑决定 |
| **逻辑独立性** | 段对应程序的逻辑结构，便于**共享和保护** |
| **按段分配** | OS 按段的需求大小在主存中分配连续空间 |

## 段表

与页表类似，段式管理使用**段表**（Segment Table）实现虚实地址转换：

| 段表项字段 | 含义 |
|-----------|------|
| **段首址**（Base） | 该段在主存中的起始物理地址 |
| **段长**（Limit / Length） | 该段的长度 |
| **有效位**（V） | 该段是否在主存中 |
| **保护位** | 访问权限控制 |

> [!warning] 越界检查
> 段式管理**必须**检查段内偏移是否超出段长，若 $D \geq \text{Limit}$ 则触发**越界中断**（地址越界）。这是段式管理与页式管理的重要区别——页式管理中页内偏移不会越界（都是固定大小）。

## 段式的优点与缺点

| 优点                                  | 缺点                                      |
| ----------------------------------- | --------------------------------------- |
| **逻辑清晰**：段对应程序的逻辑结构，便于理解和编程，**不透明** | **外部碎片**：可变长度导致主存中出现大量无法利用的碎片           |
| **共享/保护方便**：可按逻辑段设置权限、实现共享          | **分配复杂**：OS 需要更复杂的空间分配算法（如最佳适配、首次适配）    |
| **动态增长**：段长度可按需增长（如堆、栈）             | **地址转换复杂**：需要加法器（Base + Offset），不如页式拼接快 |

# 段页式虚拟存储器

**段页式虚拟存储器**（Segmented-Paged Virtual Memory）结合了**段式**的逻辑优势和**页式**的物理管理优势，是**现代操作系统的主流方案**（如 Linux、Windows）。

## 基本思想

- **先分段，再分页**：程序按逻辑划分为若干段，每个段再被划分为若干固定大小的页
- 段是**面向用户的逻辑单元**，页是**面向系统的物理单元**

## 地址结构

段页式管理的虚拟地址由三个字段组成：

- **段号 S**：标识属于哪个段
- **段内页号 P**：在该段内的虚拟页号
- **页内偏移 W**：在页内的具体位置

## 段页式的数据结构

需要**两级映射表**：

| 数据结构 | 作用 | 位置 |
|----------|------|------|
| **段表** | 每个段表项指向该段的**页表**首地址 | 每个进程一张 |
| **页表** | 每个页表项存放该页对应的**实页号** | 每个段一张 |

### 段表项格式

| 字段         | 含义              |
| ---------- | --------------- |
| **页表首址**   | 该段对应页表在主存中的起始地址 |
| **页表长度**   | 该段包含的页数         |
| **有效位**（V） | 该段是否有效/在主存      |
| **保护位**    | 访问权限            |
