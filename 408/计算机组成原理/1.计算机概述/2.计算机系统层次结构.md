# 计算机系统的组成

- **硬件系统**：计算机的实体部分，如 CPU、存储器、输入输出设备等
- **软件系统**：运行在硬件上的程序及数据，分为**系统软件**和**应用软件**
- 计算机系统的实际性能，和大程度上取决于软件对硬件资源的利用效率，而效率的实现依赖于硬件所提供的能力
- **计算机系统必须合理划分软硬件的功能边界**，对于使用频繁且硬件实现成本较低的功能宜由硬件实现
# 计算机硬件

## 冯·诺依曼机的基本思想

**存储程序**（Stored Program）是冯·诺依曼机的核心思想，包含以下几点：

- **二进制表示**：指令和数据均以二进制形式表示，存放于同一存储器，**仅凭内容无法区分**
- **存储程序**：程序（指令序列）事先存入主存储器，计算机按顺序逐条取出执行
- **五大部件**：计算机由**运算器、控制器、存储器、输入设备、输出设备**五大部分组成
- **指令由操作码和地址码组成**：操作码指明操作类型，地址码指出操作数地址

> [!important] 冯·诺依曼机的特点
> **控制流驱动方式**；指令和数据同等地位存放于存储器中，可按地址访问；指令由操作码和地址码组成；指令在存储器中按顺序存放，通常顺序执行。

## 计算机功能部件

### 中央处理器

CPU 是计算机的核心部件，内部由**数据通路**和**控制单元**两大部分组成。传统上将 CPU 描述为"运算器 + 控制器"，运算器本质上是数据通路的核心，控制器本质上是控制单元的电路实现

#### 数据通路

数据通路是**数据在 CPU 内部流动和加工的路径**，由**算数逻辑单元（ALU）**、**通用寄存器组**和互连线路构成。一条指令执行过程中，数据在数据通路中流动，依次经过各个功能部件完成加工

| 组件             | 名称        | 功能                           | 归属    |
| -------------- | --------- | ---------------------------- | ----- |
| **ALU**        | 算术逻辑单元    | 执行算术运算（加减乘除）和逻辑运算（与或非、移位、比较） | 运算部件  |
| **ACC**        | 累加器       | 存放 ALU 运算的临时结果，是使用最频繁的寄存器    | 通用寄存器 |
| **MQ**         | 乘商寄存器     | 乘法存放乘数，除法存放商                 | 专用寄存器 |
| **通用寄存器组**     | GPRs      | 存放参与运算的操作数和中间结果              | 通用寄存器 |
| **PC**         | 程序计数器     | 存放下一条指令的地址，具有自动加 1 功能        | 专用寄存器 |
| **IR**         | 指令寄存器     | 存放当前正在执行的指令，输出操作码给 CU        | 专用寄存器 |
| **MAR**        | 存储器地址寄存器  | 存放访存地址，连接地址总线                | 接口寄存器 |
| **MDR**        | 存储器数据寄存器  | 存放与主存交换的数据，连接数据总线            | 接口寄存器 |
| **多路选择器 / 总线** | MUX / Bus | 在控制信号作用下选择数据通路中的传输路径         | 互连线路  |

#### 控制单元

控制单元（CU）是 CPU 的**指挥中枢**，它不加工数据，而是根据指令的操作码和当前状态，产生**微操作控制信号**，指挥数据通路中各部件协调工作。

- CU 接收 **IR 中的操作码**作为输入
- 按照机器**时钟节拍**，在每个时钟周期发出一组控制信号
- 控制信号送到 ALU（选择运算类型）、寄存器（选择读写）、多路选择器（选择数据来源）、主存（读/写控制）等

**两种实现方式**：

| 方式                         | 原理                              | 特点              |
| -------------------------- | ------------------------------- | --------------- |
| **硬布线控制**（Hardwired）       | 用组合逻辑电路直接生成控制信号                 | 速度快，但设计复杂，不易修改  |
| **微程序控制**（Microprogrammed） | 将每条机器指令分解为一段微程序（微指令序列），存于控制存储器中 | 规整、灵活、易修改，但速度较慢 |

### 存储器

存储器是计算机中存放程序和数据的部件，按访问速度和容量可划分为**高速缓存（Cache）、主存储器（主存）和辅助存储器（外存）** 三个层次
现代内存由主存和高速缓存组成，主存作为核心工作存储器，用于存放待执行的程序和数据
外存则包括两类：一类是可与主存交换数据的磁盘、固态硬盘等；另一类是用于长期备份的海量数据存储设备，如磁带和光盘等

### 外部设备和设备控制器

#### I/O 设备

| 类型 | 功能 | 常见设备 |
|------|------|----------|
| **输入设备** | 将外部信息转换为机器可识别的电信号 | 键盘、鼠标、扫描仪、触摸屏 |
| **输出设备** | 将处理结果转换为人类可识别的形式 | 显示器、打印机、扬声器 |

> 某些设备兼具输入和输出功能，如触摸屏、磁盘。

#### 设备控制器（I/O 控制器）

- 设备不能直接与 CPU 或主存相连，需通过**设备控制器**（又称 I/O 接口/适配器）。
- **功能**：
  - 接收并解释 CPU 发来的命令
  - 控制设备与主存/CPU 之间的数据传送
  - 向 CPU 报告设备状态（就绪、忙、故障等）
- 一个控制器通常可连接**多个同类型设备**。

### 总线

**总线（Bus）** 是连接计算机各部件的一组公共信息传输线。

#### 按传输内容分类

| 总线类型 | 传输方向 | 功能 |
|----------|----------|------|
| **数据总线**（Data Bus） | 双向 | 传输数据信息；**位数反映一次可传送的数据位数** |
| **地址总线**（Address Bus） | 单向（CPU→其他部件） | 传输地址信息；**位数决定系统最大可寻址范围** |
| **控制总线**（Control Bus） | 每根线单向 | 传输控制信号和时序信号 |

#### 按连接对象分类

| 类型 | 连接范围 | 特点 |
|------|----------|------|
| **片内总线** | 芯片内部各部件 | 如 CPU 内部 ALU 与寄存器之间的总线 |
| **系统总线** | CPU、主存、I/O 接口之间 | 即通常所说的"总线"概念 |
| **通信总线** | 计算机系统之间或与外部设备之间 | 如 USB、PCIe 等 |

#### 系统总线

系统总线是连接 CPU、主存和 I/O 接口的公共通道，根据所连接部件的不同，可进一步细分为三类：

| 总线类型 | 连接对象 | 特点 |
|----------|----------|------|
| **处理器总线**（Processor Bus） | CPU ↔ 北桥 / 内存控制器 | 速度最高，与 CPU 时钟同步，负责 CPU 与主存/高速外设间的数据传输 |
| **存储器总线**（Memory Bus） | 北桥 / 内存控制器 ↔ 主存 | 传输内存读写请求与数据，决定内存带宽 |
| **I/O 总线**（I/O Bus） | 南桥 / I/O 控制器 ↔ 各类 I/O 设备 | 速度较低，兼容多种设备标准（如 SATA、USB），通过桥接芯片与处理器总线相连 |

> 三者的速度层次为：**处理器总线 > 存储器总线 > I/O 总线**。通过这种分级结构，高速部件与低速部件各行其道，避免了慢速 I/O 设备拖累 CPU 与主存之间的高速通信。

> [!info] 北桥与南桥
> **北桥（North Bridge）**：主板芯片组中的高速芯片，离 CPU 最近，连接 CPU、主存和显卡（PCIe ×16）。负责处理器总线与存储器总线之间的桥接，传输频繁、速度要求高。
> **南桥（South Bridge）**：主板芯片组中的低速芯片，通过北桥与 CPU 间接通信，连接硬盘（SATA）、USB、音频、网络等慢速 I/O 设备。负责 I/O 总线与外设的管理。
> 
> 现代处理器（如 Intel Core、AMD Ryzen）将内存控制器和部分 PCIe 通道集成进 CPU 内部，北桥功能被 CPU 吸收；南桥则演变为 **PCH**（Platform Controller Hub，Intel）或 **FCH**（Fusion Controller Hub，AMD），直接通过 DMI/PCIe 链路与 CPU 连接。

> 现代计算机大多采用**总线结构**，各部件通过总线交换信息，而非冯·诺依曼机早期的"以运算器为中心"结构。

# 计算机软件

## 系统软件

- **操作系统**（OS）：管理硬件资源，为用户提供接口，是**最核心的系统软件**。
- **语言处理程序**：汇编程序、编译程序、解释程序。
- **数据库管理系统**（DBMS）。
- **服务性程序**：调试程序、诊断程序等。

## 应用软件

- 为解决特定应用领域问题而编写的程序，如办公软件、浏览器、游戏等。

## 软件和硬件的逻辑等价性

**核心命题**：任何由硬件实现的功能，在**逻辑上**都可以用软件实现；反之亦然。软硬件在完成计算功能上是等价的，区别仅在于**实现方式**不同。

**取舍依据**——一个功能到底用硬件还是软件实现，取决于以下因素：

| 因素 | 硬件实现 | 软件实现 |
|------|----------|----------|
| **速度** | 快（专用电路并行执行） | 慢（指令序列串行执行） |
| **成本** | 高（需额外的物理电路） | 低（仅增加代码量） |
| **灵活性** | 低（电路固化后难以修改） | 高（修改程序即可） |
| **适用场景** | 使用频繁、功能固定的操作 | 不频繁、需求多变的功能 |

**基本原则**：使用频繁且硬件成本较低的功能宜用硬件实现，以提升系统整体性能；不常使用或需求多变的功能宜用软件实现，以保持灵活性和低成本。

**历史实例**：
- **浮点运算**：早期计算机无浮点硬件，全靠软件模拟（子程序），速度极慢；后来集成了 FPU（浮点运算单元），浮点运算速度大幅提升。
- **乘法/除法**：早期 CPU 只支持加减法，乘除由软件循环实现；现代 CPU 内置硬件乘法器和除法器。
- **图形渲染**：从 CPU 软件渲染 → 独立 GPU 硬件加速 → 现代 GPU 可编程着色器（软硬件融合）。
- **神经网络推理**：传统由 CPU/GPU 软件计算 → TPU/NPU 等专用硬件加速。

> **软硬件分界是动态变化的**。随着技术进步和成本下降，原本由软件实现的功能不断向硬件迁移（固化），而硬件可编程能力的提升（如 FPGA）又使"硬件"具备了软件般的灵活性。**ISA 正是这一软硬件分界面的标准化体现**。

# 计算机系统的层次结构

计算机系统是一个复杂的软硬件综合体，从底层硬件到上层应用可划分为多个层次，每层向上一层提供服务，同时屏蔽本层及以下各层的实现细节。

## 算法和编程
高级语言独立于计算机底层硬件结构，是主流软件开发语言；
低级语言则紧密依赖机器结构，特指机器语言及其符号化形式——汇编语言
### 编程语言

| 语言级别     | 特点                | 翻译方式            |
| -------- | ----------------- | --------------- |
| **机器语言** | 二进制代码，⭐**硬件直接执行** | 无需翻译            |
| **汇编语言** | 助记符表示，与机器指令一一对应   | 汇编程序（Assembler） |
| **高级语言** | 接近自然语言，可移植性好      | 编译程序 / 解释程序     |

### 翻译程序

三种语言之间的转换由翻译程序完成：

- **汇编程序（汇编器，Assembler）**：将**汇编语言**源程序翻译为**机器语言**目标程序。一句汇编指令对应一条机器指令，翻译过程相对简单。
- **编译程序（编译器，Compiler）**：将整个**高级语言**源程序一次性翻译为**汇编语言或机器语言**目标程序，然后执行。生成的目标程序可独立运行，执行效率高。
- **解释程序（解释器，Interpreter）**：逐条读取高级语言源程序中的语句，翻译一条、立即执行一条，**不产生独立的目标程序**。灵活性好，但执行效率低。

## 操作系统

操作系统（OS）是配置在硬件之上的第一层软件，向下管理硬件资源，向上为应用程序提供运行环境。

## 指令集体系结构

指令集体系结构（ISA，Instruction Set Architecture）是计算机软/硬件之间的**关键接口**，是计算机系统中**最重要的软硬件分界面**；
在程序员和编译器的视角，**完整的定义了软件可直接使用的硬件功能**，主要包括：
- **指令格式**：操作码、寻址方式、操作数的编码格式
- **操作类型**：指令支持的操作种类
- **寄存器组织**：通用寄存器、专用寄存器的数量与用途
- **寻址方式**：立即寻址、直接寻址、间接寻址、基址寻址、变址寻址等
- **地址空间**：可寻址的主存范围（由地址线位数决定）
- **数据类型**：支持的数据表示方式（整数、浮点数、字符等）

## 微体系结构

微体系结构（Microarchitecture）位于 ISA 之下，是 ISA 的**硬件实现方案**。如果说 ISA 定义了"做什么"（指令集规范），微体系结构则决定了"怎么做"（如何用硬件电路执行这些指令）。

**核心特点**：同一 ISA 可以有不同的微体系结构实现。例如同为 x86 指令集，Intel Core 和 AMD Ryzen 的微架构完全不同，但二者都能运行相同的二进制程序。

**微体系结构的设计内容**：
- **数据通路组织**：功能部件（ALU、寄存器等）之间如何互连，常见结构有单总线、双总线、三总线，决定每个时钟周期能并行完成多少次数据传送
- **控制单元实现**：采用硬布线控制还是微程序控制，直接影响指令执行速度和设计复杂度（见 [[#控制单元]]）
- **流水线级数**：将一条指令的执行切分为多少个流水线段，级数越深时钟频率越高，但流水线冲突和冒险也越复杂
- **缓存层次结构**：L1/L2/L3 Cache 的容量、关联度（直接映射/组相联/全相联）、替换策略（LRU/FIFO/随机），直接影响访存性能
- **分支预测机制**：是否支持分支预测及预测算法复杂度（静态预测/动态预测），降低控制冒险带来的流水线停顿：是否允许不按程序顺序执行指令以提升效率

> ISA 与微体系结构的分离是计算机设计的核心哲学：ISA 提供**软件兼容性**（程序不经修改即可在不同处理器上运行），微体系结构提供**硬件竞争空间**（处理器厂商可在微架构层面持续优化性能）。

# 计算机系统的工作原理

## “存储程序”工作方式

**存储程序**（Stored Program）是冯·诺依曼计算机的核心工作方式，其基本思想是：将程序（指令序列）和数据预先以二进制形式存入主存储器中，计算机运行时自动地、逐条地从主存取出指令并执行，无需人工干预。

主要特点：
- 程序和数据以同等地位存放于主存储器，**按地址访问**
- 指令在存储器中按执行顺序存放，由 **PC（程序计数器）** 指示下一条指令的地址
- 程序启动后自动运行，直到遇到停机指令或中断

## 源程序到可执行文件

一个高级语言源程序从编写到运行，需经过以下步骤：

```
源程序(.c) ─预处理─→ 预处理后源程序(.i) ─编译─→ 汇编程序(.s) ─汇编─→ 目标程序(.o) ─链接─→ 可执行文件(.exe)
```

| 阶段      | 工具   | 输入 → 输出                        | 说明                                                           |
| ------- | ---- | ------------------------------ | ------------------------------------------------------------ |
| **预处理** | 预处理器 | 源程序(.c) → 预处理后源程序(.i)          | 处理宏定义（`#define`）、文件包含（`#include`）、条件编译（`#ifdef`），展开后生成纯净的源代码 |
| **编译**  | 编译器  | 预处理后源程序(.i) → 汇编程序(.s)         | 进行词法分析、语法分析、语义分析和代码优化，生成**汇编代码**，每个语句描述一条机器指令                |
| **汇编**  | 汇编器  | 汇编程序(.s) → 目标程序(.o)            | 将助记符指令翻译为二进制机器指令，生成**可重定位目标文件**                              |
| **链接**  | 链接器  | 多个目标文件(.o) + 库函数 → 可执行文件(.exe) | 解析符号引用，合并代码段与数据段，生成完整可执行镜像                                   |


## 指令执行过程的简要描述

一条指令的执行过程称为**指令周期**，通常包含以下步骤：

1. **取指令（Fetch）**：PC 中存放的地址通过 MAR 送至地址总线，从主存读取指令到 MDR，再送入 IR；同时**PC 自动加 1**，指向下一条指令。
2. **分析指令（Decode）**：CU 对 IR 中的操作码进行译码，确定指令类型和所需的微操作序列。
3. **执行指令（Execute）**：CU 发出控制信号，指挥数据通路完成具体操作——可能是 ALU 运算、主存读写或 I/O 操作。
4. **检查中断**：当前指令执行完毕后检查是否有中断请求，若有则转去执行中断处理程序，否则继续取指。

> 以上过程**循环往复**，即冯·诺依曼机的基本工作流程。
