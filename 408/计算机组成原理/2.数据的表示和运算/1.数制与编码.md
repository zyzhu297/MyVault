# 进位计数制

> [!important]+ 计算机采用二进制的原因
> 1. **物理实现简单**：二进制只有 0 和 1 两种状态，可以用电平的高低、开关的通断、磁极的取向等物理量的两种稳定状态来表示，制造具有两个稳定状态的物理器件比制造多个稳定状态的器件容易得多
> 2. **运算规则简单**：二进制算术运算规则少，运算器电路大幅简化
> 3. **可靠性高**：两种状态差异明显，抗干扰能力强，数据传输和处理的出错概率低
> 4. **便于逻辑运算**：二进制的 0/1 与逻辑代数中的"假/真"天然对应，可直接利用布尔代数进行逻辑运算和逻辑设计
> 5. **通用性强**：二进制不仅能表示数值，还能统一编码处理字符、图像、声音、指令等各类信息

**进位计数制**（Positional Numeral System）是按进位方式计数的数制，不同位置的数字具有不同的权值
设一个 $r$ 进制数 $S$ 有 $n$ 位整数和 $m$ 位小数，则其展开式为：

$$S = \sum_{i=-m}^{n-1} K_i \times r^i$$

其中：
- $r$ 为**基数**（Radix），即每个数位上允许使用的数码个数
- $K_i$ 为第 $i$ 位的数码，$0 \leq K_i \leq r-1$
- $r^i$ 为第 $i$ 位的**位权**（Position Weight）

## 常见进制

| 进制                    | 基数  | 数码       | 后缀/前缀         | 用途             |
| --------------------- | --- | -------- | ------------- | -------------- |
| **二进制**（Binary）       | 2   | 0, 1     | `B` 或 `0b`    | 计算机内部表示        |
| **八进制**（Octal）        | 8   | 0~7      | `O` 或 `0`（前缀） | 简化二进制书写        |
| **十进制**（Decimal）      | 10  | 0~9      | `D`（可省略）      | 日常使用           |
| **十六进制**（Hexadecimal） | 16  | 0~9, A~F | `H` 或 `0x`    | 简化二进制书写、内存地址表示 |

# 不同进数值之间的相互转换

## 任意进制 → 十进制

**按权展开求和**：将每一位数码乘以其位权后相加。

- 二进制 → 十进制：$(1011.01)_2 = 1\times2^3 + 0\times2^2 + 1\times2^1 + 1\times2^0 + 0\times2^{-1} + 1\times2^{-2} = 8 + 0 + 2 + 1 + 0 + 0.25 = (11.25)_{10}$
- 十六进制 → 十进制：$(2A.8)_{16} = 2\times16^1 + 10\times16^0 + 8\times16^{-1} = 32 + 10 + 0.5 = (42.5)_{10}$

## 十进制 → 任意进制

### 整数部分：除基取余法（倒序排列）

反复除以基数 $r$，取余数，直到商为 0，余数**从下往上**（倒序）排列。

**例**：$(37)_{10}$ 转二进制
$$37 \div 2 = 18 \cdots 1$$
$$18 \div 2 = 9 \cdots 0$$
$$9 \div 2 = 4 \cdots 1$$
$$4 \div 2 = 2 \cdots 0$$
$$2 \div 2 = 1 \cdots 0$$
$$1 \div 2 = 0 \cdots 1$$

结果为 $(100101)_2$（从下往上读）。

### 小数部分：乘基取整法（顺序排列）

反复乘以基数 $r$，取整数部分，直到小数部分为 0 或达到精度要求，整数**从上往下**（顺序）排列。

**例**：$(0.6875)_{10}$ 转二进制
$$0.6875 \times 2 = 1.375 \quad → 1$$
$$0.375 \times 2 = 0.75 \quad → 0$$
$$0.75 \times 2 = 1.5 \quad → 1$$
$$0.5 \times 2 = 1.0 \quad → 1$$

结果为 $(0.1011)_2$（从上往下读）。

> [!warning] 注意
> 并非所有十进制小数都能精确转换为二进制。例如 $(0.1)_{10}$ 在二进制中是无限循环小数 $(0.0001100110011\ldots)_2$。这也是浮点数精度问题的根源。

# 定点数的编码表示

## 真值与机器数

| 概念      | 说明                                  |
| ------- | ----------------------------------- |
| **真值**  | 数学中的实际数值，**带正负号**，如 $+1011$、$-1011$ |
| **机器数** | 数值在计算机中的二进制编码表示形式，符号被数码化（0 正 1 负）   |

根据小数点的位置是否固定，机器数分为：

| 类型 | 特点 | 常用编码 |
|------|------|----------|
| **定点数** | 小数点位置**固定** | 原码、反码、补码、移码 |
| **浮点数** | 小数点位置**浮动** | IEEE 754 标准（见 [[3.浮点数的表示与运算]]） |

## 定点数的格式

### 定点整数（小数点固定在最低位之后）

$$x_n\,x_{n-1}\,\cdots\,x_1\,x_0\,.$$

- $x_n$ 为符号位，$x_{n-1} \sim x_0$ 为数值位
- $n+1$ 位定点整数可表示的范围取决于编码方式

### 定点小数（小数点固定在符号位之后）

$$x_n\,.\,x_{n-1}\,\cdots\,x_1\,x_0$$

- $x_n$ 为符号位，$x_{n-1} \sim x_0$ 为数值位
- $n+1$ 位定点小数表示范围：绝对值小于 1

> [!note] 说明
> 计算机中并没有专门的硬件来存放小数点，小数点的位置是**约定**的。定点整数和定点小数的运算规则本质相同，只是对结果的解释不同。

## 原码（Sign-Magnitude）

**规则**：最高位为符号位（0 正 1 负），其余位表示绝对值。

**例**（8 位，含符号位）：
- $[+5]_{原} = 00000101$
- $[-5]_{原} = 10000101$

| 优点               | 缺点                                                       |
| ---------------- | -------------------------------------------------------- |
| 与真值对应直观，**乘除方便** | 加减运算需判断符号位，电路复杂                                          |
| 实现简单             | 存在 **±0** 问题：$[+0]_{原} = 00000000$，$[-0]_{原} = 10000000$ |
| **表示范围关于原点对称**   |                                                          |
|                  |                                                          |

> [!warning] 原码的零
> 原码中零有两种表示，这会导致比较运算复杂化。

**表示范围**（$n+1$ 位，含符号位）：
- 定点整数：$-(2^n-1) \sim +(2^n-1)$
- 定点小数：$-(1-2^{-n}) \sim +(1-2^{-n})$

## 反码（Ones' Complement）

**规则**：
- 正数：与原码相同
- 负数：符号位为 1，其余位按位取反

**例**（8 位）：
- $[+5]_{反} = 00000101$
- $[-5]_{反} = 11111010$

| 特点 | 说明 |
|------|------|
| 反码求和 | 两个反码相加，若有循环进位需回加 |
| 零的表示 | $[+0]_{反} = 00000000$，$[-0]_{反} = 11111111$ |
| 用途 | 现代计算机已不直接使用，但反码是**求补码的中间步骤** |

**由反码求补码**：$[X]_{补} = [X]_{反} + 1$（负数时末位加 1）

## 补码（Two's Complement）

> [!important] 最重要编码
> 补码是**现代计算机中表示有符号整数的唯一方式**。

### 定义

**补码的模运算定义**（$n$ 位数值位，模 $2^{n+1}$）：
$$[X]_{补} = \begin{cases} 0,X & 0 \leq X < 2^n \\ 2^{n+1} + X = 2^{n+1} - |X| & -2^n \leq X < 0 \end{cases}$$

### 求补规则

| 情况 | 方法 |
|------|------|
| **正数** | 与原码、反码相同 |
| **负数** | 原码→补码：符号位保留 1，数值位取反后末位加 1 |
| **已知补码求真值** | 补码→反码→原码：符号位保留，数值位按位取反后末位加 1（**再次取反加 1**） |

**例**（8 位）：
- $[+5]_{补} = 00000101$
- $[-5]_{补} = 2^8 - 5 = 256 - 5 = 251 = 11111011$

计算过程：原码 $10000101$ → 反码 $11111010$ → 加 1 → $11111011$

### 补码的特性

| 特性 | 说明 |
|------|------|
| **唯一零** | $[+0]_{补} = [-0]_{补} = 00000000$（只有一种表示） |
| **多表示一个负数** | 补码范围不对称，可多表示一个最小负数 |
| **符号位参与运算** | 加减法统一为加法，简化 ALU 电路设计 |
| **$[-X]_{补}$ 快速求法** | 将 $[X]_{补}$ **连同符号位一起取反，末位加 1** |

**表示范围**（$n+1$ 位，含符号位）：
- 定点整数：$-2^n \sim +(2^n-1)$
- 定点小数：$-1 \sim +(1-2^{-n})$

> [!note] 最小负数的特殊性
> 8 位补码中：$[-128]_{补} = 10000000$。注意：$-128$ 没有对应的原码和反码（因为 8 位原码/反码只能表示 $-127 \sim +127$）。

### 补码运算的变形

从 $[X]_{补}$ 求 $[2X]_{补}$：**算术左移**一位，末位补 0（前提：不溢出）

从 $[X]_{补}$ 求 $[\frac{1}{2}X]_{补}$：**算术右移**一位，高位补符号位（即符号扩展）

### 变形补码（双符号位补码）

**变形补码**（Modified Two's Complement）又称**双符号位补码或模 4 补码**，在单符号位补码的基础上，额外增加一位符号位，用于**溢出判断**。

#### 格式

$$x_{n+1}\,x_n\,x_{n-1}\,\cdots\,x_0$$

- $x_{n+1}$ 和 $x_n$ 为**双符号位**，两符号位取值始终相同（均为 0 或均为 1）
- $x_{n-1} \sim x_0$ 为数值位
>[!warning] 变形补码的存储
>变形补码存储时只需要一个符号位，因为任何一个正确的数值，模 4 补码的两个符号位总是相同的
#### 溢出判断

两符号位 $S_{n+1}S_n$ 共有四种组合：

| 双符号位 | 含义 |
|----------|------|
| **00** | 结果为正数，**无溢出** |
| **01** | **正溢出（上溢）**：两个正数相加，结果超出了表示范围 |
| **10** | **负溢出（下溢）**：两个负数相加，结果超出了表示范围 |
| **11** | 结果为负数，**无溢出** |

> [!important] 溢出判断公式
> 设双符号位为 $S_{n+1}S_n$，则：
> $$\text{溢出} = S_{n+1} \oplus S_n$$
> 即当两符号位**相异**（01 或 10）时，发生溢出。

> [!note] 变形补码与单符号位补码的关系
> - 变形补码实际运算时，数值部分的计算规则与普通补码完全相同，只是多记录了一位符号位用于溢出检测
> - 在 ALU 设计中：两个操作数送入加法器前，将各自的符号位复制一份作为双符号位；运算结束后根据两符号位是否相等来判断溢出
> - 单符号位补码也可通过**最高位进位 $\oplus$ 次高位进位**来判断溢出，其本质与双符号位判断等价（见 [[2.运算方法与运算电路]]）

## 移码（Biased Representation / Excess Code）

**规则**：在真值 $X$ 上加一个**偏置值**（Bias），使所有数值都映射到非负数范围。
$$[X]_{移} = 2^n + X \quad (-2^n \leq X < 2^n)$$

其中偏置值通常取 $2^n$（$n$ 为数值位数）。

**例**（8 位，偏置 $2^7 = 128$）：
- $[+5]_{移} = 128 + 5 = 133 = 10000101$
- $[-5]_{移} = 128 - 5 = 123 = 01111011$

| 特点 | 说明 |
|------|------|
| 移码 = 补码符号位取反 | $[X]_{移}$ 与 $[X]_{补}$ **仅符号位相反** |
| 便于比较大小 | 移码是无符号数，移码越大真值越大 |
| 零的唯一表示 | $[+0]_{移} = [-0]_{移} = 10000000$（8 位） |
| 用途 | 浮点数的**阶码**（Exponent）表示 |

## 四种编码总结对比


| 维度            | 原码                             | 反码                             | 补码                        | 移码                        |
| ------------- | ------------------------------ | ------------------------------ | ------------------------- | ------------------------- |
| **核心规则**      | 符号位 + 绝对值                      | 正数不变；负数数值位按位取反                 | 正数不变；负数取反+1（按模取补）         | 真值 + 偏置值 $2^n$            |
| **符号位**       | 0 正 1 负，不参与运算                  | 0 正 1 负                        | 0 正 1 负，**参与运算**          | 1 正 0 负（= 补码符号位取反）        |
| **零的表示**      | ±0 两种（$0000$ / $1000$）         | ±0 两种（$0000$ / $1111$）         | **唯一零**（$0000$）           | **唯一零**（$1000$）           |
| **定点整数范围**    | $-(2^n-1) \sim +(2^n-1)$       | $-(2^n-1) \sim +(2^n-1)$       | **$-2^n \sim +(2^n-1)$**  | $-2^n \sim +(2^n-1)$      |
| **定点小数范围**    | $-(1-2^{-n}) \sim +(1-2^{-n})$ | $-(1-2^{-n}) \sim +(1-2^{-n})$ | **$-1 \sim +(1-2^{-n})$** | 不适用                       |
| **范围对称性**     | 关于原点对称                         | 关于原点对称                         | **不对称**（多一个最小负数）          | 不对称                       |
| **加法规则**      | 需判断符号，电路复杂                     | 循环进位需回加                        | **直接相加**，无需额外处理           | 直接相加                      |
| **$[-X]$ 求法** | 符号位取反                          | 全部位取反                          | **连同符号位取反+1**             | 需重新计算                     |
| **与补码的关系**    | —                              | $[X]_{补} = [X]_{反} + 1$（负数）    | —                         | $[X]_{移} = [X]_{补}$ 符号位取反 |
| **优点**        | 直观，乘除方便，范围对称                   | —                              | 唯一零、符号参与运算、硬件简单           | 便于比较大小（无符号比较）             |
| **缺点**        | ±0、加减电路复杂                      | ±0、循环进位                        | 范围不对称                     | 偏置后不为零                    |
| **主要用途**      | 浮点数尾数（IEEE 754 用原码表示尾数的绝对值部分）  | 已淘汰（求补码的中间步骤）                  | **现代计算机有符号整数的标准**         | 浮点数阶码（IEEE 754）           |


### 典型值对照（8 位，含符号位）

| 真值 | 原码 | 反码 | 补码 | 移码 |
|------|------|------|------|------|
| +127 | 01111111 | 01111111 | 01111111 | 11111111 |
| +1 | 00000001 | 00000001 | 00000001 | 10000001 |
| +0 | 00000000 | 00000000 | 00000000 | 10000000 |
| -0 | 10000000 | 11111111 | 00000000 | 10000000 |
| -1 | 10000001 | 11111110 | 11111111 | 01111111 |
| -127 | 11111111 | 10000000 | 10000001 | 00000001 |
| -128 | 无法表示 | 无法表示 | 10000000 | 00000000 |

> [!important] 关键结论
> 1. **正数**的原码、反码、补码**完全相同**
> 2. 补码的零只有一种表示：全 0；移码的零只有一种表示：$100\ldots0$
> 3. 移码与补码的区别**仅在符号位取反**
> 4. 补码能多表示一个最小负数（如 8 位的 $-128$），原码和反码无法表示
> 5. 四种编码中，**只有原码的表示范围关于原点对称**

# 整数的表示

## 无符号数的表示

**无符号整数**（Unsigned Integer）的所有位都是数值位，没有符号位。

$n$ 位无符号整数的表示范围：$0 \sim (2^n - 1)$

## 有符号数的表示

**有符号整数**（Signed Integer）需要同时表示数值的**大小**和**符号**。

### 符号位约定

计算机中，有符号数的最高位固定为**符号位**，0 表示整数，1 表示负数

- 符号位占 1 位，剩余 $n$ 位为数值位，共 $n+1$ 位
- 符号位数字化后，正负号变成了 0/1，使得符号可以**参与运算**

> [!important] 补码成为标准的原因
> 1. **唯一零**：全 0 表示零，避免了 ±0 判断的额外电路
> 2. **符号位参与运算**：$[X]_{补} + [Y]_{补} = [X+Y]_{补}$，加减法统一为加法，无需判断符号
> 3. **多表示一个负数**：补码范围 $-2^n \sim +(2^n-1)$，比原码/反码多一个最小负数，范围利用率更高
> 4. **硬件电路极简**：补码加法器 = 无符号加法器即可，不需要额外的符号处理逻辑


> [!warning] 有符号与无符号的易错点
> 同一个二进制位串，按有符号（补码）和无符号解读会得到不同数值。例如 `11111111`：按补码解读为 $-1$，按无符号解读为 $255$。

# C 语言中的整数类型及类型转换

## C 语言的整数类型

C 语言中的整数类型分为**有符号**和**无符号**两类。

| 类型               | 名称     | 典型位宽 | 表示范围（补码/无符号）                           |
| ---------------- | ------ | ---- | -------------------------------------- |
| `char`           | 字符型    | 8 位  | $-128 \sim +127$ 或 $0 \sim 255$（取决于实现） |
| `signed char`    | 有符号字符型 | 8 位  | $-128 \sim +127$                       |
| `unsigned char`  | 无符号字符型 | 8 位  | $0 \sim 255$                           |
| `short`          | 短整型    | 16 位 | $-32768 \sim +32767$                   |
| `unsigned short` | 无符号短整型 | 16 位 | $0 \sim 65535$                         |
| `int`            | 整型     | 32 位 | $-2^{31} \sim 2^{31}-1$                |
| `unsigned int`   | 无符号整型  | 32 位 | $0 \sim 2^{32}-1$                      |
| `long`           | 长整型    | 64 位 | $-2^{63} \sim 2^{63}-1$                |
| `unsigned long`  | 无符号长整型 | 64 位 | $0 \sim 2^{64}-1$                      |


> [!note] `char` 的符号性
> C 标准未规定 `char` 是有符号还是无符号，取决于编译器实现。GCC 在 x86 上默认为 `signed char`，而 ARM 上常为 `unsigned char`。

## 类型转换

### 隐式类型转换（自动转换）

C 语言在进行不同整数类型的运算时，会**自动**进行类型转换。

**整型提升**（Integer Promotion）：长度小于 `int` 的整数类型（如 `char`、`short`）在表达式中被自动转换为 `int`（或 `unsigned int`）。

**转换方向**：`char/short → int → unsigned int → long → unsigned long → long long → unsigned long long`

> [!warning] 常见陷阱
> ```c
> unsigned int a = 10;
> int b = -20;
> // a + b 的结果是什么？
> // b 被转换为 unsigned int，-20 变成 4294967276
> // a + b = 10 + 4294967276 = 4294967286（一个很大的无符号数）
> ```
> 当**有符号数和无符号数混合运算**时，有符号数被隐式转换为无符号数！

### 强制类型转换（Explicit Cast）

```c
int x = -1;
unsigned int y = (unsigned int)x;  // y = 4294967295（位模式不变，解释方式改变）
```

**强制类型转换的本质**：**位模式保持不变，但解读方式改变**。

### 位扩展与位截断

| 操作 | 有符号数（补码） | 无符号数 |
|------|----------------|----------|
| **位扩展**（短→长） | **符号扩展**：高位补符号位 | **零扩展**：高位补 0 |
| **位截断**（长→短） | 保留低位，丢弃高位；可能溢出 | 保留低位，丢弃高位；可能溢出 |

> [!important] 符号扩展（Sign Extension）
> 将有符号数从较少位扩展为较多位时，**高位全部填充符号位**。
> ⭐**若源数据为有符号数则，符号扩展；若源数据为无符号数，则零扩展**

# 数据的宽度和存储

## 数据的宽度和单位

### 基本单位

| 单位 | 英文 | 定义 | 说明 |
|------|------|------|------|
| **比特**（位） | bit（b） | 一个二进制位 | 计算机中数据的最小单位 |
| **字节** | Byte（B） | **1 Byte = 8 bit** | 计算机中信息存取的基本单位，**现代计算机按字节编址** |
| **字** | Word | 取决于机器字长 | 计算机内部进行数据处理和传输的基本单位 |

### 字长

**字长**（Word Length）是计算机一次整数运算能处理的二进制位数，定义为 **CPU 内部数据通路**和通用寄存器的宽度。

| 概念       | 说明                                                     |
| -------- | ------------------------------------------------------ |
| **机器字长** | CPU 一次能处理的二进制位数，通常等于通用寄存器位数。<br>**字长越长，计算精度越高，处理能力越强** |
| **存储字长** | 主存中一个存储单元存放的二进制位数，通常等于机器字长或 MDR 位数                     |
| **指令字长** | 一条指令的二进制位数，与机器字长无必然关系                                  |

> [!important] 字长是计算机系统结构的重要指标
> - 32 位处理器：字长 32 位，通用寄存器 32 位，一次处理 32 位数据
> - 64 位处理器：字长 64 位，通用寄存器 64 位，一次处理 64 位数据

### 二进制与十进制存储单位的区别

在描述存储器容量时，两种计量标准常被混淆：

| 标准 | 进制 | 单位换算 | 典型场景 |
|------|------|----------|----------|
| **SI 国际单位** | 十进制（$10^3$） | 1 KB = $10^3$ B, 1 MB = $10^6$ B, 1 GB = $10^9$ B | 硬盘、SSD 厂商标称 |
| **二进制惯例** | 二进制（$2^{10}$） | 1 KB = $2^{10}$ B, 1 MB = $2^{20}$ B, 1 GB = $2^{30}$ B | 操作系统、内存容量显示 |

> [!note] 这一差异导致”标称 1TB 硬盘实际显示约 931GB”——因为厂商用十进制（$10^{12}$ B），操作系统用二进制（$2^{40}$ B = 1,099,511,627,776 B），$\frac{10^{12}}{2^{40}} \approx 0.909$。

### 按字节编址

现代计算机的主存储器**按字节编址**（Byte-Addressable），即每个字节拥有唯一的内存地址。

- **字地址**：若字长为 32 位（4 字节），则字的地址通常取**最低字节的地址**，且字地址通常是 4 的倍数（**对齐**要求）
- **地址位数与寻址范围**：若地址总线为 $n$ 位，则最大可寻址 $2^n$ 个存储单元（通常为 $2^n$ 字节）

## 数据的”大端方式”和”小端方式”存储

### LSB 与 MSB

在多位数（如一个字节或一个字）中，每一位的"权重"不同，最高位和最低位有专门术语：

| 术语 | 全称 | 含义 | 位置 |
|------|------|------|------|
| **MSB** | Most Significant Bit | **最高有效位**，具有最大权值的位 | 二进制数的最左位（如 8 位数中的第 7 位，权值 $2^7$） |
| **LSB** | Least Significant Bit | **最低有效位**，具有最小权值的位 | 二进制数的最右位（如 8 位数中的第 0 位，权值 $2^0$） |

> [!note] 示例
> 对于 8 位二进制数 `10110010`（即 $128+32+16+2 = 178$）：
> 
> | 位序 | b₇ | b₆ | b₅ | b₄ | b₃ | b₂ | b₁ | b₀ |
> |------|----|----|----|----|----|----|----|----|
> | 值 | 1 | 0 | 1 | 1 | 0 | 0 | 1 | 0 |
> | 权值 | $2^7$=128 | $2^6$=64 | $2^5$=32 | $2^4$=16 | $2^3$=8 | $2^2$=4 | $2^1$=2 | $2^0$=1 |
> | 角色 | **MSB** ↑ | | | | | | | **LSB** ↑ |

**LSB 和 MSB 在大小端中的含义延伸**：

| 术语 | 全称 | 含义 |
|------|------|------|
| **MSB** | Most Significant **Byte** | **最高有效字节**，多字节数据中权值最大的字节 |
| **LSB** | Least Significant **Byte** | **最低有效字节**，多字节数据中权值最小的字节 |

> "MSB 存低地址"= 大端，"LSB 存低地址"= 小端。记忆方式：**数大端—看 MSB，数小端—看 LSB**。

### 基本概念

多字节数据（如 `int` 占 4 字节）在内存中存放时，字节排列顺序有两种方式：

> **大小端问题本质**：多字节数据在存储器中**字节的排列顺序**问题。单个字节内部没有大小端之分。

| 模式       | 英文            | 规则                      | 标准      |
| -------- | ------------- | ----------------------- | ------- |
| **大端方式** | Big Endian    | **高字节存储在低地址**，低字节存储在高地址 | 与书写顺序一致 |
| **小端方式** | Little Endian | **低字节存储在低地址**，高字节存储在高地址 | 与书写顺序相反 |

### 示例（存储 `0x12345678`）

假设 32 位数据 `0x12345678` 存入地址 `0x00` 开始的内存：

| 地址 | 0x00 | 0x01 | 0x02 | 0x03 |
|------|------|------|------|------|
| **大端** | `12` | `34` | `56` | `78` |
| **小端** | `78` | `56` | `34` | `12` |

>[!note] 直观记忆
>- **大端**（Big Endian）：数据的高位字节在**低地址**——符合人类阅读习惯，先看到”大头”
>- **小端**（Little Endian）：数据的低位字节在**低地址**——符合数学运算习惯，”小头”先取

## 数据按”边界对齐”方式存储

**边界对齐**（Alignment）要求数据的起始地址必须是其自身长度的整数倍。

| 数据类型 | 长度（字节） | 对齐要求 |
|----------|------------|----------|
| `char` | 1 | 1 的倍数（任意地址均可） |
| `short` | 2 | 2 的倍数（地址末位为 0, 2, 4, 6, 8, A, C, E） |
| `int` / `float` | 4 | 4 的倍数（地址末位为 0, 4, 8, C） |
| `long long` / `double` | 8 | 8 的倍数（地址末位为 0, 8） |
| 指针（64 位） | 8 | 8 的倍数 |

> [!important] 核心规则
> 一个 $n$ 字节的数据，其起始地址 $\bmod n = 0$。即 **”地址对长度取模，结果为 0”**。

>[!note] 为什么要对齐
>**硬件读取效率**：存储器通常按**字**（如 32 位 / 4 字节）访问。若数据未对齐，可能跨越两个字边界，需两次访存才能读取完整数据，效率减半
>**某些架构强制要求**：部分 RISC 处理器（如 ARM 早期版本、MIPS 等）访问未对齐数据会触发**硬件异常**（Alignment Fault）

> [!important] 编程启示
> 声明结构体时，**将对齐要求大的成员放在前面**，可有效减少空洞、节省内存。

**对齐 vs 不对齐的取舍**：

| 维度 | 对齐 | 不对齐 |
|------|------|--------|
| **访存速度** | 快（一次访存） | 慢（可能需两次） |
| **内存占用** | 大（有空洞） | 小（无空洞） |
| **跨平台兼容** | 好 | 差（部分架构可能崩溃） |
| **硬件要求** | 通用 | 仅 x86 等容忍未对齐的架构 |
