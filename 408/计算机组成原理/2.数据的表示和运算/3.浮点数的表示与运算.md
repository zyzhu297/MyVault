
定点数的小数点位置固定，**浮点数**（Floating-Point Number）允许小数点位置"浮动"，大幅扩展了表示范围，浮点数表示为：

$$N = (-1)^S \times M \times r^E$$

其中：
- $S$：**数符**（Sign），0 表示正，1 表示负
- $M$：**尾数**（Mantissa / Significand），是一个定点小数（纯小数），其位数决定**表示精度**
- $r$：**基数**（Radix），计算机中通常 $r = 2$（二进制），也可为 4、8、16 等
- $E$：**阶码**（Exponent），是一个定点整数，其位数决定**表示范围**

> [!important] 核心思想
> 浮点数 = 阶码 + 尾数，可类比科学计数法中的"指数"和"有效数字"。
> - **阶码**决定了**表示范围**，阶码位数越大，可表示的数值范围越广
> - **尾数**决定了**表示精度**，尾数位数越大，有效数字越多，精度越高


# IEEE 754 标准的浮点数

| 类型              | 总位数  | 数符  | 阶码   | 尾数（隐含 1 位）    | 偏置值（Bias） |
| --------------- | ---- | --- | ---- | ------------- | --------- |
| **单精度**（float）  | 32 位 | 1 位 | 8 位  | 23 位（实际 24 位） | ⭐**127**  |
| **双精度**（double） | 64 位 | 1 位 | 11 位 | 52 位（实际 53 位） | ⭐**1023** |

```
单精度格式 (32 bits):
┌───┬──────────┬─────────────────────────┐
│ S │   E (8)  │        M (23)           │
└───┴──────────┴─────────────────────────┘

双精度格式 (64 bits):
┌───┬────────────┬───────────────────────────────────────────┐
│ S │   E (11)   │              M (52)                       │
└───┴────────────┴───────────────────────────────────────────┘
```

## 移码表示阶码

IEEE 754 中阶码使用**移码**（Biased Exponent）表示：

$$E_{存储} = E_{真值} + \text{Bias}$$

| 类型 | 偏置值 Bias | 阶码真值范围 | 阶码存储值范围 |
|------|------------|-------------|---------------|
| **单精度** | 127 | $-126 \sim +127$ | $1 \sim 254$ |
| **双精度** | 1023 | $-1022 \sim +1023$ | $1 \sim 2046$ |

> [!note] 移码的优势
> - 将负指数映射为正数存储，便于硬件比较（无符号整数比较即可比较浮点数大小）
> - 阶码全 0 和全 1 被保留用于表示特殊值

## 尾数和隐藏位

IEEE 754 的尾数部分采用**规格化表示的原码**，因为规格化后尾数的最高数值位恒为 1，所以**不显式存储该位**，这样可以多获得一位精度。

- 单精度：23 位存储 → **实际精度 24 位**
- 双精度：52 位存储 → **实际精度 53 位**

## IEEE 754 浮点数的真值公式

$$N = (-1)^S \times (1.M) \times 2^{E - \text{Bias}}$$

其中 $1.M$ 表示"**1**"是隐含位，$M$ 是尾数的小数部分。

## 特殊值的表示

IEEE 754 利用阶码的全 0 和全 1 来表示特殊情况：

| 数符 $S$ | 阶码 $E$ | 尾数 $M$ | 表示的值 |
|----------|---------|---------|---------|
| 任意 | **全 0** | **全 0** | $\pm 0$（正零/负零） |
| 任意 | **全 0** | **非 0** | **非规格化数**（Denormalized）：$(-1)^S \times (0.M) \times 2^{1-\text{Bias}}$ |
| 任意 | $1 \sim 254$（单精度） | 任意 | **规格化数**：$(-1)^S \times (1.M) \times 2^{E-\text{Bias}}$ |
| 0 | **全 1** | **全 0** | $+\infty$（正无穷） |
| 1 | **全 1** | **全 0** | $-\infty$（负无穷） |
| 任意 | **全 1** | **非 0** | **NaN**（Not a Number，非数） |

> [!important] 非规格化数的作用
> 非规格化数（阶码全 0、尾数非 0）用于表示**非常接近 0 的数**，实现了**逐渐下溢**（Gradual Underflow），避免了突然跳到 0 导致的精度损失。其隐含位为 0（而非 1），指数固定为 $1 - \text{Bias}$。

## 浮点数能表示的范围

### 单精度（float，Bias = 127）

| 项目           | 阶码存储值           | 阶码真值   | 尾数    | 值（以 2 的幂表示）                                    |
| ------------ | --------------- | ------ | ----- | ---------------------------------------------- |
| **最大正数**     | $254$（11111110） | $+127$ | 全 1   | $(2 - 2^{-23}) \times 2^{127} \approx 2^{128}$ |
| **最小正规格化数**  | $1$（00000001）   | $-126$ | 全 0   | $1.0 \times 2^{-126} = 2^{-126}$               |
| **最小正非规格化数** | $0$（00000000）   | $-126$ | 末位为 1 | $2^{-23} \times 2^{-126} = 2^{-149}$           |

### 双精度（double，Bias = 1023）

| 项目           | 阶码存储值               | 阶码真值    | 尾数    | 值（以 2 的幂表示）                                      |
| ------------ | ------------------- | ------- | ----- | ------------------------------------------------ |
| **最大正数**     | $2046$（11111111110） | $+1023$ | 全 1   | $(2 - 2^{-52}) \times 2^{1023} \approx 2^{1024}$ |
| **最小正规格化数**  | $1$（00000000001）    | $-1022$ | 全 0   | $1.0 \times 2^{-1022} = 2^{-1022}$               |
| **最小正非规格化数** | $0$（00000000000）    | $-1022$ | 末位为 1 | $2^{-52} \times 2^{-1022} = 2^{-1074}$           |

> [!note] 推导要点
> - **最大正数**：阶码取最大非保留值（$254$ 或 $2046$），尾数全 1，真值 = $(1 + 1 - 2^{-n}) \times 2^{E_{\text{max}}}$
> - **最小正规格化数**：阶码 $=1$，尾数全 0，隐含位为 1，真值 = $1.0 \times 2^{1-\text{Bias}}$
> - **最小正非规格化数**：阶码 $=0$，隐含位为 0，仅末位为 1，真值 = $2^{-n} \times 2^{1-\text{Bias}}$（其中 $n$ 为尾数位数）

> [!warning] 表示范围 VS 表示个数
> 虽然浮点数表示范围远大于定点数（相同位数下），但**可表示的数的个数是相同的**（都是 $2^n$ 个），浮点数只是将数的分布变得**不均匀**——越靠近 0 越密集，越远离 0 越稀疏。

## 正负上溢与下溢

![[IEEE 754 标准单精度浮点数的表示范围.png]]

### 溢出情况

| 情况                          | 触发条件                                     | IEEE 754 处理结果            |
| --------------------------- | ---------------------------------------- | ------------------------ |
| **正上溢**（Positive Overflow）  | 正数结果 $>$ 最大正数                            | $+\infty$ 或者<br>置位浮点溢出标志 |
| **负上溢**（Negative Overflow）  | 负数结果 $<$ 最大负数（即绝对值超过最大正数）                | $-\infty$ 或者<br>置位浮点溢出标志 |
| **正下溢**（Positive Underflow） | 正数结果 $<$ 最小正非规格化数（即以 0 为中心，向正方向无法表示的极小值） | **$+0$**（逐渐下溢时可能为非规格化数）  |
| **负下溢**（Negative Underflow） | 负数结果的绝对值 $<$ 最小正非规格化数（即向负方向无法表示的极小绝对值）   | **$-0$**（逐渐下溢时可能为负非规格化数） |
> [!important] 上溢 VS 下溢的本质
> - **上溢**：阶码**太大**，超出了阶码能表示的最大值 → 产生 $\pm\infty$
> - **下溢**：阶码**太小**（绝对值太小），超出了阶码能表示的最小值 → 产生 $\pm 0$（或非规格化数）
> 
> IEEE 754 的非规格化数实现了**逐渐下溢**：当结果进入下溢区时，先以非规格化数（精度逐渐降低）表示，直到连非规格化数也无法表示时才变为 $\pm 0$。

> [!note] 非规格化数的特点
> 1. **阶码存储值全 0**，**真值固定为 $1 - \text{Bias}$（单精度 $-126$，双精度 $-1022$）**
> 2. **隐含位为 0**（规格化数隐含位为 1），真值公式为 $(-1)^S \times (0.M) \times 2^{1-\text{Bias}}$
> 3. **均匀间距**：所有非规格化数的指数相同，因此相邻两数间距恒为 $2^{-n} \times 2^{1-\text{Bias}}$（单精度 $2^{-149}$，双精度 $2^{-1074}$），是**等距分布**的
> 4. **与规格化数无缝衔接**：最大非规格化数 $(0.11\ldots1) \times 2^{-126} = (1 - 2^{-23}) \times 2^{-126}$，与最小规格化数 $1.0 \times 2^{-126}$ 之间的间距也是 $2^{-149}$，无缺口

### 浮点数表示范围

| 区间         | 范围（以 2 的幂表示）                                 | 说明                            |
| ---------- | -------------------------------------------- | ----------------------------- |
| **负上溢区**   | $(-\infty,\ -(2 - 2^{-23})\times 2^{127})$   | 负方向超出最大绝对值，结果为 $-\infty$      |
| **规格化负数**  | $[-(2 - 2^{-23})\times 2^{127},\ -2^{-126}]$ | 阶码 $\in [1, 254]$，隐含位为 1      |
| **渐进下溢负数** | $(-2^{-126},\ -2^{-149}]$                    | 非规格化数，阶码全 0，隐含位为 0，**精度逐渐降低** |
| **负下溢区**   | $(-2^{-126},\ 0)$                            | 连非规格化数也无法表示，结果为 $-0$          |
| **零**      | $0$                                          | $+0$ 或 $-0$                   |
| **正下溢区**   | $(0,\ 2^{-126})$                             | 连非规格化数也无法表示，结果为 $+0$          |
| **渐进下溢正数** | $[2^{-149},\ 2^{-126})$                      | 非规格化数，阶码全 0，隐含位为 0，**精度逐渐降低** |
| **规格化正数**  | $[2^{-126},\ (2 - 2^{-23})\times 2^{127}]$   | 阶码 $\in [1, 254]$，隐含位为 1      |
| **正上溢区**   | $((2 - 2^{-23})\times 2^{127},\ +\infty)$    | 正方向超出最大值，结果为 $+\infty$        |

# 浮点数的加减运算

## 运算步骤

浮点数加减运算遵循以下五个标准步骤：

### 第一步：对阶（Exponent Matching）

**小阶向大阶看齐**

将阶码较小的数的尾数**右移**，阶码**加 1**，重复直到两数阶码相等。

> [!note] 为什么小阶向大阶看齐？
> 若大阶向小阶看齐，需要将大阶数的尾数**左移**，可能导致尾数的高位丢失，造成严重误差。而小阶尾数右移只会丢失低位，精度损失可控。

$$\text{对阶操作：} \Delta E = |E_A - E_B|，\text{将阶码较小的尾数右移 } \Delta E \text{ 位}$$

### 第二步：尾数加减（Mantissa Addition/Subtraction）

对阶完成后，对两个尾数进行定点加减运算（**原码加减**），且需要将隐藏位还原

### 第三步：尾数规格化（Normalization）

> [!important] IEEE 754 尾数用**原码**表示
> IEEE 754 的尾数采用**原码**（Sign-Magnitude），而非补码。运算时将隐含位 1 还原，尾数变为 $1.M$ 的形式（即 $1.\underbrace{xx\ldots x}_{n\text{ 位}}$）。规格化的目标是保证尾数最高位为 **1**（即形如 $1.xx\ldots x$）。

尾数加减后可能出现以下情况，需规格化处理：

| 情况       | 尾数形式（原码，还原隐藏位后）                                | 处理                                     |
| -------- | ---------------------------------------------- | -------------------------------------- |
| **需左规**  | $0.0xx\ldots$ 或 $0.00x\ldots$（最高位为 0，即丢失了前导 1） | **左规**：尾数左移 $k$ 位直至最高位为 1，阶码 **减 $k$** |
| **需右规**  | $1x.xxx\ldots$（尾数 $\geq 2$，进位到整数部分）            | **右规**：尾数右移 1 位，阶码 **加 1**，**最多只需要一次** |
| **已规格化** | $1.xx\ldots x$（最高位恰好为 1）                       | 无需处理                                   |

> [!note] 左规与右规的触发场景
> - **左规**：常见于两个相近的数**相减**（如 $1.001 - 1.000 = 0.001$），高位出现连续的 0，需左移恢复规格化形式
> - **右规**：常见于两个同符号数**相加**（如 $1.10 + 1.10 = 11.00$），尾数溢出到整数部分，需右移 1 位

> [!warning] 左规可能伴随阶码下溢
> 左规时阶码不断减小，若阶码减到小于可表示的最小值（单精度 $-126$），则进入**非规格化数**范围（阶码全 0，隐含位变 0），或进一步触发下溢变为 $\pm 0$。

### 第四步：舍入（Rounding）

右规或对阶过程中，尾数右移可能丢弃低位，需要**舍入处理**。

#### 三个辅助位

IEEE 754 在对阶和右规过程中，为了不丢失精度信息，在尾数低位之后额外保留 **3 个辅助位**：

- **G（Guard，保护位）**：LSB 之后的第 1 位，保存移出位中的最高位。作用是提供「丢弃部分是否 $\geq$ 一半」的关键判断依据
- **R（Round，舍入位）**：LSB 之后的第 2 位，保存移出位中的次高位。作用是与 G 配合区分「恰好一半」和「超过一半」
- **S（Sticky，粘滞位）**：只要舍入位之后被移出的位中存在至少一个 1，粘滞位就置为一，作用是**一旦置 1 永不回 0**，确保"超过一半"不会被误判为"恰好一半"

#### 就近舍入（Round to Nearest）

IEEE 754 **默认舍入方式**，舍入到最接近的可表示值。由 G、R、S 三位共同决定：

| G | R | S | 丢弃部分与一半的关系 | 处理 |
|---|---|---|------|------|
| 0 | × | × | 丢弃部分 $<$ 一半 | **直接舍去**（截断） |
| 1 | 0 | 0 | 丢弃部分 $=$ 恰好一半（中间值） | **向偶数舍入**：使结果末位为 0（若末位为 1 则 +1，为 0 则舍去） |
| 1 | 0 | 1 | 丢弃部分 $>$ 一半 | **向上舍入**：尾数末位 +1 |
| 1 | 1 | × | 丢弃部分 $>$ 一半 | **向上舍入**：尾数末位 +1 |

> [!tip] 辅助位速记口诀
> - **G = 0**：不足一半 → 直接舍去
> - **G = 1, R = S = 0**：恰好一半（中间值）→ 末位凑偶（0 舍 1 入）
> - **G = 1, R 或 S = 1**：超过一半 → 向上舍入（末位 +1）

> [!note] 为什么要"向偶数舍入"
> 若中间值一律进位，在大量运算中会产生**系统性向上偏差**。向偶数舍入使得中间值有一半概率进位、一半概率截断，统计上**无偏**。

#### 正向舍入（Round toward $+\infty$）
始终向数轴正方向舍入，将浮点数表示结果取为**不超过原数的最大可表示浮点数**
#### 负向舍入（Round toward $-\infty$）
始终向数轴负方向舍入，将浮点数表示结果取为**超过原数的最小可表示浮点数**
#### 截断法（Round toward $0$）
直接丢弃多余位，正负数均向零方向截断，也**朝原点舍入**，最简单，但累积误差最大

#### 舍入后的再规格化

向上舍入可能导致尾数溢出（$1.11\ldots1 + 0.00\ldots1 = 10.00\ldots0$），此时需**再次右规**（右移 1 位，阶码 +1），并可能进一步触发阶码上溢。

### 第五步：溢出判断（Overflow Check）

浮点数的溢出只与**阶码**有关：

| 情况       | 判定                                   |
| -------- | ------------------------------------ |
| **阶码上溢** | 阶码超过最大值 → **溢出**（结果太大，无法表示，引发**异常**） |
| **阶码下溢** | 阶码小于最小值 → **机器零**（结果太小，当作 0 处理）      |

> [!note] 尾数溢出 ≠ 浮点数溢出
> 尾数加减可能溢出（双符号位出现 01 或 10），但可以通过**右规**修正，这不算浮点数溢出。只有阶码溢出才意味着真正的浮点数溢出。

## 例题

> [!example]- 例 1：两浮点数相加（需左规）
> $X = 0.5_{10}$，$Y = -0.4375_{10}$，求 $X + Y$
>
> 设浮点数格式：阶码 4 位（含 1 位阶符，补码），尾数 8 位（含 1 位数符，补码，采用双符号位则运算时尾数为 9 位）
>
> **解**：
>
> **（1）十进制 → 机器数**
>
> $X = 0.5 = 0.5 \times 2^0$（已规格化）
> - 阶码：$0$ → 补码 $0000$
> - 尾数：$+0.5 = 0.1000000$（双符号位：$00.1000000$）
>
> $Y = -0.4375 = -0.0111_2$（未规格化）
> - 规格化：$-0.4375 = -0.875 \times 2^{-1}$
> - 阶码：$-1$ → 补码 $1111$
> - 尾数：$-0.875 = -0.1110000$ → 补码 $1.0010000$（双符号位：$11.0010000$）
>
> **（2）对阶**
>
> $\Delta E = E_X - E_Y = 0 - (-1) = 1 > 0$，$E_Y < E_X$，将 $Y$ 向 $X$ 看齐。
>
> $Y$ 尾数右移 $1$ 位（算术右移，符号位不变）：$11.0010000 \rightarrow 11.1001000$
>
> $Y$ 阶码 $+1$：$-1 + 1 = 0$（$0000$），与 $X$ 阶码相同 ✓
>
> **（3）尾数相加（补码加法，双符号位）**
>
> $$\begin{aligned}
>   &\ 00.1000000 \quad (X)\\
> + &\ 11.1001000 \quad (Y)\\
> \hline
>   &1\cancel{0}0.0001000
> \end{aligned}$$
>
> 进位丢弃（双符号位运算中，超出双符号位的进位自动丢弃），结果为 $00.0001000$。
>
> 双符号位为 $00$ → 结果为正，无溢出 ✓
>
> **（4）结果规格化（左规）**
>
> 尾数为 $00.0001000$，不满足规格化条件（符号位后第一位为 $0$），需**左规**：
>
> - 尾数左移 $3$ 位：$00.0001000 \rightarrow 00.1000000$
> - 阶码减 $3$：$0 - 3 = -3$ → 补码 $1101$
>
> **（5）最终结果**
>
> 阶码 $1101$（$-3$），尾数 $0.1000000$（$+0.5$）
>
> $$X + Y = 0.5 \times 2^{-3} = 0.0625$$
>
> 验证：$0.5 + (-0.4375) = 0.0625$ ✓

> [!example]- 例 2：IEEE 754 单精度表示
> 将 $-12.625_{10}$ 表示为 IEEE 754 单精度浮点数。
>
> **解**：
>
> **（1）转换为二进制**
>
> $12_{10} = 1100_2$
>
> $0.625_{10}$：$0.625 \times 2 = 1.25 \rightarrow 1$；$0.25 \times 2 = 0.5 \rightarrow 0$；$0.5 \times 2 = 1.0 \rightarrow 1$
>
> 所以 $0.625_{10} = 0.101_2$
>
> $12.625_{10} = 1100.101_2$
>
> **（2）规格化**
>
> $1100.101_2 = 1.100101_2 \times 2^3$
>
> - 数符 $S = 1$（负数）
> - 尾数小数部分 $M = 10010100000000000000000$（23 位，去掉隐含的 1）
> - 阶码真值 $E = 3$，存储值 $= 3 + 127 = 130 = 10000010_2$
>
> **（3）组装为 32 位**
>
> $$1\ |\ 10000010\ |\ 10010100000000000000000$$
>
> 即十六进制：`0xC1480000`

# C 语言中的浮点数类型

## 基本类型

| C 语言类型 | IEEE 754 对应 | 位数 | 有效数字位数（十进制） | 取值范围（近似） |
|-----------|--------------|------|---------------------|-----------------|
| `float` | 单精度 | 32 位 | 6~7 位 | $\pm 3.4 \times 10^{\pm 38}$ |
| `double` | 双精度 | 64 位 | 15~16 位 | $\pm 1.8 \times 10^{\pm 308}$ |
| `long double` | 扩展双精度 | ≥ 80 位 | 18~19 位 | 平台相关 |

## 类型转换

### 强制类型转换（Casting）

C 语言中不同类型数据的混合运算和赋值操作会触发**自动类型转换**或**强制类型转换**。

转换遵循**低精度 → 高精度**方向（隐式转换不会损失精度）：

```
char → short → int → long → float → double → long double
```

> [!warning] 转换中的精度损失
> - `int` → `float`：int 有 32 位有效位，float 尾数仅 24 位，**可能舍入**
> - `float`或`double` → `int`：**截断小数部分**（向零取整，即 truncate）
> - `double` → `float`：**可能溢出或舍入**
> - `int`或`float` → `double`：double 尾数 53 位 > int 32 位，**精确转换**（无损失）

### 关键转换规则

| 转换 | 行为 |
|------|------|
| `float`/`double` → `int` | 截断小数部分，**向零取整**（不四舍五入） |
| `int` → `float` | 可能舍入（`int` 有效位 > 24 位时） |
| `int` → `double` | 精确（$2^{53}$ 以内） |

## 常见陷阱

### 1. 浮点数比较

```c
float a = 0.1 + 0.2;
if (a == 0.3)  // ❌ 通常为 false！
```

> [!important] 原因
> 0.1 和 0.2 在二进制中都是无限循环小数，存储时就产生了舍入误差。正确的比较方式：
> ```c
> if (fabs(a - 0.3) < 1e-6)  // ✓ 使用容差比较
> ```

### 2. 大数吃小数

```c
float a = 1.0e10;
float b = 1.0;
float c = a + b;  // c 仍然等于 1.0e10，b 被"吃掉了"
```

原因是：对阶时 $b$ 的尾数需要右移 34 位（$10^{10} \approx 2^{34}$），而单精度尾数只有 24 位，$b$ 的有效位全部被移出，变成了 0。

### 3. 结合律不成立

浮点数加法和乘法**不满足结合律**：

$$(a + b) + c \neq a + (b + c)$$

$$(a \times b) \times c \neq a \times (b \times c)$$

这是舍入误差导致的。

## 考点总结

| 考点 | 要点 |
|------|------|
| **浮点数格式** | 数符 + 阶码 + 尾数，与科学计数法的对应关系 |
| **规格化** | 左规/右规的条件与操作；原码和补码规格化的判定 |
| **IEEE 754** | 移码表示阶码；隐藏位；偏置值（127/1023）；特殊值（0、∞、NaN、非规格化数） |
| **加减运算步骤** | 对阶 → 尾数加减 → 规格化 → 舍入 → 溢出判断 |
| **溢出判定** | 只有**阶码溢出**才算浮点数溢出，尾数溢出可通过右规修正 |
| **类型转换** | `int→float` 可能舍入；`float→int` 向零取整；`int→double` 精确 |
| **舍入方法** | 0舍1入、恒置1、截断、IEEE 754向偶数舍入 |
