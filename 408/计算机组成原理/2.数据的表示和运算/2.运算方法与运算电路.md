# 基本运算部件

运算器由算术逻辑单元（Arithmetic Logic Unit，ALU）、移位器、状态存储器（PSW）和通用寄存器组等组成；运算器的基本功能包括加减乘除四则运算，与或非异或等逻辑运算，以及**移位、求补**等操作；ALU 的核心部件是加法器
## 一位全加器

**一位全加器**（Full Adder，简称 FA）是实现两个 1 位二进制数及低位进位相加的基本单元

### 输入与输出

| 信号 | 符号 | 说明 |
|------|------|------|
| **被加数** | $A_i$ | 第 $i$ 位的加数，1 位 |
| **加数** | $B_i$ | 第 $i$ 位的加数，1 位 |
| **低位进位** | $C_{i-1}$ | 来自第 $i-1$ 位的进位输入，1 位 |
| **本位和** | $S_i$ | 第 $i$ 位的和输出，1 位 |
| **向高位进位** | $C_i$ | 向第 $i+1$ 位的进位输出，1 位 |

### 逻辑表达式

$$S_i = A_i \oplus B_i \oplus C_{i-1}$$

$$C_i = A_iB_i + (A_i \oplus B_i)C_{i-1}$$

> [!note] 理解 $C_i$ 的表达式
> - $A_iB_i$：当两位加数都为 1 时，必然产生进位（进位生成）
> - $(A_i \oplus B_i)C_{i-1}$：当 $A_i$、$B_i$ 中恰有一位为 1 且低位有进位时，产生进位（进位传递）

## 串行进位加法器

**串行进位加法器**（Ripple Carry Adder，简称 RCA）将 $n$ 个一位全加器**串联**，高位的进位输入来自低位的进位输出。

### 结构特点

- $n$ 位串行进位加法器需要 $n$ 个一位全加器
- 低位 FA 的 $C_{out}$ 连接高位 FA 的 $C_{in}$
- 最低位 $C_0 = 0$，最高位 $C_n$ 为最终进位输出

### 延迟分析

> [!warning] 关键问题：进位传播延迟
> 每个 FA 的 $C_i$ 必须等待 $C_{i-1}$ 稳定后才能确定，进位信号**逐级传递**。

- 设每个 FA 的进位延迟为 $t_c$（从 $C_{i-1}$ 到 $C_i$ 的延迟）
- $n$ 位 RCA 的**最坏情况进位延迟** = $n \times t_c$

| 优点 | 缺点 |
|------|------|
| 结构简单，硬件开销小（$n$ 个 FA） | 进位串行传递，**速度慢** |
| 规则性好，便于集成电路实现 | 位宽越大，延迟线性增长 |

## 并行进位加法器

**并行进位加法器**（Carry Look-ahead Adder，简称 CLA）的核心思想是：**进位信号不逐级传递，而是根据各位的输入直接并行生成所有进位**。

### 进位生成与进位传递

定义两个辅助函数：

| 函数 | 表达式 | 含义 |
|------|--------|------|
| **进位生成** $G_i$ | $G_i = A_i B_i$ | 当 $A_i$ 和 $B_i$ 均为 1 时，**必然产生进位** |
| **进位传递** $P_i$ | $P_i = A_i \oplus B_i$ | 当 $A_i$、$B_i$ 中恰有一个为 1 时，**传递低位进位** |

则进位 $C_i$ 可写为：

$$C_i = G_i + P_i \cdot C_{i-1}$$

### 并行进位展开

将 $C_i$ 逐级展开为仅关于 $G_i$、$P_i$ 和初始进位 $C_0$ 的表达式：

$$\begin{aligned}
C_1 &= G_1 + P_1 C_0 \\[4pt]
C_2 &= G_2 + P_2 G_1 + P_2 P_1 C_0 \\[4pt]
C_3 &= G_3 + P_3 G_2 + P_3 P_2 G_1 + P_3 P_2 P_1 C_0 \\[4pt]
C_4 &= G_4 + P_4 G_3 + P_4 P_3 G_2 + P_4 P_3 P_2 G_1 + P_4 P_3 P_2 P_1 C_0
\end{aligned}$$

> [!important] CLA 的关键优势
> 所有进位信号 $C_1, C_2, C_3, C_4$ 都可以**同时**从 $G_i$、$P_i$ 和 $C_0$ 计算出（与门-或门实现），不再需要等待低位进位。理论延迟为**常数级**（两级门延迟）。

### 多级分组并行进位

随着位数增加，进位表达式的门扇入数急剧增大（例如 $C_4$ 需要 4 输入或门），实际实现时会采用**分组分级**策略：

| 分组方式 | 说明 |
|----------|------|
| **组内并行，组间串行** | 每组 4 位用 CLA，组间进位串行传递（如 74LS283） |
| **组内并行，组间并行** | 再设一组 $G^*$、$P^*$ 函数，实现组间也并行（完全先行进位） |

> 典型的 ALU 采用**4 位一组**的 CLA 结构。

## 带标志加法器

**带标志加法器**在完成加法/减法运算的同时，生成**标志位**（Flags），用于后续的条件判断和程序控制。

### 核心标志位

在 $n$ 位加法器中，设 $A$、$B$ 为操作数，$C_{in}$ 为最低进位输入，$C_{out}$ 为最高进位输出，结果为 $F$

| 标志     | 名称                  | 含义          | 生成逻辑                         |
| ------ | ------------------- | ----------- | ---------------------------- |
| **CF** | 进位/借位标志（Carry Flag） | 无符号运算的进位/借位 | $CF = C_{out} \oplus C_{in}$ |
| **ZF** | 零标志（Zero Flag）      | 结果是否为零      | $ZF = 1$ 当且仅当 $F = 0$        |
| **SF** | 符号标志（Sign Flag）     | 结果最高位（符号位）  | $SF = F_{n-1}$               |
| **OF** | 溢出标志（Overflow Flag） | 有符号运算是否溢出   | $OF = C_{n-1} \oplus C_n$    |

> [!important] CF 与 OF 的本质区别
> | 标志 | 适用场景 | 判断标准 |
> |------|----------|----------|
> | **CF** | **无符号数**运算 | 是否超出无符号数表示范围 |
> | **OF** | **有符号数**（补码）运算 | 是否超出有符号数表示范围 |

### 溢出标志 OF 的判断

#### 单符号位进位判断法

$$OF = C_n \oplus C_{n-1}$$

- $C_n$：符号位产生的进位
- $C_{n-1}$：最高数值位向符号位产生的进位
- 两进位**不同**时发生溢出

| $C_n$ | $C_{n-1}$ | OF | 含义 |
|-------|-----------|-----|------|
| 0 | 0 | 0 | 无溢出（正 + 正，结果为正） |
| 0 | 1 | 1 | 溢出（正 + 正 → 超出上限） |
| 1 | 0 | 1 | 溢出（负 + 负 → 超出下限） |
| 1 | 1 | 0 | 无溢出（负 + 负，结果为负） |

#### 双符号位法

使用**变形补码**（双符号位 $S_{n+1}S_n$），详见 [[1.数制与编码#变形补码（双符号位补码）]]。

$$OF = S_{n+1} \oplus S_n$$

- 双符号位为 00 或 11：无溢出
- 双符号位为 01：**正溢出**
- 双符号位为 10：**负溢出**

#### 符号判断法

根据两个操作数和结果的符号位直接判断：

$$OF = \overline{A_s} \cdot \overline{B_s} \cdot F_s \; + \; A_s \cdot B_s \cdot \overline{F_s}$$

- 两个正数相加得负数 → 溢出
- 两个负数相加得正数 → 溢出
- ⭐一个正一个负相加**绝不会溢出**

### 进/借位标志 CF 的判断

无符号数的加减同样使用补码加法器，但用 **CF 标志**判断溢出：

$$CF = Sub \oplus C_{out}$$

其中：
- $Sub$：加减控制信号（$Sub = 0$ 表示加法，$Sub = 1$ 表示减法）
- $C_{out}$：加法器最高位产生的进位输出（$C_n$）

| Sub   | $C_{out}$ | CF    | 含义                          |
| ----- | --------- | ----- | --------------------------- |
| 0（加法） | 0         | 0     | 无符号加法结果不超出范围                |
| 0（加法） | 1         | 1     | 无符号加法结果超出 $2^n-1$，溢出        |
| 1（减法） | 0         | **1** | 有借位（$A < B$，结果为负，按无符号是一个大数） |
| 1（减法） | 1         | **0** | 无借位（$A \geq B$，结果正确）        |

> [!important] 理解减法时 $CF = \overline{C_{out}}$
> 减法电路实际执行 $A + \overline{B} + 1$：
> - 若 $A \geq B$，则 $A + \overline{B} + 1 \geq 2^n$，有进位 $C_{out} = 1$，$CF = 0$（无借位）
> - 若 $A < B$，则 $A + \overline{B} + 1 < 2^n$，无进位 $C_{out} = 0$，$CF = 1$（有借位）
>
> **为什么减法时进位和借位恰好相反？**
>
> 关键在于 $\overline{B} = (2^n - 1) - B$，所以：
> $$A + \overline{B} + 1 = A + (2^n - 1 - B) + 1 = A - B + 2^n$$
>
> 补码减法等价于给差值加了一个 $2^n$ 的偏移：
> - $A \geq B$ 时，$A - B \geq 0$，加上 $2^n$ 后 $\geq 2^n$ → **有进位**，但实际减法**无借位** → $C_{out}$ 被"翻转"
> - $A < B$ 时，$A - B < 0$，加上 $2^n$ 后 $< 2^n$ → **无进位**，但实际减法**有借位** → $C_{out}$ 被"翻转"
>
> 因此 $CF = Sub \oplus C_{out}$：
> - **加法**（$Sub=0$）：$CF = C_{out}$，进位即溢出
> - **减法**（$Sub=1$）：$CF = \overline{C_{out}}$，进位取反才是借位
>
> 用一条异或门就同时处理了加减两种情形，无需在加法器和减法器之间切换。

> [!note] 减法中 CF 的含义
> 对于无符号减法 $A - B$：若 $CF = 0$（无借位），说明 $A \geq B$，结果正确；若 $CF = 1$（有借位），说明 $A < B$，结果为负数（按无符号解读是一个很大的数）。

## 算术逻辑单元（ALU）

**ALU**（Arithmetic Logic Unit）是 CPU 中执行算术运算和逻辑运算的核心部件

### ALU 的基本结构

- **输入端**：两个 $n$ 位操作数 $A$ 和 $B$，一组控制信号（功能选择线）
- **输出端**：$n$ 位运算结果 $F$，以及标志位（CF、OF、SF、ZF）
- **核心部件**：$n$ 位加法器 + 逻辑运算电路 + 多路选择器

### ALU 的功能

| 类型 | 运算 | 说明 |
|------|------|------|
| **算术运算** | 加法 ADD | $F = A + B$ |
| | 带进位加法 ADC | $F = A + B + CF$ |
| | 减法 SUB | $F = A - B = A + \overline{B} + 1$（补码） |
| | 自增 INC | $F = A + 1$ |
| | 自减 DEC | $F = A - 1$ |
| | 取负 NEG | $F = -A = \overline{A} + 1$ |
| **逻辑运算** | 与 AND | $F = A \land B$（按位） |
| | 或 OR | $F = A \lor B$（按位） |
| | 非 NOT | $F = \overline{A}$（按位） |
| | 异或 XOR | $F = A \oplus B$（按位） |
| **移位运算** | 左移/右移 | 详见 [[#定点数的移位运算]] |

> [!note] ALU 数据宽度
> ALU 的数据宽度通常等于**机器字长**。字长越长，一次能处理的数据范围越大、精度越高，但硬件开销也越大。

# 定点数的移位运算

移位运算（Shift Operation）分为**逻辑移位**、**算术移位**和**循环移位**三种，由移位寄存器或移位逻辑电路实现。

## 逻辑移位

**逻辑移位**（Logical Shift）将操作数视为**无符号数**，所有位统一参与移位，空的位补 0。

| 操作 | 规则 | 示例（8 位：`10110010`） |
|------|------|--------------------------|
| **逻辑左移** SHL | 所有位向左移一位，最高位移入 CF，最低位补 0 | `10110010` → `01100100` |
| **逻辑右移** SHR | 所有位向右移一位，最低位移入 CF，最高位补 0 | `10110010` → `01011001` |

> [!note] 逻辑移位与乘除
> 对于无符号数：左移 1 位 ≈ × 2（不移出有效位）；右移 1 位 ≈ ÷ 2（舍入）

## 算术移位

**算术移位**（Arithmetic Shift）将操作数视为**有符号数**（补码），移位时**符号位保持不变**。

| 操作 | 符号位 | 数值位的处理 | 移入位 |
|------|--------|-------------|--------|
| **算术左移** SAL | 不变 | 所有数值位左移，数值部分高位丢弃 | 最低位补 **0** |
| **算术右移** SAR | 不变 | 所有数值位右移，数值部分低位丢弃 | 最高数值位补**符号位**（符号扩展） |

### 补码算术移位规则

> [!important] 补码算术移位的核心规则
> - **左移**：末位补 0（有溢出风险）
> - **右移**：高位补**符号位**（保持补码值正确）

> [!warning] 左移溢出的判断
> 算术左移后，若**符号位发生变化**（即移出的最高数值位 ≠ 原符号位），则发生溢出。
> 例如 `01000000`（+64）左移 1 位 → `10000000`（-128），符号位由 0 变 1，发生溢出。

### 原码、反码、补码的移位对比

| 编码 | 左移 | 右移 |
|------|------|------|
| **原码** | 数值部分左移，末位补 0；符号位不变 | 数值部分右移，高位补 0；符号位不变 |
| **反码** | 数值部分左移，末位补符号位（正补 0，负补 1） | 数值部分右移，高位补符号位 |
| **补码** | 数值部分左移，末位补 0 | 数值部分右移，高位补符号位 |

### 循环移位

**循环移位**（Rotate Shift）将移出的位回填到另一端的空位，形成循环。

| 操作 | 规则 | 说明 |
|------|------|------|
| **不带 CF 循环左移** ROL | 最高位移入最低位和 CF | 纯循环 |
| **不带 CF 循环右移** ROR | 最低位移入最高位和 CF | 纯循环 |
| **带 CF 循环左移** RCL | CF 作为"额外的第 9 位"参与循环 | 9 位循环 |
| **带 CF 循环右移** RCR | CF 作为"额外的第 9 位"参与循环 | 9 位循环 |

# 定点数的加减运算

> [!important] 补码加减法的核心优势
> 补码加减法使用**同一套加法电路**即可实现加法和减法，不需要额外的减法器，也不需要在加减前判断符号位。

## 补码加减法公式

### 补码加法

$$[X+Y]_{补} = [X]_{补} + [Y]_{补} \pmod{2^{n+1}}$$

- 两个补码**直接相加**，所得结果即为和的补码
- 符号位**参与运算**，与数值位一视同仁

### 补码减法

$$[X-Y]_{补} = [X]_{补} + [-Y]_{补} \pmod{2^{n+1}}$$

- 将减法转化为加法：$X - Y = X + (-Y)$
- $[-Y]_{补}$ 的求法：将 $[Y]_{补}$ **连同符号位一起取反，末位加 1**

> [!note] 求 [-Y]补 的具体步骤
> 例：已知 $[Y]_{补} = 10000101$，求 $[-Y]_{补}$
> 1. 连同符号位取反：`01111010`
> 2. 末位加 1：`01111011`
> 3. 所以 $[-Y]_{补} = 01111011$

## 无符号数大小的比较

无符号数的比较通过**减法**（$A - B$）实现，然后用标志位判断结果。

### 判断依据

| 条件         | 标志位                 | 含义              |
| ---------- | ------------------- | --------------- |
| $A = B$    | **ZF = 1**          | 结果为 0，两数相等      |
| $A > B$    | ZF = 0 且 **CF = 0** | 无借位，$A - B > 0$ |
| $A < B$    | ZF = 0 且 **CF = 1** | 有借位，$A - B < 0$ |

### x86 条件转移指令

| 指令                      | 含义      | 判断条件            |
| ----------------------- | ------- | --------------- |
| **JE** / **JZ**         | 相等则跳转   | ZF = 1          |
| **JNE** / **JNZ**       | 不等则跳转   | ZF = 0          |
| **JA**（Jump if Above）   | 无符号大于   | CF = 0 且 ZF = 0 |
| **JAE**（Jump if Above or Equal）| 无符号大于等于 | CF = 0 |
| **JB**（Jump if Below）   | 无符号小于   | CF = 1 且 ZF = 0 |
| **JBE**（Jump if Below or Equal）| 无符号小于等于 | CF = 1 或 ZF = 1 |

> [!note] 只依赖 CF 和 ZF
> 无符号数比较**不涉及** OF 和 SF，因为无符号数本身没有符号概念，溢出仅由 CF 判断。

## 有符号数大小的比较

有符号数（补码）的比较同样通过**减法**（$A - B$）实现，但判断时需要同时考虑**溢出**。

### 核心难点

有符号减法 $A - B = A + (-B)$ 可能发生溢出，导致结果的 SF（符号位）**不等于**真正的正负。

> [!important] 关键结论
> 有符号数比较时，结果的真实符号 = **$SF \oplus OF$**。
> - $SF \oplus OF = 0$：真正结果 $> 0$
> - $SF \oplus OF = 1$：真正结果 $< 0$；
> - 若发生溢出（OF = 1），SF 是"反的"——溢出使得 SF 与真实符号相反，因此异或 OF 即可纠正

### 判断依据

| 条件              | 标志位                                   | 说明              |
| --------------- | ------------------------------------- | --------------- |
| $A = B$         | **ZF = 1**                            | 结果为 0           |
| $A > B$         | ZF = 0 且 **$SF \oplus OF = 0$**       | 真实结果为正，且非零      |
| $A \geq B$      | **$SF \oplus OF = 0$**                | 真实结果为非负（含零）     |
| $A < B$         | ZF = 0 且 **$SF \oplus OF = 1$**       | 真实结果为负，且非零      |
| $A \leq B$      | ZF = 1 或 **$SF \oplus OF = 1$**       | 结果为 0 或负（真实结果为非正） |

### x86 条件转移指令

| 指令                                        | 含义        | 判断条件                     |
| ----------------------------------------- | --------- | ------------------------ |
| **JE** / **JZ**                           | 相等则跳转     | ZF = 1                   |
| **JNE** / **JNZ**                         | 不等则跳转     | ZF = 0                   |
| **JG**（Jump if Greater）                  | 有符号大于     | ZF = 0 且 $SF \oplus OF = 0$ |
| **JGE**（Jump if Greater or Equal）        | 有符号大于等于   | $SF \oplus OF = 0$        |
| **JL**（Jump if Less）                     | 有符号小于     | $SF \oplus OF = 1$        |
| **JLE**（Jump if Less or Equal）           | 有符号小于等于   | ZF = 1 或 $SF \oplus OF = 1$ |

> [!important] 有符号与无符号条件跳转的本质差异
> | 比较类型 | 依赖标志 | 关键逻辑 |
> |----------|----------|----------|
> | **无符号数** | CF + ZF | 借位（CF）直接反映两数大小：$A-B$ 有借位 → $A<B$ |
> | **有符号数** | SF + OF + ZF | 溢出会使 SF 反向，必须用 $SF \oplus OF$ 纠偏才能得到真实符号 |


## 加减运算电路

补码加减法的硬件核心是：**一个加法器 + 一个多路选择器 + 一个控制信号（Sub）**。

| 控制信号 Sub | 操作 | Cin | 输入到加法器的 Y 端 | 实现 |
|-------------|------|-----|-------------------|------|
| **0**（加法） | $X + Y$ | 0 | $[Y]_{补}$（直通） | $[X]_{补} + [Y]_{补}$ |
| **1**（减法） | $X - Y$ | 1 | $\sim[Y]_{补}$（按位取反） | $[X]_{补} + \sim[Y]_{补} + 1 = [X]_{补} + [-Y]_{补}$ |

> [!important] 核心原理
> 减法时：$\sim[Y]_{补} + 1 = [-Y]_{补}$（取反加一），所以 $[X]_{补} + \sim[Y]_{补} + Cin = [X]_{补} + [-Y]_{补} = [X-Y]_{补}$
> 所以**输入加法器**的是 X 的原码，Y 的反码和 Sub

### 关键部件

| 部件 | 作用 |
|------|------|
| **$n$ 位加法器** | 执行核心加法运算，通常由 $n$ 个全加器（FA）级联构成（行波进位） |
| **多路选择器**（MUX） | 根据 Sub 信号选择：加法时直通 $[Y]_{补}$，减法时输出 $\sim[Y]_{补}$ |
| **异或门阵列** | 实际实现中，Sub 信号与 $[Y]_{补}$ 的每一位异或：Sub=0 直通，Sub=1 取反 |
| **Cin（进位输入）** | 最低位全加器的进位输入，加法时为 0，减法时为 1 |

## 原码加减法（了解）

原码的符号位和数值位是分离的，因此加减法**不能直接套用补码电路**。

### 原码加减法的流程

1. **比较符号**：两数符号 $X_s$、$Y_s$
2. **若同号**（$X_s = Y_s$）：数值部分做**加法**，结果的符号取原符号
3. **若异号**（$X_s \neq Y_s$）：先**比较绝对值大小**
   - 若 $|X| \geq |Y|$：$|X| - |Y|$，结果符号取 $X_s$
   - 若 $|X| < |Y|$：$|Y| - |X|$，结果符号取 $Y_s$

### 所需硬件

| 部件 | 用途 |
|------|------|
| **比较器** | 比较两数绝对值大小 |
| **加法器** | 同号相加 |
| **减法器**（或加补码后取绝对值的电路） | 异号时大减小 |
| **符号逻辑电路** | 根据符号和比较结果确定最终符号 |

# 定点数的乘除运算

## 原码乘法

### 基本原理

原码乘法的特点是**符号位和数值位分别处理**，分为两步：1.乘积的符号位由两个乘数的符号位异或得到；2.相当于两个乘数的绝对值相乘，归结为无符号数的相乘

**核心思想**：乘数从最低位开始逐位判断，为 1 则加上被乘数，为 0 则不加。每步累加后，**部分积右移一位**（相当于被乘数左移的相对关系）

### 无符号数乘法流程

设被乘数 $X$，乘数 $Y$，结果 $P$。

**符号处理**：$P_s = X_s \oplus Y_s$（**符号位单独异或处理**）

**数值部分**：$|P| = |X| \times |Y|$

**运算规则**（$n$ 位数值位）：

1. 部分积 $A$ 初始化为 0，附加位记录乘数 $|Y|$
2. 检查乘数最低位：
   - 若为 **1**：部分积 $A = A + |X|$
   - 若为 **0**：部分积 $A$ 不变
3. 部分积连同乘数**右移一位**
4. 重复步骤 2-3 共 $n$ 次

> [!example] 原码一位乘法示例
> 设 $X = +0.1101$，$Y = +0.1011$（定点小数，4 位数值）
>
> | 步 | 部分积 A | 乘数最低位 | 操作 |
> |----|----------|-----------|------|
> | 初 | 0.0000 | ,101**1** | |
> | 1 | 0.1101 | | +\|X\| |
> | 1 | **0.0110** | 1,10**1** | 右移 |
> | 2 | 1.0011 | | +\|X\| |
> | 2 | **0.1001** | 11,1**0** | 右移 |
> | 3 | 0.1001 | | +0 |
> | 3 | **0.0100** | 111,**1** | 右移 |
> | 4 | 1.0001 |  | +\|X\| |
> | 4 | 0.1000 | 1111, | 右移 |
>
> 最终：$P_s = 0 \oplus 0 = 0$，$|P| = 0.10001111$
>
> 结果 $= +0.10001111$（$0.1101 \times 0.1011 = 0.10001111$）✓

### 无符号整数的乘法运算电路

> 本质：**移位 + 加法**的迭代结构，每次根据乘数最低位决定是否加上被乘数。
![[无符号整数的乘法运算电路.png]]
**核心部件**：

| 部件            | 位宽                         | 作用                  |
| ------------- | -------------------------- | ------------------- |
| **乘积寄存器 P**   | $n$ 位                      | 存放部分积，初始为 0         |
| **乘数寄存器 Y**   | $n$ 位                      | 存放乘数，右移时 LSB 移出用于判断 |
| **被乘数寄存器 X**  | $n$ 位                      | 存放被乘数，固定不变          |
| **$n$ 位加法器**  | $n$ 位                      | 执行 $P + X$          |
| **计数器 C$_n$** | $\lceil \log_2 n \rceil$ 位 | 控制循环次数 $n$          |

**工作流程**（$n$ 次迭代）：
1. 检查 Y 最低位：为 1 → $P \leftarrow P + X$；为 0 → 不变
2. P 与 Y **联合逻辑右移 1 位**（P 最高位补 0，Y 最低位移入 CF 或丢弃）
3. 计数器 -1，归零则结束

### 结果与溢出判断

**结果存放**：$n$ 位 × $n$ 位 → 乘积最多 $2n$ 位，存放在 {P, Y} 中（P 存高 $n$ 位，Y 存低 $n$ 位）。

**溢出判断**：无符号 $n$ 位乘法的乘积必然可以用 $2n$ 位完整表示（最大值为 $(2^n-1)^2 = 2^{2n} - 2^{n+1} + 1 < 2^{2n}$）。

> **溢出发生在截断时**：若只取低 $n$ 位作为结果，则 $P \neq 0$ 即表示溢出（高 $n$ 位有有效数据，截断会丢失精度）。

**硬件处理**：通常不额外检查溢出，直接给出完整 $2n$ 位积交由软件判断。部分指令集（如 x86 的 MUL）会自动设置 CF/OF 标志——$P \neq 0$ 时 OF = CF = 1。

## 补码乘法（Booth 算法）

**Booth 算法**（布斯算法）是一种**直接处理补码**的乘法算法，不需要将负数转换为原码，运算过程中全程使用补码。

### Booth 一位乘法原理

Booth 算法的核心在于将乘数 $Y$ 重新编码为 $\pm 1$ 和 0 的序列：

$$Y = \sum_{i=0}^{n-1} (y_{i-1} - y_i) \cdot 2^i$$

其中 $y_{-1} = 0$（附加位）。

### 操作规则（Booth 一位乘）

设乘数 $Y$ 的补码表示（含符号位共 $n+1$ 位），附加位 $y_{-1} = 0$：

| $y_i$ | $y_{i-1}$ | 操作                         |
| ----- | --------- | -------------------------- |
| 0     | 0         | 部分积不变，右移 1 位               |
| 0     | 1         | 部分积 **$+ [X]_{补}$**，右移 1 位 |
| 1     | 0         | 部分积 **$- [X]_{补}$**，右移 1 位 |
| 1     | 1         | 部分积不变，右移 1 位               |

> [!important] Booth 算法的关键
> 1. 判断的是 $y_{i} y_{i-1}$（前一位减当前位），而非单一位
> 2. 初始附加位 $y_{-1} = 0$，因此第一步只看 $y_0$
> 3. 符号位**自然参与**运算，结果即为补码乘积，无需单独处理符号

### Booth 一位乘法流程

1. 部分积 $A$ 初始化为 0（补码，与 $[X]_{补}$ 同宽）
2. 附加位 $y_{-1} = 0$
3. 检查 $y_0$ 和 $y_{-1}$，按上表执行操作
4. **算术右移**部分积和乘数（高位补符号位）
5. 重复步骤 3~4 共 $n+1$ 次，但**第 $n+1$ 次不右移**（即累加 $n+1$ 次，移位 $n$ 次）

### 有符号整数的乘法运算电路

> 基于 Booth 算法的硬件实现，可直接处理补码。与无符号乘法电路相比，核心差异在**移位方式**和**操作选择逻辑**。
![[有符号整数的乘法运算电路.png]]
**核心部件**（与无符号电路对比）：

| 部件             | 无符号乘法           | 有符号乘法（Booth）                    |
| -------------- | --------------- | ------------------------------- |
| **累加器 P**      | $n$ 位，初始 0      | $n+1$ 位（多 1 位符号），初始 0           |
| **乘数寄存器 Y**    | $n$ 位           | $n+1$ 位 + **1 位附加位** $y_{-1}=0$ |
| **被乘数寄存器 X**   | $n$ 位（绝对值）      | $n+1$ 位（补码）                     |
| **移位方式**       | **逻辑右移**（高位补 0） | **算术右移**（高位补符号位）                |
| **操作选择**       | Y 最低位 = 1 → +X  | 查 $y_i y_{i-1}$ 编码表             |
| **迭代次数 C$_n$** | $n$ 次（累加 + 移位各 $n$ 次） | **累加 $n+1$ 次，移位 $n$ 次**（最后一次不移位） |

**关键电路差异**：
- 需要 $\pm X$ 选择逻辑（X 或 $\overline{X}$ 通过多路选择器送入加法器，减时 $C_{in}=1$）
- 移位为**算术右移**（P 最高位不变，即符号扩展），保证补码正确性

### 结果与溢出判断

**结果存放**：64 位乘积存储在寄存器 \[ P : Y \] 中，其中 P 为高 32 位，Y 为低 32 位

**溢出判断**：
- 补码乘法理论上，两个 $n+1$ 位补码相乘（真值范围 $-2^n \sim 2^n-1$），乘积可能占 $2n+1$ 位，但实际只取低 $2n+1$ 位（含符号位）
- **判断方法**：检查 P 是不是低 32 位结果 Y 的符号扩展，若不是则溢出

### 乘法运算的三种实现方式

#### 迭代式乘法器

> 即前述移位-加法器，通过**ALU + 移位寄存器 + 计数器**反复迭代完成。

| 特点     | 说明                                   |
| ------ | ------------------------------------ |
| **硬件** | 1 个加法器 + 3 个寄存器（P, Y, X）+ 计数器 + 控制逻辑 |
| **速度** | 慢，$n$ 个时钟周期（$n$ 位）                   |
| **面积** | 小                                    |
| **代表** | 原码一位乘、Booth 一位乘                      |

适用于**低成本、低功耗**场景。

#### 阵列乘法器

> 用**组合逻辑**（大量全加器排列成阵列）在一个时钟周期内直接计算出乘积。

**原理**：将乘法展开为 $n \times n$ 个部分积的求和，每行由 AND 门生成部分积，各行之间用全加器级联求和。

| 特点 | 说明 |
|------|------|
| **硬件** | $n \times n$ 个 AND 门 + $(n-1) \times n$ 个全加器 |
| **速度** | 快，**1 个时钟周期**（仅组合逻辑延迟） |
| **面积** | 大，$O(n^2)$ |
| **代表** | Wallace 树乘法器、Dadda 乘法器 |

#### 移位-加减法

> 同一套 **ALU（加法器 + 移位器）**，通过**微程序/控制信号**循环执行移位和加减，**不额外增加乘法专用硬件**。

| 特点 | 说明 |
|------|------|
| **硬件** | 复用 ALU（零额外乘法硬件） |
| **速度** | 最慢，每条指令需大量微操作 |
| **适用** | 早期 CPU、极低成本嵌入式 |

> [!note] 与前两种的关系
> 迭代式乘法器是其**硬件化**版本（加寄存器+计数器）；阵列乘法器是其**并行化**版本。三者本质相同——乘法 = 部分积的移位累加。

## 除法运算

除法运算相较于乘法更为复杂，需要考虑**商的逐位确定**和**余数的处理**，除法首先需要对被除数和除数的取值进行预判以识别异常和确定结果是否为 0：
- 若被除数为 0，除数为 0 → **异常**（$0 \div 0$ 未定义）
- 若被除数不为 0，除数为 0 → **异常**（除零错）
- 若被除数为 0，除数不为 0 → **商 = 0，余数 = 0**，直接结束
仅当被除数和除数均不为 0 且 $|\text{被除数}| \ge |\text{除数}|$ 时，才进入正常的逐位试商流程

### 无符号整数的除法运算原理

> 核心：**试商 → 减除数 → 判断余数 → 商位确定**，迭代 $n$ 次。与乘法不同，除法**余数左移**（而非部分积右移）。

#### 恢复余数法

**规则**：先减除数，余数 $\ge 0$ 则商上 1；余数 $< 0$ 则商上 0，并加回除数（恢复余数）。

| 步骤 | 操作 |
|------|------|
| 1 | 余数 $R \leftarrow R - Y$（首次直接减） |
| 2 | $R \ge 0$ → 商上 **1**；$R < 0$ → 商上 **0**，$R \leftarrow R + Y$（恢复） |
| 3 | 余数左移 1 位，重复步骤 1~2 共 $n$ 次 |

**缺点**：不够减时需额外一次加法恢复，效率低。

#### 不恢复余数法（加减交替法）

**改进思路**：余数为负时不恢复，下一步直接做加法，省去恢复步骤。

推导：若 $R_i < 0$，恢复后得 $R_i + Y$，左移再减 → $2(R_i + Y) - Y = 2R_i + Y$，等价于直接 $R_{i+1} = 2R_i + Y$。

| 上一步商 | 下一步操作 |
|----------|------------|
| 商 = 1（够减） | $R \leftarrow 2R - Y$（**减**） |
| 商 = 0（不够减） | $R \leftarrow 2R + Y$（**加**） |

> [!important] 最后一步若余数为负，需**恢复一次**（$R \leftarrow R + Y$）得到正确余数。

#### 商的修正

原码除法最后一步商上 1，若除不尽则**末位恒置 1**以保证精度。

### 无符号整数除法运算电路

![[无符号整数除法运算电路.png]]

**核心部件**（与乘法电路对称）：

| 部件           | 位宽                         | 作用               |
| ------------ | -------------------------- | ---------------- |
| **余数寄存器 R**  | $n$ 位                      | 存放当前余数，**初始为 0** |
| **除数寄存器 Y**  | $n$ 位                      | 存放除数，固定不变        |
| **商寄存器 Q**   | $n$ 位                      | 逐位存入商，**初始为被除数** |
| **$n$ 位加法器** | $n$ 位                      | 执行 $R \pm Y$     |
| **计数器**      | $\lceil \log_2 n \rceil$ 位 | 控制 $n$ 次迭代       |

**工作要点**：
- 每次先试商（$R - Y$ 判断符号），再根据结果决定加减
- 余数寄存器 R 与商寄存器 Q **联合逻辑左移**（与乘法右移方向相反）
- 加减交替法由**单个加减控制信号**（$Sub$）切换

#### 结果与异常处理

**结果存放**：$n$ 位 ÷ $n$ 位 → 商在 Q 中（$n$ 位），余数在 R 中（$n$ 位）
**异常处理**：

| 异常类型 | 条件 | 处理 |
|----------|------|------|
| **除零错** | 除数 = 0 | 触发除法异常中断（Division Error） |

> [!important] 无符号除法不溢出
> 两个 $n$ 位无符号数相除，最大商为 $(2^n-1) \div 1 = 2^n-1$，始终可用 $n$ 位表示，**不会发生溢出**。唯一的异常就是除零。

## 补码除法

### 补码除法运算的工作原理

> 补码除法直接处理补码操作数，符号位自然参与运算，无需单独处理；核心同样是**加减交替**。

**操作规则**（被除数 $[X]_{补}$，除数 $[Y]_{补}$）：

**第一步**：根据符号关系决定初操作
- 若 $[X]_{补}$ 与 $[Y]_{补}$ **同号**：$R \leftarrow X - Y$（减）
- 若 $[X]_{补}$ 与 $[Y]_{补}$ **异号**：$R \leftarrow X + Y$（加）

**后续各步**：

| 比较 | 商上 | 下一步操作 |
|------|------|------------|
| $R$ 与 $Y$ **同号** | 商上 **1** | $R \leftarrow 2R - Y$（减） |
| $R$ 与 $Y$ **异号** | 商上 **0** | $R \leftarrow 2R + Y$（加） |
> 其中是**算术左移**

> [!important] 补码 vs 原码除法的关键区别
> | 维度 | 原码除法 | 补码除法 |
> |------|----------|----------|
> | 上商依据 | 余数正负（$\ge 0$ 上 1） | 余数与除数**同号则上 1** |
> | 符号处理 | 单独异或 | 自然参与运算 |
> | 商末位 | 恒置 1 | 恒置 1 |

**商的修正**：补码除法末位同样**恒置 1**

#### 异常处理

| 异常类型    | 条件                      | 说明            |
| ------- | ----------------------- | ------------- |
| **除零错** | 除数 = 0                  | 与无符号除法相同，触发中断 |
| **商溢出** | $X = -2^{n}$ 且 $Y = -1$ | **唯一溢出情形**    |

> [!important] 补码除法的溢出仅此一种
> $n+1$ 位补码（$n$ 位数值）范围 $[-2^n, 2^n-1]$。任何两个在此范围内的数相除，商的绝对值 $\le$ 被除数的绝对值，**只有被除数为最小负数 $-2^n$ 且除数为 $-1$ 时商 $= 2^n$ 超出范围**。

# 运算方法与运算电路知识体系总结

> [!summary] 本章重点
> 1. **基本运算部件**：全加器（$S_i = A_i \oplus B_i \oplus C_{i-1}$）→ RCA（进位串行，慢）→ CLA（并行进位，$G_i/P_i$，常数级延迟）→ 带标志加法器（OF / CF / SF / ZF，$OF = C_n \oplus C_{n-1}$，$CF = Sub \oplus C_{out}$）→ ALU（加法器 + 逻辑电路 + 移位器）
> 2. **定点数的移位运算**：逻辑移位（无符号，补 0）、算术移位（补码，左移末位补 0，右移高位补符号位）、循环移位（ROL/ROR/RCL/RCR）；算术左移符号位变 → 溢出
> 3. **定点数的加减运算**：补码加减法统一用加法器（$[X-Y]_{补} = [X]_{补} + [-Y]_{补}$）；OF 三种判断法（进位异或 / 双符号位 / 符号判断）等价；CF 用于无符号数
> 4. **定点数的乘法运算**：原码一位乘（符号异或 + 绝对值移位加）、Booth 算法（$y_{i-1} - y_i$ 编码，累加 $n+1$ 次移位 $n$ 次）、三种实现方式（迭代式 / 阵列式 / 移位-加减法）；无符号乘法截断时 $P \neq 0$ 溢出，补码乘法看 P 高两位
> 5. **定点数的除法运算**：恢复余数法（不够减则恢复）→ 加减交替法（不够减不恢复，下一步加）；补码除法（同号商 1、异号商 0）；异常：无符号仅除零错，补码除零 + $-2^n \div (-1)$ 商溢出
